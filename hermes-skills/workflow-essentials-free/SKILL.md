## 快速开始

### 自动化的核心原理（<60秒理解）

```text
自动化 = 触发器 + 操作步骤 + 错误处理

触发器：什么事件启动工作流？（定时/事件/手动）
操作步骤：执行什么动作？（收集/处理/输出）
错误处理：出问题时怎么办？（通知/重试/跳过）
```

```

## 核心能力
### Step 1：识别自动化机会

**自动化黄金法则**：每周重复两次以上、不需要创造性思维的任务，都值得自动化。

**适合自动化的四特征**：

| 特征 | 说明 | 示例 |
|------|------|------|
| 重复性 | 每次步骤相同 | 每周格式相同的周报 |
| 规则性 | 规则驱动，无需判断 | 付款后创建发票 |
| 高频率 | 每日或每周执行 | 表单数据入库 |
| 耗时长 | 单次10分钟以上 | 数据手动汇总 |

**不适合自动化的任务**：

- 需要创造性思维（写定制内容）
- 需要人际互动（客户访谈）
- 罕见任务（每月不到1次）
- 需要综合判断（产品决策）

**快速识别清单**：

```text
问自己：
  □ 这个任务每次步骤都一样吗？
  □ 这个任务每周至少做两次吗？
  □ 单次耗时超过10分钟吗？
  □ 不需要创造性判断吗？

四个都"是" → 立即自动化
三个"是" → 建议自动化
两个以下"是" → 手动即可
```

**输入**: 用户提供Step 1：识别自动化机会所需的指令和必要参数。
**处理**: 按照skill规范执行Step 1：识别自动化机会操作,遵循单一意图原则。
**输出**: 返回Step 1：识别自动化机会的执行结果,包含操作状态和输出数据。

### Step 2：设计自动化方案

在构建之前，先用纸笔画出方案。

**工作流设计模板**：

```text
┌─────────────────────────────────────┐
│         工作流设计方案               │
├─────────────────────────────────────┤
│                                     │
│  触发器：什么事件启动？              │
│  （定时/文件变化/新数据/手动）       │
│                                     │
│  条件（可选）：满足什么条件才执行？  │
│                                     │
│  操作步骤：                          │
│    步骤1：__________                │
│    步骤2：__________                │
│    步骤3：__________                │
│                                     │
│  错误处理：失败时怎么办？            │
│  （通知我/重试/跳过继续）            │
│                                     │
└─────────────────────────────────────┘
```

**设计示例（表单→CRM→邮件）**：

```text
触发器：网站新表单提交

条件：邮箱字段不为空

操作：
  步骤1：添加到CRM
  步骤2：发送欢迎邮件
  步骤3：创建3天后跟进任务
  步骤4：发消息通知"新线索来了"

错误处理：步骤1失败时，发邮件告警给我
```

**三条设计原则**：

1. **从简单开始**：先做2-3步，跑通后再加复杂度
2. **逐步测试**：每步单独测试成功后再串联
3. **必加错误通知**：出问题要能知道，否则静默失败

**输入**: 用户提供Step 2：设计自动化方案所需的指令和必要参数。
**处理**: 按照skill规范执行Step 2：设计自动化方案操作,遵循单一意图原则。
**输出**: 返回Step 2：设计自动化方案的执行结果,包含操作状态和输出数据。

### Step 3：构建与验证

**工具选择速查**：

| 工具 | 适合 | 价格 | 难度 |
|------|------|------|------|
| Zapier | 简单2-3步工作流 | $20-50/月 | 简单 |
| Make | 可视化多步骤工作流 | $9-30/月 | 中等 |
| n8n | 复杂工作流，自托管 | 免费或$20/月 | 较难 |

**新手建议**：从Zapier开始，最容易上手。

**构建步骤（通用流程）**：

1. 选择触发器应用
2. 连接账户（OAuth认证）
3. 测试触发器（确认数据能过来）
4. 添加第一个操作
5. 映射字段（匹配数据到目标）
6. 测试操作（确认结果正确）
7. 重复添加更多操作
8. 开启工作流

**验证清单**：

- [ ] 提交测试数据，触发器是否响应
- [ ] 每个操作是否正确执行
- [ ] 数据是否映射到正确字段
- [ ] 边界情况测试（空字段、特殊字符）
- [ ] 错误处理测试（故意制造失败）

**常见问题速修**：

| 问题 | 修复方法 |
|------|----------|
| 工作流不触发 | 检查触发条件，放宽过滤 |
| 操作失败 | 检查API限流，添加延迟 |
| 数据错误 | 检查字段映射是否正确 |
| 重复执行 | 基于唯一ID去重 |

**输出**: 返回Step 3：构建与验证的执行结果,包含操作状态和输出数据。

### 场景一：第一个自动化（新手入门）

用户想开始自动化但不知道从哪做起。

```text
推荐：从"低垂果实"开始——表单提交自动通知

1. 识别 → 表单提交后手动查看，每周多次 → 适合自动化

用户在两个工具间手动复制数据。

```text
1. 识别 → 每天手动从A工具复制到B工具 → 适合自动化
2. 设计 → 触发：A工具新数据 → 操作：写入B工具
3. 构建 → 连接两个工具 → 映射字段 → 测试

效果：数据自动同步，不再手动复制
```

### 场景三：定时提醒（任何人）

用户经常忘记定期做某事。

```text

3. 构建 → 定时触发+消息通知 → 测试

效果：每周自动提醒，不再遗忘
```

## 自动化思维入门

### 什么时候该想到自动化？

- 你第二次做相同操作时
- 你说"每次都要..."时
- 你在复制粘贴时
- 你在手动导出导入时
- 你在同一时间做同一件事时

### 自动化的三个层次

```text
层次1：手动做（每次重复全部步骤）
  ↓ 自动化
层次2：半自动（部分步骤用工具辅助）
  ↓ 自动化
层次3：全自动（触发后无需干预）
```

**目标**：把高频任务从层次1推进到层次3。

## FAQ

### Q1：我完全不懂技术，能用自动化吗？

可以。现代无代码自动化工具（如Zapier）通过可视化界面连接应用，不需要写代码。你只需描述"我想做什么"，Agent会帮你设计方案并指导构建。从2步的简单工作流开始，逐步建立信心。

### Q2：三步精要框架够用吗？

对于构建第一个自动化工作流，三步框架（识别→设计→构建）完全够用。它能帮你快速上手，完成80%的常见自动化需求。当你需要维护多个工作流、设计复杂场景或计算ROI时，专业版提供完整的六步框架和进阶内容。

### Q3：哪个自动化工具最好？

没有"最好"，只有"最适合"。新手从Zapier开始（最简单）；需要可视化多步骤用Make；有技术背景且想免费自托管用n8n。建议先用免费版试用，跑通后再决定是否付费。

### Q4：自动化安全吗？会不会出问题？

自动化本身是安全的，但需要做好错误处理：设置失败通知（出问题你会知道）、用测试数据验证（上线前测试）、保留手动备份方案。专业版提供完整的故障排查表，覆盖常见问题。

### Q5：免费版和专业版有什么区别？

免费版提供三步精要框架（识别→设计→构建）和三个基础场景。专业版解锁六步完整框架（新增维护和进阶）、六种自动化设计模式、四种进阶场景、维护策略、常见错误清单、多角色指南和完整故障排查表。专业版还提供工具深度对比和迁移指南。
