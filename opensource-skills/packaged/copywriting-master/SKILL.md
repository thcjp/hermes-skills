# 营销文案大师

基于经典文案框架与模板,生成以转化为导向的营销文案。不只写文字,更关注转化效果。

## 适用场景

| 场景 | 输入 | 输出 |
|---|---|---|
| 广告标题 | 产品信息+目标受众+渠道 | 5-10个高点击率标题候选 |
| 产品描述 | 产品功能+卖点+目标受众 | FAB结构详情页文案(300-800字) |
| 落地页 | 产品+转化目标+品牌调性 | 完整落地页文案(钩子+痛点+方案+证据+CTA) |
| EDM邮件 | 邮件目的+受众+产品 | 主题行+正文+CTA(50字主题+300字正文) |
| 社交文案 | 平台+产品+受众+话题 | 平台适配文案(微博140字/小红书500-1000字) |
| 销售信 | 产品+痛点+证据+报价 | 说服型长文案(1000-3000字) |

### 不适用于
- 品牌VI设计与视觉创意(请使用设计工具)
- 视频脚本分镜与拍摄指导(请使用视频脚本工具)
- SEO技术优化(关键词布局/外链建设,本Skill仅做文案层)
- 数据分析与转化漏斗诊断(请使用分析工具)
- 多语言翻译执行(本Skill输出中文文案,翻译请使用翻译服务)
- 法律合规审核(医疗/金融等敏感行业需法务审核)

## 核心能力

1. **四大框架灵活切换**:AIDA(注意力-兴趣-欲望-行动)/PAS(问题-激化-解决)/FAB(特性-优势-利益)/BAB(现状-理想-桥梁),根据产品类型与受众痛点自动选择
2. **12种标题模板批量生成**:数字型/问题型/利益型/好奇型/对比型/恐惧型/权威型/故事型/反常识型/时效型/社交证明型/稀缺型,每场景输出5-10个候选
3. **全渠道文案适配**:搜索广告(30/90字)、微博(140字)、小红书(500-1000字)、微信公众号(800-2000字)、邮件主题(50字内)、落地页(不限)、产品详情(300-800字)
4. **CTA优化与A/B测试建议**:动词+利益+紧迫感组合、测试元素建议(标题/钩子/CTA/证据)、可读性与说服力评分
5. **文案原则内置**:用户视角(写"你"不写"我们")、具体胜于抽象、一个文案一个核心信息、CTA明确、简洁有力

## 使用流程

### Step 1: 需求分析
1. 产品/服务理解:功能/卖点/差异化/定价
2. 目标受众画像:人口统计/痛点/欲望/语言风格
3. 转化目标:点击/注册/购买/分享
4. 投放渠道:搜索引擎/社交/邮件/落地页
5. 品牌调性:正式/活泼/专业/亲切

### Step 2: 框架选择
1. AIDA(注意力-兴趣-欲望-行动):适用广告/落地页/销售信
2. PAS(问题-激化-解决):适用痛点驱动的产品
3. FAB(特性-优势-利益):适用产品描述/技术产品
4. BAB(现状-理想-桥梁):适用转化型文案

### Step 3: 文案生成
1. **标题生成**(5-10个候选):数字型/问题型/利益型/好奇型/对比型
2. **正文撰写**:钩子(首句)→痛点(共鸣)→方案(产品)→证据(案例/数据)→CTA(行动)
3. **多版本生成**:不同框架/不同角度/不同语气

### Step 4: 渠道适配
1. 根据渠道调整长度与风格(见渠道适配表)
2. 嵌入关键词(SEO友好,如落地页)
3. 平台特性适配(微博话题/小红书关键词/微信排版)

### Step 5: 优化与A/B测试
1. 文案评分:可读性/说服力/转化潜力
2. A/B测试建议:哪些元素值得测试
3. CTA优化:动词+利益+紧迫感

## 文案框架详解

### AIDA(注意力-兴趣-欲望-行动)
- Attention:标题抓住注意力
- Interest:引发兴趣(痛点/利益)
- Desire:激发欲望(场景/证据)
- Action:明确行动号召
- 适用:广告/落地页/销售信

### PAS(问题-激化-解决)
- Problem:提出问题
- Agitate:激化痛点(后果/感受)
- Solve:给出解决方案
- 适用:痛点驱动的产品

### FAB(特性-优势-利益)
- Feature:产品特性
- Advantage:相比的优势
- Benefit:用户的利益
- 适用:产品描述/技术产品

### BAB(Before-After-Bridge)
- Before:现状(有问题)
- After:理想状态(无问题)
- Bridge:桥梁(产品)
- 适用:转化型文案

## 文案原则

1. **用户视角**:写"你"不写"我们",关注用户利益
2. **具体胜于抽象**:用数字/案例,不用空泛形容词
3. **一个文案一个核心信息**:不贪多,聚焦
4. **CTA 明确**:告诉用户下一步做什么
5. **简洁有力**:删减冗余,每字都有价值

## 错误处理
- 边界输入处理: 空输入返回提示信息, 超长输入自动截断
- 降级策略: 异常时返回默认值, 确保流程不中断
- 重试机制: 失败时自动重试, 最多3次

| 错误场景 | 原因 | 处理方式 |
|:------|------:|:------|
| 文案空泛无说服力 | 产品信息不足/卖点模糊 | 追问3个核心卖点+1个差异化,要求提供具体数据/案例 |
| 渠道不适配 | 长度/风格与平台不符 | 按渠道适配表重新调整,提供平台规范参考 |
| CTA模糊 | 未明确转化目标 | 追问转化目标(点击/注册/购买),生成动词+利益+紧迫感CTA |
| 文案同质化 | 与竞品文案雷同 | 提供竞品文案参考,切换框架或角度,强化差异化卖点 |
| 品牌调性不符 | 风格与品牌冲突 | 确认调性(正式/活泼/专业/亲切),重新生成匹配风格 |
| 敏感行业合规风险 | 医疗/金融/教育违规用词 | 标注风险词,提示法务审核,提供合规替代文案 |
| 多版本差异不大 | 框架选择单一 | 强制使用不同框架(AIDA/PAS/FAB/BAB)生成差异化版本 |
