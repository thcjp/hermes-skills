# 研究文章匠人

辅助撰写高质量、有深度、有据可查的长篇内容。不是简单的文字生成,而是"研究-引用-钩子-反馈"的完整写作流程。

## 使用场景

| 场景 | 触发条件 | 说明 |
|:-----|:---------|:-----|
| 技术博客 | 需要技术深度文章 | 研究文档+代码示例+引用 |
| 行业分析 | 市场趋势/竞品分析 | 数据支撑+权威引用 |
| 白皮书 | 企业/产品白皮书 | 结构化论证+专业语调 |
| 研究报告 | 学术/商业研究 | 文献综述+方法论+结论 |
| 深度评测 | 产品/技术评测 | 实测数据+对比分析 |

## 工作流

### 1. 选题与研究

1. **明确主题与角度**:主题、目标读者、文章长度、发布平台
2. **关键词研究**:核心关键词+长尾关键词+语义相关词
3. **资料搜集**:搜索权威来源(官方文档/学术论文/行业报告)
4. **来源分层**:一手资料 > 二手资料 > 评论观点,标注可信度
5. **研究简报**:汇总关键发现、数据点、引用片段

### 2. 结构设计

1. **大纲生成**:引言 → 核心论点(2-4个) → 案例/数据 → 结论
2. **钩子设计**:首段用问题/数据/故事/反常识抓住注意力
3. **段落分配**:每段一个核心观点,200-400字
4. **确认大纲**:输出结构化大纲供用户确认

### 3. 分段撰写与反馈

1. **逐段撰写**:按大纲顺序撰写,每段聚焦一个观点
2. **引用嵌入**:在论述中嵌入权威引用,标注来源
3. **分段反馈**:每段写完提供改进建议(论据是否充分/逻辑是否连贯/语言是否精炼)
4. **过渡衔接**:段与段之间用过渡句衔接,保证行文流畅

### 4. 钩子与标题优化

1. **标题候选**:生成 5-8 个标题候选,涵盖不同风格
2. **首段钩子**:提供 3 个首段版本(数据型/故事型/问题型)
3. **结尾行动号召**:根据文章目的设计 CTA
4. **SEO 优化**:标题含核心关键词,正文关键词密度 1-2%

### 5. 全文校审

1. **事实核查**:核对所有数据和引用的准确性
2. **逻辑审查**:论证链是否完整,有无跳跃
3. **语言润色**:去除冗余,提升可读性
4. **格式规范**:标题层级/列表/引用块/代码块统一
5. **原创性检查**:确保内容原创,引用部分规范标注

## 质量标准

- **深度**:每个论点有数据或案例支撑,非空泛论述
- **可信**:所有数据可溯源,引用标注来源
- **可读**:Flesch 可读性评分 > 50,段落长度适中
- **结构**:清晰的层级,读者可扫描式阅读
- **原创**:非拼凑,有独立观点和分析

## 输出规范

- 文章正文:`output/{article-slug}/article.md`
- 研究简报:`output/{article-slug}/research.md`
- 引用列表:`output/{article-slug}/references.md`
- 大纲:`output/{article-slug}/outline.md`
- 标题候选:`output/{article-slug}/titles.md`

## 依赖说明

### 运行环境
- **Agent平台**: Claude Code / Cursor / Codex / Gemini CLI / Windsurf 等支持SKILL.md的任意Agent
- **操作系统**: Windows / macOS / Linux
- **运行时**: 需要Agent支持exec(命令行执行)能力

### 第三方依赖
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| LLM API | API | 必需 | 由Agent内置LLM提供写作 |
| 联网搜索 | 工具 | 可选 | WebSearch/WebFetch收集资料 |

### API Key 配置
- **LLM_API_KEY**: 必需(通常由Agent内置) - 研究文章撰写
- 配置方式: 在Agent的环境变量中设置

### 纯Markdown使用说明
本Skill需要联网搜索能力收集研究资料。核心写作由Agent LLM完成。

只需将SKILL.md文件放入Agent的skills目录即可直接使用。
如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。

### 可用性分类
- **分类**: MD+EXEC
- **说明**: 纯Markdown,但需要exec能力(命令行执行),用于文件读写和命令调用
