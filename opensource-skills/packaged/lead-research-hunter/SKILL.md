# 销售线索猎手

分析你的产品定位,主动搜索匹配的目标公司,识别高质量线索,并提供可执行的外联策略。让销售从"找线索"转向"谈合作"。

## 使用场景

| 场景 | 触发条件 | 说明 |
|:-----|:---------|:-----|
| 线索挖掘 | 需要新客户 | 主动搜索匹配目标公司 |
| 客户分级 | 线索太多需筛选 | 评级排序,聚焦高质量 |
| 外联准备 | 准备联系客户 | 决策人+痛点+话术 |
| 竞品客户转化 | 抢竞品客户 | 分析竞品客户痛点 |
| 市场拓展 | 进入新行业 | 行业线索批量发现 |

## 工作流

### 1. 产品定位分析

1. **价值主张提炼**:
   - 解决什么问题(痛点)
   - 给谁解决(ICP - 理想客户画像)
   - 怎么解决(差异化)
   - 价值量化(省X时间/赚Y钱)
2. **ICP 定义**:
   - 行业/垂直领域
   - 公司规模(人数/营收)
   - 地域
   - 技术栈/工具栈
   - 业务模式(B2B/B2C/电商/SaaS)
3. **触发事件识别**:
   - 融资/扩张(需要扩容)
   - 招聘特定岗位(暗示需求)
   - 技术栈变更(迁移机会)
   - 负面新闻(痛点加剧)

### 2. 目标公司搜索

1. **多渠道搜索**:
   - LinkedIn(公司/人员)
   - Crunchbase(融资/规模)
   - 行业目录/协会
   - 招聘网站(岗位信号)
   - 新闻/公关(触发事件)
2. **筛选条件**:
   - 匹配 ICP(行业/规模/地域)
   - 有触发事件(近期变化)
   - 有预算信号(融资/招聘/采购)
3. **去重与合并**:同一公司多来源信息合并

### 3. 线索评级

| 评级 | 标准 | 处理方式 |
|:-----|:-----|:---------|
| A(热线索) | 完全匹配ICP+有触发事件+有预算 | 立即外联,高优先级 |
| B(温线索) | 匹配ICP+部分信号 | 纳入外联序列,中等优先 |
| C(冷线索) | 部分匹配ICP | 培育序列,长期跟进 |
| D(不合格) | 不匹配ICP | 不外联,存档 |

### 4. 决策人识别

1. **关键角色**:
   - 决策者(CEO/CTO/VP)
   - 影响者(部门负责人)
   - 使用者(一线员工)
   - 把关人(助理/采购)
2. **联系信息**:LinkedIn/邮箱/电话(合规获取)
3. **背景调研**:近期动态/发文/演讲(找外联切入点)

### 5. 外联策略

1. **痛点匹配**:
   - 基于公司情况推断可能痛点
   - 将产品价值与痛点对应
2. **外联话术**:
   - 主题行:引发好奇(非推销)
   - 开场:提及触发事件(证明做过功课)
   - 价值:一个具体利益点(非功能列表)
   - CTA:低摩擦下一步(非"约个会")
3. **多触点序列**:
   - 第1封:价值引入
   - 第2封(3天后):案例/证据
   - 第3封(7天后):不同角度
   - 第4封(14天后):breakup(最后一封)
4. **渠道组合**:邮件+LinkedIn+电话

## 合规原则

1. **数据合规**:仅使用公开信息,遵守 GDPR/个人信息保护法
2. **退订尊重**:提供退订选项,及时处理
3. **频率控制**:不轰炸,合理间隔
4. **诚实透明**:不冒充、不欺骗

## 输出规范

- 线索清单:`output/{campaign}/leads.csv`(含评级)
- ICP 文档:`output/{campaign}/icp.md`
- 外联话术:`output/{campaign}/outreach-scripts.md`
- 跟进序列:`output/{campaign}/sequence.md`
- 公司情报:`output/{campaign}/company-intel/`

## 依赖说明

### 运行环境
- **Agent平台**: Claude Code / Cursor / Codex / Gemini CLI / Windsurf 等支持SKILL.md的任意Agent
- **操作系统**: Windows / macOS / Linux
- **运行时**: 需要Agent支持exec(命令行执行)能力

### 第三方依赖
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| LLM API | API | 必需 | 由Agent内置LLM提供分析和判断 |
| 联网搜索 | 工具 | 可选 | WebSearch/WebFetch工具收集线索 |

### API Key 配置
- **LLM_API_KEY**: 必需(通常由Agent内置) - 线索评估和报告生成
- 配置方式: 在Agent的环境变量中设置

如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。
