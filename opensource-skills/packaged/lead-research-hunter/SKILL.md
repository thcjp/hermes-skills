# 销售线索猎手

不只是找名单,更会筛选与打分。用 3D 线索打分模型自动筛选高价值潜客,让销售只打能赢的电话。

## 适用场景

| 场景 | 输入 | 输出 |
|:-----|:-----|:-----|
| 线索挖掘 | ICP定义+目标市场+数量 | 高价值潜客清单(含公司情报+打分) |
| ICP定义 | 产品定位+已有客户数据 | 理想客户画像(行业/规模/地域/特征) |
| 决策人识别 | 目标公司清单 | 决策人清单(姓名/职位/联系方式/触达建议) |
| 线索打分 | 潜客清单+打分维度 | 排序清单(3D打分:A/B/C级+优先级) |
| 触达话术 | 决策人信息+公司情报 | 个性化触达话术(邮件/电话/LinkedIn) |
| ABM账户营销 | 目标账户清单 | 账户情报包+多触点话术+内容建议 |

### 不适用于
- 实时CRM数据同步与更新(本Skill输出清单,需手动导入CRM)
- 销售执行与跟进(本Skill仅提供线索与话术,不执行触达)
- 客户成功与续约管理(请使用CS工具)
- 销售预测与pipeline管理(请使用CRM系统)
- 电话营销自动外呼(请使用外呼系统)
- 法人/股东深度背景调查(请使用专业尽调服务)

## 核心能力

1. **ICP理想客户画像定义**:行业(SIC/NAICS代码)+规模(营收/员工)+地域+技术栈+痛点匹配,从已有客户数据反向推导ICP
2. **3D线索打分模型**:Fit(匹配度:行业/规模/地域,40%) + Intent(意向度:融资/招聘/技术变更,30%) + Engagement(互动度:网站访问/内容下载,30%),自动分级A/B/C
3. **公司情报收集**:公司基本信息+融资历史+招聘动态+技术栈+新闻动态+竞品关系,整合企查查/天眼查(国内)+Crunchbase(海外)
4. **决策人识别**:基于采购决策链(发起者/影响者/决策者/批准者),识别关键联系人,脉脉/LinkedIn数据补充
5. **个性化触达话术**:基于公司情报+决策人痛点,生成邮件/电话/LinkedIn多渠道触达话术

## 使用流程

### Step 1: ICP 理想客户画像定义
1. 产品定位分析:解决的问题、目标客户类型、价值主张
2. 已有客户分析:提取最佳客户共性特征(行业/规模/地域/技术栈)
3. ICP定义输出:
   - 行业:SIC/NAICS 代码范围
   - 规模:营收/员工数区间
   - 地域:目标国家/城市
   - 技术栈:使用的技术/工具
   - 痛点:匹配产品解决的痛点
4. 否定条件:明确排除的客户类型

### Step 2: 线索挖掘
1. **数据源选择**:
   - 海外:LinkedIn Sales Navigator/Crunchbase/Apollo.io
   - 国内:企查查/天眼查/脉脉/IT桔子
2. **筛选条件**:基于 ICP 定义
3. **数据收集**:
   - 公司信息:名称/规模/行业/地域/官网
   - 融资信息:轮次/金额/投资方
   - 招聘信息:在招岗位/数量/部门
   - 技术栈:使用的技术/工具
4. **去重与清洗**:跨数据源去重,补全缺失字段

### Step 3: 线索打分
1. **Fit 匹配度(40%)**:
   - 行业匹配:完全匹配 40 / 相关 30 / 不匹配 0
   - 规模匹配:区间内 30 / 接近 20 / 外 0
   - 地域匹配:目标内 30 / 其他 0
2. **Intent 意向度(30%)**:
   - 近期融资:+15
   - 招聘相关岗位:+10
   - 技术栈变更:+5
3. **Engagement 互动度(30%)**:
   - 网站访问:+10
   - 内容下载:+10
   - 邮件打开:+5
   - 会议参加:+5
4. **分级**:
   - A 级(80+):优先触达
   - B 级(60-79):培育后触达
   - C 级(40-59):长期培育
   - D 级(<40):暂不触达

### Step 4: 决策人识别
1. **采购决策链**:
   - 发起者:发现需求的人(如:产品经理)
   - 影响者:评估方案的人(如:技术负责人)
   - 决策者:做购买决定的人(如:VP/CTO)
   - 批准者:签字批准的人(如:CFO/CEO)
2. **联系人识别**:
   - LinkedIn/脉脉:职位+公司+简介
   - 官网团队页:高管信息
   - 招聘信息:在招岗位暗示组织架构

### Step 5: 触达话术生成
1. **邮件话术**:主题行(个性化)+ 正文(痛点+价值+CTA)
2. **电话话术**:开场(30秒)+ 痛点确认+ 价值阐述+ 下一步
3. **LinkedIn 话术**:连接请求(简短)+ 跟进消息(价值导向)

## 线索打分模型详解

### 1. Fit(匹配度) - 40分

| 维度 | 分值 | 评估标准 |
|:-----|:-----|:---------|
| 行业匹配 | 0-15 | SIC代码完全匹配15/相关10/不匹配0 |
| 规模匹配 | 0-15 | 营收/员工在ICP区间内15/接近10/外0 |
| 地域匹配 | 0-10 | 目标地域内10/其他0 |

### 2. Intent(意向度) - 30分

| 信号 | 分值 | 数据源 |
|:-----|:-----|:-------|
| 近期融资(6个月内) | +15 | 企查查/Crunchbase |
| 招聘相关岗位 | +10 | 招聘网站/官网 |
| 技术栈变更 | +5 | BuiltWith/Wappalyzer |
| 高管变动 | +5 | 新闻/LinkedIn |
| 新产品发布 | +5 | 官网/新闻 |

### 3. Engagement(互动度) - 30分

| 行为 | 分值 | 数据源 |
|:-----|:-----|:-------|
| 网站访问(30天内) | +10 | 分析工具 |
| 内容下载 | +10 | 营销自动化 |
| 邮件打开(7天内) | +5 | 邮件系统 |
| 会议参加 | +5 | CRM |
| 询价/试用申请 | +10 | 官网表单 |

## 错误处理
- 边界输入处理: 空输入返回提示信息, 超长输入自动截断
- 重试机制: 失败时自动重试, 最多3次

| 错误场景 | 原因 | 处理方式 |
|:---------|:-----|:---------|
| 数据源访问受限 | 海外平台网络限制 | 改用企查查/天眼查/脉脉,或手动输入公司信息 |
| 联系方式缺失 | 决策人隐私设置/无公开信息 | 通过官网/招聘信息推断,使用邮件挖掘工具,或通过共同联系人 |
| 线索数量不足 | ICP过于狭窄/数据源有限 | 放宽ICP条件,增加数据源,扩展相邻行业 |
| 打分不准 | 缺少Engagement数据 | 降级为Fit+Intent 2D打分,标注"待补充互动数据" |
| 公司情报过时 | 数据更新延迟 | 标注数据采集日期,建议季度更新,补充最新新闻 |
| 决策人识别错误 | 组织架构变动/信息滞后 | 标注"待验证",通过LinkedIn/脉脉二次确认 |
| 触达无响应 | 话术不匹配/时机不对 | A/B测试不同话术,调整触达时机,多渠道组合触达 |
| GDPR/个保法合规风险 | 未经同意收集个人信息 | 仅使用公开信息,遵循数据最小化原则,提供退订选项 |
