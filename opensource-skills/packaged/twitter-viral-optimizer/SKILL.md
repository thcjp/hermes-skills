# 社媒爆款优化器

基于 Twitter(X)/微博/小红书 公开推荐算法的洞察,分析并优化社交媒体内容,最大化触达与互动率。支持海外(Twitter)与国内(微博/小红书/抖音)双场景内容优化。

## 核心能力

1. **算法因子评分**:基于各平台公开算法因子(互动权重/内容信号/用户亲和度/新鲜度)打分(0-100)
2. **内容重写**:钩子优化、长度控制、互动诱导、标签策略、媒体建议,生成 3-5 个 A/B 测试版本
3. **发布时机优化**:基于目标受众时区与活跃时段,推荐最佳发布时间窗口
4. **线程/笔记结构优化**:首推钩子、节奏控制、高潮前置、结尾行动号召
5. **多平台适配**:Twitter 海外 + 微博/小红书/抖音国内,按平台特性差异化优化

## 适用场景

| 场景 | 输入 | 输出 |
|:-----|:-----|:-----|
| 推文/微博重写 | 草稿内容 + 目标受众 + 平台 | 3-5 个重写版本 + 评分卡 + A/B 测试建议 |
| 发布时机规划 | 目标受众时区 + 平台 | 一周发布日历 + 最佳时间窗口 |
| 话题标签策略 | 内容主题 + 目标流量 | 高流量低竞争标签组合(1-3 个) |
| 线程/笔记优化 | 多条内容草稿 | 优化后的线程结构 + 钩子 + CTA |
| 账号诊断 | 近期 10-20 条内容 + 互动数据 | 互动低迷原因分析 + 改进建议 |

**不适用于**:
- 黑帽刷量与买粉买赞(仅支持合规白帽内容优化)
- 视频内容制作(专注于文字内容优化,视频需专用工具)
- 直播带货话术(使用电商专用培训)
- 私域社群运营(微信群/Discord,使用专用工具)
- 政治与争议性内容(避免合规风险)
- 虚假信息与误导性内容(违反平台政策)

## 使用流程

### Step 1: 内容分析
1. **接收内容**:文本、图片(可选)、发布时间、目标受众
2. **确定平台**:Twitter(X)海外 / 微博 / 小红书 / 抖音图文
3. **算法因子评分**:基于平台算法公开因子打分
   - 互动权重(回复/转发/点赞/书签的相对权重)
   - 内容负面信号(过多标签/外链/敏感词)
   - 用户亲和度(作者与潜在读者的互动历史)
   - 新鲜度(发布后的时间衰减)
4. **输出评分卡**:当前内容的算法友好度评分(0-100)

### Step 2: 内容重写
1. **钩子优化**:首行用问题/数据/反常识/故事抓住注意力
2. **长度控制**:Twitter 短推(280 字符内)优先,长内容用线程;微博 140 字+长微博;小红书 300-800 字
3. **互动诱导**:结尾设计提问/投票/引发回复的钩子
4. **标签策略**:Twitter 1-2 个高相关标签;微博 2-3 个话题;小红书 5-10 个标签
5. **媒体建议**:建议配图/GIF/视频,多媒体内容互动率更高
6. **生成 3-5 个重写版本**:不同风格供 A/B 测试

### Step 3: 发布时机
1. **受众活跃分析**:基于目标受众时区和生活习惯
2. **最佳时间窗口**:
   - Twitter: 工作日 9-11点、13-15点、18-20点(受众时区)
   - 微博: 工作日 12-13点、18-22点;周末 10-12点、20-22点
   - 小红书: 工作日 7-9点、12-13点、20-23点;周末全天
3. **避开竞争时段**:避免与大V同时发布被淹没
4. **发布日历**:规划一周发布节奏

### Step 4: 线程/笔记结构优化
1. **首推钩子**:线程第一条必须强钩子,决定打开率
2. **节奏控制**:每条推文/笔记段落一个完整观点,避免断句
3. **高潮前置**:核心价值在前 3 条,不要埋太深
4. **结尾行动号召**:最后一条引导关注/转发/访问链接

### Step 5: 互动预测与复盘
1. **互动预测**:基于历史数据和算法因子预测互动区间
2. **发布后追踪**:T+1h/T+24h/T+7d 互动数据对比
3. **偏差分析**:预测 vs 实际,校准预测模型
4. **改进建议**:基于复盘数据优化下一篇

## 错误处理

| 错误场景 | 原因 | 处理方式 |
|:---------|:-----|:---------|
| Twitter 国内无法访问 | 网络限制 | 切换至微博/小红书平台优化,或使用代理 |
| 算法更新导致评分失准 | 平台调整算法 | 标注算法版本日期,定期更新权重 |
| 互动数据获取失败 | API 限制或权限不足 | 引导用户手动提供近期互动数据 |
| 内容触发风控 | 敏感词/外链/频繁发布 | 检测敏感词,建议修改后重试 |
| 标签流量数据缺失 | 第三方工具不可用 | 基于经验值推荐,标注数据来源 |
| 多平台内容差异化不足 | 一稿多投 | 按平台特性差异化(长度/标签/语气) |
| A/B 测试样本不足 | 内容曝光过低 | 建议投放付费推广积累数据 |
| 跨时区发布时间混乱 | 用户与受众时区不同 | 统一转换为受众时区,标注 UTC 偏移 |

### Q4: 一稿多平台如何差异化?
A: 差异化要点:
- **Twitter**:280字符,1-2标签,英文/中文,开发者语气
- **微博**:140字+长微博,2-3话题#,口语化
- **小红书**:300-800字,5-10标签,emoji + 个人体验
- **知乎**:1500字+,专业深度,无标签
- 核心:同一观点,不同表达,适配平台调性

## 已知限制

- Twitter(X)在国内无法直接访问,需使用代理或切换至国内平台
- 微博/小红书算法未公开,评分为经验推测(置信度中)
- 无法获取实时互动数据(需用户提供或使用第三方工具)
- 不涉及视频内容制作与优化(专注于图文/线程)
- 不覆盖付费推广与广告投放优化(SEM/信息流广告)
- 算法会更新,评分模型需定期校准(建议每季度)
- 不涉及账号涨粉策略(专注于单条内容优化)

## 安全声明

- API Key(Twitter/微博)仅通过环境变量读取,绝不硬编码
- 不收集用户密码或登录凭证
- 互动数据分析时自动脱敏用户 ID 与昵称
- 输出的内容建议不包含任何敏感词或违规内容
- 不支持刷量、买粉、虚假互动等黑帽操作
- 不分析政治、色情、暴力等违规内容
