# 文档处理大师

全面处理 Word 文档。从创建到编辑,从格式到修订,覆盖正式文档全生命周期。

## 使用场景

| 场景 | 触发条件 | 说明 |
|:-----|:---------|:-----|
| 文档创建 | 从零生成文档 | 大纲 → 格式化文档 |
| 文档编辑 | 修改现有文档 | 读取/编辑/输出 |
| 协作审阅 | 多人修订批注 | 修订模式+批注 |
| 邮件合并 | 批量个性化文档 | 模板+数据源 |
| 格式转换 | 格式统一/规范化 | 样式/目录/页眉页脚 |

## 工作流

### 1. 文档规划

1. **明确文档类型**:合同/报告/方案/手册/论文
2. **确定结构**:标题层级、章节划分、目录结构
3. **样式规范**:字体/字号/行距/段距/对齐
4. **页面设置**:纸张大小/页边距/页眉页脚

### 2. 内容生成与编辑

1. **标题层级**:
   - H1:章(标题1,16pt加粗)
   - H2:节(标题2,14pt加粗)
   - H3:小节(标题3,12pt加粗)
   - 正文(11pt,1.5倍行距)
2. **段落格式**:
   - 首行缩进/段前段后距
   - 对齐方式(左/居中/两端)
   - 列表(项目符号/编号)
3. **特殊元素**:
   - 表格(边框/底纹/合并单元格)
   - 图片(插入/环绕/题注)
   - 代码块(等宽字体/底纹)
   - 引用块(缩进/斜体)

### 3. 修订与批注

1. **修订模式**:
   - 开启修订跟踪
   - 插入/删除/格式更改均被记录
   - 作者标识+时间戳
2. **批注**:
   - 选中文字添加批注
   - 批注者+批注内容
   - 回复批注(线程化)
3. **接受/拒绝修订**:逐条或批量处理

### 4. 高级功能

1. **目录生成**:
   - 基于标题样式自动生成
   - 多级目录(1-3级)
   - 页码引用+超链接
   - 更新目录(内容变更后)
2. **邮件合并**:
   - 主文档(含占位符)
   - 数据源(Excel/CSV)
   - 合并生成个性化文档(信件/证书/邀请函)
3. **交叉引用**:图表编号/章节引用/页码引用
4. **书签与超链接**:内部跳转+外部链接

### 5. 页面与输出

1. **页眉页脚**:
   - 页码(居中/外侧)
   - 文档标题/章节名
   - 不同节不同页眉
2. **分节符**:不同页面方向/页边距
3. **封面页**:独立设计,无页码
4. **输出**:.docx(可编辑)/.pdf(分享)

## 输出规范

- Word 文件:`output/{document-name}/document.docx`
- PDF 导出:`output/{document-name}/export.pdf`
- 大纲:`output/{document-name}/outline.md`
- 修订记录:`output/{document-name}/revisions.md`

## 依赖说明

### 运行环境
- **Agent平台**: Claude Code / Cursor / Codex / Gemini CLI / Windsurf 等支持SKILL.md的任意Agent
- **操作系统**: Windows / macOS / Linux
- **运行时**: 需要Agent支持exec(命令行执行)能力

### 第三方依赖
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| Python 3.8+ | 运行时 | 可选 | python-docx库(pip install python-docx) |
| LLM API | API | 可选 | 由Agent内置LLM提供内容生成 |

### API Key 配置
- 本Skill无需额外API Key配置

### 纯Markdown使用说明
本Skill为文档处理方法论指导。实际生成docx文件需要python-docx库,Agent通过exec调用。

只需将SKILL.md文件放入Agent的skills目录即可直接使用。
如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。

### 可用性分类
- **分类**: MD+EXEC
- **说明**: 纯Markdown,但需要exec能力(命令行执行),用于文件读写和命令调用
