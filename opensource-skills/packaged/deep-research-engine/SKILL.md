# 深度研究引擎

执行自主多步研究。从搜索到验证,从综合到报告,全流程自动化。坚持来源分层与交叉验证,确保研究质量。

## 适用场景

| 场景 | 输入 | 输出 |
|---|---|---|
| 市场分析 | 目标市场+品类+地域 | 市场规模/趋势/竞争格局研究报告 |
| 竞品调研 | 竞品清单+关注维度 | 产品/定价/策略/差异分析报告 |
| 文献综述 | 研究主题+时间范围 | 文献搜索+综述撰写+引用清单 |
| 投资尽调 | 标的公司+行业+风险点 | 全面风险评估报告+数据支撑 |
| 技术选型 | 候选方案+评估维度 | 对比/评估/推荐报告+决策矩阵 |
| 行业趋势 | 行业+时间范围+地域 | 趋势分析+驱动因素+预测报告 |

### 不适用于
- 实时新闻追踪(本Skill为深度研究,非实时监控)
- 内部数据库查询(请使用数据分析工具)
- 问卷调查与用户访谈执行(本Skill基于公开信息)
- 财务模型构建与估值(本Skill提供数据,不建模)
- 法律意见与合规判断(请咨询律师)
- 学术原创研究(本Skill为综合研究,非原创实验)

## 核心能力

1. **多步搜索策略**:第一轮广度搜索(搜索引擎/学术/行业/新闻)→第二轮深度搜索(一手资料/专家观点)→第三轮交叉验证(多来源验证/冲突标注/缺口识别)
2. **来源分层与评估**:一手(官方文档/财报/专利,高可信)→二手(行业报告/权威媒体,中高)→评论(博客/社交媒体,中)→未验证(单一来源,低),按层级处理
3. **信息综合**:提取关键数据点、按主题归类、交叉验证、冲突处理(标注各方观点)、缺口识别
4. **报告生成**:执行摘要(1页)+详细分析(分章)+数据支撑(图表表格)+结论建议+来源清单(分层)+局限性声明
5. **研究质量标准**:可溯源(每个数据点可追溯)、多源验证(关键数据至少2个独立来源)、时效性(优先近2年)、客观性(多方观点)、透明度(标注不确定性)

## 使用流程

### Step 1: 研究规划
1. 明确研究问题:核心问题 + 子问题分解
2. 确定范围:时间范围/地域范围/深度要求
3. 识别信息源:权威来源清单(见依赖说明中外对照)
4. 制定研究计划:搜索策略 + 步骤分解

### Step 2: 多步搜索
1. **第一轮:广度搜索**
   - 搜索引擎:Google/Bing/百度
   - 学术:Google Scholar/arXiv/知网/万方
   - 行业:行业报告/白皮书/艾瑞/IDC
   - 新闻:近期相关报道
2. **第二轮:深度搜索**
   - 基于第一轮发现,深入特定方向
   - 查找一手资料(官方文档/财报/专利)
   - 查找专家观点(论文/博客/访谈)
3. **第三轮:交叉验证**
   - 多来源验证关键数据
   - 标记有争议的信息
   - 补充缺失信息

### Step 3: 来源分层与评估
1. 一手来源(高可信):官方文档/财报/专利/原始数据 → 优先引用
2. 二手来源(中高):行业报告/权威媒体/学术论文 → 交叉验证后引用
3. 评论来源(中):博客/评论/社交媒体 → 仅作参考,标注
4. 未验证来源(低):单一来源/匿名信息 → 不引用或明确标注风险

### Step 4: 信息综合
1. 信息提取:从每个来源提取关键数据点
2. 分类整理:按主题/维度归类
3. 交叉验证:多来源数据一致性检查
4. 冲突处理:标注冲突,说明各方观点
5. 缺口识别:标注信息不足的领域

### Step 5: 报告生成
1. 执行摘要:1页核心发现(给决策者)
2. 详细分析:按主题分章深度分析
3. 数据支撑:图表+表格+引用
4. 结论与建议:基于证据的结论与行动建议
5. 来源清单:完整引用,按层级标注
6. 局限性:研究的局限与不确定性

## 研究质量标准

1. **可溯源**:每个数据点可追溯到来源
2. **多源验证**:关键数据至少 2 个独立来源
3. **时效性**:优先近 2 年数据,标注旧数据
4. **客观性**:呈现多方观点,不预设立场
5. **透明度**:标注不确定性,不假装全知

## 错误处理

| 错误场景 | 原因 | 处理方式 |
|----|:--:|---:|
| 搜索结果不足 | 关键词过于狭窄/信息源有限 | 扩展关键词,使用同义词,增加信息源(见中外对照) |
| 信息冲突 | 不同来源数据不一致 | 标注冲突,呈现各方观点,注明来源层级,优先一手来源 |
| 信息过时 | 数据陈旧,最新数据未公开 | 标注数据年份,寻找最新预测,建议定期更新 |
| 无法访问海外源 | 网络限制/订阅墙 | 使用国内替代(知网/万方/艾瑞/企查查),手动输入资料 |
| 来源可信度存疑 | 单一来源/匿名信息/营销内容 | 降级为"未验证",不引用或明确标注风险,寻找第二来源验证 |
| 研究范围过广 | 主题过大,无法深入 | 分解为子问题,优先解决核心问题,标注次要问题为"后续研究" |
| 数据缺失 | 关键数据未公开/需付费 | 标注"数据不足",提供估算范围,建议付费数据源 |
| LLM分析偏差 | 模型预设立场/训练数据偏差 | 多角度验证,人工复核关键结论,标注"AI分析待验证" |

## 常见问题

### Q1: 如何确保研究质量?如何避免信息茧房?
A: (1)来源分层:强制使用一手来源(官方/财报),二手来源交叉验证,评论来源仅参考。(2)多源验证:关键数据至少2个独立来源,冲突数据必须标注。(3)多角度搜索:主动搜索对立观点,避免确认偏差。(4)时效性:优先近2年数据,标注旧数据。(5)透明度:明确标注不确定性,不假装全知。(6)人工复核:关键结论需人工验证,不盲信LLM分析。

### Q2: 没有海外信息源访问权限怎么办?
A: 国内替代方案已内置对照表:(1)学术:知网/万方/维普替代Google Scholar(2)搜索:百度/搜狗替代Google(3)行业报告:艾瑞/易观/IDC中国替代McKinsey(4)企业信息:企查查/天眼查/IT桔子替代Crunchbase(5)金融数据:Wind/同花顺替代Bloomberg(6)财报:巨潮资讯网/交易所替代SEC EDGAR。对于必须使用海外源的场景,可让用户提供资料手动输入。

### Q3: 研究报告的结论可信度如何判断?
A: 本Skill对每个结论标注置信度:(1)高置信:3+独立一手来源一致,数据时效<1年。(2)中置信:2个来源一致,或1个一手+1个二手,数据时效<2年。(3)低置信:单一来源,或来源为评论级,数据时效>2年。报告中"局限性"章节明确标注不确定性与数据缺口,决策者应结合自身判断。

### Q4: 深度研究与快速搜索有什么区别?
A: (1)搜索深度:快速搜索1轮(仅搜索引擎),深度研究3轮(广度→深度→验证)。(2)来源质量:快速搜索不区分来源层级,深度研究强制分层评估。(3)验证:快速搜索不交叉验证,深度研究关键数据至少2个独立来源。(4)报告:快速搜索返回摘要,深度研究生成结构化报告(摘要+分析+数据+来源+局限)。(5)耗时:快速搜索<5分钟,深度研究30分钟-2小时。建议:初步了解用快速搜索,决策支撑用深度研究。

## 已知限制

- 研究质量依赖公开信息,无法获取内部数据/未公开数据
- 海外信息源(Google Scholar/arXiv等)在国内访问可能受限
- 付费数据库(Wind/知网等)需用户自行订阅,本Skill不提供凭证
- LLM分析可能存在训练数据偏差,关键结论需人工复核
- 实时性有限,最新事件(近1周)可能覆盖不全
- 不执行原创研究(实验/调研/访谈),仅综合已有信息
- 学术原创性要求高的研究需人工补充

## 安全与合规

- 本Skill不存储任何API Key,LLM由Agent内置
- 搜索工具(WebSearch/WebFetch)由Agent平台提供,无需额外凭证
- 付费数据库凭证由用户自行配置,不进入Skill上下文
- 研究数据仅用于本次分析,不持久化原始来源内容
- 引用遵循版权法,标注来源,不大量复制原文
- 涉及个人信息的来源(如社交媒体)仅作参考,不引用个人信息
- 投资尽调报告不构成投资建议,需专业机构二次验证
- 研究日志记录搜索过程,便于追溯与复现
