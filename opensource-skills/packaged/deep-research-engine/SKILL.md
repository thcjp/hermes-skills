# 深度研究引擎

执行自主多步研究。从搜索到验证,从综合到报告,全流程自动化。坚持来源分层与交叉验证,确保研究质量。

## 使用场景

| 场景 | 触发条件 | 说明 |
|:-----|:---------|:-----|
| 市场分析 | 进入新市场前 | 市场规模/趋势/竞争格局 |
| 竞品调研 | 了解竞争对手 | 产品/定价/策略/差异 |
| 文献综述 | 学术/技术研究 | 文献搜索+综述撰写 |
| 投资尽调 | 投资决策前 | 全面风险评估 |
| 技术选型 | 选择技术方案 | 对比/评估/推荐 |

## 工作流

### 1. 研究规划

1. **明确研究问题**:核心问题 + 子问题分解
2. **确定范围**:时间范围/地域范围/深度要求
3. **识别信息源**:权威来源清单
4. **制定研究计划**:搜索策略 + 步骤分解

### 2. 多步搜索

1. **第一轮:广度搜索**
   - 搜索引擎:Google/Bing
   - 学术:Google Scholar/arXiv/PubMed
   - 行业:行业报告/白皮书
   - 新闻:近期相关报道
2. **第二轮:深度搜索**
   - 基于第一轮发现,深入特定方向
   - 查找一手资料(官方文档/财报/专利)
   - 查找专家观点(论文/博客/访谈)
3. **第三轮:交叉验证**
   - 多来源验证关键数据
   - 标记有争议的信息
   - 补充缺失信息

### 3. 来源分层与评估

| 层级 | 来源类型 | 可信度 | 处理方式 |
|:-----|:---------|:-------|:---------|
| 一手 | 官方文档/财报/专利/原始数据 | 高 | 优先引用 |
| 二手 | 行业报告/权威媒体/学术论文 | 中高 | 交叉验证后引用 |
| 评论 | 博客/评论/社交媒体 | 中 | 仅作参考,标注 |
| 未验证 | 单一来源/匿名信息 | 低 | 不引用或明确标注风险 |

### 4. 信息综合

1. **信息提取**:从每个来源提取关键数据点
2. **分类整理**:按主题/维度归类
3. **交叉验证**:多来源数据一致性检查
4. **冲突处理**:标注冲突,说明各方观点
5. **缺口识别**:标注信息不足的领域

### 5. 报告生成

1. **执行摘要**:1页核心发现(给决策者)
2. **详细分析**:按主题分章深度分析
3. **数据支撑**:图表+表格+引用
4. **结论与建议**:基于证据的结论与行动建议
5. **来源清单**:完整引用,按层级标注
6. **局限性**:研究的局限与不确定性

## 研究质量标准

1. **可溯源**:每个数据点可追溯到来源
2. **多源验证**:关键数据至少 2 个独立来源
3. **时效性**:优先近 2 年数据,标注旧数据
4. **客观性**:呈现多方观点,不预设立场
5. **透明度**:标注不确定性,不假装全知

## 输出规范

- 研究报告:`output/{research-topic}/report.md`
- 执行摘要:`output/{research-topic}/executive-summary.md`
- 来源清单:`output/{research-topic}/sources.md`(分层标注)
- 数据集:`output/{research-topic}/data/`(CSV/JSON)
- 图表:`output/{research-topic}/charts/`(PNG)
- 研究日志:`output/{research-topic}/research-log.md`(搜索过程)

## 依赖说明

### 运行环境
- **Agent平台**: Claude Code / Cursor / Codex / Gemini CLI / Windsurf 等支持SKILL.md的任意Agent
- **操作系统**: Windows / macOS / Linux
- **运行时**: 需要Agent支持exec(命令行执行)能力

### 第三方依赖
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| LLM API | API | 必需 | 由Agent内置LLM提供深度分析 |
| 联网搜索 | 工具 | 可选 | WebSearch/WebFetch工具 |

### API Key 配置
- **LLM_API_KEY**: 必需(通常由Agent内置) - 研究分析和报告生成
- 配置方式: 在Agent的环境变量中设置

### 纯Markdown使用说明
本Skill需要Agent具备联网搜索能力以获取最新信息。核心分析由Agent LLM完成。

只需将SKILL.md文件放入Agent的skills目录即可直接使用。
如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。

### 可用性分类
- **分类**: MD+EXEC
- **说明**: 纯Markdown,但需要exec能力(命令行执行),用于文件读写和命令调用
