# 合规管理器

构建企业级合规操作系统,将合规从一次性审计转变为持续运营。从控制项管理到证据收集,从审计就绪到持续监控,全流程合规管理。

## 使用场景

| 场景 | 触发条件 | 说明 |
|:-----|:---------|:-----|
| ISO 27001 | 信息安全管理体系认证 | 风险评估/控制项/声明适用性/ISMS |
| SOC 2 | 服务组织控制报告 | 五大信任准则:安全/可用/处理完整/保密/隐私 |
| GDPR | 欧盟数据保护合规 | 数据映射/DPIA/数据主体权利/跨境传输 |
| HIPAA | 医疗数据隐私合规 | PHI保护/BAA/安全规则/隐私规则 |
| 审计准备 | 即将接受审计 | 证据收集/控制测试/差距分析/补救计划 |
| 持续合规 | 日常合规运营 | 控制监控/事件管理/变更管理/培训 |

## 工作流

### 1. 合规框架搭建

1. **范围界定**
   - 确定合规框架(ISO 27001/SOC 2/GDPR/HIPAA)
   - 界定系统范围(哪些服务/数据/流程)
   - 识别数据流(数据映射)
   - 确定利益相关者
2. **风险评估**
   - 资产识别与分类
   - 威胁与漏洞评估
   - 风险评分(可能性 x 影响)
   - 风险处置计划(接受/缓解/转移/规避)
3. **控制项设计**
   - 选择适用控制项(Annex A / Trust Criteria)
   - 定义控制项描述与实施方式
   - 分配控制项所有者
   - 设定控制项频率(持续/月度/季度/年度)

### 2. 控制项管理

1. **控制项目录**
   - 访问控制(认证/授权/密码/会话)
   - 加密(传输/存储/密钥管理)
   - 网络安全(防火墙/IDS/分段)
   - 运营安全(变更管理/补丁/备份)
   - 人员安全(背景调查/培训/终止)
   - 物理安全(门禁/监控/环境)
   - 供应商管理(尽职调查/合同/监控)
   - 事件管理(检测/响应/报告)
2. **控制项实施**
   - 技术控制:自动化实施(WAF/SIEM/加密)
   - 管理控制:政策与流程文档
   - 物理控制:设施安全措施
   - 纠正控制:漏洞修复/事件响应
3. **控制项监控**
   - 自动化监控(配置扫描/日志分析)
   - 定期测试(渗透测试/控制测试)
   - 偏差识别与补救
   - 控制有效性评估

### 3. 证据管理

1. **证据收集**
   - 自动收集:API/日志/配置导出
   - 手动收集:截图/文档/签核
   - 证据分类:设计证据/运行证据
   - 证据存储:集中化证据库
2. **证据维护**
   - 时间戳与版本控制
   - 证据链完整性
   - 过期证据识别
   - 定期证据更新

### 4. 审计就绪

1. **审计前准备**
   - 审计范围确认
   - 控制项差距分析
   - 证据完整性检查
   - 审计员沟通
2. **审计支持**
   - 证据快速检索
   - 控制项演示
   - 访谈准备
   - 发现项响应
3. **审计后跟进**
   - 发现项分类(观察项/发现项/严重发现)
   - 补救计划制定
   - 补救跟踪与验证
   - 报告归档

### 5. 持续合规运营

1. **变更管理**
   - 系统变更合规影响评估
   - 新服务合规审查
   - 控制项更新
2. **事件管理**
   - 安全事件分类与响应
   - 合规影响评估
   - 监管报告(GDPR 72小时)
   - 事件后改进
3. **培训与意识**
   - 年度安全培训
   - 钓鱼模拟
   - 政策签核
   - 合规文化推广

## 异常处理

| 异常场景 | 处理方式 |
|:---------|:---------|
| 控制项失败 | 立即评估风险,启动补救计划,记录例外 |
| 审计发现 | 分类严重程度,制定补救时间表,跟踪验证 |
| 法规变更 | 影响评估,控制项更新,培训通知 |
| 数据泄露 | 启动事件响应,72小时内监管报告,受影响方通知 |
| 供应商违规 | 合同审查,控制项重新评估,替换评估 |
| 证据缺失 | 补充收集,控制项重新测试,记录偏差 |

## 示例

### 输入:SOC 2 审计准备

```
用户请求:为SaaS公司准备SOC 2 Type II审计,列出需要准备的控制项和证据

输出:
- 信任准则:Security(必须)+ Availability + Confidentiality + Privacy
- 控制项清单(30+):访问控制/加密/变更管理/事件响应/备份...
- 每个控制项的:
  - 控制描述
  - 实施方式
  - 证据类型(政策/日志/截图/签核)
  - 收集频率
- 证据收集计划(12个月证据)
- 差距分析(当前 vs 要求)
- 补救优先级排序
```

### 输入:GDPR数据主体权利

```
用户请求:实现GDPR数据主体权利(访问/删除/更正/可携)的流程

输出:
- 数据映射:个人数据存储位置/用途/共享方
- 权利实现流程:
  - 身份验证流程
  - 数据检索(30天内)
  - 数据导出格式(JSON/CSV)
  - 数据删除(含备份处理)
  - 第三方数据处理通知
- DPIA模板
- 跨境传输机制(SCC/BCR)
```
