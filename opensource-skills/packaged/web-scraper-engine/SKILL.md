# 网页抓取引擎

基于 Firecrawl API,实现全流程网页数据采集。从搜索发现到结构化提取,从单页抓取到多步骤浏览器交互,无需维护爬虫基础设施。

## 使用场景

| 场景 | 触发条件 | 说明 |
|:-----|:---------|:-----|
| 竞品分析 | 竞品网站数据采集 | 批量抓取产品/价格/评论 |
| 价格监控 | 电商价格追踪 | 定时抓取商品页面 |
| 内容采集 | 文章/博客聚合 | 站点地图爬取+内容提取 |
| Lead Generation | 潜在客户信息 | 公司网站信息提取 |
| 市场调研 | 行业数据收集 | 搜索+批量抓取 |
| 结构化提取 | 非结构化页面转结构化 | 自动识别字段/JSON输出 |

## 工作流

### 1. 搜索发现

1. **关键词搜索**
   - `POST /v1/search`:输入关键词,返回相关网页
   - 参数:query/limit/scrapeOptions
   - 输出:URL列表+标题+摘要
2. **站点地图发现**
   - 获取目标站点所有URL
   - 过滤不需要的页面
   - 构建爬取队列

### 2. 单页抓取

1. **基础抓取**
   - `POST /v1/scrape`:抓取单个URL
   - 输出:Markdown/HTML/纯文本
   - 自动处理JS渲染
2. **结构化提取**
   - 使用 Schema 定义提取字段
   - 自动识别页面内容
   - 输出JSON格式
3. **截图**
   - 全页截图
   - 可视化验证

### 3. 批量爬取

1. **异步爬取**
   - `POST /v1/crawl`:提交URL列表
   - 返回job ID,异步处理
   - `GET /v1/crawl/{id}`:查询状态
2. **爬取策略**
   - 深度控制:仅首页/2层/全站
   - 路径过滤:include/exclude规则
   - 速率限制:并发数控制
3. **结果处理**
   - 统一格式输出
   - 去重
   - 导出CSV/JSON

### 4. 多步骤浏览器交互

1. **点击操作**
   - 点击按钮/链接
   - 等待页面加载
   - 处理弹窗
2. **表单填写**
   - 填充输入框
   - 选择下拉菜单
   - 提交表单
3. **翻页**
   - 点击"下一页"
   - 等待新数据加载
   - 收集每页数据
4. **登录状态**
   - 注入Cookie/Token
   - 处理认证流程
   - 会话保持

### 5. 数据处理

1. **清洗**
   - 移除HTML标签
   - 去除空白字符
   - 统一编码(UTF-8)
2. **结构化**
   - JSON Schema 映射
   - 字段类型转换
   - 数据验证
3. **存储**
   - 导出CSV/JSON/Parquet
   - 写入数据库
   - 上传S3/R2

## 依赖说明

| 依赖类型 | 要求 | 说明 |
|:---------|:-----|:-----|
| LLM | 任意支持 Agent Skills 的 LLM | Claude/GPT/Gemini 等 |
| API Key | Firecrawl API Key | 从 firecrawl.dev 获取(免费500页/月) |
| 运行环境 | Python 3.10+ 或 Node.js 18+ | SDK 运行环境 |
| SDK | firecrawl-sdk (pip/npm) | 官方 SDK |
| 可选 | 代理IP | 大规模爬取需要(避免IP封禁) |
| 可选 | 数据库 | 存储抓取结果(关系型数据库/MongoDB) |

## 异常处理

| 异常场景 | 处理方式 |
|:---------|:---------|
| 页面加载超时 | 设置超时(30s),重试3次,跳过失败页 |
| JS渲染失败 | 切换渲染引擎,增加等待时间 |
| 反爬虫检测 | 使用代理IP,降低速率,添加延迟 |
| 结构化提取失败 | 回退到Markdown,LLM辅助提取 |
| 登录状态过期 | 刷新Token/Cookie,重新认证 |
| 数据格式不一致 | 字段映射容错,默认值处理 |

## 示例

### 输入:竞品价格监控

```
用户请求:抓取3个竞品电商网站的产品价格,每周更新

输出:
1. 搜索:搜索产品页URL
2. 抓取:Firecrawl scrape 获取页面
3. 提取:Schema { product_name, price, currency, availability, url }
4. 批量:异步crawl所有产品页
5. 存储:写入关系型数据库,按日期记录
6. 对比:与上周价格对比,标记涨跌
7. 报告:生成价格变动报告
```

### 输入:多步骤表单提交

```
用户请求:自动填写某网站的查询表单,翻页采集结果

输出:
1. 加载页面
2. 填写表单(选择日期/输入关键词)
3. 点击"搜索"
4. 等待结果加载
5. 提取第一页结果
6. 点击"下一页"
7. 重复直到最后一页
8. 合并所有结果
9. 导出CSV
```
