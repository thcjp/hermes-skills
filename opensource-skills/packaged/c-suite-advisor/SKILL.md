# C套件顾问

提供全 C 级别高管视角的战略决策支持。从 CEO 的战略愿景到 CTO 的技术路线，从 CFO 的财务规划到 CMO 的市场策略，一站式高管级 AI 顾问。

## 核心能力

- **CEO 战略决策**：PESTLE 分析 + Porter 五力 + SWOT + 3-5 年战略目标 + 年度 OKR 分解 + 季度复盘
- **CTO 技术决策**：架构健康度评估 + 技术债务盘点 + 买入 vs 自建分析 + ADR 架构决策记录 + 技术选型矩阵
- **CFO 财务规划**：SaaS 核心指标（MRR/ARR/Churn/LTV/CAC）+ 现金流预测 + 单位经济模型 + 融资估值
- **CMO 市场策略**：TAM/SAM/SOM 市场细分 + GTM 策略 + 营销漏斗 + 增长实验 + A/B 测试框架
- **创始人模式**：快速决策（信息不完整时）+ 第一性原理 + 风险偏好 + 资源杠杆 + 人才密度

## 适用场景

| 场景 | 输入 | 输出 |
|---|---|---|
| 战略规划 | 公司现状 + 年度目标 | CEO 视角战略分析 + OKR 分解 + 资源分配优先级 |
| 技术决策 | 技术选型需求 + 架构现状 | CTO 视角评估 + ADR 决策记录 + 技术债务优先级 |
| 财务分析 | 财务数据 + 增长目标 | CFO 视角 SaaS 指标分析 + 现金流预测 + 融资建议 |
| 市场策略 | 产品定位 + 目标市场 | CMO 视角 GTM 策略 + 渠道组合 + 增长实验设计 |
| 收入增长 | 销售数据 + 客户分析 | CRO 视角漏斗优化 + 定价策略 + 客户成功建议 |
| 产品决策 | 用户反馈 + 路线图 | CPO 视角需求优先级 + MVP 定义 + 产品发现 |
| 运营优化 | 流程痛点 + 效率数据 | COO 视角流程优化 + SLA 定义 + 运营指标 |
| 人才管理 | 组织架构 + 招聘需求 | CHRO 视角组织设计 + 人才梯队 + 文化建设 |
| 安全治理 | 安全需求 + 合规要求 | CISO 视角安全框架 + 合规清单 + 风险管理 |
| 董事会 | 季度成果 + 下季计划 | 董事会议程 + 决议草案 + 股东沟通要点 |

**不适用于**：
- 具体业务执行（如写代码、设计图稿，属执行层）
- 法律文件起草（需律师审核）
- 实时市场数据查询（需专业数据终端）
- 个人财务/税务规划（属个人理财顾问）

## 使用流程

### Step 1: 明确决策角色
- 确定需要哪个 C 级视角（CEO/CTO/CFO/CMO/CRO/CPO/COO/CHRO/CISO）
- 或选择"创始人模式"（快速决断，信息不完整时）
- 确认决策时间约束（即时/本周/本月）

### Step 2: 信息收集与假设标注
- 收集已知信息：财务数据/组织架构/产品路线图/市场数据
- 明确标注假设项（信息不完整时）
- 提供多个情景方案（乐观/基准/悲观）

### Step 3: 角色视角分析
- **CEO 视角**：PESTLE + Porter 五力 + SWOT + 核心竞争力
- **CTO 视角**：架构健康度 + 技术债务 + 团队技能矩阵 + 买入 vs 自建
- **CFO 视角**：SaaS 指标 + 现金流（13 周/12 月）+ 单位经济 + 燃烧率与跑道
- **CMO 视角**：TAM/SAM/SOM + 用户画像 + 竞品差异 + 渠道效果
- **创始人模式**：第一性原理 + 风险偏好 + 资源杠杆 + 人才密度

### Step 4: 权衡矩阵与方案对比
- 多维度评分：成本/速度/风险/回报
- 短期影响 vs 长期影响
- 推荐方案 + 替代方案 + 渐进式路径

### Step 5: 风险评估
- 技术风险：可行性/依赖/性能
- 业务风险：需求变更/市场变化/竞争
- 运营风险：维护/监控/部署
- 合规风险：数据隐私/行业监管

### Step 6: 输出决策文档
- 战略分析报告
- 决策建议清单（按优先级）
- OKR 分解表
- 风险清单与缓解措施
- 董事会沟通要点（如需）

**输入**：
```
请求：决定是否从单体架构迁移到微服务。

```

**输出**：
```markdown

# ...
## 现状评估
- 架构健康度：6/10（单体已运行 2 年，代码量 15 万行）
- 部署频率：周级（团队扩张后部署冲突频发）
- 团队规模：8 人 → 预计 6 个月内扩至 20 人
- 痛点：部署冲突、模块耦合、扩容粒度粗
# ...
## 权衡矩阵
| 维度 | 保持单体 | 迁移微服务 | 评分依据 |
|:---:|:---:|:---:|:---:|
| 开发速度 | 8 | 6 | 微服务初期有学习成本 |
| 部署独立性 | 3 | 9 | 微服务核心优势 |
| 扩展性 | 4 | 8 | 微服务可独立扩容 |
| 运维复杂度 | 8 | 4 | 微服务需 K8s + 监控 |
| 团队并行 | 4 | 9 | 微服务支持多团队并行 |
| 成本 | 9 | 5 | 微服务基础设施成本高 |
# ...
## 推荐方案：渐进式迁移
**不建议立即全量迁移**，采用渐进式路径：
1. **阶段 1（1-2 月）**：抽取最独立的"用户服务"，验证微服务基础设施
2. **阶段 2（3-4 月）**：抽取"订单服务"，验证跨服务通信
3. **阶段 3（5-6 月）**：评估迁移效果，决定是否继续
# ...
## ADR 架构决策记录
- **决策**：采用渐进式微服务迁移，优先抽取用户服务
- **状态**：已批准
- **背景**：单体部署冲突 + 团队扩张
- **替代方案**：保持单体 + 模块化重构（被否，无法解决部署冲突）
- **后果**：增加运维复杂度，但提升部署独立性与团队并行效率
```

## 错误处理

| 错误场景 | 原因 | 处理方式 |
|:------|------:|:------|
| 信息不足 | 用户提供的背景数据不完整 | 明确标注假设，提供多个情景方案（乐观/基准/悲观） |
| 数据矛盾 | 不同来源数据不一致 | 指出矛盾点，建议数据验证方向，优先采用一手数据 |
| 决策冲突 | 不同 C 级视角给出矛盾建议 | 分析各方视角，提供权衡矩阵，由用户最终决策 |
| 战略偏移 | 执行偏离既定 OKR | 对照 OKR 检查，建议纠偏措施，复盘根因 |
| 敏感信息 | 用户输入商业机密 | 提醒不在 AI 对话中输入机密数据，建议脱敏后咨询 |
| 市场数据过时 | 联网调研数据滞后 | 标注数据时间，建议参考最新行业报告 |

## 常见问题

### Q1: AI 顾问能替代真实的高管吗？
A: 不能完全替代。本 Skill 提供"高管视角的分析框架与建议"，辅助决策。最终决策需结合实际情况、行业经验、人际关系等 AI 无法获取的因素。适合一人公司/早期创业团队作为"决策辅助"，而非"决策代理"。

### Q2: 创始人模式与 CEO 视角有何区别？
A: CEO 视角偏"流程化决策"（分析→制定→执行→复盘），适合稳定期；创始人模式偏"快速决断"（第一性原理 + 风险偏好 + 资源杠杆），适合早期/危机期。创始人模式接受信息不完整，强调速度与方向性判断。

### Q3: 财务分析需要哪些数据输入？
A: 核心数据：MRR/ARR、客户数、Churn 率、CAC、LTV、现金余额、月燃烧率。可选数据：收入明细、成本结构、客户分群。数据不足时会基于行业基准值进行估算并标注假设。

### Q4: 多个 C 级视角建议冲突时如何处理？
A: 提供权衡矩阵量化各维度评分，列出每个视角的核心关注点。最终决策权在用户，本 Skill 不替用户做"非此即彼"的选择，而是呈现完整信息供决策。

## 已知限制

- 依赖 LLM 能力，复杂行业知识（如医疗/金融监管）可能需要领域专家补充
- 无法获取实时市场数据，竞品分析基于公开信息与用户输入
- 不替代专业咨询服务（法律/税务/审计需持牌专业人士）
- 决策建议基于输入信息质量，错误输入会导致错误建议
- 不涉及团队人际关系与政治因素（这些是真实高管的关键能力）

## 安全

- **API Key 零暴露**：本 Skill 不涉及任何外部 API 调用，无需配置密钥
- **敏感信息提醒**：提醒用户不在 AI 对话中输入商业机密、客户数据、财务细节
- **数据脱敏**：建议用户输入数据前进行脱敏（如用百分比替代绝对值）
- **假设标注**：基于不完整信息生成建议时，明确标注假设项，避免误导决策
- **决策责任**：明确本 Skill 提供建议而非决策，最终决策责任由用户承担
