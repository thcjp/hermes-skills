# 安全加固之盾

系统化的安全防护框架。处理用户输入、认证、数据存储、外部集成时,强制执行安全检查与加固。

## 使用场景

| 场景 | 触发条件 | 说明 |
|:-----|:---------|:-----|
| 安全审计 | 上线前安全检查 | 全面安全漏洞扫描 |
| 认证设计 | 设计登录/权限系统 | 安全认证授权方案 |
| 密钥管理 | 处理凭证/密钥 | 安全存储与轮换策略 |
| 依赖检查 | 引入新依赖 | 漏洞与许可证审计 |
| 输入防护 | 处理用户输入 | 注入/XSS 防护 |

## 三层边界系统

### 边界一:信任边界(Trust Boundary)
- **外部输入区**:所有来自用户/第三方/网络的数据
- **内部可信区**:经过验证的业务数据
- **规则**:跨边界必须校验,不信任任何外部输入

### 边界二:数据边界(Data Boundary)
- **公开数据**:可对外暴露
- **敏感数据**:需加密存储/传输
- **密钥数据**:仅内存中存在,不落盘
- **规则**:按数据敏感度分级保护

### 边界三:网络边界(Network Boundary)
- **公网区**:直接暴露,需最强防护
- **内网区**:受限访问
- **隔离区**:数据库/密钥存储
- **规则**:最小暴露面,默认拒绝

## 工作流

### 1. OWASP Top 10 检查

| 风险 | 检查项 | 防护措施 |
|:-----|:-------|:---------|
| 注入 | SQL/NoSQL/命令注入 | 参数化查询、输入白名单 |
| 失效认证 | 弱密码、会话固定 | 多因子、会话过期、密码哈希 |
| 敏感数据泄露 | 明文传输/存储 | TLS、加密存储、密钥管理 |
| XML 外部实体 | XXE 注入 | 禁用外部实体解析 |
| 失效访问控制 | 越权访问 | 服务端权限校验、默认拒绝 |
| 安全配置错误 | 默认配置/错误信息泄露 | 安全基线、自定义错误页 |
| XSS | 反射/存储型 XSS | 输出编码、CSP 策略 |
| 不安全反序列化 | 远程代码执行 | 输入校验、白名单类 |
| 已知漏洞组件 | 依赖漏洞 | 依赖审计、及时升级 |
| 日志监控不足 | 攻击无感知 | 安全日志、异常告警 |

### 2. 认证与授权

1. **认证方案选择**:
   - 密码认证:bcrypt/argon2 哈希,禁用 MD5/SHA1
   - OAuth 2.0:第三方登录
   - 多因子认证:TOTP/短信/硬件密钥
2. **会话管理**:
   - 会话 ID 随机且不可预测
   - 登录后重置会话 ID(防固定)
   - 设置合理过期时间
   - HttpOnly + Secure + SameSite Cookie
3. **授权模型**:
   - RBAC(基于角色)/ ABAC(基于属性)
   - 服务端强制校验,不依赖前端隐藏
   - 默认拒绝,显式允许

### 3. 密钥与凭证管理

1. **密钥分级**:
   - 主密钥:最高权限,极少使用
   - 数据加密密钥:加密业务数据
   - 会话密钥:临时,短期有效
2. **存储方式**:
   - 禁止硬编码在源码
   - 禁止提交到版本控制
   - 使用密钥管理服务(KMS/Vault/环境变量)
3. **轮换策略**:
   - 定期轮换(90天/180天)
   - 泄露后立即轮换
   - 支持双密钥平滑过渡

### 4. 依赖审计

1. **漏洞扫描**:`npm audit` / `pip-audit` / `snyk test`
2. **许可证检查**:确保兼容(避免 GPL 污染)
3. **供应链安全**:
   - 锁定版本(lockfile)
   - 验证包完整性(checksum)
   - 警惕安装脚本(npm install scripts)
4. **升级策略**:
   - 安全补丁:立即升级
   - 次要版本:评估后升级
   - 主版本:计划性迁移

### 5. 输入校验与输出编码

1. **输入校验**(信任边界):
   - 白名单优于黑名单
   - 校验类型、长度、格式、范围
   - 拒绝不合规输入,返回明确错误
2. **输出编码**(数据边界):
   - HTML 上下文:HTML 实体编码
   - JS 上下文:JS 编码
   - URL 上下文:URL 编码
   - SQL 上下文:参数化查询
3. **CSP 策略**:限制脚本来源,防 XSS
