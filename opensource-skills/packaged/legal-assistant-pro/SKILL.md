# 法律助手Pro

为法律团队提供专业辅助。从合同审查到引文核验,从版本对比到状态报告,提升法务效率。

> **重要声明**:本技能为辅助工具,不构成法律意见。所有法律判断须由执业律师做出。

## 使用场景

| 场景 | 触发条件 | 说明 |
|:-----|:---------|:-----|
| NDA 分诊 | 收到 NDA 需审查 | 快速识别风险条款 |
| 版本对比 | 合同多版本协商 | 精确标记变更 |
| 引文核验 | 法律文书引用核对 | 验证引文准确性 |
| 会议简报 | 法律会议记录 | 结构化会议简报 |
| 状态报告 | 定期法务汇报 | 汇总本周工作状态 |

## 工作流

### 1. NDA 分诊审查

1. **NDA 分类识别**:
   - 单向 NDA(一方披露)
   - 双向 NDA(互相披露)
   - 主 NDA + 任务单结构
2. **关键条款审查**:
   - 保密信息定义(范围宽窄)
   - 保密义务期限(年限合理性)
   - 除外条款(已有/合法获取/法院命令)
   - 许可使用范围(限制 vs 过宽)
   - 返还/销毁义务
   - 违约救济(赔偿/禁令)
   - 管辖法律与争议解决
3. **风险标记**:
   - 红色:高风险(不可接受)
   - 黄色:需协商(可接受但需修改)
   - 绿色:标准条款(可接受)
4. **修改建议**:针对风险条款给出修改文本

### 2. 多方版本差异对比

1. **版本收集**:原始版/对方修改版/我方修改版
2. **逐条对比**:
   - 新增条款(标记来源)
   - 删除条款(标记影响)
   - 修改条款(精确到字词级 diff)
3. **变更分类**:
   - 实质性变更(权利义务变化)
   - 编辑性变更(措辞/格式)
   - 中立性变更(同义替换)
4. **影响评估**:每个实质性变更的影响分析
5. **建议回复**:针对对方修改的接受/反驳/反建议

### 3. 法律引文核验

1. **引文识别**:从文书中提取所有法律引文
2. **引文类型**:
   - 法条(法律名称+条款号)
   - 判例(案件名称+年份+页码)
   - 法规(法规名称+条款)
   - 司法解释
3. **核验维度**:
   - 引文是否存在(法律/判例是否真实)
   - 引文是否准确(条款号/页码是否正确)
   - 引文是否现行(是否已废止/修订)
   - 引文是否相关(是否支持论述)
4. **核验报告**:每个引文的核验结果+原始来源链接

### 4. 会议简报生成

1. **会议信息**:时间/地点/参会人/议题
2. **内容结构**:
   - 讨论要点(按议题)
   - 决议事项(明确决定)
   - 待办事项(责任人+截止日)
   - 分歧点(未达成一致)
3. **特权标注**:标注可能涉及律师-客户特权的内容
4. **行动清单**:提取所有行动项,按责任人分组

### 5. 周五状态合成

1. **本周工作汇总**:
   - 进行中的案件/项目
   - 本周完成事项
   - 下周计划事项
   - 阻塞与风险
2. **数据来源**:时间记录/邮件/文档/会议
3. **格式统一**:按团队标准模板输出
4. **分发准备**:适合邮件/Slack 分发的格式

## 特权与保密

1. **律师-客户特权**:识别并标注可能涉及特权的内容
2. **工作产品原则**:律师工作产品标记
3. **保密处理**:敏感信息脱敏后再输出
4. **日志留痕**:记录审查操作,便于追溯

## 实务领域参考

| 领域 | 常见文书 | 审查重点 |
|:-----|:---------|:---------|
| 公司法 | 章程/股东协议/股权转让 | 权利义务/退出机制 |
| 劳动法 | 劳动合同/竞业协议 | 合规性/公平性 |
| 知识产权 | 许可协议/转让协议 | 权属/范围/期限 |
| 商事合同 | 买卖/服务/租赁 | 违约/救济/管辖 |
| 合规 | 合规报告/审计报告 | 法规对齐/风险 |

## 依赖说明

### 运行环境
- **Agent平台**: Claude Code / Cursor / Codex / Gemini CLI / Windsurf 等支持SKILL.md的任意Agent
- **操作系统**: Windows / macOS / Linux
- **运行时**: 需要Agent支持exec(命令行执行)能力

### 第三方依赖
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| LLM API | API | 必需 | 由Agent内置LLM提供法律分析 |
| 联网搜索 | 工具 | 可选 | WebSearch/WebFetch查找法律条文 |

### API Key 配置
- **LLM_API_KEY**: 必需(通常由Agent内置) - 法律文书起草和合规分析
- 配置方式: 在Agent的环境变量中设置

### 纯Markdown使用说明
本Skill为法律辅助方法论。不替代专业律师,所有法律建议需专业人士复核。

只需将SKILL.md文件放入Agent的skills目录即可直接使用。
如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。

### 可用性分类
- **分类**: MD+EXEC
- **说明**: 纯Markdown,但需要exec能力(命令行执行),用于文件读写和命令调用
