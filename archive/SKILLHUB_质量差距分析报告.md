## 0. 概述与评分方法

本报告对已上传到 SkillHub 平台的 26 个 skill 进行 8 维度质量评估，并与 `scores` 表中final 评分最高（48 分）和最低（34 分）的 skill 做横向对比，识别质量差距。

**8 个评估维度**（每维度 0-6 分，满分 48，与 `scores` 表满分对齐）：

| 维度 | 评估要点 |
|------|----------|
| 1. description 字段质量 | 是否清晰描述触发场景、是否含'use when'/适用场景、长度 50-200 字符、不含宽泛词、列出核心能力、有具体名词 |
| 2. frontmatter 完整性 | slug/name/version/displayName/summary/license/description/tags/tools 9 个字段是否齐全 |
| 3. 正文结构 | 是否有'核心能力/架构总览'、'使用场景'、'依赖说明/运行环境'三章 |
| 4. 正文长度 | <50 行（2 分）/50-150 行（4 分）/150-300 行（6 分）/300-500 行（5 分）/>500 行（4 分） |
| 5. 工作流程/步骤 | 是否有'快速开始'、编号式步骤、代码/命令示例 |
| 6. 示例（examples） | 是否有 3+ 场景示例、5+ 代码块、输入/输出说明 |
| 7. 错误处理指引 | 是否有'错误处理'章节、错误码表/故障排查表、重试/降级/回退策略 |
| 8. 触发关键词精确性 | 是否显式列出触发关键词、数量 5-15 个、不含过宽词 |

**红牌（red flag）检测**：description 宽泛、正文过短、缺工作流程、无代码示例、中英混杂、疑似自动生成模板、含大量模板化套话、缺 FAQ、缺定价说明。
