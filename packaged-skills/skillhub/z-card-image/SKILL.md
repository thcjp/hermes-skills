# z-card-image

将用户提供的文案渲染成 PNG 卡片图。
支持短文案封面图、长文分页图、X 风格帖子分享长图,以及公众号文章封面图。
只要输入是帖子类型数据并希望导出成 X 风格长图,都应走 `x-like-posts`。

---

## 环境要求

- Python 3
- Google Chrome(macOS 位于 `/Applications/Google Chrome.app`;Linux 需将脚本中的 chromium 路径调整为本机实际路径)

---

## 渲染管线

### 环境检测

用户触发时检测一次,有问题给提示,不中止后续渲染环节:

- `python3 --version` 失败则提示未检测到 Python 3,渲染可能失败
- 检查 Chrome 路径,失败则提示安装

### 场景识别

根据输入特征识别渲染场景:

- 短文案封面图走 `poster-3-4`
- 长文分页图走 `article-3-4`
- X 风格帖子分享图、帖子长图、帖子类型数据走 `x-like-posts`
- 公众号文章封面图走 `wechat-cover-split`

### 查模板规则

根据识别到的模板在模板索引中找到对应规范文档,读取后按其规则处理文案与参数。

高亮规则:

- 整行高亮用 `--hl1`、`--hl2`、`--hl3`
- 按词高亮用 `--highlight-words`(逗号分隔)
- 两者可同时使用,不能忽略

### 识别平台

按平台预设自动设置配色。

- 用户提到"小红书配图"时使用小红书预设
- "小绿书"指公众号配图,使用公众号预设
- 推特长图默认使用 Twitter 风格蓝白灰配色
- 未明确指定时默认公众号

### 渲染输出

- `poster-3-4` 执行 `render_card.py`
- `article-3-4` 执行 `render_article.py`
- `x-like-posts` 执行 `render_x_like_posts.py`
- `wechat-cover-split` 执行 `render_card.py`
- 默认 `--out` 填 `tmp/...png`;用户指定导出位置时直接传绝对路径或相对路径

### 输出产物

生成 PNG 到指定路径,供后续发送、裁切或复用。
如需给外部工具上传,避免写入系统 `/tmp/`。

## 模板索引

| 模板名 | 比例 | 尺寸 | 用途 |
| --- | --- | --- | --- |
| poster-3-4 | 3:4 | 900x1200 | 文字海报(金句、大字报、封面) |
| article-3-4 | 3:4 | 900x1200 | 长文分页卡片 |
| x-like-posts | 自适应长图 | 900px 宽 | X 风格帖子分享长图 |
| wechat-cover-split | 335:100 | 1340x400 | 公众号文章封面长条图(左标题右 icon) |

## 输入校验

- 比例不存在:驳回请求,告知当前支持的比例列表,询问是否新增模板
- 文案超出模板字数上限:先自动拆分或缩写后再渲染,不直接塞入
- 帖子过多:按规范拆成多张 Part 1、Part 2,不把超长内容强行塞进一张
- 公众号封面标题过长:先压缩成 2 到 3 行短标题再渲染,不把完整长标题硬塞进模板

---

## 新增模板

1. 新建 `assets/templates/<name>.html`
2. 在 `render_card.py` 的 `size_map` 里注册尺寸
3. 在模板索引中添加一行
4. 创建对应 `references/<name>.md`,记录该模板的参数、字数上限、配图选取规则

## 适用场景

### 社媒帖子分享图

将 X 或类 X 平台的帖子数据渲染为可分享的长图,
适合内容创作者把文字动态转为图片在不能直接转发的渠道传播。

### 公众号文章封面

为公众号文章生成 1340x400 的封面长条图,

### 文字海报与金句卡

将金句、大字报等短文案渲染为 3:4 海报,
配合整行高亮突出核心句,适合做每日金句卡片或活动封面。

### 长文分页卡片

把长文章按模板字数上限拆分成多页卡片,
适合把长内容拆解为适合移动端纵向浏览的图文序列。

## 异常处理

### Python 3 未安装

`python3 --version` 失败,渲染脚本无法执行。
提示用户未检测到 Python 3,渲染可能失败,
建议安装 Python 3 后重新触发。检测不中止,但渲染会失败。

### Chrome 路径不存在

检查 Chrome 路径失败,渲染脚本依赖 Chrome 进行截图。
macOS 确认 `/Applications/Google Chrome.app` 存在,
Linux 需将脚本中的 chromium 路径改为本机实际路径,
Windows 需确认 Chrome 安装路径并在脚本中正确配置。

### 比例不存在

用户请求的比例不在模板索引中。
驳回请求,告知当前支持的比例列表,
询问是否需要新增模板,不强行使用错误比例渲染。

### 文案超出字数上限

文案长度超过模板字数上限,直接塞入会导致溢出或截断。
先自动拆分或缩写,再按拆分后的内容渲染,
不把超长文案硬塞进单张图。

### 帖子过多

帖子数量超过单张长图承载能力。
按规范拆成多张 Part 1、Part 2,
不把超长内容强行塞进一张图,保证可读性。

### 公众号封面标题过长

完整长标题硬塞进 wechat-cover-split 会导致排版错乱。
先压缩成 2 到 3 行短标题再渲染。

### 高亮参数被忽略

用户要求高亮但未传入 --hl 或 --highlight-words 参数,
导致渲染结果无高亮效果。
确认高亮规则:整行用 --hl1、--hl2、--hl3,按词用 --highlight-words,
两者可同时使用,不能忽略用户的高亮要求。

### 渲染脚本执行失败

render_card.py 等脚本执行报错。
检查 Python 环境与 Chrome 是否就绪,
确认 --out 路径所在目录存在且有写权限,
避免写入系统 /tmp/ 导致权限问题。

## 常见问题

### 如何选择模板?

根据输入特征判断:短文案封面用 poster-3-4,长文分页用 article-3-4,
帖子类型数据导出长图用 x-like-posts,公众号封面用 wechat-cover-split。
不确定时按用途对号入座。

### 小绿书和小红书有什么区别?

小绿书指公众号配图,使用公众号预设(背景 #e6f5ef、高亮 #22a854);
小红书使用小红书预设(背景 #fdecea、高亮 #e53935)。
用户提到"小红书配图"时切换到小红书预设,否则默认公众号。

### 整行高亮和按词高亮能同时用吗?

可以。整行用 --hl1、--hl2、--hl3,按词用 --highlight-words(逗号分隔),
两者同时使用时各自生效,满足既强调整行又标注关键词的需求。

### 输出路径有什么要求?

默认 --out 填 tmp/...png,用户指定时可直接传绝对或相对路径。
如需给外部工具上传,避免写入系统 /tmp/,建议使用项目内或用户目录下的路径。

### 如何新增模板?

新建 assets/templates/<name>.html,在 render_card.py 的 size_map 注册尺寸,
在模板索引添加一行,并创建 references/<name>.md 记录参数、字数上限与配图规则。

### X 风格长图的输入格式是什么?

x-like-posts 场景的输入 JSON 格式、可显示字段、时间规则与导出规则
记录在 references/x-like-posts.md 中,命中该路线时需要先读取该文档。

## 已知限制

- 依赖 Python 3 与 Google Chrome,缺少任一将导致渲染失败
- Linux 环境需手动调整脚本中的 chromium 路径
- 仅支持内置四种模板,新增模板需要手动修改脚本与文档
- 不处理动图与视频,仅输出静态 PNG
- 公众号封面标题需人工或自动压缩,超长标题不会自动适应
- 渲染为本地执行,不提供云端渲染能力
