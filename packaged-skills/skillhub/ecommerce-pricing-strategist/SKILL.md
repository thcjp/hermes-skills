# 电商定价军师

整合竞品监控与定价策略规划的一站式电商定价决策工具。自动抓取对标商品数据(标题/价格/销量/评价),结合5种定价模型、利润优化策略、大促定价公式和多平台差异化定价矩阵,输出可执行的今日行动清单。

## 使用场景

| 场景 | 触发条件 | 说明 |
|:-----|:---------|:-----|
| 竞品监控 | 选品前/定期监控 | 抓取对标商品标题/价格/销量/评价+差异化建议 |
| 新品定价 | 新品上架前确定价格 | 选择定价模型+计算建议价 |
| 利润优化 | 毛利率不达标 | 输出动态调价+捆绑+满减策略 |
| 大促定价 | 618/双11定价规划 | 计算大促到手价+预售定金+价保 |
| 多平台定价 | 多平台差异化定价 | 输出各电商平台定价方案 |
| 价格弹性分析 | 需要评估调价影响 | 计算价格弹性系数+预期销量变化 |
| 今日行动清单 | 日常运营决策 | 综合竞品数据+定价模型输出当日行动项 |

## 工作流

### 阶段一: 竞品监控(competitor)

1. **读取竞品关键词或URL列表**
   - 输入校验: keywords非空且长度<=50,platform在允许列表内
   - 失败返回CA-ERR-04
2. **调用电商平台数据API搜索竞品**
   - 参数: keywords关键词列表, platform平台, sort_by排序方式, count数量
   - 抓取被限流返回CA-ERR-01,自动降频重试
3. **调用竞品趋势分析API分析竞品趋势**
   - AI分析标题公式、价格区间、评价关键词分析、竞争建议
   - 字段缺失使用默认值(价格=0,销量=0),标记data_incomplete=true
   - 全部字段缺失返回CA-ERR-02
4. **评价分析流程**
   - 调用电商平台评价API获取商品评价内容
   - 调用评价分析API分析好评/差评情感+关键词提取+改进建议
   - 输出: positive_keywords好评关键词 / negative_keywords差评关键词 / positive_ratio好评率
   - 输出: improvement_suggestions改进建议 / competitive_advantages竞争优势
   - LLM不可用时基于关键词统计生成摘要分析
5. **获取市场概览**
   - 包含: avg_price / price_range / best_performing_title
6. **输出竞品分析报告**
   - 写入JSON文件(competitor_reports/YYYY-MM-DD.json)

### 阶段二: 定价策略(pricing)

1. **接收输入** - 输入成本+竞品价格+平台+操作类型
   - 验证: cost为正数, competitor_prices为列表, platform为有效平台, action为strategy/promotion/bundle/competitor/action_list之一
   - 异常: 无效输入返回INVALID_INPUT
2. **生成定价策略(action=strategy)** - 选择模型+计算建议价+毛利率
3. **生成大促方案(action=promotion)** - 计算大促到手价+预售定金+价保
4. **生成捆绑方案(action=bundle)** - 输出捆绑销售+满减策略
5. **竞品分析(action=competitor)** - 执行阶段一竞品监控全流程
6. **今日行动清单(action=action_list)** - 综合竞品数据+定价模型输出当日行动项
7. **输出JSON** - 结构化返回结果

## 5种定价模型

### 模型概览

| 模型 | 适用场景 | 计算公式 | 优点 | 缺点 |
|:-----|:---------|:---------|:-----|:-----|
| 成本加成法 | 标准品/有明确成本 | 售价 = 成本 * (1 + 加成率) | 简单/保利润 | 忽略竞争 |
| 竞争导向法 | 竞品明确的红海市场 | 售价 = 竞品均价 * (1 +/- 浮动%) | 有竞争力 | 可能牺牲利润 |
| 价值定价法 | 品牌品/差异化产品 | 售价 = 客户感知价值 * 折扣 | 高利润 | 难量化 |
| 渗透定价法 | 新品抢占市场 | 售价 = 成本 * (1 + 微利率) | 快速获客 | 初期低利 |
| 动态定价法 | 库存/销量波动大 | 售价 = 基准价 * 动态系数 | 实时优化 | 需数据支撑 |

### 模型选择决策树
```
有明确成本? -> 是 -> 竞品价格明确? -> 是 -> 竞争导向法
                                  -> 否 -> 成本加成法
            -> 否 -> 品牌差异化? -> 是 -> 价值定价法
                            -> 否 -> 渗透定价法
库存/销量波动大? -> 是 -> 动态定价法(可与上述模型叠加)
```

### 加成率参考

| 品类 | 加成率范围 | 说明 |
|:-----|:-----------|:-----|
| 虚拟商品(充值/会员) | 50-200% | 成本极低,主要时间成本 |
| 数码3C | 10-25% | 价格透明,竞争激烈 |
| 服饰鞋包 | 40-80% | 季节性强,库存风险 |
| 美妆护肤 | 50-100% | 品牌溢价高 |
| 食品生鲜 | 20-40% | 毛利低,走量为主 |
| 家居日用 | 30-60% | 中等毛利 |

## 利润优化策略

### 毛利率目标
- **健康毛利率**: >=30%
- **警戒毛利率**: 20-30%(需优化)
- **危险毛利率**: <20%(需提价或降本)
- **暴利毛利率**: >80%(检查是否虚高)

### 动态调价策略(动态定价法)
基于三维度动态调整价格:
- **库存维度**: 库存>安全库存*2 -> 降价10-15% / 库存<安全库存 -> 涨价5-10%
- **销量维度**: 近7天销量环比下降>30% -> 降价5-10% / 上升>50% -> 涨价3-8%
- **竞品维度**: 竞品降价>10% -> 跟进降价 / 竞品涨价 -> 观察后跟进

### 捆绑销售策略

| 策略 | 公式 | 适用场景 |
|:-----|:-----|:---------|
| 买A送B | 套餐价 = A价 / (A+B)成本比 | 关联品/清库存 |
| A+B套餐 | 套餐价 = (A+B) * 0.85 | 提升客单价 |
| 阶梯满减 | 满100减10/满200减30/满500减80 | 刺激凑单 |
| 第二件半价 | 均价 = (首件+半件) / 2 = 0.75首件 | 快速走量 |

### 满减策略设计
- **门槛设计**: 满减门槛 = 平均客单价 * 1.3-1.5(刺激凑单)
- **力度设计**: 满减力度 = 门槛金额 * 10-15%(吸引力足够)
- **层级设计**: 2-3级满减(如满99减10/满199减30/满299减60)

## 大促定价策略

### 大促定价公式
```
到手价 = 原价 * 折扣率 - 平台券 - 店铺券 - 补贴
```
- **折扣率**: 618/双11常规5-8折 / 秒杀3-5折
- **平台券**: 跨店满减(如满300减50)
- **店铺券**: 店铺专属优惠券
- **补贴**: 平台补贴/品牌补贴

### 预售定金策略

| 项目 | 公式/规则 | 说明 |
|:-----|:---------|:-----|
| 定金 | 预售价 * 10-20% | 定金翻倍抵扣 |
| 尾款 | 预售价 - 定金抵扣额 | 尾款期支付 |
| 定金翻倍 | 定金*2抵扣(如10元定金抵20元) | 刺激预售 |
| 预售优惠 | 预售价 < 正式价(低5-15%) | 预售专属价 |

### 价保策略
- **价保周期**: 大促期间15-30天
- **价保规则**: 买贵退差价(自动/申请)
- **价保目的**: 消除观望心理,提前转化
- **价保成本**: 预估2-5%订单需退差,计入定价成本

## 多平台差异化定价

### 多平台定价矩阵

| 平台类型 | 定价策略 | 价格区间 | 核心目标 |
|:---------|:---------|:---------|:---------|
| 二手交易类 | 低价引流 | 成本*1.1-1.3 | 引流/清库存/测款 |
| 短视频带货类 | GPM优化 | 成本*1.5-2.5 | 内容带货/冲动消费 |
| 综合电商类 | 搜索权重 | 成本*1.8-3.0 | 搜索流量/稳定销售 |
| 拼团社交类 | 拼团价 | 成本*1.2-1.6 | 拼团走量/社交裂变 |
| 品质电商类 | 品质溢价 | 成本*2.0-3.5 | 品质信任/会员体系 |
| 直播电商类 | 性价比 | 成本*1.3-2.0 | 老铁经济/复购 |

### 多平台定价注意事项
- **最低价管控**: 避免某平台价格过低影响其他平台(建议最低价差<=15%)
- **平台佣金**: 不同平台佣金差异需计入成本
- **包装差异**: 不同平台可用不同规格/包装做差异化
- **活动同步**: 大促期间各平台价格需协调一致

## 数据指标

| 指标 | 定义 | 健康值 | 计算公式 |
|:-----|:-----|:-------|:---------|
| 毛利率 | 利润占售价比 | >=30% | (售价-成本) / 售价 * 100% |
| 客单价 | 平均订单金额 | 因品类而异 | GMV / 订单数 |
| 价格弹性 | 销量对价格的敏感度 | 因品类而异 | 销量变化率 / 价格变化率 |
| 转化率 | 访客转订单比 | >=3% | 订单数 / 访客数 * 100% |
| 利润率 | 净利润占营收比 | >=15% | (营收-成本-费用) / 营收 * 100% |

## 异常处理

### 竞品分析异常

| 异常编号 | 错误代码 | 触发条件 | 处理方式 | 恢复策略 |
|:---------|:---------|:---------|:---------|:---------|
| CA-ERR-01 | RATE_LIMITED | 抓取被限流 | 降低频率,切换代理IP | 等待60s后重试,最多3次 |
| CA-ERR-02 | DATA_INCOMPLETE | 数据不完整(>50%字段缺失) | 使用已有数据分析,标记data_incomplete | 扩大关键词范围重新搜索 |
| CA-ERR-03 | COMPETITOR_DELISTED | 竞品已下架 | 标记"已下架",使用历史数据 | 从JSON文件读取上次分析结果 |
| CA-ERR-04 | INVALID_INPUT | keywords为空或platform无效 | 返回错误,提示修正输入 | 等待用户提供有效输入 |
| CA-ERR-05 | API_UNAVAILABLE | 电商平台API不可用 | 返回错误,提示检查API服务 | 检查API服务状态,重启后重试 |
| CA-ERR-06 | NO_REVIEWS | 评价数据为空 | 跳过评价分析,继续后续流程 | 标记review_analysis=null,不影响主流程 |
| CA-ERR-07 | REVIEW_RATE_LIMITED | 评价抓取限流 | 使用5分钟缓存,避免重复抓取 | 等待5分钟后重试,降级为comments数量 |

### 定价策略异常

| 异常类型 | 错误代码 | 处理方式 |
|:---------|:---------|:---------|
| 成本为空或非正数 | INVALID_INPUT | 返回错误,提示cost应为正数 |
| 竞品价格格式错误 | INVALID_COMPETITOR | 使用成本加成法,提示竞品格式应为逗号分隔数字 |
| 平台无效 | INVALID_PLATFORM | 返回错误,提示可选平台类型 |
| action无效 | INVALID_ACTION | 返回错误,提示可选competitor/strategy/promotion/bundle/action_list |

竞品数据可手动输入,不依赖特定电商平台API。LLM不可用时基于关键词统计生成摘要。

只需将SKILL.md文件放入Agent的skills目录即可直接使用。
如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。

### 可用性分类
- **分类**: MD+EXEC
- **说明**: 纯Markdown,但需要exec能力(命令行执行),用于文件读写和命令调用

## 示例

### 竞品分析示例
```bash
python pricing_strategist.py \
  --action competitor \
  --keywords '["AI代写文案","AI绘画头像"]' \
  --platform ecommerce \
  --sort_by sales \
  --count 10
```

### 新品定价策略示例
```bash
python pricing_strategist.py \

  --platform ecommerce
```

