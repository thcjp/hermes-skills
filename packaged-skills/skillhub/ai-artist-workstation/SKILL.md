# AI接单画师工作站

> 定位: 多引擎AI绘画+接单SOP全流程工作站
> 设计: 双引擎路由(写实→鹧应AI写真98%面部保持 / 艺术风格→通用绘画引擎)+接单交付闭环

## 使用场景
- 电商平台客户下单AI绘画服务后自动生成图片并交付
- 内容矩阵发布AI绘画作品展示引流
- 批量生成商品图/头像/壁纸用于电商销售
- AI写真定制: 参考图+写实风格,98%面部保持

## 工作流
1. 接收画图请求
   - 输入: style(头像/商业图/风格图/壁纸/商品图), prompt(描述), count(数量,1-10), size(尺寸)
   - 验证: style在允许范围内, prompt非空, count≤10
   - 异常: 参数缺失→提示补全;count超限→截断为10

2. 选择生成引擎
   - 头像/人像→写实引擎(鹧应AI写真,98%面部保持)或通用绘画引擎(5种场景模板)
   - 商品图/通用→通用绘画引擎(通用/商品图模式)
   - 风格图/古风→通用绘画引擎(古风插画模式)
   - 视频需求→视频生成引擎(T2V)或角色一致性引擎
   - 异常: 首选引擎不可用→降级到备选引擎链

2.5 头像风格选择(仅style=头像时)
   - 有参考图(face_image/agent_id): 调用一致性写真生成
     - **双引擎路由**: 写实风格→鹧应AI写真(98%面部保持,0.08元/张) | 艺术风格→通用绘画引擎(免费,无面部保持)
     - 降级链: 鹧应AI写真→通用绘画引擎T2I
     - 参考图获取方式: face_image参数(路径/URL) 或 agent_id(从配置读取基础图)
   - 无参考图: 调用通用绘画引擎纯text-to-image(面部一致性弱)

   | 风格 | style参数值 | 面部ID保持 | 推荐引擎 |
   |:-----|:----------|:----------|:---------|
   | 二次元 | anime | 否(艺术风格) | 通用绘画 T2I |
   | 写实 | realistic | 是(98%,鹧应) | 鹧应AI写真→通用绘画 |
   | 水彩 | watercolor | 否(艺术风格) | 通用绘画 T2I |
   | 国风 | chinese_ink | 是(98%,鹧应) | 鹧应AI写真→通用绘画 |
   | 赛博朋克 | cyberpunk | 否(艺术风格) | 通用绘画 T2I |
   | 像素 | pixel | 否(艺术风格) | 通用绘画 T2I |
   | 油画 | oil_painting | 否(艺术风格) | 通用绘画 T2I |
   | 3D卡通 | 3d_cartoon | 否(艺术风格) | 通用绘画 T2I |

3. 生成图片
   - 调用对应绘画引擎生成图片
   - 等待生成完成(超时180秒)
   - 下载图片到本地
   - 异常: 生成失败→重试1次→仍失败则换引擎→全部失败返回错误

4. 敏感词审核
   - 审核prompt和生成结果描述
   - 替换敏感词,确保合规
   - 异常: prompt含违禁内容→拒绝生成并退款

5. 交付
   - 电商订单: 发送预览图+发送高清原图网盘链接
   - 内容矩阵: 生成作品展示帖用于社交平台发布
   - 异常: 发送失败→记录待发送队列

```json
{

  "face_image": "d:/photos/selfie.jpg",
  "avatar_style": "cyberpunk",
  "template_id": "25f62a71-8255-431f-b0be-4e56f6b41921",
  "count": 1,
  "size": "1:1",
  "channel": "ecommerce",
  "order_id": "EC20240115002"
}
```

## 异常处理
| 异常场景 | 处理方式 | 错误码 |
|:---------|:---------|:-------|
| 参数缺失 | 提示补全 | MISSING_PARAMS |
| 首选引擎不可用 | 降级到备选引擎链 | ENGINE_FALLBACK |
| 生成超时(180s) | 重试1次后换引擎 | GENERATION_TIMEOUT |
| prompt含违禁内容 | 拒绝生成+退款 | FORBIDDEN_CONTENT |
| 发送失败 | 记录待发送队列 | DELIVERY_FAILED |

## 依赖说明

### 运行环境
- **Agent平台**: Claude Code / Cursor / Codex / Gemini CLI / Windsurf 等支持SKILL.md的任意Agent
- **操作系统**: Windows / macOS / Linux
- **运行时**: 需要Agent支持exec(命令行执行)能力

### 第三方依赖
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| LLM API | API | 必需 | 任意LLM服务商,由Agent内置LLM提供 |
| AI绘画引擎 | API | 必需 | 通用绘画引擎(免费)/鹧应AI写真(面部保持) |
| 图像生成API | API | 可选 | 视频生成/角色一致性引擎 |

### API Key 配置
- **LLM_API_KEY**: 必需(通常由Agent内置) - prompt优化和敏感词审核
- **IMAGE_API_KEY**: 可选 - AI绘画引擎(通用绘画引擎可免费使用)
- **PORTRAIT_API_KEY**: 可选 - 鹧应AI写真98%面部保持
- 配置方式: 在Agent的环境变量中设置

### 纯Markdown使用说明
通用绘画引擎支持免费text-to-image。鹧应AI写真用于写实头像98%面部保持。

只需将SKILL.md文件放入Agent的skills目录即可直接使用。
如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。

### 可用性分类
- **分类**: MD+EXEC
- **说明**: 纯Markdown,但需要exec能力(命令行执行),用于文件读写和命令调用

## 示例
电商平台客户下单"AI头像定制":
1. 接收订单→提取需求(头像,写实风格,附自拍)
2. 有参考图+写实风格→调用一致性写真生成(face_image=自拍, style=realistic)
3. 鹧应AI写真生成98%面部保持的写实头像(降级到通用绘画引擎)
4. 敏感词审核通过
5. 发送预览+网盘链接交付

无参考图时:
1. 接收订单→提取需求(头像,赛博朋克风格程序员)
2. 无参考图→调用通用绘画引擎纯text-to-image
3. 生成1张1:1头像图(面部一致性弱)
4. 敏感词审核通过
5. 发送预览+网盘链接交付

## 引擎降级链
- 头像(有参考图,写实): 鹧应AI写真(98%面部保持)→通用绘画 T2I(无面部保持)
- 头像(有参考图,艺术): 通用绘画 T2I(无面部保持)
- 头像(无参考图): 通用绘画→备选引擎A→备选引擎B
- 商品图: 通用绘画→备选引擎
- 视频图: 视频引擎A→视频引擎B→视频引擎C
- 注意: 通用绘画引擎不支持image参数(图生图),仅可用于text-to-image
