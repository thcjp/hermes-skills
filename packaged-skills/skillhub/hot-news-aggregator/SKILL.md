## 核心能力

- 国内外社会、科技、军事、财经、体育新闻多源聚合与汇总
- 自动搜索：根据关键词、主题、时间范围从多个新闻源检索相关报道
- 智能筛选：基于相关性评分、来源权威度、时效性对新闻进行排序和过滤
- 要点提取：自动生成新闻摘要，提取关键人物、事件、数据、影响分析
- 去重合并：识别同一事件的多源报道，合并为一条带多来源引用的新闻条目
- 分类标注：自动识别新闻类别（社会/科技/军事/财经/国际/教育/健康）
- 趋势分析：识别热点话题的发展趋势、舆论走向和关联事件
- 多语言支持：中文新闻源与英文国际新闻源并行检索

### 流程详解：每日新闻简报生成

**步骤 1：指定检索参数**

```
查询参数示例:
- 日期范围: 2024-07-23 至 2024-07-24
- 关注领域: 科技, 军事, 国际
- 关键词: AI芯片, 军演, 中东局势 (可选)
- 语言: 中文 + 英文
- 每领域条数: 10
- 排序方式: 时效性 + 重要性
```

**步骤 2：多源检索与去重**

系统会从以下类型的新闻源并行检索：

| 新闻源类型 | 示例来源 | 特点 |
|:-----------|:---------|:-----|
| 国内主流媒体 | 新华网、人民网、央视新闻 | 权威性高、官方视角 |
| 国内科技媒体 | 36氪、虎嗅、钛媒体 | 深度分析、行业前沿 |
| 国际通讯社 | Reuters、AP、Bloomberg | 全球视角、时效性强 |
| 科技博客 | TechCrunch、The Verge、Ars Technica | 技术深度、产品评测 |
| 社交媒体热搜 | 微博热搜、知乎热榜、Twitter Trending | 舆论风向、民间视角 |
| 军事专业媒体 | 简氏防务、军事观察网 | 专业分析、装备动态 |

去重策略：
- 标题相似度 > 80% 视为同一事件
- 发布时间相近（< 2小时）且主体内容相似 > 70% 合并
- 合并后保留最早发布的来源作为主来源，其他来源列为相关报道

**步骤 3：摘要生成与要点提取**

对每条新闻提取以下结构化信息：

```json
{
  "title": "新闻标题",
  "summary": "100-200字摘要，概括核心事实",
  "category": "科技",
  "importance": "high",
  "key_entities": ["涉及人物", "涉及机构", "涉及地点"],
  "key_data": {"关键数据名": "数值"},
  "timeline": "事件发展脉络",
  "impact": "潜在影响分析",

    {"name": "来源名", "url": "原文链接", "publish_time": "发布时间"}
  ],
  "tags": ["标签1", "标签2"]
}
```

| 语法 | 含义 | 示例 |

### 重要性评分算法

每条新闻的重要性评分（0-100）基于以下维度：

| 维度 | 权重 | 评分依据 |
|:-----|:-----|:---------|
| 来源权威度 | 25% | 官方媒体 > 主流媒体 > 自媒体 |
| 报道广度 | 25% | 被多少独立来源报道 |
| 时效性 | 20% | 距当前时间越近分越高 |
| 影响范围 | 20% | 涉及的人口/地域/行业规模 |
| 互动热度 | 10% | 社交媒体讨论量与转发量 |

## 最佳实践

### 高效使用建议

1. **明确关注领域**：指定 2-4 个关注领域，避免检索结果过于分散
2. **设置时间窗口**：每日简报用 24 小时窗口，专题跟踪用 7-30 天窗口
3. **利用关键词过滤**：添加排除词减少噪音（如排除"广告"、"推广"类内容）
4. **交叉验证**：对重要新闻查看多个来源的报道，避免单一来源偏差
5. **定期调整**：根据简报质量调整新闻源和关键词配置

| 使用场景 | 推荐格式 | 原因 |

| 数据分析处理 | JSON | 便于程序解析和二次处理 |
| 汇报展示 | Markdown + 趋势图 | 直观展示变化趋势 |
| 存档检索 | JSON + 全文索引 | 便于后续搜索和引用 |

## 常见问题

### Q1: 如何开始使用hot-news-aggregator？
A: 直接描述你想了解的新闻主题即可。例如"帮我汇总今天的科技和军事新闻"或"搜索最近一周关于AI芯片的新闻"。系统会自动从多个新闻源检索、去重、筛选，生成包含摘要、关键信息和来源链接的结构化新闻简报。你可以指定关注领域、时间范围和输出格式。

### Q2: 遇到错误怎么办？
A: 常见错误及处理方式：1) 搜索结果为空 - 尝试扩大时间范围或使用更宽泛的关键词；2) 新闻源不可达 - 系统会自动切换备用源，不影响整体结果；3) 网络超时 - 检查网络连接后重试；4) 结果质量不佳 - 调整关注领域和排除词，减少噪音内容。

### Q3: hot-news-aggregator有什么限制？
A: 1) 新闻时效性取决于新闻源的更新频率，部分来源可能有 1-2 小时延迟；2) 搜索结果受新闻源覆盖范围限制，小众话题可能报道较少；3) 摘要基于AI生成，重要数据建议对照原文核实；4) 不提供实时推送功能，需主动发起查询；5) 部分新闻源可能有访问频率限制。
## 错误处理

| 错误场景(续)| 原因 | 处理方式 |
|----:|:----|----:|
| LLM响应超时或无响应 | 网络延迟或模型负载过高 | 重试请求；确认Agent平台LLM服务正常 |

| 命令执行失败 | 运行环境不满足要求或权限不足 | 确认运行环境符合依赖说明中的要求；检查命令权限设置 |
