# Design

自适应设计偏好引擎。通过观察用户选择与反馈,自动学习并沉淀视觉偏好,适配 UI、图形、视频、印刷等多媒介创作。偏好档案随使用持续进化.
**范围外**（本技能不做）: 3D 建模与动画制作、设计稿自动渲染输出、设计工具操作控制、品牌资产管理系统.

## 偏好档案结构

偏好档案分为四大分区,初始为空,随使用自动填充:

### Aesthetic（美学风格）

记录用户对色彩、排版、留白、字体、质感等美学元素的偏好.
示例条目格式（精简紧凑）:
```
- 配色: 偏好低饱和度暖色调,避免高对比霓虹色
- 排版: 左对齐为主,行高 1.6,段间距 16px
- 字体: 无衬线优先,标题用 Inter,正文用 Noto Sans
- 留白: 大量负空间,内容区不超过视口 60%
```

### By Medium（按媒介细分）

按不同设计媒介记录差异化偏好:

```
- UI: 圆角 8-12px,阴影柔和,深色模式优先
- 图形: 扁平化插画,几何构图,限定 5 色调色板
- 视频: 16:9 为主,字幕底部居中,转场用淡入淡出
- 印刷: CMYK 色域,出血 3mm,正文字号不小于 9pt
```

### Brands（品牌专属偏好）

为不同品牌维护独立偏好配置:

```
- 品牌A: 主色 #1a73e8,辅色 #34a853,风格偏科技感
- 品牌B: 主色 #e8344e,风格偏年轻活泼,多用插画
```

### Never（禁忌清单）

记录用户明确不喜欢的视觉元素:

```
- 不用: 渐变背景、毛玻璃效果、Emoji 图标
- 避免: 全大写标题、斜体正文、纯黑纯白
- 禁止: 抖动动画、自动播放视频、弹窗遮罩
```

## 偏好学习机制

### 检测信号

从以下用户行为中提取偏好信号:

1. **选择行为**: 用户在多个方案中选择某一个时,记录被选方案的视觉特征
2. **反馈表达**: 用户对设计输出给出评价（喜欢/不喜欢/修改意见）时,提取评价指向的视觉元素
3. **修改行为**: 用户要求修改某个视觉元素时,记录修改前后的差异
4. **反应强度**: 用户对某个设计表现出强烈情绪反应时,标记为高权重信号

### 确认规则

- 同一偏好方向出现 **2 次以上** 一致信号后,自动确认写入偏好档案
- 仅 1 次出现的信号标记为"待观察",不写入正式档案
- 出现矛盾信号时（如既选了圆角又要求直角）,保留两条记录并标注冲突
- 已确认的偏好可被用户显式覆盖或删除

### 条目维护规则

- **精简紧凑**: 每条偏好不超过 1 行,避免冗长描述
- **具体可执行**: 记录具体数值或明确指令（如"圆角 8px"而非"圆角适中"）
- **分类归档**: 按 Aesthetic / By Medium / Brands / Never 四区归档
- **定期整理**: 当条目超过 50 条时,合并相似条目,清理过时条目

## 参考文件

### dimensions.md

维护偏好类别维度,定义可观察的视觉属性范围:

- 色彩维度: 色相、饱和度、明度、对比度、配色方案
- 排版维度: 对齐、行高、字距、字号层级、字体族
- 构图维度: 网格、留白、比例、对称性、视觉重心
- 质感维度: 圆角、阴影、渐变、纹理、透明度
- 动效维度: 时长、缓动、触发方式、过渡类型

### criteria.md

维护条目格式规范:

- 格式: `- 属性: 具体描述`
- 属性用中文,描述含具体数值或明确指令
- 多值用顿号分隔: `圆角 8px、12px、16px`
- 禁忌标注: `不用: xxx` / `避免: xxx` / `禁止: xxx`

## 适用场景

### 场景一: UI 设计风格沉淀

设计师在多个项目中反复选择低饱和度配色与大量留白,引擎检测到 2 次以上一致偏好后自动写入 Aesthetic 分区,后续设计输出自动遵循该风格.
### 场景二: 多品牌偏好管理

团队同时服务多个品牌,每个品牌有独立视觉规范。引擎为每个品牌维护 Brands 分区条目,切换品牌上下文时自动加载对应偏好.
### 场景三: 视频视觉风格适配

视频创作者偏好 16:9 画幅与底部居中字幕,引擎在 By Medium 的视频分类下记录该偏好,后续视频设计自动应用.
### 场景四: 禁忌清单积累

用户多次要求移除毛玻璃效果与渐变背景,引擎在 2 次确认后写入 Never 分区,后续设计不再使用这些元素.
## 使用流程

1. 检查偏好档案是否为空,为空则进入观察模式
2. 在设计交互中观察用户的选择、反馈、修改与反应行为
3. 提取视觉特征信号,与已有偏好比对
4. 同一偏好方向出现 2 次以上一致信号后,按 criteria.md 格式写入对应分区
5. 后续设计输出前,先读取偏好档案,确保输出符合已确认偏好
6. 用户显式否定某偏好时,从档案中移除或标注冲突
7. 定期整理档案,合并相似条目,清理过时条目

## 案例展示

### 案例一： UI 配色偏好学习

**场景**: 设计师在连续 3 个项目中都选择了低饱和度暖色调配色方案

**观察记录**:
```
信号1: 项目A - 用户选择了 #f5f0eb 背景 + #8b7355 主色,反馈"这个配色舒服"
信号2: 项目B - 用户从 3 个方案中选了暖灰系方案,弃选高对比方案
信号3: 项目C - 用户要求将冷蓝配色改为暖棕色调
```

**确认写入** Aesthetic 分区:
```
- 配色: 偏好低饱和度暖色调（米色/棕色系）,避免高对比霓虹色
```

**说明**: 3 次一致信号超过 2 次阈值,自动确认。后续 UI 设计输出默认使用低饱和暖色调,除非用户显式指定其他配色.
### 案例二： 多品牌偏好切换

**场景**: 团队服务品牌A（科技感）与品牌B（年轻活泼）,需在不同项目中切换视觉风格

**Brands 分区记录**:
```
- 品牌A: 主色 #1a73e8,辅色 #34a853,风格偏科技感,无衬线字体,几何图形
- 品牌B: 主色 #e8344e,辅色 #ff9800,风格偏年轻活泼,多用插画,圆角 16px
```

**切换行为**: 当用户提到"品牌A的项目"时,引擎自动加载品牌A偏好;提到"品牌B"时切换到品牌B偏好。避免风格混淆.
**说明**: Brands 分区为每个品牌维护独立条目,切换时按品牌名索引加载。已确认的品牌偏好不会被其他品牌的信号覆盖.
### 案例三： 禁忌清单积累

**场景**: 用户在多次设计中明确表示不喜欢毛玻璃效果和全大写标题

**观察记录**:
```
信号1: 设计稿A - 用户要求移除毛玻璃卡片效果,反馈"毛玻璃太花哨"
信号2: 设计稿B - 用户要求标题改为首字母大写,反馈"全大写太正式"
信号3: 设计稿C - 用户再次否决毛玻璃方案
```

**确认写入** Never 分区:
```
- 不用: 毛玻璃效果（backdrop-blur）、全大写标题
```

**说明**: 毛玻璃效果出现 2 次否定信号后确认写入禁忌清单。全大写标题仅 1 次信号但用户反馈强烈,提前确认。后续设计不再使用这些元素.

## 常见问题

### Q1: 偏好档案什么时候开始有内容?
A: 偏好档案初始为空。引擎在用户进行设计交互时观察选择、反馈、修改与反应行为,同一偏好方向出现 2 次以上一致信号后自动写入。通常经过 2-3 次设计交互后开始有初始条目.
### Q2: 偏好档案会冲突怎么办?
A: 当同一维度出现矛盾信号（如既选圆角又要求直角）时,引擎保留两条记录并标注冲突。后续设计输出时提示用户显式选择,确认后删除被否决的条目.
### Q3: 可以手动编辑偏好档案吗?
A: 可以。用户可以显式添加、修改或删除偏好条目。手动编辑的条目优先级与自动检测的条目相同,但不会被自动覆盖。建议遵循 criteria.md 中的格式规范.
### Q4: dimensions.md 和 criteria.md 的作用是什么?
A: dimensions.md 维护可观察的视觉属性维度（色彩、排版、构图、质感、动效）,帮助引擎系统化识别偏好信号。criteria.md 维护条目格式规范,确保偏好条目精简紧凑且具体可执行.
### Q5: 多个品牌的偏好会互相干扰吗?
A: 不会。Brands 分区为每个品牌维护独立条目,切换品牌上下文时按品牌名索引加载。品牌A的偏好信号不会覆盖品牌B的已确认条目.
### Q6: Never 分区的禁忌元素会被自动移除吗?
A: 不会自动移除。Never 分区记录用户明确不喜欢的视觉元素,后续设计输出时会主动避免使用这些元素。用户可以显式删除 Never 条目以解除禁忌.

## 已知限制

1. **需 LLM 支持**: 偏好检测依赖 LLM 理解用户反馈语义,无 LLM 环境无法使用
2. **初始空档案**: 首次使用时偏好档案为空,需 2-3 次交互后才有初始条目
3. **确认阈值固定**: 2 次以上一致信号才确认,急性用户可能觉得学习速度偏慢
4. **媒介类型有限**: 预置 UI/图形/视频/印刷四类,其他媒介需归入最接近分类
5. **条目上限 50 条**: 超过后触发整理流程,可能丢失低频但有用的条目
6. **复杂场景需人工辅助**: 偏好冲突或品牌定位调整时可能需要人工判断
7. **性能取决于底层模型**: 偏好检测准确度受 Agent 内置 LLM 能力影响
