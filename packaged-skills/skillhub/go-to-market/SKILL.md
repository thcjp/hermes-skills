# 上市策略

GTM(上市)策略是触达客户并产生收入的计划。回答:客户是谁?信息是什么?如何触达?如何销售?对独立创业者,精准GTM策略避免在无效渠道或信息上浪费精力.

## 核心能力

### 1. 定义目标客户(ICP)
ICP(理想客户画像)模板覆盖四维:
- 人口统计(B2C):年龄、收入、地区、职业
- 企业画像(B2B):行业、公司规模、收入范围、地区
- 心理特征:痛点、目标、价值观、恐惧
- 行为特征:出没地点、购买决策方式、触发寻找解决方案的因素

验证:访谈5-10位匹配ICP的人,询问痛点、现有方案与切换诱因。若答案与假设不符,修正ICP.
**输出**: 返回定义目标客户(ICP)的处理结果,包含执行状态码、结果数据和执行日志.
### 2. 打造定位
定位陈述模板(内部用,非客户面向):
```
For [目标客户],
Who [有此问题],
[你的产品] is a [类别]
That [关键收益/独特价值].
Unlike [竞品或替代方案],
[你的产品] [关键差异点].
```
测试:能否1-2句话解释?听者是否立即理解面向谁、为何不同?不能则简化.
**输入**: 用户提供打造定位所需的指令和必要参数.
**处理**: 解析打造定位的输入参数,执行核心处理逻辑,返回结构化结果和执行状态.
**输出**: 返回打造定位的处理结果,包含执行状态码、结果数据和执行日志.
### 3. 选择GTM运动模式
| 模式 | 适用场景 | 特征 |
|:---:|:---:|:---:|
| 产品驱动(PLG) | 低价($0-$100/月),自助 | 免费试用/freemium,用户自注册自上手,无销售介入 |
| 销售驱动 | 高价($500+/月或$5K+订单),复杂 | 演示、提案、人工跟进,长销售周期 |
| 混合 | 中端市场($100-500/月) | 小客户自助,大客户销售辅助 |
| 社区驱动 | 网络效应或同侪影响重要 | 先建社区,通过社区变现 |

选择指南:简单便宜→PLG;复杂昂贵→销售驱动;受益用户互动→社区驱动;同时服务SMB与企业→混合.
**处理**: 解析选择GTM运动模式的输入参数,执行核心处理逻辑,返回结构化结果和执行状态.
**输出**: 返回选择GTM运动模式的处理结果,包含执行状态码、结果数据和执行日志.
### 4. 选择渠道
渠道选择框架:
| 渠道 | 最适合 | ROI周期 | 成本 |
|:------|------:|:------|:------|
| SEO/内容 | 高意图搜索,长期增长 | 3-6个月 | 低(时间) |
| 付费广告 | 快速流量,测试,扩量 | 即时 | 高 |
| 社交媒体 | 建立受众,品牌认知 | 1-3个月 | 低(时间) |
| 邮件营销 | 自有受众,培育线索 | 即时(若有列表) | 低 |
| 社区/论坛 | 垂直受众,信任建立 | 1-6个月 | 低(时间) |
| 合作/分销 | 借力他人受众 | 1-3个月 | 中(分成) |
| 外联(冷邮/私信) | B2B,精准定向 | 即时 | 低(时间) |
| 产品目录站 | 科技/SaaS产品,发布峰值 | 即时 | 低 |

选择方法:1)ICP在哪出没?2)预算多少?3)时间线多紧?选2个主要渠道,精通后再加第三个.
**输入**: 用户提供选择渠道所需的指令和必要参数.
**输出**: 返回选择渠道的处理结果,包含执行状态码、结果数据和执行日志.

每周检查:指标是否达标?不达标需调整什么?

- 异常时参考错误处理章节进行恢复

### 6. 设定定价与优惠
定价考量:交付价值、竞品价格、目标CAC(获客成本)、优化销量(低价)还是利润(高价).
发布优惠(可选但有效):
- 早鸟折扣(前100客户20-30% off)
- 终身买断(一次性付费,利于现金流,不利于MRR)
- 免费层或试用(降低摩擦,增加注册)
原则:不要为"快速获客"而低价。廉价客户往往低质量高流失。按价值定价.
**输入**: 用户提供设定定价与优惠所需的指令和必要参数.
**处理**: 解析设定定价与优惠的输入参数,执行核心处理逻辑,返回结构化结果和执行状态.
**输出**: 返回设定定价与优惠的处理结果,包含执行状态码、结果数据和执行日志.
### 7. 度量与迭代
| 指标 | 含义 | 追踪方式 |
|---:|:---|---:|
| CAC(获客成本) | 获取一个客户的花费 | 营销支出/新客户数 |
| 转化率 | 线索/访客转为客户的比例 | 分析+CRM |
| 渠道ROI | 哪些渠道盈利 | 每渠道收入/每渠道支出 |
| 首客时间 | 从发布到首单的时长 | 手动追踪 |
| 客户反馈 | 是否解决正确问题 | 调研、访谈、工单 |

月度GTM审查(30分钟):是否达成90天里程碑?哪些渠道表现最好?哪些不佳?加倍投入什么?砍掉什么?学到了什么?

**输入**: 用户提供度量与迭代所需的指令和必要参数.
**输出**: 返回度量与迭代的处理结果,包含执行状态码、结果数据和执行日志.
### 8. GTM错误规避
- ICP定义不清:"所有人"不是目标客户,必须收窄
- 同时铺太多渠道:分散=处处平庸，最多2-3个
- 无定位发布:不知道自己有何不同,客户也不知道
- 不跟踪指标:无法改进未度量的事物,发布前建好分析
- 过早放弃:多数渠道需30-90天见效,别第一周就慌
- 忽略客户反馈:客户说信息混乱或定价不对,听并调整

**输入**: 用户提供GTM错误规避所需的指令和必要参数.
**输出**: 返回GTM错误规避的处理结果,包含执行状态码、结果数据和执行日志.
### 输出格式

完成响应以Markdown格式返回,包含任务状态(成功/失败)、解析摘要和具体输出数据。失败时返回错误码和错误信息,便于定位问题。- 验证返回数据的完整性和格式正确性

1. 定义ICP,访谈5-10位匹配者验证
2. 撰写定位陈述,1-2句话测试
3. 按价格与复杂度选择GTM运动模式

## 错误处理

| 错误场景 | 原因 | 处理方式 |
|----|:--:|---:|
| ICP定义过宽,"所有人"都是客户 | 恐惧错失机会 | 收窄到1个核心画像,验证后再扩展;访谈5-10人确认痛点真实 |
| 同时铺5+渠道,处处平庸 | 贪多嚼不烂 | 砍到2个主渠道,精通后再加第三个;按ROI排名砍掉低效渠道 |
| 定位陈述听者不懂面向谁 | 差异化不清晰 | 用定位模板重写,1-2句测试;删掉形容词,保留具体收益 |
| 渠道30天无效果就放弃 | 多数渠道需30-90天见效 | 区分"未见效"与"无潜力";SEO需3-6个月,付费广告才该快砍 |
| 未建分析就发布,无法度量 | 追踪后置 | 发布前配好分析:转化漏斗、渠道归因、CAC计算 |
| 客户反馈定价过高直接降价 | 未理解价值 | 访谈确认是价格问题还是价值传达问题;考虑分期或缩减功能版 |
| 试用转付费率<10% | 上手摩擦或价值未兑现 | 缩短首次价值时间,加引导序列,排查注册到激活漏斗 |

## 常见问题

### Q1: ICP访谈5-10人如何招募?
A: 从现有客户中筛选最满意的;在ICP出没的社区发帖邀约;通过职业社交平台定向私信。访谈聚焦:痛点、现有方案、切换诱因。避免引导性问题,让对方多说.
### Q2: PLG与销售驱动如何选择?
A: 价格$0-$100/月且产品简单→PLG,用户自注册自上手;价格$500+/月或产品复杂→销售驱动,需演示与提案。中间地带($100-500)用混合:小客户自助,大客户销售辅助.
### Q3: 90天计划目标如何设定?
A: 基于渠道ROI倒推。如目标$2,450 MRR(50客户@$49),冷外联15%回复率、25%转付费,需发500条私信;SEO 12篇博文目标月均2,000访客。目标要可分解为tactic级metric.
### Q4: 早鸟折扣20-30%会损害品牌吗?
A: 限时限量(前100客户)的早鸟折扣是合理的获客手段,不损害品牌。但永久降价或为"快速获客"而低价会吸引低质量高流失客户。折扣必须有明确的截止条件.
### Q5: 月度GTM审查30分钟如何分配?
A: 里程碑达成检查5分钟、渠道ROI排名10分钟、低效渠道决策5分钟、ICP/定位新洞察5分钟、下月tactic调整5分钟。超时项目单独跟进.
### Q6: 渠道ROI如何计算?
A: 每渠道ROI = (该渠道带来的收入 - 该渠道支出) / 该渠道支出。需配置渠道归因(UTM参数、CRM来源字段)。SEO的时间成本按小时折算,付费广告按实际支出计.
