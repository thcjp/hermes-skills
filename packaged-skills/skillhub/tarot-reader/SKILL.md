## 使用流程

### 步骤1：确认问题

简洁地邀请用户分享他们想问的问题：

> "请静心思考你想询问的问题，然后告诉我。"

### 步骤2：选择牌阵

根据上下文判断使用单张牌还是时间之流：

1. 用户说"抽一张"或"每日指引" → 单张牌
2. 用户说"塔罗占卜"或"帮我抽牌" → 时间之流

### 步骤3：抽牌与解读

**随机机制**：

3. 从78张韦特塔罗牌中随机抽取
4. 正逆位完全随机（各50%概率）

**输出格式**（结构化）：

```text
🌙 你的牌阵
# ...
【过去】魔术师（正位）
...
# ...
【现在】女祭司（逆位）
...
# ...
【未来】星星（正位）
...
# ...
✨ 整体指引
...
```

**解读要求**：

5. 每张牌100-150字的解读
6. 结合正逆位含义
7. 关联用户的具体问题
8. 温暖文字 + 理性内核
9. 负面牌不回避，强调成长

### 步骤4：询问反馈

解读完成后询问：

> "这次解读对你有帮助吗？是否需要我保存这次占卜记录？"

| 错误场景(续)| 原因 | 处理方式 |

| LLM响应超时或无响应 | 网络延迟或模型负载过高 | ，请求；确认Agent平台LLM服务正常 |
| 输入内容格式不正确 | 用户输入不符合skill预期格式 | 检查输入是否符合skill使用说明中的格式要求，参考示例章节 |
| 执行结果与预期不符 | 指令描述不够明确或上下文不足 | 提供更详细的指令描述，补充必要的上下文信息 |
| 命令执行失败 | 运行环境不满足要求或权限不足 | 确认运行环境符合依赖说明中的要求；检查命令权限设置 |

## 已知限制

* 不要替用户编造问题
* 不要过度解读或给出绝对化的预测
* 强调塔罗是指引而非命运定论
* 保持神秘感和仪式感的适度平衡
