# SEO体检医生

## 使用场景

1. **发布前SEO优化**: 对标题、描述、标签进行全方位优化,提升搜索排名和曝光量
2. **关键词研究**: 基于BERT语义模型提取核心关键词,挖掘长尾关键词
3. **标题A/B测试**: 生成3-5个优化标题版本,提供搜索排名预估
4. **热搜关键词匹配**: 获取当前热搜关键词,结合热度生成关键词策略
5. **发布后排名追踪**: 监控已发布内容的搜索排名变化,提供优化建议
6. **品牌关键词注入**: 将品牌词以最高优先级注入SEO关键词和标签体系
7. **竞品分析**: 对标竞品内容,生成差异化SEO策略

---

## 工作流

### 工作流1: 关键词优化 (发布前)

1. **读取原始内容**: 获取标题、正文、图片信息
2. **BERT语义关键词提取**: 使用预训练BERT模型提取内容核心关键词
3. **品牌关键词注入**: 读取品牌关键词配置,将品牌词前置到核心关键词列表(最高权重),执行大小写不敏感去重
4. **热搜关键词匹配**: 通过热搜数据API获取当前热搜关键词,与内容关键词交叉匹配
5. **关键词策略生成**: 综合语义提取、品牌注入、热搜匹配,输出关键词方案(核心词+次要词+长尾词)

### 工作流2: 标题优化 (发布前)

1. **基于关键词策略生成优化标题**: 生成3-5个SEO友好的标题版本
2. **生成SEO描述和标签**: 根据平台规则生成描述和标签
3. **搜索排名预估**: 基于关键词竞争度和内容质量预估排名区间
4. **竞品分析**: 对标竞品URL,分析关键词覆盖差距
5. **输出完整SEO方案**: 标题候选+描述+标签+关键词+预估排名+SEO评分

### 工作流3: SEO排名追踪 (发布后)

1. **接收已发布内容URL和目标关键词列表**
2. **调用搜索API**: 对每个目标关键词执行搜索查询
3. **检测排名位置**: 检查内容URL是否出现在搜索结果中,记录排名序号
4. **趋势对比**: 与历史排名数据对比,计算变化趋势(上升/下降/新入榜/跌出)
5. **生成优化建议**:
   - 排名下降 -> 建议调整关键词密度或标题
   - 未入榜 -> 建议增加外链和互动数据
   - 排名上升 -> 建议保持当前策略
6. **输出追踪结果**: 排名位置+趋势变化+优化建议+汇总统计

## 输入格式

### 优化模式 (工作流1+2)

```json
{
  "title": "原始标题",
  "content": "正文内容",
  "platform": "xiaohongshu",
  "target_keywords": ["AI代写", "文案生成"],
  "competitor_urls": ["对标链接1", "对标链接2"],
  "brand_keywords": ["品牌词1", "品牌词2"]
}
```

| 字段 | 类型 | 必填 | 说明 |
|:-----|:-----|:----:|:-----|
| title | string | 是 | 原始标题 |
| content | string | 是 | 正文内容 |
| platform | string | 是 | 目标平台 |
| target_keywords | string[] | 否 | 目标关键词列表 |
| competitor_urls | string[] | 否 | 竞品内容URL |
| brand_keywords | string[] | 否 | 品牌关键词,优先级最高 |

### 追踪模式 (工作流3)

```json
{
  "mode": "track",
  "content_url": "已发布内容的URL",
  "target_keywords": ["AI代写", "文案生成"],
  "platform": "xiaohongshu",
  "history_data": {
    "AI代写": {"last_rank": 5, "date": "2026-05-27"},
    "文案生成": {"last_rank": 12, "date": "2026-05-27"}
  }
}
```

## SEO评分模型

| 评分维度 | 权重 | 检查项 |
|:---------|:-----|:-------|
| 关键词覆盖 | 25% | 核心词密度/长尾词覆盖/品牌词注入 |
| 标题质量 | 20% | 关键词前置/长度合规/吸引力 |
| 内容质量 | 20% | 原创性/深度/可读性 |
| 标签优化 | 15% | 标签数量/相关性/热门标签覆盖 |
| 结构化数据 | 10% | meta描述/H标签/alt属性 |
| 用户体验 | 10% | 加载速度/移动适配/内链结构 |

## 验证步骤

SEO优化完成后执行以下验证:

| 步骤 | 验证项 | 验证方式 | 通过标准 |
|:-----|:-------|:---------|:---------|
| V1 | 外部API可用 | 检查关键词API返回 | 返回非空结果 |
| V2 | 关键词有效 | 检查搜索量>0 | 至少3个关键词有搜索量 |
| V3 | 优化后标题合规 | 重新审核优化标题 | 无敏感词+长度合规 |

## 异常处理

| 异常场景 | 处理方式 | 降级策略 | 错误码 |
|:---------|:---------|:---------|:-------|
| 内容为空 | 返回错误,提示输入内容 | 无法降级,必须提供内容 | SEO_ERR_01 |
| 关键词API超时 | 使用本地关键词库 | 从历史关键词数据中读取替代 | SEO_ERR_02 |
| 标题过长 | 自动截断到平台限制 | 按平台最大长度截断,保留核心关键词 | SEO_WARN_01 |
| 标签违规 | 自动过滤敏感标签 | 移除违规标签,用同义安全标签替换 | SEO_WARN_02 |
| 热搜API不可用 | 跳过热搜关键词辅助 | 仅使用BERT语义提取结果 | SEO_ERR_03 |
| BERT模型加载失败 | 使用规则匹配提取关键词 | 降级为正则+词频统计方式 | SEO_ERR_04 |
| 竞品URL无法访问 | 跳过竞品分析步骤 | 仅基于自身内容生成SEO方案 | SEO_WARN_03 |
| 搜索API不可用 | 返回追踪失败 | 标注"排名追踪不可用",仅输出优化方案 | SEO_ERR_05 |
| 搜索结果未找到URL | 记录为"未入榜" | 输出trend="未入榜"+增加权重建议 | SEO_WARN_04 |
| 历史排名数据缺失 | 视为首次追踪 | trend="新入榜",不显示change值 | SEO_WARN_05 |
| 品牌关键词为空 | 跳过品牌词注入 | 使用原始关键词方案 | SEO_WARN_06 |

## 依赖说明

### 运行环境
- **Agent平台**: Claude Code / Cursor / Codex / Gemini CLI / Windsurf 等支持SKILL.md的任意Agent
- **操作系统**: Windows / macOS / Linux
- **运行时**: 需要Agent支持exec(命令行执行)能力

### 第三方依赖
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| LLM API | API | 必需 | 任意LLM服务商,由Agent内置LLM提供 |
| BERT模型 | 模型 | 可选 | 语义关键词提取(加载失败降级为正则+词频统计) |
| 热搜数据API | API | 可选 | 热搜关键词(不可用时仅用BERT结果) |
| 搜索API | API | 可选 | 排名追踪(不可用时仅输出优化方案) |

### API Key 配置
- **LLM_API_KEY**: 必需(通常由Agent内置) - SEO优化/标题生成/关键词策略
- **SEARCH_API_KEY**: 可选 - 搜索排名追踪
- 配置方式: 在Agent的环境变量中设置

### 纯Markdown使用说明
BERT模型加载失败时降级为正则+词频统计。热搜API不可用时跳过热搜匹配。

只需将SKILL.md文件放入Agent的skills目录即可直接使用。
如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。

### 可用性分类
- **分类**: MD+EXEC
- **说明**: 纯Markdown,但需要exec能力(命令行执行),用于文件读写和命令调用
