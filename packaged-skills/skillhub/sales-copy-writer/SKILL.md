# 卖货文案手 v1.1.0

解决内容营销中"文案干瘪无转化"的核心痛点,基于FAB法则+情绪钩子+CTA话术生成高转化卖货文案。

## 核心能力

1. **FAB法则卖点提炼**: Feature(特性)+Advantage(优势)+Benefit(利益)三段式结构化提炼3-5个核心卖点,逻辑清晰有说服力
2. **6种情绪钩子生成**: 痛点共鸣(pain_resonance)/好奇悬念(curiosity_suspense)/紧迫感(urgency)/社交证明(social_proof)/对比反差(contrast)/权威背书(authority),按场景智能选择
3. **4种CTA话术注入**: follow(关注)/dm(私信)/order(下单)/group(加群),匹配不同转化目标
4. **5平台风格自动适配**: 小红书活泼种草/抖音口语化/公众号深度种草/微博简洁观点/电商促转化,自动调整字数与风格
5. **活动促销文案**: 结合紧迫感和社交证明,生成限时活动文案

## 使用流程

### Step 1: 环境准备
- 确认Agent已加载本SKILL.md,且支持exec工具
- 在Agent环境变量中配置 `LLM_API_KEY`(必需)
- 本Skill纯LLM驱动,无额外API依赖

### Step 2: 卖点提炼(FAB法则)
- 接收 `topic`(主题)和 `brand_info`(品牌信息,可选)
- 用LLM提炼3-5个FAB结构化卖点(Feature特性+Advantage优势+Benefit利益)
- 异常处理: 品牌信息缺失→生成无品牌版本,标注warning

### Step 3: 情绪钩子生成
- 根据 `hook_type` 选择钩子类型,生成营销文案框架
- 6种钩子类型: pain_resonance(痛点共鸣)/curiosity_suspense(好奇悬念)/urgency(紧迫感)/social_proof(社交证明)/contrast(对比反差)/authority(权威背书)
- 异常处理: 钩子类型无效→默认使用pain_resonance,标注warning

### Step 4: 平台适配+CTA注入
- 根据 `platform` 调整文案风格和字数(小红书300-500字/抖音50-100字/公众号800-1500字/微博140字内/电商200-400字)
- 根据 `cta_type` 注入对应CTA话术(follow关注/dm私信/order下单/group加群)
- 异常处理: 平台不支持→降级为通用文案,标注warning;CTA类型无效→默认使用follow

### Step 5: 输出完整营销文案
- 整合卖点、情绪钩子、平台适配文案和CTA
- 输出结构化结果: selling_points[]+hook+adapted_content+cta

## 输入格式

```json
{
  "topic": "AI代写文案服务",
  "brand_info": "品牌名:XX, slogan:让AI帮你写",
  "platform": "xiaohongshu",
  "hook_type": "pain_resonance",
  "cta_type": "order"
}
```

| 字段 | 类型 | 必填 | 说明 |
|:-----|:-----|:----:|:-----|
| topic | string | 是 | 内容主题/产品名称 |
| brand_info | string | 否 | 品牌信息(品牌名+slogan) |
| platform | string | 是 | 目标平台(xiaohongshu/douyin/wechat/weibo/ecommerce) |
| hook_type | string | 否 | 情绪钩子类型,默认pain_resonance |
| cta_type | string | 否 | CTA类型,默认follow |

## 异常处理

| 异常场景 | 原因 | 处理方式 |
|:---------|:-----|:---------|
| 主题为空 | 未提供topic参数 | 返回success=false, error="主题不能为空",无法降级 |
| 平台不支持 | 输入的platform不在5个支持范围内 | 降级为通用文案模板,标注warning,不针对平台优化 |
| 品牌信息缺失 | 未提供brand_info | 生成无品牌版本,标注warning,聚焦产品卖点 |
| 钩子类型无效 | hook_type不在6种支持范围内 | 默认使用pain_resonance,标注warning |
| CTA类型无效 | cta_type不在4种支持范围内 | 默认使用follow,标注warning |
| LLM调用失败 | LLM服务不可用或超时 | 返回success=false, error="文案生成LLM调用失败" |
| 卖点提炼失败 | LLM返回格式异常或为空 | 使用预设卖点模板(特性/优势/利益通用模板) |
| 文案过长 | 生成内容超出平台字数限制 | 按平台最大长度截断,保留核心卖点和CTA |
| LLM_API_KEY未配置 | 环境变量缺失 | 返回error,提示配置环境变量 |

## 常见问题

### Q1: 如何开始使用卖货文案手?
A: 两步启动:(1)将SKILL.md放入Agent的skills目录;(2)确认Agent环境变量已配置LLM_API_KEY(通常由Agent内置)。调用时传入topic(必填)和platform(必填)即可生成文案,其他参数可选。纯LLM驱动,无额外API依赖。

### Q2: 6种情绪钩子如何选择?
A: 按场景选择:(1)痛点共鸣(pain_resonance)→解决用户痛点的产品,通用性强;(2)好奇悬念(curiosity_suspense)→新品/黑科技类,引发好奇;(3)紧迫感(urgency)→限时促销/活动;(4)社交证明(social_proof)→已有大量用户的成熟产品;(5)对比反差(contrast)→性价比突出的产品;(6)权威背书(authority)→专业度要求高的产品(如教育/医疗)。

### Q3: 5平台风格有什么区别?
A: 小红书活泼种草(emoji丰富,300-500字)、抖音口语化(前3秒强hook,50-100字)、微信公众号深度种草(逻辑清晰,800-1500字)、微博简洁观点(话题标签,140字内)、电商平台卖点突出(促转化,200-400字)。选择平台后会自动调整文案风格和字数。

### Q4: 品牌信息缺失会影响文案质量吗?
A: 会一定程度影响。品牌信息缺失时,系统生成无品牌版本,聚焦产品卖点,但缺少品牌辨识度。建议尽量提供brand_info(品牌名+slogan),让文案更有品牌调性。如暂无品牌信息,可后续手动补充品牌元素。

### Q5: 生成的文案可以直接发布吗?
A: 文案结构完整(卖点+钩子+正文+CTA),可直接发布或微调后发布。建议发布前:(1)核对产品参数和价格是否准确;(2)检查CTA链接是否有效;(3)根据账号调性微调语气;(4)补充真实用户评价或数据增强说服力。

## 错误处理


| 错误场景 | 原因 | 处理方式 |
|---------|------|---------|
| LLM响应超时或无响应 | 网络延迟或模型负载过高 | 执行ping命令测试网络连通性,检查防火墙和代理设置连接，执行ping命令测试网络连通性,检查防火墙和代理设置连接后重新执行命令请求；确认Agent平台LLM服务正常 |
| 输入内容格式不正确 | 用户输入不符合skill预期格式 | 检查输入是否符合skill使用说明中的格式要求，参考示例章节 |
| 执行结果与预期不符 | 指令描述不够明确或上下文不足 | 提供更详细的指令描述，补充必要的上下文信息 |
| 命令执行失败 | 运行环境不满足要求或权限不足 | 确认运行环境符合依赖说明中的要求；检查命令权限设置 |

## 已知限制

- **平台覆盖**: 仅支持5个国内平台(小红书/抖音/公众号/微博/电商),不支持Instagram/TikTok国际版/LinkedIn等海外平台风格
- **字数控制**: LLM生成的字数可能偏离建议范围(±15%),需人工微调至平台限制内
- **卖点准确性**: FAB卖点基于LLM理解,可能与产品实际参数有偏差,发布前需人工核对
- **品牌调性**: 无品牌信息时生成的文案偏通用,缺少品牌独特调性,需补充品牌信息后再生或人工调整
- **合规限制**: 不生成虚假宣传/绝对化用语/医疗承诺等违规文案,但平台规则可能更新,发布前建议复核平台最新规范
- **多语言**: 仅支持中文文案生成,不支持英文/其他语种
- **图片素材**: 仅生成文字文案,不包含配图建议(cover_suggestion需配合其他工具)

## 安全

### API Key 零暴露原则
- **环境变量注入**: LLM_API_KEY必须通过Agent环境变量注入,严禁在SKILL.md、配置文件或代码中硬编码
- **本文件无Key示例**: 本SKILL.md中不包含任何真实或示例API Key,所有引用均使用 `$env:LLM_API_KEY` 占位
- **日志脱敏**: exec工具记录日志时,自动过滤包含"key"/"token"/"secret"字段的值

### 内容安全
- **广告合规**: 自动过滤虚假宣传/绝对化用语(最/第一/唯一)/医疗承诺等违规表述
- **平台规范**: 适配各平台广告规范,小红书避免硬广、抖音避免违禁词、电商遵守广告法
- **版权提示**: 生成的文案为原创内容,但如引用他人案例/数据,需标注来源,商业使用前建议查重
- **品牌授权**: 使用品牌信息(brand_info)时,需确认用户有权使用该品牌,避免侵犯他人商标权
