# 3秒留人术

## 使用场景

| 场景 | 触发条件 | 说明 |
|:-----|:---------|:-----|
| **视频创作前设计** | 短视频策划阶段 | 根据内容主题生成前3秒留人钩子 |
| **完播率优化** | 视频完播率低于预期 | 诊断开头问题,生成优化方案 |
| **多平台适配** | 同一内容跨平台发布 | 针对不同平台生成差异化开场钩子 |
| **钩子效果追踪** | 视频发布后 | 追踪3秒留存率和完播率,优化后续钩子策略 |
| **A/B测试** | 钩子方案对比 | 生成多个钩子候选,支持A/B测试选择最优 |

## 工作流

### 步骤1: 接收内容主题与参数

接收输入参数: `content_topic`(内容主题)、`content_keywords`(内容关键词列表)、`platform_list`(目标平台列表)、`hook_types`(钩子类型列表,可选,默认全部6类)。

验证参数完整性: content_topic非空、content_keywords至少3个、platform_list至少1个。参数不完整时返回错误提示,列出缺失项。

### 步骤2: AI生成6类钩子候选

调用LLM为每类钩子生成2-3个候选文案,每个候选控制在15字以内(前3秒口播字数):

| 钩子类型 | 标识 | 策略 | 示例 |
|:---------|:-----|:-----|:-----|
| **悬念钩子** | curiosity_suspense | 制造信息差,引发好奇 | "你绝对想不到接下来发生了什么..." |
| **痛点钩子** | pain_resonance | 直击用户痛点,引发共鸣 | "你是不是也经常对着镜头不知道说什么?" |
| **反差钩子** | contrast_reversal | 打破预期,制造惊讶 | "别再这样做饭了,99%的人都搞错了" |
| **情绪钩子** | emotional_trigger | 激发强烈情绪反应 | "看到第3秒我直接泪崩了" |
| **视觉钩子** | visual_impact | 用画面冲击力留住观众 | "这个画面我看了10遍还是觉得震撼" |
| **提问钩子** | question_hook | 用问题引发思考 | "为什么富人越来越富?答案出人意料" |

### 步骤3: 合规性检测

对每个钩子候选文案执行内容合规性检测:
- 通过 -> 进入下一步
- 不通过 -> 标记违规词并重新生成该候选(最多重试2次)
- 2次重试仍不通过 -> 丢弃该候选,记录告警

### 步骤4: 一致性校验

对每个通过合规检测的钩子进行内容一致性校验:
- 计算钩子关键词与内容关键词的交集数量
- 计算相似度分数: `similarity = 交集数 / max(钩子关键词数, 内容关键词数)`
- 校验标准: 交集>=3个 且 similarity>=0.6
- 不通过 -> 重新生成该钩子(从步骤2重新开始,最多3次)
- 3次仍不通过 -> 保留相似度最高的候选,标记为低一致性

### 步骤5: 按平台分类适配

根据平台特性对钩子进行差异化适配:

| 平台分类 | 平台 | 钩子策略 | 前3秒要点 |
|:---------|:-----|:---------|:---------|
| **强留人型** | 抖音/快手 | 悬念+反差,3秒内必须出核心信息 | 口播+字幕+画面同步冲击 |
| **内容型** | 小红书/B站 | 痛点+价值预告,3秒说明看完能获得什么 | 清晰价值点+视觉吸引 |
| **深度型** | 微信视频号/知乎 | 提问+权威感,3秒建立专业可信 | 专业场景+逻辑引入 |
| **电商型** | 电商平台 | 视觉+卖点,3秒展示产品核心亮点 | 产品特写+使用场景 |

### 步骤6: 完播率优化建议

基于钩子类型和内容特征,生成完播率优化建议:

1. 分析钩子与正文的衔接流畅度
2. 评估前3秒信息密度是否合理
3. 检查钩子承诺是否在正文中兑现
4. 生成留存曲线优化建议:
   - 0-3秒: 钩子冲击力是否足够
   - 3-15秒: 价值前置,快速兑现钩子承诺
   - 15-30秒: 节奏变化,防止流失
   - 结尾: 引导互动,提升复播率

### 步骤7: 钩子效果追踪

钩子生成并植入视频后,记录使用关系用于后续留存分析:

1. 记录内容: hook_id + content_id + 目标平台 + 钩子类型 + 记录时间
2. 后续更新留存数据(3秒留存率/完播率/互动率/复播率)
3. 查询单个钩子统计和全部钩子对比
4. 追踪数据存储为JSON文件,路径: `hook_tracking/{hook_id}.json`

**留存率计算**:
- 3秒留存率 = 3秒后仍在观看人数 / 总曝光人数
- 完播率 = 完整观看人数 / 总曝光人数
- 互动率 = (点赞+评论+转发) / 总曝光人数

**告警阈值**:
- 3秒留存率 < 30% -> 钩子吸引力不足,建议重新设计
- 完播率 < 15% -> 内容节奏问题,建议优化中段
- 互动率 < 3% -> 引导不足,建议加强结尾CTA

## 输入格式

```json
{
  "content_topic": "少儿编程启蒙教程",
  "content_keywords": ["少儿编程", "编程启蒙", "Scratch", "逻辑思维", "AI教育"],

  "hook_types": ["curiosity_suspense", "pain_resonance", "question_hook"],
  "video_duration": 45,
  "generate_retention_advice": true
```

| 字段 | 类型 | 必填 | 说明 |
|:-----|:-----|:----:|:-----|
| content_topic | string | 是 | 内容主题 |
| content_keywords | string[] | 是 | 内容关键词列表(>=3个) |
| platform_list | string[] | 是 | 目标平台列表(>=1个) |
| hook_types | string[] | 否 | 钩子类型列表,默认全部6类 |
| video_duration | int | 否 | 视频时长(秒),用于完播率优化建议 |
| generate_retention_advice | bool | 否 | 是否生成完播率优化建议,默认true |

## 异常处理

| 异常类型 | 错误代码 | 处理方式 |
|:---------|:---------|:---------|
| 内容合规检测服务不可用 | COMPLIANCE_UNAVAILABLE | strict模式返回失败,不生成任何钩子 |
| 一致性校验不通过 | CONSISTENCY_FAILED | 重新生成(最多3次),3次后保留最高相似度候选 |
| LLM生成失败 | LLM_GENERATION_FAILED | 返回失败,记录错误日志 |
| 参数不完整 | INVALID_PARAMS | 返回缺失参数列表,提示补充 |
| 完播率数据缺失 | RETENTION_DATA_MISSING | 跳过完播率建议,仅输出钩子候选 |
| 钩子效果追踪服务不可用 | TRACK_UNAVAILABLE | 返回钩子文案但不持久化追踪数据,提示手动记录 |

## 依赖说明

### 运行环境
- **Agent平台**: Claude Code / Cursor / Codex / Gemini CLI / Windsurf 等支持SKILL.md的任意Agent
- **操作系统**: Windows / macOS / Linux
- **运行时**: 需要Agent支持exec(命令行执行)能力

### 第三方依赖
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| LLM API | API | 必需 | 任意LLM服务商,由Agent内置LLM提供 |
| 内容合规检测 | API | 可选 | 合规性检测(不可用时strict模式返回失败) |
| JSON文件存储 | 文件系统 | 可选 | exec工具创建hook_tracking/目录 |

### API Key 配置
- **LLM_API_KEY**: 必需(通常由Agent内置) - 6类钩子候选生成/完播率优化建议
- 配置方式: 在Agent的环境变量中设置

### 纯Markdown使用说明
合规检测服务不可用时strict模式返回失败。追踪服务不可用时返回钩子文案但不持久化。

只需将SKILL.md文件放入Agent的skills目录即可直接使用。
如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。

### 可用性分类
- **分类**: MD+EXEC
- **说明**: 纯Markdown,但需要exec能力(命令行执行),用于文件读写和命令调用
