## 核心能力

此技能赋予 AI 自主上网搜索和学习的能力，使其能够：

* 主动从互联网获取最新信息
* 学习新知识并整合到回答中
* 查询新闻、天气、事件等信息
* 获取特定主题的详细资料
* 多源交叉验证信息的准确性
* 结构化组织获取的知识并标注来源

## 搜索策略方法论

### 查询构建原则

有效的搜索查询是获取高质量信息的前提。系统采用以下策略构建搜索查询：

| 策略 | 说明 | 示例 |
|:-----|:------|:------|
| 关键词精准化 | 使用专业术语而非口语表达 | "React Server Components hydration" 而非 "React新功能怎么用" |
| 时间限定 | 添加时间范围限定获取最新结果 | "Rust 1.80 release notes 2026" |
| 排除干扰 | 使用减号排除不相关结果 | "Python asyncio -async/await教程" 排除基础教程 |
| 站点定向 | 在特定权威站点内搜索 | "CVE-2026-1234 site:nvd.nist.gov" |
| 迭代深化 | 从宽泛到具体逐步缩小范围 | 先搜"K8s网络策略"，再搜"K8s NetworkPolicy default deny" |

### 迭代搜索流程

```
初始查询 → 评估结果质量
    │           │
    │     ┌─────┴─────┐
    │     │ 结果充足？ │
    │     └─────┬─────┘
    │      是   │   否
    │     ┌─────┴─────┐
    │     │ 提取信息   │ ← 调整查询关键词
    │     └─────┬─────┘   扩大/缩小范围
    │           │
    └───────────┘
```

## 信息源评估标准（CRAAP框架）

系统对搜索到的每个信息源进行以下五维评估，只有通过评估的来源才会被采纳：

| 维度 | 评估问题 | 权重 | 高质量示例 | 低质量示例 |
|:-----|:------|:------|:------|:------|
| **C**urrency (时效性) | 信息是否最新？是否有过时内容？ | 20% | 2026年发布的官方文档 | 2018年的技术教程 |
| **R**elevance (相关性) | 信息是否直接回答用户问题？ | 20% | 针对性强的技术文档 | 泛泛而谈的博客文章 |
| **A**uthority (权威性) | 作者/发布者是否可信？ | 25% | 官方文档、学术论文、知名媒体 | 匿名论坛帖子、内容农场 |
| **A**ccuracy (准确性) | 信息是否有证据支撑？可验证？ | 20% | 含数据来源、引用链接 | 无来源的主观断言 |
| **P**urpose (目的性) | 信息是否存在偏见或商业目的？ | 15% | 中立的技术评测 | 软文、付费推广 |

**来源可信度分级**:

| 级别 | 来源类型 | 信任度 | 示例 |
|:-----|:------|:------|:------|
| S级 | 官方文档/标准规范/学术论文 | 95% | MDN Web Docs, RFC, arXiv |
| A级 | 权威媒体/知名技术社区 | 85% | Hacker News, Stack Overflow高赞回答 |
| B级 | 个人技术博客/教程网站 | 70% | Medium技术文章, Dev.to |
| C级 | 社交媒体/论坛讨论 | 50% | Reddit讨论, Twitter帖子 |
| D级 | 内容农场/聚合站点 | 20% | 自动生成内容, SEO垃圾站 |

## 信息提取与综合流程

### 步骤1：页面抓取与解析

系统访问搜索结果的URL，提取以下内容：
- **正文内容**: 去除导航、广告、侧边栏等噪声
- **结构化数据**: 表格、列表、代码块等格式化内容
- **元信息**: 标题、作者、发布日期、最后更新时间
- **引用链接**: 文中引用的外部资源链接

### 步骤2：关键信息提取

从抓取内容中提取与用户查询相关的关键信息：
- 事实陈述（数据、日期、事件）
- 观点判断（专家评价、用户反馈）
- 操作步骤（教程、指南）
- 代码示例（技术文档）

### 步骤3：多源交叉验证

对同一主题的信息，从至少2-3个独立来源进行交叉验证：

```
来源A: "K8s 1.30支持Sidecar Container"
来源B: "Kubernetes 1.30引入原生Sidecar支持"
来源C: K8s官方Changelog确认该特性
→ 结论: 高置信度确认，标注三个来源
```

当来源之间存在矛盾时：
- 优先采信更高级别的来源（官方文档 > 博客）
- 标注分歧观点，不强行统一
- 向用户说明信息存在争议

### 步骤4：知识综合与结构化

将验证后的信息综合为结构化输出：

```json
{
  "query": "K8s 1.30 Sidecar Container特性",
  "summary": "Kubernetes 1.30正式引入原生Sidecar Container支持...",
  "key_points": [
    "Sidecar容器通过restartPolicy: Always字段标识",
    "在主容器启动前启动，主容器退出后终止",
    "解决了Job/Pod生命周期管理问题"
  ],
  "sources": [
    {"url": "https://kubernetes.io/...", "level": "S", "title": "K8s 1.30 Release Notes"},
    {"url": "https://blog.example.com/...", "level": "B", "title": "Sidecar实践指南"}
  ],
  "confidence": 0.95,
  "last_verified": "2026-07-24"
}
```

## 降级处理与异常恢复

### 常见访问障碍及处理

| 障碍类型 | 症状 | 处理策略 |
|:-----|:------|:------|
| 付费墙 | 内容被遮挡，要求订阅 | 搜索相同信息的免费来源（如Google Cache、archive.org） |
| 地理限制 | 返回403或地区不可用 | 使用替代来源，标注"原文在部分地区不可访问" |
| 网站宕机 | 连接超时或502 | 查询Web Archive缓存，或搜索镜像站点 |
| 反爬机制 | 返回403或验证码 | 降低访问频率，使用搜索引擎缓存页面 |
| 内容删除 | 404 Not Found | 查询Web Archive历史快照，搜索转载内容 |

### 信息不完整时的处理

当无法获取完整信息时，系统会：
1. 标注信息缺口（"以下信息未能从公开来源获取"）
2. 提供部分可验证的信息
3. 建议用户参考的备选来源
4. 标注信息的时效性（"截至2026年7月的公开信息"）

## 引用规范

系统输出的信息遵循以下引用规范：

- **内联引用**: 在信息后标注来源编号，如"K8s 1.30支持Sidecar容器 [1]"
- **来源列表**: 在回答末尾列出所有引用来源，含URL、标题、访问日期
- **置信度标注**: 对关键数据标注置信度（高/中/低）
- **时效声明**: 注明信息的获取时间和最后更新时间

```
来源:
[1] Kubernetes 1.30 Release Notes - https://kubernetes.io/... (访问: 2026-07-24)
[2] Sidecar Container实践指南 - https://blog.example.com/... (访问: 2026-07-24)
```

## 常见问题

### Q1: 如何开始使用Web Learner？
A: 直接提出你想了解的问题即可，例如"2026年最新的AI大模型有哪些"、"查询今天上海的天气"、"学习Rust的所有权系统"。系统会自动搜索互联网、抓取页面、交叉验证并综合输出结构化的回答，附带信息来源链接。

### Q2: 搜索结果的信息是否可靠？
A: 系统采用CRAAP框架（时效性、相关性、权威性、准确性、目的性）对每个信息源进行五维评估，并按S/A/B/C/D五级标注来源可信度。关键信息会从至少2-3个独立来源交叉验证后才输出。每个知识点都标注置信度（高/中/低），低于阈值的来源不会被采纳。

### Q3: 遇到付费墙或无法访问的页面怎么办？
A: 系统会自动降级处理：（1）搜索相同信息的免费来源；（2）查询Google Cache或Web Archive的缓存版本；（3）搜索转载该内容的站点。如果所有途径都无法获取，系统会标注"该信息来源需要付费/受限访问"，并提供已获取的部分信息。

### Q4: 能否获取实时数据（股价、天气、新闻）？
A: 可以。系统对实时性数据采用直接查询策略，优先访问提供实时API的来源。天气数据从气象服务获取，股价从财经数据源获取，新闻从主流新闻聚合源获取。实时数据会在回答中标注获取时间，并建议用户以官方数据源为准做最终确认。

### Q5: 如何确保获取的信息是最新的？
A: 系统在搜索时默认添加时间限定，优先展示最近6个月的结果。对于技术版本信息，会验证最新版本号和发布日期。对于新闻事件，按发布时间倒序排列。回答中会明确标注信息的发布日期和获取日期，让用户判断时效性。

### Q6: 获取的信息会被存储吗？如何保护隐私？
A: 系统仅在当前会话中临时存储获取的信息用于综合分析，不会持久化存储用户的搜索历史。引用的来源URL会包含在回答中供用户验证，但不关联用户身份信息。建议用户对敏感查询（如个人健康、财务信息）自行评估是否适合通过本技能获取。
