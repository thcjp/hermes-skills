# 爆款拆解师

## 使用场景

1. **周度爆款复盘**: 每周分析过去7天内容数据,提取爆款特征和公式,指导下周内容策略
2. **新平台拓展**: 进入新平台前,分析该平台已有爆款内容,快速掌握内容偏好
3. **内容模板优化**: 基于爆款公式更新创作模板,提升内容爆款概率
4. **竞品爆款学习**: 分析竞品爆款内容,提取可复用的创作模式
5. **创作指南迭代**: 将爆款公式沉淀到创作指南,持续优化内容方向

---

## 爆款定义标准

| 指标 | 爆款阈值 | 超级爆款阈值 |
|:------|:---------|:--------------|
| 播放量 | > 10万 | > 100万 |
| 点赞数 | > 5000 | > 5万 |
| 评论数 | > 500 | > 5000 |
| 转发数 | > 200 | > 2000 |
| **满足项数** | >=3项 | >=3项 |

> 阈值基于运营经验估算,可根据平台和行业调整。

---

## 工作流

### 步骤1: 接收分析请求

接收分析参数: `time_range`(时间范围)、`platform`(目标平台)、`min_views`(最小播放量)、`extract_formulas`(是否提取爆款公式)、`update_guidelines`(是否更新创作指南)。

验证时间范围有效性,无效时返回错误。

### 步骤2: 数据收集

1. 通过数据接口从各平台获取内容数据
2. 按爆款阈值过滤候选集(播放>10万+点赞>5000+评论>500+转发>200,满足3项)
3. 输出候选内容列表

### 步骤3: 特征提取

提取3类特征,形成特征字典:

| 特征类别 | 特征项 | 说明 |
|:---------|:-------|:-----|
| **内容特征** | 类型/时长/封面/标题 | 内容本身属性 |
| **创作特征** | 发布时间/BGM/场景/视角/节奏 | 创作手法属性 |
| **互动特征** | 播放量/点赞数/评论数/转发数/收藏数 | 用户反馈数据 |

通过LLM协作分析,提取关键特征组合。

### 步骤4: 模式识别

1. LLM分析所有爆款内容的共同特征
2. 识别正相关特征组合(同时出现时爆款概率高)
3. 识别反模式特征(出现时爆款概率低)
4. 生成爆款模式描述

### 步骤5: 爆款公式生成

生成3-5个可复制的爆款公式:

**公式格式**: `[因素A] + [因素B] + [因素C] -> 高概率爆款`

每个公式附带:
- `confidence`: 置信度(0-1,基于样本量和特征一致性)
- `applicable_platforms`: 适用平台列表
- `sample_count`: 支撑该公式的样本数量

### 步骤6: 知识沉淀

1. 将分析结果写入分析记录存储(JSON文件格式)
2. 可选更新创作指南,将爆款公式纳入创作参考
3. 输出记录结果

### 步骤7: 返回分析报告

输出完整分析报告: 爆款清单 + 特征统计 + 爆款公式 + 行动建议

## 输入格式

```json
{
  "time_range": "7d",
  "platform": "all",
  "min_views": 100000,
  "extract_formulas": true,
  "update_guidelines": false
}
```

| 字段 | 类型 | 必填 | 说明 |
|:-----|:-----|:----:|:-----|
| time_range | string | 是 | 时间范围: 7d(7天)/30d(30天)/90d(90天) |
| platform | string | 否 | 目标平台: all(全部)/douyin/xiaohongshu/bilibili等,默认all |
| min_views | int | 否 | 最小播放量阈值,默认100000 |
| extract_formulas | bool | 否 | 是否提取爆款公式,默认true |
| update_guidelines | bool | 否 | 是否更新创作指南,默认false |

## 异常处理

| 异常编号 | 错误代码 | 触发条件 | 处理方式 |
|:---------|:---------|:---------|:---------|
| VIR-ERR-01 | NO_DATA | 无数据可分析 | 返回空结果,提示先发布内容 |
| VIR-ERR-02 | DATA_SOURCE_ERROR | 数据源不可用 | 使用缓存的上次分析结果,标记数据陈旧 |
| VIR-ERR-03 | INSUFFICIENT_SAMPLES | 爆款样本不足(<3) | 扩大时间范围或降低阈值重新分析 |
| VIR-ERR-04 | FORMULA_GENERATION_ERROR | LLM公式生成失败 | 返回特征统计(不含公式),不影响基础功能 |
| VIR-ERR-05 | KNOWLEDGE_SAVE_ERROR | 知识沉淀写入失败 | 返回分析结果但不持久化,提示手动保存 |

## 依赖说明

### 运行环境
- **Agent平台**: Claude Code / Cursor / Codex / Gemini CLI / Windsurf 等支持SKILL.md的任意Agent
- **操作系统**: Windows / macOS / Linux
- **运行时**: 需要Agent支持exec(命令行执行)能力

### 第三方依赖
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| LLM API | API | 必需 | 任意LLM服务商,由Agent内置LLM提供 |
| 内容数据接口 | API | 可选 | 各平台内容数据(可手动输入) |
| JSON文件存储 | 文件系统 | 可选 | exec工具保存分析记录 |

### API Key 配置
- **LLM_API_KEY**: 必需(通常由Agent内置) - 特征提取/模式识别/爆款公式生成
- 配置方式: 在Agent的环境变量中设置

### 纯Markdown使用说明
数据源不可用时使用缓存的上次分析结果。LLM公式生成失败时返回特征统计(不含公式)。

只需将SKILL.md文件放入Agent的skills目录即可直接使用。
如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。

### 可用性分类
- **分类**: MD+EXEC
- **说明**: 纯Markdown,但需要exec能力(命令行执行),用于文件读写和命令调用

## 示例

### 示例1: 抖音平台周度爆款分析

**输入**:
```json
{
  "time_range": "7d",
  "platform": "douyin",
  "min_views": 100000,
  "extract_formulas": true,
  "update_guidelines": false
}
```

**输出**:
- 分析周期: 2026-05-07 ~ 2026-05-14
- 总分析内容数: 120
- 发现爆款数: 5
- 爆款率: 4.2%
- TOP爆款: 教程类视频,35万播放,1.8万点赞,1200评论,800转发
- 特征统计: 常见类型(教程/生活妙招/旅行),常见时长(30-60秒),常见BGM(轻松),常见视角(第一人称)
- 爆款公式1: [教程类]+[30秒内]+[轻松BGM]+[第一人称] -> 高概率爆款 (置信度0.82, 适用抖音/快手)
- 爆款公式2: [痛点引入]+[3步解决]+[结果展示] -> 高转化爆款 (置信度0.78, 适用抖音/小红书)
- 行动建议: 增加教程类内容占比至35%/控制视频时长在30秒内/使用轻松BGM风格
