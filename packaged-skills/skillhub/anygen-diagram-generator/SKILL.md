# AnyGen 图表生成器

通过 AnyGen CLI 调用 `www.anygen.io` 服务端绘图引擎，将文本描述转换为流程图、架构图、时序图、思维导图等可视化图表。核心操作类型为 `smart_draw`，由 `anygen-workflow-generate` 依赖技能编排完整生成流程.

## 认证配置

AnyGen CLI 提供两种认证方式，任选其一：

**方式一：API Key 直接认证**

```bash
anygen auth login --api-key sk-your-api-keyxxxxxxxxxxxx
```

也可通过环境变量持久化配置：

```bash
export ANYGEN_API_KEY=sk-your-api-keyxxxxxxxxxxxx
```

**方式二：浏览器交互认证**

```bash
anygen auth login --no-wait
```

执行后 CLI 会输出一个授权链接，引导用户在浏览器中完成登录授权。授权完成后即可在终端正常使用所有命令。当任何命令返回认证错误时，重新执行此命令并等待用户完成浏览器授权，授权成功后重试原命令.

## 支持的图表类型

smart_draw 操作支持以下图表类型，由智能引擎根据文本描述自动识别，也可在描述中显式指定：

- **流程图（Flowchart）**：业务流程、审批流程、算法流程，支持判断分支、循环、并行、子流程等结构
- **系统架构图（Architecture）**：微服务架构、部署架构、技术栈架构，支持分层布局与组件依赖关系
- **时序图（Sequence）**：API 交互、协议通信、服务调用时序，支持同步/异步消息、参与者、激活区
- **思维导图（Mind Map）**：知识体系、产品规划、头脑风暴，支持放射状层级展开
- **组织结构图（Org Chart）**：团队架构、部门层级、汇报关系
- **状态机图（State Machine）**：订单状态、任务状态、协议状态转换
- **ER 图（Entity Relationship）**：数据库表结构、领域模型实体关系

### 场景一：技术文档架构图配图

在编写技术设计文档、API 文档、部署文档时，需要将文字描述的系统架构、模块依赖、数据流向转化为可视化图表。用户提供架构文字描述，本技能通过 smart_draw 操作生成架构图，直接嵌入文档.

产品需求评审或业务流程梳理时，需要将业务流程可视化为标准流程图。用户提供业务步骤描述，本技能生成包含判断分支、循环、并行等结构的流程图，辅助团队评审讨论.
### 场景三：API 交互时序图

设计或文档化 API 接口时，需要绘制客户端与服务端的交互时序图。用户提供接口调用顺序与参与方描述，本技能生成标准 UML 时序图，明确展示消息流向与时序关系.
### 场景四：知识体系思维导图

整理学习笔记、产品规划、竞品分析等结构化知识时，需要将层级关系可视化为思维导图。用户提供主题与分支结构描述，本技能生成放射状思维导图，便于全局把握知识脉络.
## 使用案例

### 案例一：微服务架构图生成

用户描述系统由网关层、用户服务、订单服务、支付服务、消息队列、数据库组成，各服务通过网关统一入口对外提供 REST API，订单服务异步调用支付服务并写入消息队列。本技能调用 smart_draw 操作，生成包含服务节点、调用关系、数据流向的微服务架构图。生成结果可直接下载并嵌入架构设计文档.
### 案例二：订单退款流程图生成

用户描述电商退款流程包含用户发起退款申请、客服审核、风控校验、退款执行、通知用户五个阶段，其中客服审核与风控校验存在拒绝分支，退款执行失败需重试。本技能生成包含起止节点、判断菱形、重试循环的标准流程图，每个节点附带简短说明文本，便于业务团队评审流程合理性.
### 案例三：OAuth2 授权时序图生成

用户描述 OAuth2 授权码模式的交互流程涉及客户端、授权服务器、资源服务器、资源所有者四方，包含授权请求、用户确认、颁发授权码、换取令牌、访问资源等步骤。本技能生成标准 UML 时序图，按时间轴展示四方消息交互顺序，明确同步与异步调用的时序关系，作为接口对接文档的核心配图.
## 命令执行规范

生成图表时遵循以下执行顺序：

1. 检查 AnyGen CLI 认证状态，未认证则执行 `anygen auth login` 系列命令

5. 将结果返回用户，必要时提供图表内容说明与后续编辑建议

## 图表描述编写规范

smart_draw 操作的生成质量高度依赖文本描述的清晰度。编写描述时遵循以下规范可提升生成效果：

**结构化描述**：按"参与方/节点 → 关系/连线 → 约束/属性"的顺序组织描述。先列出图表中所有参与者或节点，再描述它们之间的连接关系，最后补充属性约束（如方向、标签、分组）.
**显式标注图表类型**：虽然 smart_draw 能自动识别类型，但在描述开头显式声明（如"生成一个微服务架构图"）可减少识别歧义，特别是描述中包含多种图表特征交叉时.
**控制节点规模**：单次生成的节点数量建议控制在合理范围内（通常 20 个以内）。节点过多会导致布局拥挤、连线交叉，降低可读性。大规模系统建议拆分为总览图加多张子图.
**标注关键关系**：对于架构图与时序图，明确标注调用方向（单向/双向）、调用类型（同步/异步）、数据流向。对于流程图，明确标注判断条件与分支走向。这些标注直接影响 smart_draw 的连线渲染.
## 异常处理

### 认证失败（auth error）

命令返回认证错误时，说明当前未登录或 API Key 失效。处理方式：执行 `anygen auth login --no-wait`，引导用户在浏览器完成授权，授权成功后重新执行原命令。若使用 API Key 方式，检查 `ANYGEN_API_KEY` 环境变量是否正确设置，或重新执行 `anygen auth login --api-key` 命令.
### 依赖技能未安装

执行 smart_draw 操作时报错提示 `anygen-workflow-generate` 技能不存在。处理方式：根据当前 Agent 平台执行 `anygen skill install --platform <平台标识> -y` 完成安装，安装完成后图表生成请求。安装前确认网络可达 AnyGen 技能仓库.
### 网络连接超时

调用 `www.anygen.io` 服务端时出现连接超时或不可达。处理方式：检查本机网络连接与代理配置，确认能正常访问外网。若处于受限网络环境，配置 HTTP 代理后。服务端渲染大图可能耗时较长，适当延长请求超时阈值.
### smart_draw 操作参数缺失

发起 smart_draw 请求时未提供足够的图表描述信息，导致服务端无法识别目标图表类型。处理方式：补充图表类型（流程图/架构图/时序图等）、参与方、关系描述等关键信息后重新发起请求。描述越具体，生成质量越高.
### 图表生成结果与预期不符

生成的图表在结构、节点、关系上与用户描述存在偏差。处理方式：在原描述基础上补充约束条件（如节点数量、层级深度、布局方向），或明确指定图表类型后重新生成。复杂图表建议拆分为多个子图分别生成再组合.
### CLI 未安装或版本过低

执行 anygen 命令时提示命令不存在或功能不可用。处理方式：参照 AnyGen 官方文档安装最新版 CLI，或升级已安装版本。版本过低可能导致 smart_draw 操作不被支持，升级后.
### 服务端渲染失败

服务端返回渲染错误，通常由图表描述过于复杂或包含不支持的元素引起。处理方式：简化图表描述，减少节点数量与嵌套层级，拆分为多个简单图表分别生成。若持续失败，记录错误码并联系 AnyGen 支持团队.
### 浏览器授权未完成

执行 `anygen auth login --no-wait` 后用户未在浏览器完成授权，导致后续命令持续返回认证错误。处理方式：重新执行登录命令获取新的授权链接，提醒用户在有效时间内完成浏览器授权操作，授权成功后再执行业务命令.
## 常见问题

### Q1：AnyGen CLI 支持哪些操作系统？

AnyGen CLI 支持 Windows、macOS、Linux 三大主流操作系统。安装方式与系统相关，具体安装步骤参考 AnyGen 官方文档对应平台的说明。CLI 运行依赖 Node.js 运行时环境，安装前请确认 Node.js 版本满足最低要求.
### Q2：API Key 认证与浏览器认证有什么区别？

API Key 认证适合自动化脚本与无头环境，通过 `--api-key` 参数或 `ANYGEN_API_KEY` 环境变量直接传入密钥，无需人工交互。浏览器认证适合首次使用或交互式终端，通过 `--no-wait` 参数启动浏览器授权流程，适合不希望暴露 API Key 的场景。两种方式功能等价，按使用环境选择即可.
### Q3：生成的图表支持哪些输出格式？

AnyGen 服务端渲染支持 PNG、SVG、PDF 等常见图片格式。具体可用格式取决于 smart_draw 操作的配置参数与当前 AnyGen 账户的权限等级。生成的图片可直接下载到本地或通过链接在线访问，便于嵌入文档与分享.
### Q4：如何生成复杂的分页或分层图表？

对于节点数量超过单图合理展示范围的复杂图表，建议将整体结构拆分为多个子图：先生成总览图展示模块间关系，再为每个模块单独生成详细子图。通过描述中明确标注层级与模块边界，smart_draw 操作会按层级布局渲染。拆分生成比单图承载所有节点效果更好.
### Q5：图表生成后可以编辑修改吗？

生成的图表以图片形式输出，不能直接在 AnyGen CLI 中编辑。若需调整，修改原始文本描述后重新执行 smart_draw 操作重新生成。建议保留原始描述文本，便于后续迭代修改时基于原文调整描述重新生成，避免从零开始描述.
### Q6：智能绘图（smart_draw）与手动指定图表类型有什么区别？

smart_draw 操作由智能引擎根据文本描述自动识别图表类型与布局，无需用户显式指定。对于描述明确、图表类型清晰的场景，也可在描述中显式指定图表类型（如"生成流程图"），引擎会按指定类型渲染。smart_draw 适合快速原型与不确定图表类型的探索场景，显式指定适合对图表类型有严格要求的设计场景.

## 已知限制

- 需要联网访问 `www.anygen.io` 服务端，纯离线环境无法使用
- 依赖 AnyGen CLI 与 `anygen-workflow-generate` 技能，两者缺一不可
- 图表生成质量取决于文本描述的清晰度，模糊描述会导致生成结果偏差
- 单次生成的图表节点数量存在上限，超大规模图表需拆分生成
- 生成的图表为图片输出，不支持生成后直接在 CLI 中二次编辑
- 服务端渲染耗时随图表复杂度增加，复杂图表可能需要数秒到数十秒
- API Key 与浏览器授权均存在有效期，过期需重新认证
