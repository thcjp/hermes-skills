# 代码委派基础版

## 核心能力

- 编程任务的委派与编排：接收自然语言描述的编程需求，转化为可执行的子任务序列
- 任务分类与简单分解：将复杂需求拆分为基础代码生成、代码补全、简单重构等可管理单元
- 基础代码生成与输出校验：根据子任务生成对应代码片段，并对输出做基础语法与格式校验
- 执行日志记录：记录每个子任务的执行状态与耗时，便于回溯问题
#
## 快速开始

1. 确认运行环境满足依赖说明中的要求
2. 在AI Agent对话中调用本技能,提供必要的输入参数
3. 检查输出结果,根据需要进行后续处理

> 详细的输入输出格式请参考下方章节说明。

## 适用场景

| 场景 | 输入 | 输出 |
|---|---|---|
| 代码生成 | 自然语言描述的函数需求 | 对应语言的函数实现 |
| 代码补全 | 已有代码片段与补全意图 | 补全后的完整代码 |
| 简单重构 | 待重构代码与重构目标 | 重构后的代码 |
| 文档生成 | 代码文件路径 | 函数级注释或 API 文档 |

**不适用于**：需要跨多仓库深度分析、大规模批量改写、复杂架构决策的场景（请使用高级版）

## 使用流程

1. 接收用户的自然语言编程需求，明确目标语言与运行环境
2. 将需求分解为若干子任务，按依赖顺序排列
3. 依次执行子任务，生成代码片段并做基础语法校验
4. 汇总结果，输出完整代码与执行日志
5. 若执行中出现错误，按异常处理章节的指引重试或回退

## 输入格式

| 参数名 | 类型 | 必填 | 说明 |
|:-----|:-----|:-----|:-----|
| task_description | string | 是 | 自然语言描述的编程任务 |
| target_language | string | 否 | 目标编程语言，可选: python/java/go/java（请参考skill目录中的脚本文件），默认: python |
| output_format | string | 否 | 输出格式，可选: code/markdown/json，默认: code |
| max_retries | integer | 否 | 单步最大重试次数，默认: 1 |

## 异常处理

| 错误场景 | 原因 | 处理方式 |
|---:|---:|---:|
| 需求不清晰 | task_description 过于简略 | 提示用户补充目标语言与具体功能点 |
| 语法校验失败 | 生成的代码不符合目标语言规范 | 在 max_retries 范围内重新生成，并附带校验错误信息 |
| 运行时错误 | 运行环境不满足 | 确认目标语言运行环境已正确安装 |
| 网络错误 | 连接超时或不可达 | 检查网络连接和配置后重试 |

## 常见问题

### Q1: 如何开始使用代码委派基础版？
A: 查看使用流程章节，确认运行环境满足依赖说明中的要求，然后按"输入格式"提供任务描述即可.
### Q2: 遇到语法校验失败怎么办？
A: 请参考异常处理章节。基础版会在 max_retries 范围内自动重试一次；若仍失败，建议细化任务描述或拆分为更小的子任务.
### Q3: 代码委派基础版有什么限制？
A: 基础版不支持多步门控、断点续传与复杂重试策略，如需更稳健的执行能力，可升级至高级版.
### Q4: 如何获取多步门控、断点续传等高级能力？
A: 这些属于高级版能力。基础版聚焦于任务委派的核心流程，如需在复杂项目中使用门控校验、断点续传与多级重试，请升级至高级版.
## 错误处理

| 错误场景(续)| 原因 | 处理方式 |
|:---------|---------:|:---------|
| LLM响应超时或无响应 | 网络延迟或模型负载过高 | 检查网络连接和配置后重试；确认Agent平台LLM服务正常 |
| 输入内容格式不正确 | 用户输入不符合skill预期格式 | 检查输入是否符合skill使用说明中的格式要求，参考示例章节 |
| 执行结果与预期不符 | 指令描述不够明确或上下文不足 | 提供更详细的指令描述，补充必要的上下文信息 |
| 命令执行失败 | 运行环境不满足要求或权限不足 | 确认运行环境符合依赖说明中的要求；检查命令权限设置 |

## 已知限制

- 单步最大重试次数为 1 次，复杂场景可能需要人工介入
- 不支持多步门控（gates）机制，无法在关键步骤进行人工或自动校验
- 不支持断点续传，任务中断后需从头执行
- 不支持跨多仓库的深度代码分析与批量改写
- ，复杂任务建议拆分为多个小任务
