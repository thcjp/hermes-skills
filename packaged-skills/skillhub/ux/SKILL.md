## Nielsen 十大可用性启发式

对产品界面进行系统性UX审计时，基于以下10条启发式逐项评估：

| 编号 | 启发式 | 评估问题 | 常见问题 |
|:-----|:------|:------|:------|
| H1 | 系统状态可见 | 用户是否随时知道系统在做什么？ | 缺少Loading状态、操作无反馈 |
| H2 | 贴近真实世界 | 语言是否用户熟悉？ | 使用技术术语而非用户语言 |
| H3 | 用户控制与自由 | 用户能否撤销/退出？ | 无返回按钮、无法取消操作 |
| H4 | 一致性与标准 | 相同操作是否用相同方式？ | 不同页面按钮位置/样式不一致 |
| H5 | 防错 | 是否减少犯错机会？ | 无确认对话框、无输入限制 |
| H6 | 识别而非回忆 | 选项是否可见？ | 隐藏菜单、需记住操作路径 |
| H7 | 灵活性与效率 | 是否支持快捷操作？ | 无键盘快捷键、无批量操作 |
| H8 | 美观与极简 | 是否只展示必要信息？ | 界面拥挤、无关内容干扰 |
| H9 | 错误恢复 | 错误信息是否清晰？ | 错误提示模糊如"操作失败" |
| H10 | 帮助与文档 | 是否提供帮助？ | 无引导、无FAQ、帮助难找到 |

**评分标准**: 每项0-4分（0=严重违反, 4=完全符合），总分/40换算为百分比。

```

         │      │      │      │      │
用户目标: 了解   比较   下单   体验   分享
         │      │      │      │      │
触点:    搜索   对比页  结账   产品   社交
         结果          流程   界面   媒体
         │      │      │      │      │
行为:    关键词  读评论  填表   开箱   评分
         点击   看演示  支付   配置   发帖
         │      │      │      │      │

         │      │      │      │      │

         │      │      │      │      │

```

### 旅程地图分析要点

| 分析维度 | 关键问题 | 输出 |
|:------|:------|:------|
| 情绪低谷 | 哪个阶段用户情绪最低？ | 优先优化该阶段体验 |
| 痛点频率 | 哪些痛点在多个阶段重复出现？ | 全局性问题，需系统性解决 |
| 流失风险 | 哪个阶段最可能放弃？ | 增加挽留机制或简化流程 |
| 机会缺口 | 哪个阶段存在未被满足的需求？ | 产品创新机会点 |

## 交互设计原则

### 费茨定律（Fitts's Law）

目标越大、距离越近，操作越快。应用实践：
- 主要操作按钮放在拇指可达区域（移动端底部）
- 高频操作按钮加大尺寸（最小44x44px）
- 危险操作按钮远离常用按钮，防止误触

### 席克定律（Hick's Law）

选项越多，决策越慢。应用实践：
- 每屏不超过7个主要选项（米勒定律）
- 复杂流程分步骤展示（向导模式）
- 提供合理默认值，减少用户选择

### 认知负荷优化

| 策略 | 方法 | 示例 |
|:------|:------|:------|
| 减少内在负荷 | 分块呈现信息 | 表单分多步而非一页 |
| 减少外在负荷 | 消除无关元素 | 移除装饰性动画和广告 |
| 增加相关负荷 | 引导用户投入 | 进度条、成就系统 |

## 常见问题

### Q1: 如何开始使用UX？
A: 描述你的产品或界面场景即可，例如"分析电商App结账流程的UX问题"或"设计一个新手引导流程"。系统会基于Nielsen启发式进行评估，输出问题清单、优先级排序和改进建议。也可以提供用户行为数据（如转化率、跳出率），系统会结合数据给出分析。

### Q2: UX分析和UI设计有什么区别？
A: UX（用户体验）关注"用户为什么这样做"和"流程是否顺畅"，输出的是分析报告、旅程地图和交互方案。UI（用户界面）关注"界面长什么样"，输出的是视觉设计稿和前端代码。UX是骨架，UI是皮肤。建议先做UX分析确定交互流程，再做UI设计实现视觉。

### Q3: 如何确定UX优化的优先级？
A: 系统使用"影响度 x 置信度 / 努力"（ICE框架）对优化建议排序。影响度基于该问题影响的用户比例和严重程度；置信度基于数据支撑强度（用户测试数据 > 行为分析 > 专家判断）；努力度基于实现复杂度。优先做高影响、高置信、低努力的项目。

### Q4: 用户旅程地图适用于什么场景？
A: 适用于以下场景：（1）新产品设计前梳理完整用户路径；（2）转化率下降时定位流失节点；（3）竞品分析时对比体验差异；（4）跨团队对齐用户认知。旅程地图的核心价值是将抽象的用户体验可视化，帮助团队发现盲点和机会。
