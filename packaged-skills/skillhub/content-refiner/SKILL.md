# 内容洗稿师

## 使用场景

解决跨平台内容同质化风险。相同内容直接多平台发布会被平台检测为搬运/重复内容,导致降权甚至封号。本工具在发布前对内容进行语义改写,保持核心信息不变但改变表达方式,实现:

1. **跨平台差异化发布**: 同一内容适配多平台风格,避免重复检测
2. **查重规避**: 通过语义改写降低内容相似度,规避平台查重
3. **风格迁移**: 将内容从一种风格转换为另一种风格(如专业转口语化)
4. **原创度提升**: 通过深度语义改写,使内容原创度达99%以上
5. **批量内容处理**: 对同质化内容批量进行差异化改写

---

## 工作流

### 步骤1: 接收原始内容和目标平台

接收输入: `content`(原始内容) + `platform`(目标平台) + `use_llm`(是否使用LLM模式)

验证: 内容非空,平台在支持列表内。验证不通过时返回错误码和缺失项。

### 步骤2: 选择改写模式

| 模式 | 标识 | 特点 | 适用场景 |
|:-----|:-----|:-----|:---------|
| LLM模式 | use_llm=true | 深度语义改写,高质量,消耗500-1000 tokens/次 | 需要高原创度的正式内容 |
| 本地模式 | use_llm=false | 同义替换,0 Token消耗,速度快 | 简单内容快速改写 |

### 步骤3: 生成改写提示词

根据目标平台风格配置生成改写提示词,指导改写方向。

**平台风格配置**:

| 平台 | 风格 | 语气 | 长度 | 改写策略 |
|:-----|:-----|:-----|:-----|:---------|
| zhihu(知乎) | 专业理性 | 客观分析 | 中长文 | 增加论据,强化逻辑链 |
| xiaohongshu(小红书) | 活泼种草 | 亲切分享 | 短文+emoji | 增加emoji,口语化表达 |
| douyin(抖音) | 口语化 | 直接有力 | 短文案 | 精简表达,增强节奏感 |
| weibo(微博) | 简洁观点 | 犀利 | 短文 | 提炼核心观点,增加话题标签 |
| 技术社区 | 技术深度 | 专业分享 | 中长文 | 增加技术细节,代码示例 |
| csdn | 技术教程 | 详细讲解 | 长文 | 增加步骤说明,注意事项 |
| bilibili(B站) | 趣味科普 | 轻松 | 中短文 | 增加趣味性,降低理解门槛 |
| 电商平台 | 商品卖点 | 吸引购买 | 短文案 | 突出卖点,增强购买欲望 |

### 步骤4: 执行改写

**LLM模式**:
- 通过API网关调用LLM服务进行深度语义改写
- 改写策略: 保持核心信息不变,改变句式结构/词汇选择/表达顺序
- 原创度目标: 99%以上

**本地模式**:
- 执行同义替换: 推荐->安利/好用->实用/很好->不错 等
- 句式变换: 主动句改被动句,陈述句改反问句
- 词汇替换: 同义词词典匹配,保持语义一致

### 步骤5: SimHash去重检测

对改写后内容执行SimHash相似度检测:
- 计算原始内容与改写内容的SimHash指纹
- 计算汉明距离,判断相似度
- 相似度 > 80% -> 标记warning,建议使用LLM模式重新改写
- 相似度 <= 80% -> 改写合格,输出结果

### 步骤6: 返回改写结果

输出: original(原始内容) + rewritten(改写内容) + platform(目标平台) + mode(改写模式) + originality_score(原创度评分)

## 输入格式

```json
{
  "content": "推荐一个好用的AI工具,可以快速生成文案,效率提升10倍",
  "platform": "xiaohongshu",
  "use_llm": false
}
```

| 字段 | 类型 | 必填 | 说明 |
|:-----|:-----|:----:|:-----|
| content | string | 是 | 原始内容文本 |
| platform | string | 是 | 目标平台(zhihu/xiaohongshu/douyin/weibo/tech_community/csdn/bilibili/ecommerce) |
| use_llm | bool | 否 | 是否使用LLM模式,默认false(本地模式) |

## 异常处理

| 错误码 | 异常场景 | 处理方式 | 降级策略 |
|:-------|:---------|:---------|:---------|
| REWRITE_VAL_ERR | 内容为空/平台未知 | 返回JSON错误+exit(1) | 无法降级,必须提供有效输入 |
| REWRITE_FALLBACK | LLM服务调用失败 | 自动降级为本地同义替换 | 标注warning,使用本地模式完成改写 |
| REWRITE_TIMEOUT | LLM服务超时 | 降级为本地同义替换 | 标注warning,提示LLM模式不可用 |
| REWRITE_LOW_ORIGINALITY | SimHash相似度>80% | 标记warning,建议使用LLM模式 | 输出结果但标注"原创度不足" |
| REWRITE_ERR | 其他未预期异常 | 返回JSON错误+exit(2) | 记录错误日志,返回原始内容 |

## 依赖说明

### 运行环境
- **Agent平台**: Claude Code / Cursor / Codex / Gemini CLI / Windsurf 等支持SKILL.md的任意Agent
- **操作系统**: Windows / macOS / Linux
- **运行时**: 需要Agent支持exec(命令行执行)能力

### 第三方依赖
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| LLM API | API | 可选 | 任意LLM服务商(use_llm=true时必需,由Agent内置LLM提供) |
| SimHash | 算法 | 必需 | 相似度检测(内置算法,无需额外依赖) |

### API Key 配置
- **LLM_API_KEY**: 可选(use_llm=true时必需) - 深度语义改写
- 配置方式: 在Agent的环境变量中设置

### 纯Markdown使用说明
支持本地模式(use_llm=false)和LLM模式(use_llm=true)。本地模式0 Token消耗,使用同义替换。LLM服务不可用时自动降级为本地模式。

只需将SKILL.md文件放入Agent的skills目录即可直接使用。
如果Skill中包含exec工具调用,需要Agent支持命令行执行能力。

### 可用性分类
- **分类**: MD+EXEC
- **说明**: 纯Markdown,但需要exec能力(命令行执行),用于文件读写和命令调用
