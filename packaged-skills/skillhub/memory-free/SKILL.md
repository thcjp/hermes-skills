# 记忆管理器免费版

基础分类长期记忆系统，与 Agent 内置记忆并行扩展.
## 输入格式

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| input | string | 是 | 记忆管理器免费版处理的输入数据或指令 |
| options | object | 否 | 附加配置选项,如模式选择、格式偏好等 |
| callback_url | string | 否 | 异步处理完成后的回调通知URL |

## 核心能力

### 1. 预设分类存储
提供三个预设类别目录，满足基础存储需求.
- **参数**：类别名称固定为 projects、people、knowledge
- **用法**：在 `~/memory/` 下创建预设类别文件夹
- **输出**：每个类别目录下生成对应的条目文件（`.md` 格式）

预设类别：

| 类别 | 路径 | 适用内容 |
|:-----|:-----|:-----|
| 项目 | `~/memory/projects/` | 项目背景、进展、待办 |
| 人脉 | `~/memory/people/` | 联系人信息、互动记录 |
| 知识 | `~/memory/knowledge/` | 领域知识、学习笔记 |

**处理**: 解析预设分类存储的输入参数,执行核心处理逻辑,返回结构化结果和执行状态.
### 2. 自动索引维护
每个类别目录自动维护 `INDEX.md`，列出所有条目.
- **参数**：无额外参数，写入条目时自动更新
- **用法**：条目写入后同步更新对应类别的 `INDEX.md`
- **输出**：包含名称、状态、更新时间、文件名的表格

**处理**: 解析自动索引维护的输入参数,执行核心处理逻辑,返回结构化结果和执行状态.
### 3. 即时写入机制

用户分享重要信息时立即写入对应文件，不等待、不批处理.
- **参数**：信息内容、目标类别（仅限三个预设类别）
- **用法**：识别信息归属类别 → 写入条目文件 → 更新索引 → 回复用户
- **输出**：已写入的文件路径与更新后的索引

### 4. 关键词检索

通过 grep 命令在记忆目录中搜索关键词.
- **参数**：检索关键词
- **用法**：`grep -r "关键词" ~/memory/`
- **输出**：匹配的文件内容

- 参考`关键词检索`的配置文档进行参数调优

## 使用流程

### 第一步：创建目录结构

在 `~/memory/` 下创建三个预设类别目录（projects、people、knowledge）与对应的 `INDEX.md` 文件.
### 第二步：即时写入记忆

用户分享信息时，识别归属类别，写入对应条目文件，同步更新类别 `INDEX.md`，然后回复用户.
### 第三步：检索记忆

使用 `grep -r "关键词" ~/memory/` 搜索记忆内容，或直接查看类别 `INDEX.md` 浏览条目列表.
**结果验证**: 任务完成后,查看输出确认状态。成功时返回摘要和数据;失败时根据错误信息排查,参考恢复章节获取修复步骤.
## 错误处理

| 错误类型 | 原因 | 处理方式 |
|---:|---:|---:|
| 误改内置记忆 | 操作了 Agent 的 MEMORY.md 或工作区 memory/ 文件夹 | 严格隔离 `~/memory/` 与内置记忆路径，本系统仅在 `~/memory/` 下操作 |
| 索引缺失 | 写入条目后未更新 INDEX.md | 每次写入条目后立即更新对应类别的 INDEX.md |
| 延迟写入丢失信息 | 用户分享信息后攒着批量写导致遗忘 | 用户分享即写入，不等待不批处理 |

## FAQ

### Q1：免费版支持几个类别？

免费版提供三个预设类别：projects（项目）、people（人脉）、knowledge（知识）。如需自定义类别（如 decisions、collections 等），请升级到付费版.
### Q2：能从内置记忆同步信息吗？

免费版不支持内置记忆同步功能。如需将内置记忆中的特定信息同步到本系统，请升级到付费版.
### Q3：记忆积累多了怎么检索？

免费版支持关键词检索（`grep -r "关键词" ~/memory/`）和查看类别 INDEX.md。如需层级导航、分层扩展等高级检索能力，请升级到付费版.

## 已知限制

- 仅提供三个预设类别，不支持自定义类别
- 不支持内置记忆单向同步
- 不支持分层扩展（类别超 100 条目无法自动拆分子类别）
- 检索仅支持关键词匹配，不支持层级导航
- 跨设备同步需用户自行处理
