# Creator Alpha Feed Free 创作者 AI 日报(免费版)

## 输入格式

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| input | string | 是 | 创作者AI日报(免费版)处理的输入数据或指令 |
| options | object | 否 | 附加配置选项,如模式选择、格式偏好等 |
| callback_url | string | 否 | 异步处理完成后的回调通知URL |

## 概述

Creator Alpha Feed Free 是 AI 内容采集与排名 Skill 的免费版本,提供基础的每日 X 平台 AI 内容采集、三档排名与 Obsidian 报告写入能力。适合个人创作者快速搭建每日选题工作流.
## 核心能力

### 1. 三阶段顺序采集
按固定顺序执行采集:

- **阶段一 - X 主页 feed**:拉取登录用户的主页时间线,过滤 AI 相关内容.
- **阶段二 - 白名单账号**:逐个访问白名单账号主页,采集最近 6 小时推文.
- **阶段三 - 关键词搜索**:对配置的关键词列表执行 X 搜索补充内容.
**输出**: 返回三阶段顺序采集的处理结果,包含执行状态码、结果数据和执行日志.
### 2. 浏览器标签页上限管控
- 最多 7 个并发标签页
- 完成后立即关闭,再开新标签页
- 任务结束时标签页数归零

**输入**: 用户提供浏览器标签页上限管控所需的指令和必要参数.
**处理**: 解析浏览器标签页上限管控的输入参数,执行核心处理逻辑,返回结构化结果和执行状态.
**输出**: 返回浏览器标签页上限管控的处理结果,包含执行状态码、结果数据和执行日志.
### 3. 三档基础排名
- **KOL TOP3(近 6 小时)**:白名单 KOL 互动量最高的 3 条
- **实用/教程/观点 TOP10**:实用教程类与观点类内容前 10
- **行业 TOP3(近 6 小时)**:行业动态前 3 条

**输入**: 用户提供三档基础排名所需的指令和必要参数.
**处理**: 解析三档基础排名的输入参数,执行核心处理逻辑,返回结构化结果和执行状态.
**输出**: 返回三档基础排名的处理结果,包含执行状态码、结果数据和执行日志.
### 4. Obsidian 报告写入
完整报告以 `YYYY-MM-DD_HHMM.md` 命名写入 Obsidian Vault 的 `Skill平台/项目/AI内容日报/01-日报/` 目录,包含每条内容的原文摘要、作者、链接.
**输入**: 用户提供Obsidian 报告写入所需的指令和必要参数.
**处理**: 解析Obsidian 报告写入的输入参数,执行核心处理逻辑,返回结构化结果和执行状态.
**输出**: 返回Obsidian 报告写入的处理结果,包含执行状态码、结果数据和执行日志.
### 输出格式

完成响应以Markdown格式返回,包含任务状态(成功/失败)、解析摘要和具体输出数据。失败时返回错误码和错误信息,便于定位问题。- 验证返回数据的完整性和格式正确性
- 参考`输出格式`的配置文档进行参数调优

## 使用流程

### 步骤一：配置采集参数

编辑 Obsidian Vault 中的采集配置文件(路径通过 `${OBSIDIAN_CONFIG_PATH}` 指定,默认 `<vault>/Skill平台/项目/AI内容日报/采集配置.md`),配置白名单账号与关键词列表.
### 步骤二：确认 Obsidian 目录

检查 Vault 中存在 `Skill平台/项目/AI内容日报/01-日报/` 目录,缺失则创建.
### 步骤三：执行采集

向 Agent 发送触发指令(如"采集今日 AI 内容日报"),Skill 按顺序执行采集,标签页管控在 7 个以内.
### 步骤四：查阅 Obsidian 报告

报告以 `YYYY-MM-DD_HHMM.md` 命名写入 `01-日报/` 目录,在 Obsidian 中打开即可查阅.
**结果验证**: 任务完成后,查看输出确认状态。成功时返回摘要和数据;失败时根据错误信息排查,参考恢复章节获取修复步骤.
**使用步骤**:
1. 阅读依赖说明章节,确认运行环境已就绪
2. 根据任务需求,参考核心能力章节选择对应能力
3. 按照能力描述提供输入参数,执行操作
4. 查看输出结果,确认任务完成状态

## 异常处理

### 1. OBSIDIAN_CONFIG_PATH 未配置

**原因**: 环境变量未设置且默认路径不存在.
**处理**: 设置 `OBSIDIAN_CONFIG_PATH` 环境变量指向采集配置.md,或在默认 Vault 路径创建该文件。建议在 `03-文档/` 目录提供配置模板.
### 2. 浏览器标签页超过 7 个上限

**原因**: 未及时关闭已完成任务的标签页.
**处理**: 每次开新标签页前检查并发数,已达 7 则等待最早标签页关闭后再开;任务结束前强制关闭全部临时标签页确保归零.
### 3. X 登录拦截

**原因**: 采集源需登录才能查看完整内容.
**处理**: 免费版不自动等待登录,跳过该源并在报告中标注"源 X 因登录未采集"。建议用户提前在浏览器登录 X 后再执行采集。若需自动等待登录与恢复,请升级付费版.
### 4. Obsidian Vault 路径不可写

**原因**: 路径权限不足或磁盘已满.
**处理**: 写入前先创建测试文件验证可写性;若不可写,回退将报告写入 workspace 镜像目录;提示用户检查 Vault 路径权限.
### 5. 采集内容去重后为空

**原因**: 全部采集源无返回(网络中断或登录失效).
**处理**: 不生成空报告,提示"今日采集失败:全部源无内容返回,请检查网络连接和配置后重试";建议手动访问 X 确认是否为平台故障.
## 常见问题

### Q1: 如何新增白名单 KOL 账号?

编辑采集配置文件,在 `whitelist_accounts` 列表中追加账号 handle(如 `@new_kol`),下次采集自动纳入.
### Q2: 报告文件名为什么用 YYYY-MM-DD_HHMM 格式?

日期在前便于按日聚合,HHMM 精确到分钟避免同日多次采集文件名冲突,下划线分隔避免冒号在文件系统中的兼容问题.
### Q3: 免费版支持群频道推送吗?

不支持。免费版仅写入 Obsidian 报告,不推送群频道。若需群频道精简版推送与 `@提问者` 提及,请升级付费版.
### Q4: 采集频率建议多少?

建议每日 1-2 次,高频采集可能触发 X 限流.
## 错误处理

| 错误场景 | 原因 | 处理方式 |
|---:|---:|---:|
| LLM响应超时或无响应 | 网络延迟或模型负载过高 | 检查网络连接和配置后重试；确认Agent平台LLM服务正常 |
| 输入内容格式不正确 | 用户输入不符合skill预期格式 | 检查输入是否符合skill使用说明中的格式要求，参考示例章节 |
| 执行结果与预期不符 | 指令描述不够明确或上下文不足 | 提供更详细的指令描述，补充必要的上下文信息 |
| 命令执行失败 | 运行环境不满足要求或权限不足 | 确认运行环境符合依赖说明中的要求；检查命令权限设置 |

## 已知限制

- 仅支持 X(原 Twitter)平台采集,不支持其他内容源
- 不支持群频道推送(仅写入 Obsidian)
- 不支持必追账号状态校验(@xiaohu @dotey @marclou)
- 不支持 API 优先与登录拦截自动等待
- 三档排名基于启发式规则,非语义级内容质量评估
- 浏览器采集依赖 X 的 DOM 结构,X 改版可能导致采集失败
