使用 `catt`（Cast All The Things）工具控制局域网内的投屏协议设备。支持设备发现、媒体投放、播放控制、音量调节、队列管理、字幕加载和多设备别名管理.

## 核心能力

### 1. 设备发现

通过 `catt scan` 命令扫描局域网内的所有投屏设备。返回设备名称和IP地址列表。使用 `-d <device>` 参数通过设备名称或IP地址指定目标设备。适用于初次配置或多设备环境下的设备定位。- 验证返回数据的完整性和格式正确性
- 参考`设备发现`的配置文档进行参数调优
### 2. 媒体投放
通过 `catt cast <url>` 命令将视频、音频或网页投放到目标设备。支持YouTube视频/播放列表、Vimeo、Twitch直播、本地文件（MP4/MKV/WebM）和直接视频URL。使用 `-b` 参数在后台投放，`-y format=best` 指定最佳格式。适用于家庭影院和会议演示场景.
**处理**: 解析用户提供的媒体URL或文件路径,验证格式兼容性后通过catt命令投放到目标Chromecast设备,自动处理设备发现和连接.
**输出**: 返回媒体投放的处理结果,包含执行状态码、结果数据和执行日志。- 验证返回数据的完整性和格式正确性
- 参考`媒体投放`的配置文档进行参数调优
### 3. 播放控制
通过 `catt play`、`catt pause`、`catt play_toggle` 和 `catt stop` 命令控制播放状态。`play_toggle` 切换播放/暂停，`stop` 完全停止播放，`skip` 跳至内容末尾。适用于播放过程中的即时控制需求.
**输入**: 用户提供播放控制所需的指令和必要参数.
**处理**: 接收播放控制指令(play/pause/stop/skip),映射到对应的catt子命令并执行,返回播放状态变更确认.
**输出**: 返回播放控制的处理结果,包含执行状态码、结果数据和执行日志.
### 4. 进度跳转
通过 `catt seek` 命令跳转到指定时间点，支持秒数（如 `catt seek 300` 跳至5分钟）和HH:MM:SS格式（如 `catt seek 01:30:00` 跳至1小时30分）。`catt ffwd 30` 快进30秒，`catt rewind 30` 后退30秒。适用于长视频的快速定位和章节跳转.
**输入**: 用户提供进度跳转所需的指令和必要参数.
**处理**: 解析时间参数(秒数或HH:MM:SS格式),通过catt seek/ffwd/rewind命令调整播放位置,返回当前播放进度.
**输出**: 返回进度跳转的处理结果,包含执行状态码、结果数据和执行日志。- 验证返回数据的完整性和格式正确性
- 参考`进度跳转`的配置文档进行参数调优
### 5. 音量控制
通过 `catt volume <0-100>` 设置绝对音量（0-100范围），`catt volumeup 10` 增加10，`catt volumedown 10` 减少10。`catt volumemute on` 静音，`catt volumemute off` 取消静音。适用于不同内容间的音量统一和静音管理.
**输入**: 用户提供音量控制所需的指令和必要参数.
**处理**: 解析音量参数(绝对值0-100或相对增减),通过catt volume命令调整设备音量,返回设置后的音量值.
**输出**: 返回音量控制的处理结果,包含执行状态码、结果数据和执行日志。- 验证返回数据的完整性和格式正确性
- 参考`音量控制`的配置文档进行参数调优
### 6. 队列管理
通过 `catt add <url>` 将媒体添加到播放队列，`-n` 参数将媒体添加为下一首播放。`catt remove <url>` 从队列移除，`catt clear` 清空整个队列。队列功能主要支持YouTube播放列表。适用于连续播放和多视频队列场景.
**处理**: 管理YouTube播放队列,通过catt add/remove/clear命令维护队列内容,返回当前队列状态.
**输出**: 返回队列管理的处理结果,包含执行状态码、结果数据和执行日志。- 验证返回数据的完整性和格式正确性
- 参考`队列管理`的配置文档进行参数调优
### 7. 状态管理

通过 `catt save` 命令保存当前播放状态（进度、音量等），`catt restore` 恢复到上次保存的状态。适用于跨设备的播放状态迁移或中断后恢复播放。- 验证返回数据的完整性和格式正确性
- 参考`状态管理`的配置文档进行参数调优
### 8. 设备信息查询
通过 `catt status` 获取简要信息（当前时间、音量、静音状态），`catt info` 获取完整信息（标题、URL、播放器状态、媒体类型等）。适用于播放状态监控和设备状态查询.
**处理**: 调用catt status或catt info命令获取设备信息,返回设备名称、IP地址、播放状态、当前媒体等详情.
**输出**: 返回设备信息查询的处理结果,包含执行状态码、结果数据和执行日志。- 验证返回数据的完整性和格式正确性
- 参考`设备信息查询`的配置文档进行参数调优
### 9. 字幕加载
通过 `catt cast -s ./subtitles.srt ./video.mp4` 命令在投放本地视频时加载SRT格式字幕文件。使用 `--no-subs` 参数禁用字幕。`-t 01:30:00` 参数指定从特定时间点开始播放。适用于外语视频观看和字幕校对场景.
**处理**: 解析字幕文件路径和视频文件路径,通过catt cast -s命令加载字幕,返回加载状态和字幕格式信息.
**输出**: 返回字幕加载的处理结果,包含执行状态码、结果数据和执行日志。- 验证返回数据的完整性和格式正确性
- 参考`字幕加载`的配置文档进行参数调优
### 10. 设备别名管理
通过 `catt -d "Living Room TV" set_default` 设置默认设备，`catt -d 192.168.1.163 set_alias tv` 为设备设置别名（如 `tv`）。`catt -d tv del_alias` 删除别名，`catt del_default` 取消默认设备。别名保存在 `~/.config/catt/catt.cfg` 中。适用于多设备环境下的快速设备切换.
**输入**: 用户提供设备别名管理所需的指令和必要参数.
**处理**: 管理Chromecast设备别名,通过catt set_alias/del_alias命令配置,更新本地配置文件catt.cfg.

完成响应以Markdown格式返回,包含任务状态(成功/失败)、解析摘要和具体输出数据。失败时返回错误码和错误信息,便于定位问题。- 验证返回数据的完整性和格式正确性

## 使用流程

1. 运行 `catt scan` 发现局域网内的投屏设备，记录设备名称或IP地址
2. 使用 `catt -d <device> set_default` 设置默认设备，或使用别名快速引用
3. 通过 `catt cast <url>` 或 `catt cast ./video.mp4` 投放媒体内容
4. 使用 `catt play`/`pause`/`stop` 控制播放，`catt seek`/`ffwd`/`rewind` 调整进度
5. 通过 `catt volume <0-100>` 调节音量，`catt add <url>` 管理播放队列
6. 使用 `catt status` 或 `catt info` 查询当前播放状态

**结果验证**: 任务完成后,查看输出确认状态。成功时返回摘要和数据;失败时根据错误信息排查,参考恢复章节获取修复步骤.
## 示例

### 示例1：发现设备并投放YouTube视频

```bash
# 扫描局域网设备
catt scan
# 输出示例:
# 192.168.1.163 - Living Room TV
# 192.168.1.200 - Bedroom Speaker
# ...
# 设置默认设备
catt -d "Living Room TV" set_default
# ...
# 投放YouTube视频
catt cast "https://www.youtube.com/watch?v=dQw4w9WgXcQ"
# ...
# 查看播放状态
catt status
# 输出: Time: 01:23 / 03:33 | Volume: 50 | Muted: False
```

### 示例2：投放本地视频并加载字幕

```bash
# 设置设备别名
catt -d 192.168.1.163 set_alias tv
# ...
# 投放本地视频并加载SRT字幕
catt -d tv cast -s ./subtitles.srt ./movie.mp4
# ...
# 从1小时30分处开始播放
catt -d tv cast -t 01:30:00 ./movie.mp4
# ...
# 快进30秒
catt -d tv ffwd 30
# ...
# 设置音量为75
catt -d tv volume 75
# ...
# 保存当前播放状态
catt -d tv save
```

## 错误处理

| 错误场景 | 原因 | 处理方式 |
|:---:|:---:|:---:|
| `catt scan` 找不到设备 | 网络阻止mDNS发现 | 使用设备IP地址直接连接：`catt -d 192.168.1.163 cast <url>` |
| 本地文件投放失败 | TCP端口45000-47000被防火墙阻止 | 开放端口45000-47000范围，或使用网络共享URL代替本地文件 |
| 字幕加载失败 | SRT文件格式错误或路径不对 | 确认字幕文件为标准SRT格式，使用绝对路径指定文件 |
| 设备别名冲突 | 别名已被其他设备占用 | 先删除旧别名 `catt -d <old_device> del_alias`，再设置新别名 |
| 队列操作失败 | 仅YouTube支持队列功能 | 队列管理仅适用于YouTube视频/播放列表，其他源不支持 `add`/`remove` |
| 音量参数超出范围 | 传入了0-100之外的值 | 确保音量值在0-100范围内，使用 `volumeup`/`volumedown` 增减 |
| seek时间格式错误 | 传入了非HH:MM:SS格式或负数 | 使用秒数（如 `300`）或HH:MM:SS格式（如 `01:30:00`），不支持负数 |
| 播放列表为空 | YouTube播放列表URL无效或无视频 | 使用 `-r` 参数确认播放列表URL正确，检查播放列表是否公开可访问 |

## 常见问题

### Q1: 如何发现局域网中的投屏设备？

运行 `catt scan` 命令扫描局域网。返回结果包含设备名称和IP地址。如果扫描不到设备，可能是网络阻止了mDNS协议，此时可以直接使用设备IP地址连接：`catt -d 192.168.1.163 cast <url>`.
### Q2: 支持哪些媒体源？

通过yt-dlp内部引擎支持YouTube（视频/播放列表/直播）、Vimeo、Dailymotion、Twitch、直接视频URL（MP4/MKV/WebM）和本地文件（视频/音频/图片）。使用 `catt cast_site <url>` 可投放整个网页.
### Q3: 如何投射本地文件？

使用 `catt cast ./video.mp4` 命令投射本地文件。需要确保TCP端口45000-47000开放（catt通过这些端口传输文件数据）。如果端口被阻止，可以将文件放在网络共享上使用URL投射.
### Q4: 如何添加字幕？

使用 `catt cast -s ./subtitles.srt ./video.mp4` 命令加载SRT格式字幕文件。字幕文件必须与视频在同一台计算机上。使用 `--no-subs` 参数可以禁用字幕.
### Q5: 如何管理多个设备？

使用 `catt -d <device>` 参数指定目标设备，或通过 `catt -d <name_or_ip> set_alias <alias>` 设置别名后用别名引用。在 `~/.config/catt/catt.cfg` 配置文件中可以设置默认设备和多个别名。使用 `catt del_default` 取消默认设备.
### Q6: 队列功能支持哪些平台？

队列管理（`catt add`/`remove`/`clear`）主要支持YouTube视频和播放列表。使用 `-n` 参数将媒体添加为下一首播放。其他平台（如Vimeo、本地文件）不支持队列操作.
## 已知限制

- 投屏设备与计算机必须在同一局域网内
- 本地文件投射需要TCP端口45000-47000开放
- 队列管理功能仅支持YouTube平台
- 部分网络环境可能阻止mDNS设备发现
- 播放状态保存/恢复可能因设备重启而失效
