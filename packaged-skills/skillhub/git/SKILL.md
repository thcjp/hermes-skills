## 适用场景

| 场景 | 输入 | 输出 |
|:-----|:-----|:-----|
| 提交管理 | 暂存文件和提交信息 | Git提交记录和变更摘要 |
| 分支操作 | 分支名和合并策略 | 分支创建/合并/变基结果 |
| 冲突解决 | 冲突文件和合并上下文 | 冲突解决方案和合并结果 |

**不适用于**：非Git版本控制系统(如SVN/Mercurial)的操作

## 使用流程

1. 确认运行环境满足依赖说明中的要求
2. 根据适用场景选择合适的使用方式
3. 执行操作并检查输出结果
4. 如遇错误，参考错误处理章节

| 参数名 | 类型 | 必填 | 说明 |

| operation | string | 否 | Git操作, 可选: commit/branch/merge/rebase, 默认: commit |

## 错误处理

| 错误场景 | 原因 | 处理方式 |
|:------|------:|:------|
| LLM响应超时或无响应 | 网络延迟或模型负载过高 | ，请求；确认Agent平台LLM服务正常 |
| 输入内容格式不正确 | 用户输入不符合skill预期格式 | 检查输入是否符合skill使用说明中的格式要求，参考示例章节 |
| 执行结果与预期不符 | 指令描述不够明确或上下文不足 | 提供更详细的指令描述，补充必要的上下文信息 |
| 命令执行失败 | 运行环境不满足要求或权限不足 | 确认运行环境符合依赖说明中的要求；检查命令权限设置 |
