## 适用场景

### 场景 1：跨场内预测市场套利监控
专业交易员监控 Polymarket 与 Limitless 同主题市场的价差，付费端点返回每对市场的 yes-mid 价差（bps）与套利方向。专业版的跨场内 JOIN 能力单场内透传无法实现，配合支出控制确保单次任务成本可控.
### 场景 2：链上代理信誉评估
企业采购链上代理服务前，调用 `/agent/score` 付费端点获取 0-100 信誉分，综合 ERC-8004 注册、USDC 结算速度与链上反馈注册表。返回包含 SHA-256 + preimage + 规范化 JCS 字符串，验证者可无回调重新派生字节精确结果.
### 场景 3：专业交易员深度分析
Hyperliquid 永续合约交易员调用付费端点评估交易员技能分、金库表现、清算记录。配合胜率/回撤/利润因子等高级 MCP工具，构建交易员画像，辅助跟单决策.
### 场景 4：企业级链上数据批量采集
数据团队批量调用地址解析端点，获取 x402 Base 子图上的地址生命周期统计（付款人/收款人角色）、近期 10 笔双向支付、facilitator 元数据与索引区块高度。配合预算熔断避免超支.
## 使用流程

> 上手时间：< 120 秒。专业版支持付费端点，建议先配置支出控制再启用自主模式.
### 步骤 1：查询免费配额

```text
GET https://graph-query.example/quota?sender=0xAPI_KEY
```

返回当日剩余免费查询次数与下一调用是否付费.
### 步骤 2：付费端点 402 预览

每个付费端点的 402 响应体含 `output_example` 字段，预览付款后将收到的数据结构。检查形状是否符合需求，不符合则放弃付费.
```text
HTTP/1.1 402 Payment Required
X-Payment: { "amount": "0.01", "currency": "USDC", "network": "base",
             "recipient": "0x...", "challenge": "<nonce>" }
```

### 步骤 3：按次审批（推荐）

配置运行时将 402 挑战呈现给用户，每次付费前 y/N 确认。适用于持有非小额余额的钱包.
### 步骤 4：自主模式前配置支出控制

自主模式前必须配置：(1) 专用低余额钱包；(2) 单次上限与总预算；(3) 停止条件（次数上限/时间上限/余额下限）；(4) 首次探测模式.

## 异常处理

| 现象 | 可能原因 | 解决步骤 | 优先级 |
|:---:|:---:|:---:|:---:|
| 402 反复出现 | 免费配额耗尽 | 升级付费或次日 | P2 |
| 付费后无收据头 | 链上结算未完成 | 反馈处理，无结算不收费 | P1 |
| 路由置信度低 | 提问不具体 | 补充链名、协议名、地址 | P2 |
| 自主模式失控 | 未配置停止条件 | 立即停止，配置支出控制后重启 | P1 |
| 跨场价差为空 | 两场无同主题市场 | 换主题或检查市场命名 | P3 |
| 地址解析无结果 | 地址无 x402 活动 | 确认地址参与过 x402 支付 | P3 |
| 钱包余额不足 | 注入余额过低 | 补充低余额钱包余额 | P1 |
| 缓存不命中 | 缓存过期 / 参数不一致 | 检查 cache_for_seconds 与参数 | P3 |

### 依赖说明(补充)
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:------|------:|:------|:------|
| LLM API | API | 必需 | 由 Agent 平台内置 LLM 提供 |

### API Key 配置
- **免费层**：无需 API Key
- **付费端点**：通过 x402 协议自动结算，无需额外 API Key；需配置支持 x402 的 Agent 运行时
- **钱包私钥**：存储于系统密钥管理工具或硬件钱包，禁止硬编码于 SKILL.md 或脚本
- **禁止**：在 SKILL.md 或脚本中硬编码私钥、助记词或资金库钱包凭证

### 可用性分类
- **分类**：MD+EXEC（纯 Markdown 指令，部分功能需要 exec 命令行执行能力）

## 常见问题

### Q1：付费端点返回 402 但我不确定是否值得付费？

A：检查 402 响应体的 `output_example` 字段，预览付款后的数据结构。若不符合需求，直接放弃，不产生费用.
### Q2：自主模式下如何避免失控支出？

A：必须配置：(1) 专用低余额钱包；(2) `maxAmountPerCall` 与 `maxTotalSpend`；(3) 至少一个停止条件；(4) 首次探测模式记录支出.
### Q3：收据 `x-payment-response` 头如何使用？

A：记录该头的链上结算凭证，用于审计与对账。若调用扣款但无该头，属于异常，应反馈处理.
### Q4：哪些端点永不收费？

A：`/.well-known/agent-card.json`、`/agents/capabilities.json`、`/mcp/catalog`、`/llms.txt`、`/dashboard`、`/chat`、`/quota`、`GET /` 永不触发付费.
### Q5：跨场内价差为何单场内透传无法返回？

A：跨场内 JOIN 需要同时关联两个场内的数据，单场内透传只能访问单一数据源，结构上无法返回跨场结果。必须使用 `/predmarket/spread` 专用端点.
### Q6：代理信誉评分如何验证？

A：`/agent/score` 返回包含 SHA-256 + preimage + 规范化 JCS 字符串，验证者可无回调重新派生字节精确结果，无需回查服务.
### Q7：Polymarket 路由规则？

A：常见查询（市场、OHLCV、活动、持仓、P&L、平台统计）优先用 token-api；高级查询（订单簿深度、价差、争议市场、UMA 解决、胜率/回撤、CTF 拆分合并）路由到 graph-polymarket MCP工具.
### Q8：请求跨两个服务怎么办？

A：Skill 会同时调用两个服务并合并结果。如"Hyperliquid 原始市场数据 + 派生技能分"，前者走 token-api，后者走 `/hyperliquid/*` 付费端点.
### Q9：如何审计每日支出？

A：定时轮询 `/quota` 端点，记录每日每发送者的免费配额使用与付费调用次数，结合收据汇总支出.
### Q10：专业版是否支持自定义路由策略？

A：支持。通过配置文件指定优先服务、付费阈值、缓存策略，适合批量采集与自动化分析场景.
