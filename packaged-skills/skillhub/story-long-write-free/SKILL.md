# 长篇网文写作(免费版)

从零开始写一本长篇网络小说,从选题确认到大纲搭建再到正文输出。先抓情绪,再用验证过的方法可靠地交付这个情绪,灵感只做素材来源.
## 输入格式

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| input | string | 是 | 长篇网文写作(免费版)处理的输入数据或指令 |
| options | object | 否 | 附加配置选项,如模式选择、格式偏好等 |
| callback_url | string | 否 | 异步处理完成后的回调通知URL |

## 核心能力

### 1. 核心方法
- 先定情绪,再定故事:每个场景必须服务于明确的情绪目标,说不清交付什么情绪的场景不该存在
- 从验证过的模式出发:扫榜找方向,拆文找模块,对标找节奏
- 用模块组装:把对标书的具体角色看成功能位(对手/盟友/催化剂),映射到自己的角色,用自己素材填充
- 只加载必需信息:写每章时只加载"不知道就会写错"的信息

**输入**: 用户提供核心方法所需的指令和必要参数.
**处理**: 解析核心方法的输入参数,执行核心处理逻辑,返回结构化结果和执行状态.
**输出**: 返回核心方法的处理结果,包含执行状态码、结果数据和执行日志.
### 2. 题材情绪映射
| 题材 | 核心情绪 | 重点参考 |
|:-----|:-----|:-----|
| 打脸/逆袭 | 爽感释放 | genre-writing-formulas.md |
| 身份反转 | 震撼+痛快 | reversal-toolkit.md |
| 感情拉扯 | 意难平 | emotional-methods.md |
| 悬疑/惊悚 | 紧张+好奇 | hooks-suspense.md |
| 日常装逼 | 期待感 | hooks-chapter.md |

**输入**: 用户提供题材情绪映射所需的指令和必要参数.
**处理**: 解析题材情绪映射的输入参数,执行核心处理逻辑,返回结构化结果和执行状态.
**输出**: 返回题材情绪映射的处理结果,包含执行状态码、结果数据和执行日志.
### 3. 场景路由
| 场景 | 触发条件 | 执行流程 |
|---:|---:|---:|
| 开书 | "帮我开书"/项目目录为空 | Phase 1→2→3,默认停在细纲交付,不自动写正文 |
| 写指定章 | "写第N章" | Phase 4 单章写作,写完Phase 5检查后停止 |
| 补纲/扩纲 | "出细纲/补细纲/规划剧情"且已有大纲 | Phase 3 中途补纲小流程,默认停在细纲交付 |
| 日更续写 | "日更/续写/继续写"且已有正文+追踪 | 加载 `references/workflow-daily.md` |

匹配优先级:大修 → 写指定章 → 补纲/扩纲 → 日更续写 → 开书.
**输出**: 返回场景路由的处理结果,包含执行状态码、结果数据和执行日志.
### 4. 裸调用与停靠点(防失控)
裸调用(无明确意图)时只做项目状态诊断并列出选项,不自动进入正文写作:
- 空项目 → 建议"帮我开书"
- 已有设定/大纲但无正文 → 建议"写第1章"
- 已有正文+追踪 → 展示最后章节与下一章细纲状态

正文批量上限:未给数量时单章默认1章;日更默认2-3章;用户给N时单轮最多3章,超过拆成本轮3章.
- 参考`裸调用与停靠点(防失控)`的配置文档进行参数调优

**输出**: 返回裸调用与停靠点(防失控)的处理结果,包含执行状态码、结果数据和执行日志.
### 输出格式

完成响应以Markdown格式返回,包含任务状态(成功/失败)、解析摘要和具体输出数据。失败时返回错误码和错误信息,便于定位问题。- 验证返回数据的完整性和格式正确性
- 参考`输出格式`的配置文档进行参数调优

## 适用场景

| 场景 | 输入 | 输出 |
|:---:|:---:|:---:|
| 开书 | 选题方向或对标书 | 项目结构+核心设定+卷纲+前10章细纲 |
| 写指定章 | 章节号 | 该章正文+质量检查报告 |
| 补纲/扩纲 | 剧情方向 | 新剧情单元卡+滚动细纲 |

不适用于:短篇创作、纯技术文档、非虚构写作、学术写作.
## 使用流程

1. 识别场景:开书/写指定章/补纲/日更/大修,按匹配优先级
2. 裸调用时只做项目状态诊断,不自动进入正文
3. 开书走Phase 1→2→3,默认停在细纲交付
4. 写正文走Phase 4,写完Phase 5同轮清零
5. 补纲走中途补纲小流程,默认停在细纲交付

**结果验证**: 任务完成后,查看输出确认状态。成功时返回摘要和数据;失败时根据错误信息排查,参考恢复章节获取修复步骤.

## 错误处理

| 错误场景 | 原因 | 处理方式 |
|:------|------:|:------|
| 对标主产物缺失 | `剧情/情绪模块.md`或`剧情/节奏.md`不存在 | 停止,设置 `missing_primary_contract: true`,要求重跑分析流程或重新导入 |
| 细纲缺失无法写正文 | 未建对应章细纲 | 先补Phase 1→3再写点名章,不跳章 |
| 章节号冲突 | 细纲与正文文件名编号不一致 | 按细纲编号修正正文文件名,保持一致 |
| 大纲安全审查契约破坏 | 剧情违反读者契约或主角代理权 | 必须修纲,补交换/铺垫/成本后再继续 |

## 常见问题

### Q1: 首批细纲为何封顶10章?
A: 避免单次对话强行产出30章完整细纲导致质量下降。首批前10章封顶,可跨剧情单元。写到单元末段(剩余未建细纲≤2章)即滚动补下一单元整批。用户显式给章数时按要求执行,单批仍建议≤10章.
### Q2: 低压+过场合计不超约15%如何把握?
A: 章节定位分高压/推进/修炼试错/关系回收/低压生活/信息整理。低压章允许无显性爽点,重点写功能(喘息、关系、铺垫、转场)。但低压+过场合计不超约15%,否则读者失去往下看的理由。底线是每章给读者一个往下看的理由,相邻章不情绪趋同.
### Q3: 批末预算核对如何执行?
A: 每批建完必做。统一核对各章「预算合计」行 Σ∈[章目标, ×1.1]。不合格章补密点(增加情节)或压疏点(精简过渡)后再交付。不要求逐章写作时反复心算,批次级核对即可.
