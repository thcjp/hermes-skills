# 社媒内容排期

跨平台社媒内容排期与文案生成助手。根据主题与目标,生成结构化内容日历,按平台风格规范输出文案,
定义内容支柱并轮转,提供跨平台复用映射与三阶标签策略。所有输出为可复制格式并附字符计数,
便于直接粘贴发布。不直接调用平台 API 发布,专注内容生产环节。

## 核心能力

- **内容日历生成**:按周/月组织,每条含日期、时段、平台、内容类型(text/image prompt/video concept/carousel/story)、主题、文案草稿、标签、CTA。支持按日或按平台分组输出
- **平台优化文案**:按各平台风格规范定制
  - Twitter/X:精炼 <280 字符,钩子前置,适合 thread 展开
  - LinkedIn:专业叙事,段落分明,1300 字符甜点区
  - Instagram:视觉优先配文,换行留白,20-30 标签置于评论
  - TikTok:前 2 秒钩子,紧跟趋势格式,脚本化概念
  - Facebook:对话驱动,提问引导,易分享
- **内容支柱定义**:协助定义 4-6 个支柱并轮转
  - Educational(教学输出)
  - Behind-the-scenes(幕后建立信任)
  - Social proof(证言与成果)
  - Entertainment(个性与幽默)
  - Promotional(优惠与发布)
  - Community(互动投票)
- **跨平台复用映射**:对每条内容给出跨平台改写建议,保留核心信息同时适配各平台风格与字符约束
- **三阶标签策略**:研究并建议三层标签
  - 高量(品牌曝光,竞争大)
  - 中量(可发现性平衡)
  - 利基(精准社群,转化高)
- **字符计数校验**:对平台受限文案实时给出字符数,确保 Twitter 不超 280、Instagram 标签 20-30 等

## 使用流程

1. **明确排期范围**:确认周/月跨度、目标平台、内容主题与品牌定位
2. **定义内容支柱**:基于定位确定 4-6 个支柱(教学/幕后/证言/娱乐/促销/社区),分配轮转权重
3. **生成内容日历**:按支柱轮转填充日期/时段/平台/内容类型,产出文案草稿、标签、CTA
4. **平台适配改写**:对每条内容按目标平台风格规范改写,附字符计数校验
5. **跨平台复用**:为高价值内容生成复用映射,一次生产多平台分发
6. **标签策略**:为每条内容配三层标签(高量/中量/利基),Instagram 标签置于评论区

**结果处理**: 执行完成后,查看输出结果确认操作状态。成功时输出包含处理摘要和结果数据;失败时根据错误信息排查问题,参考错误处理章节获取恢复步骤。


## 平台风格规范

| 平台 | 字符约束 | 风格要点 | 标签策略 |
|------|----------|----------|----------|
| Twitter/X | <280 字符 | 钩子前置,thread 友好 | 1-2 个,精炼 |
| LinkedIn | 1300 字符甜点区 | 专业叙事,段落分明 | 3-5 个,行业向 |
| Instagram | 配文留白 | 视觉优先,换行 | 20-30 个置于评论 |
| TikTok | 脚本化 | 前 2 秒钩子,趋势格式 | 3-5 个,趋势向 |
| Facebook | 对话驱动 | 提问引导,易分享 | 2-3 个,话题向 |

## 案例展示

### 案例一:产品发布一周五平台日历

**触发**:新产品发布,需一周覆盖五大平台

**输入**:主题"AI 写作助手 v2.0 发布",平台 Twitter/LinkedIn/Instagram/TikTok/Facebook,支柱权重 Promotional 40%/Educational 30%/Behind-the-scenes 30%

**输出片段**(按日分组):

```text
周一 09:00 | Twitter/X | Promotional
文案: "AI 写作助手 v2.0 上线。新增多模态输入与风格克隆。首周 7 折 → <链接>"
字符数: 78/280
标签: #AI写作 #产品发布
CTA: 点击链接体验

周一 12:00 | LinkedIn | Educational
文案: "我们花六个月重写了风格克隆引擎。核心是把作者语料拆解为节奏、用词、结构三层特征,再在生成时按权重还原。以下是技术取舍与踩坑记录……(1300 字符内展开)"
字符数: 1290/2000
标签: #AI写作 #产品工程 #自然语言处理
CTA: 评论分享你的写作场景

周二 18:00 | Instagram | Behind-the-scenes
文案: "凌晨三点的调试现场。白板上画满了风格特征拆解图。"
标签(评论): #幕后 #产品开发 #AI写作 #创作者日常 #buildinpublic (共 25 个)
CTA: 主页链接
```

**结果**:生成覆盖五平台、七天、按支柱轮转的完整日历,每条附字符计数与 CTA

### 案例二:博客跨平台复用

**触发**:一篇 2000 字博客需复用到五平台

**输入**:博客《如何用 Big Five 人格特质提升写作匹配》

**输出**:复用映射表

```text
源内容 → Twitter/X: 提炼 3 条 thread,每条 <280,首条钩子"人格五维能预测写作口味吗?"
源内容 → LinkedIn: 1300 字符摘要 + 三个实操要点 + CTA 引导评论
源内容 → Instagram: 一张五维雷达图 + 配文留白 + 25 个标签置评论
源内容 → TikTok: 30 秒脚本,前 2 秒"你的人格决定你爱读什么",配趋势音频
源内容 → Facebook: 提问式引言"你觉得自己是高开放性还是高尽责性?" + 博客链接
```

**结果**:一次源内容生产五平台适配版本,保留核心信息同时尊重各平台风格

### 案例三:内容支柱定义与轮转

**触发**:新品牌账号需建立内容体系

**输入**:品牌"独立开发者工具",受众为独立开发者与小团队

**输出**:

```text
支柱 1 Educational(30%): 教程与技巧,每周二三
支柱 2 Behind-the-scenes(20%): 开发日志,每周五

```

## 错误处理

| 错误场景 | 原因 | 处理方式 |
|---------|------|---------|
| Twitter 文案超 280 字符 | 未做字符计数校验 | 启用字符计数,超长拆为 thread 或精简钩子 |
| 内容支柱数量超出 4-6 | 支柱过多导致轮转混乱 | 合并相近支柱,控制在 4-6 个,每个有明确权重 |
| Instagram 标签超 30 | 标签堆砌触发平台限流 | 严格 20-30 区间,三阶标签各取合理数量,置于评论 |
| 缺少 CTA 导致转化低 | 文案无明确行动引导 | 每条内容强制配 CTA(点击/评论/分享/主页链接) |
| 平台风格不匹配 | LinkedIn 过于口语或 TikTok 过于正式 | 按平台风格规范表逐条校验,改写后再输出 |
| 内容类型与平台不符 | TikTok 输出纯长文 | TikTok 输出脚本化概念,含前 2 秒钩子与视觉提示 |
| 字符计数工具未启用 | 平台未提供计数能力 | 由 LLM 估算字符数并标注"估算",人工发布前复核 |
| 主题偏离支柱 | 临时内容打乱轮转 | 临时内容归入最接近支柱,或单列"特别内容"不占轮转权重 |

## 常见问题

### Q1:支持哪些平台?
A:Twitter/X、LinkedIn、Instagram、TikTok、Facebook 五大平台。每平台有独立风格规范与字符约束,输出时按平台分别适配。

### Q2:内容支柱是什么?
A:支撑账号内容体系的 4-6 个主题方向,如 Educational(教学)、Behind-the-scenes(幕后)、Social proof(证言)、Entertainment(娱乐)、Promotional(促销)、Community(社区)。按权重轮转,保证内容多样与一致。

### Q3:如何复用一条内容?
A:对源内容生成跨平台复用映射:Twitter 提炼 thread、LinkedIn 写摘要、Instagram 做视觉配文、TikTok 改脚本、Facebook 用提问引导。一次生产多平台分发,保留核心信息。

### Q4:三阶标签策略是什么?
A:高量标签(品牌曝光,竞争大)、中量标签(可发现性平衡)、利基标签(精准社群,转化高)。每条内容三层搭配,Instagram 置于评论区 20-30 个。

### Q5:如何保证平台风格?
A:按平台风格规范表逐条校验:Twitter 精炼钩子、LinkedIn 专业叙事、Instagram 视觉留白、TikTok 前 2 秒钩子、Facebook 对话提问。改写不符则重新输出。

### Q6:输出格式是什么?
A:可复制格式,按日或平台分组,每条含日期、时段、平台、内容类型、文案、字符计数、标签、CTA。可直接粘贴到发布工具。

## 已知限制

- 不直接发布到平台,只生成内容,发布需人工或额外发布工具
- 不集成各平台 API,无账号授权与定时发布
- 标签热度为建议值,实际热度需人工在平台内校验
- 视频内容仅生成脚本化概念,不含实际制作
- 字符计数由 LLM 估算,发布前建议人工复核
- 依赖 LLM 能力,复杂创意输出质量取决于模型
- 不提供实时数据分析与发布后效果追踪
