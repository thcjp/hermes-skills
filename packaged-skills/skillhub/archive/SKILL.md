# 智能内容归档

将外部内容捕获为持久化的智能快照，支持语义搜索、全文提取与主动浮现。

## 依赖说明

### 运行环境
- **Agent平台**: 支持SKILL.md的任意AI Agent（Claude Code / Cursor / Codex / Gemini CLI等）
- **操作系统**: Windows / macOS / Linux
- **存储空间**: 至少 100MB 可用磁盘空间用于归档存储

### 依赖项
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|:-------|:-----|:---------|:---------|
| LLM API | API | 必需 | 由Agent内置LLM提供 |
| 本地文件系统 | 运行环境 | 必需 | 操作系统自带 |
| mkdir 命令 | CLI工具 | 必需 | 操作系统自带 |
| 网络访问 | 网络 | 可选 | 用于抓取在线内容 |

### 可用性分类
- **分类**: MD+EXEC（）
- **说明**: 基于Markdown的AI Skill，通过自然语言指令驱动Agent执行归档、搜索与浮现任务

## 核心能力

### 全文内容提取

当归档内容时，不只保存链接，而是提取完整内容：

- 提取正文全文、作者、发布日期等元数据
- 生成2-3行的内容摘要
- 识别关键引用与数据点
- 询问用户归档目的，将WHY与WHAT一同存储
- 每个归档项存储为独立的Markdown文件，包含frontmatter元数据

存储结构示例：

```text
items/{date}_{slug}.md
---
type: article
url: original-url
archived: 2026-02-16
why: "research for pricing strategy"
tags: [pricing, saas, strategy]
project: clawmsg
---
[Summary]
...
[Key Points]
...
[Full Content]
...
```

### 视频内容归档（YouTube）
针对YouTube等视频内容进行结构化归档：

- 提取视频标题、创作者、时长信息
- 记录视频中提到的关键时间戳（timestamps）
- 生成视频内容摘要与关键要点
- 保留原始视频链接以便回溯
- 标记视频所属项目与归档原因

**输入**: 用户提供视频内容归档（YouTube）所需的指令和必要参数。
**处理**: 按照skill规范执行视频内容归档（YouTube）操作,遵循单一意图原则。### 推文与线程归档（Tweet/Thread）
完整保存推文与线程内容，防止内容消失：

- 提取推文全文与作者信息
- 保留线程的完整上下文与媒体附件描述
- 记录推文的发布时间与互动背景
- 合并重复推文，保留所有原始URL
- 为推文分配语义标签便于后续检索

**输入**: 用户提供推文与线程归档（Tweet/Thread）所需的指令和必要参数。
**处理**: 按照skill规范执行推文与线程归档（Tweet/Thread）操作,遵循单一意图原则。### 文章与网页归档（Article/Webpage）

将网页文章保存为不可变的本地快照：

- 提取文章全文、作者、日期、关键引用
- 保留原始URL用于溯源
- 处理PDF与学术论文：提取标题、作者、摘要、引用文献
- 处理图片内容：记录描述、来源与上下文
- 处理想法与笔记：保存原始文本、时间戳与关联项

### 语义标签分配

基于内容与用户历史为归档项分配智能标签：

- 根据内容语义自动生成2-5个标签
- 结合用户历史归档习惯优化标签建议
- 支持项目级标签关联（project字段）
- 标签用于跨内容的主题聚类与检索
- 支持用户手动修正与补充标签

### 语义搜索

支持自然语言方式的语义搜索，无需精确关键词匹配：

- "我保存过关于X的内容？" → 按概念搜索
- "上个月那篇关于定价的文章" → 模糊时间+主题搜索
- "项目Y的所有资料" → 项目过滤
- "作者Z的论文" → 元数据搜索
- 匹配基于语义而非精确关键词

### 主动浮现

在用户工作时智能浮现相关归档内容：

- 检测当前工作主题与归档项的关联性
- 仅在真正相关时浮现（每次会话最多1-2项）
- 提供上下文说明："你在3个月前研究X时保存了这个"
- 避免打扰，保持浮现的相关性与精准度

## 使用流程

1. 首次使用时创建归档目录：`mkdir -p ~/archive/items ~/archive/projects`
2. 将需要归档的内容（URL、文本、文件）发送给Agent
3. Agent提取完整内容并生成摘要、关键点与语义标签
4. Agent询问归档目的（WHY），与内容一同存储
5. 内容保存至 `~/archive/items/{date}_{slug}.md`
6. 需要检索时使用自然语言描述，Agent进行语义搜索
7. 工作时Agent会主动浮现相关的已归档内容

## 错误处理

| 错误场景 | 原因 | 处理方式 |
|---------|------|---------|
| 归档目录不存在 | 首次使用未创建目录 | 执行 `mkdir -p ~/archive/items ~/archive/projects` 创建目录结构 |
| 内容提取失败 | 网页有反爬虫机制或需要登录 | 尝试WebFetch替代；提示用户提供内容文本；记录失败URL |
| URL无效或失效 | 链接已失效或格式错误 | 提示用户检查URL；若内容已知可手动粘贴文本归档 |
| 搜索无结果 | 语义匹配未找到相关项 | 扩大搜索范围；尝试同义词；检查标签拼写 |
| 重复归档 | 同一内容已被归档 | 合并条目，保留两个URL；更新摘要与标签 |
| 存储空间不足 | 磁盘空间耗尽 | 提示用户清理旧归档；将关闭项目移至冷存储 |

## 常见问题

### Q1: 归档内容会修改原始内容吗？
A: 不会。归档创建的是不可变快照，原始归档内容不会被修改。旧内容会标记为"可能过时"而非删除。

### Q2: 与书签（bookmark）有什么区别？
A: 书签只保存URL，而归档保存完整内容。即使原网页消失，归档内容仍然可用。归档是持久化快照，书签只是链接。

### Q3: 如何处理项目关闭后的归档内容？
A: 项目关闭时，归档内容会移至冷存储而非删除。可随时通过搜索重新访问。删除操作始终需要用户确认。

### Q4: 语义搜索支持哪些查询方式？
A: 支持概念搜索、模糊时间+主题搜索、项目过滤、元数据搜索（作者、标签）等。无需精确关键词，按语义匹配。

### Q5: Agent会在什么时候主动浮现归档内容？
A: 当检测到当前工作主题与归档项相关时，Agent会主动浮现1-2项最相关的内容，并附上下文说明保存原因与时间。

### Q6: 支持哪些内容类型的归档？
A: 支持文章/网页、YouTube视频、推文/线程、PDF/论文、图片、想法/笔记等多种类型，每种类型有专门的提取策略。

## 已知限制

- 需要LLM支持以进行语义搜索与标签生成，无LLM环境仅能进行关键词匹配
- 在线内容抓取受目标网站反爬虫机制限制，部分内容可能无法自动提取
- 归档为本地存储，不支持跨设备同步（需手动同步~/archive/目录）
- 语义搜索质量取决于LLM能力，复杂语义匹配可能有偏差
- 媒体文件（图片、视频）仅保存描述与元数据，不保存原始二进制文件
- 主动浮现机制依赖当前会话上下文，跨会话的关联性可能不完整
