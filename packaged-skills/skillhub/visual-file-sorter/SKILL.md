## 视觉识别分类逻辑

### 图片文件识别流程

1. **文件类型判定**: 首先通过扩展名和MIME类型筛选图片文件（.png/.jpg/.jpeg/.webp/.bmp/.gif）
2. **视觉内容分析**: 对图片调用视觉模型，识别以下维度：
   - 内容类型：截图/照片/设计稿/图表/表情包
   - 主体内容：文字/人物/风景/UI界面/产品/文档
   - 文字提取：OCR识别图片中的关键文字（如发票号、日期、标题）
   - 场景上下文：时间信息、来源标识
3. **语义命名**: 基于识别结果生成描述性文件名

### 命名规则

| 文件类型 | 命名模板 | 示例 |
|:------|:------|:------|
| 截图 | `截图_{界面描述}_{日期}.png` | `截图_设置面板_20260724.png` |
| 发票 | `发票_{公司名}_{金额}_{日期}.jpg` | `发票_阿里云_299元_20260715.jpg` |
| 合同 | `合同_{对方公司}_{类型}_{日期}.pdf` | `合同_腾讯云_服务协议_20260601.pdf` |
| 照片 | `照片_{场景描述}_{日期}.jpg` | `照片_团队合影_20260720.jpg` |
| 表情包 | `表情_{描述}.gif` | `表情_点赞猫咪.gif` |
| 设计稿 | `设计稿_{页面名}_{版本}.png` | `设计稿_登录页_v2.png` |

### 非图片文件处理

对于非图片文件（.pdf, .docx, .txt, .py等），系统通过以下方式识别内容：
- **文本文件**: 读取文件头部内容，提取关键词和主题
- **PDF文档**: 提取文本层，识别文档类型（发票/合同/报告）
- **代码文件**: 解析文件扩展名和内容特征，识别语言和用途
- **压缩包**: 读取文件列表，推断内容主题

## 处理工作流

### 步骤1：扫描目标目录

```bash
# 扫描下载文件夹
scan --target ~/Downloads --recursive=false

# 扫描桌面（含子目录）
scan --target ~/Desktop --recursive=true --max-depth=2
```

### 步骤2：视觉识别与分类

对每个文件执行识别，输出分类建议：

```json
{
  "file": "IMG_20260724_001.png",
  "original_path": "~/Downloads/IMG_20260724_001.png",
  "recognition": {
    "type": "screenshot",
    "content": "VS Code编辑器界面，显示Python代码",
    "extracted_text": "def main(): print('Hello World')",
    "confidence": 0.92
  },
  "suggested_name": "截图_VSCode_Python代码_20260724.png",
  "suggested_category": "图片/截图",
  "suggested_path": "~/Sorted/图片/截图/截图_VSCode_Python代码_20260724.png"
}
```

### 步骤3：执行归档

```bash
# 预览模式（不实际移动文件，仅显示建议）
sort --target ~/Downloads --dry-run=true

# 执行模式（实际移动并重命名文件）
sort --target ~/Downloads --dry-run=false

# 自定义归档根目录
sort --target ~/Downloads --output-root ~/Documents/Sorted
```

### 步骤4：处理冲突与去重

当目标路径已存在同名文件时，按以下策略处理：

| 策略 | 参数 | 说明 |
|:------|:------|:------|
| 跳过 | `--conflict=skip` | 保留原文件不动，记录到日志 |
| 覆盖 | `--conflict=overwrite` | 替换已有文件 |
| 重命名 | `--conflict=rename` | 添加后缀（_2, _3...） |
| 去重比较 | `--conflict=dedup` | 计算MD5哈希，内容相同则删除重复 |

## 输入格式

| 参数名 | 类型 | 必填 | 说明 |
|---:|---:|---:|---:|
| target | string | 是 | 待整理的目标目录路径 |
| output_root | string | 否 | 归档根目录, 默认 ~/Sorted |
| recursive | boolean | 否 | 是否递归扫描子目录, 默认 false |
| max_depth | integer | 否 | 递归最大深度, 默认 1 |
| dry_run | boolean | 否 | 预览模式, 不实际移动文件, 默认 true |
| conflict | string | 否 | 冲突处理策略: skip/overwrite/rename/dedup, 默认 rename |
| custom_rules | object | 否 | 自定义分类规则JSON |

## 常见问题

### Q1: 如何开始使用Visual File Sorter？
A: 指定要整理的目录即可，例如"整理我的下载文件夹"或"sort ~/Downloads"。建议首次使用时启用 `dry_run=true` 预览模式，查看分类建议后再实际执行。系统会逐个识别文件内容，生成语义化文件名并归档到对应分类目录。

### Q2: 视觉识别的准确率如何？
A: 图片识别准确率通常在85-95%之间，取决于图片内容的清晰度和类型。截图、发票等结构化内容的识别率较高（90%+），抽象图片或模糊照片的识别率较低。每个识别结果都附带置信度评分，低于0.6的结果会标记为"待确认"，建议人工复核。

### Q3: 如何自定义分类规则？
A: 通过 `custom_rules` 参数传入JSON格式的规则配置。规则支持按文件扩展名、内容关键词、视觉特征进行匹配。例如可以定义"所有含'发票'关键词的PDF归档到财务/发票目录"，或"所有分辨率大于1920x1080的图片归档到高清图片目录"。

### Q4: 处理大量文件时速度如何？
A: 视觉识别速度约为2-5秒/文件（取决于图片大小和网络延迟）。100个文件约需5-8分钟。系统支持批量处理，会在处理过程中显示进度条。对于非图片文件（文本/PDF/代码），识别速度更快（<1秒/文件）。建议在闲时运行大规模整理任务。

### Q5: 误分类的文件如何恢复？
A: 系统在执行归档前会生成操作日志（`~/Sorted/.sorter-log.json`），记录每个文件的原始路径和新路径。误分类后可通过日志逆向恢复。也可以使用 `--dry-run=true` 先预览所有分类建议，确认无误后再执行实际操作。
