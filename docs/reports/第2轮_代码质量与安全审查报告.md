### C2. 定价双写无事务保护的代码实锤

**关联第1轮问题#2（Critical）**

**文件**：`d:\skills\skill-registry\pricing_engine.py`
**行号**：第406-484行（cmd_apply） + 第487-527行（cmd_update_db）

**问题描述**：
定价信息分两步写入：先写SKILL.md文件，再写数据库。两步操作无事务保护，无原子性。如果第一步成功第二步失败，会导致文件和数据库数据不一致。

**问题代码**：
```python
# cmd_apply：写SKILL.md（第481行）
skill_md.write_text(new_content, encoding='utf-8')  # 无异常处理
updated += 1

# cmd_update_db：写数据库（第514-520行）
c.execute("""
    UPDATE skills 
    SET suggested_price = ?, pricing_category = ?, pricing_rationale = ?, 
        pricing_tier = ?, is_paid = ?
    WHERE slug = ?
""", (result['final_price'], result['category'], result['rationale'],
      result['tier'], 1 if result['is_paid'] else 0, result['slug']))
```

**失败场景分析**：
1. `cmd_apply` 成功修改100个SKILL.md，`cmd_update_db` 执行时数据库锁定 → 文件已改，数据库未改
2. `cmd_apply` 修改到第50个文件时磁盘满 → 前49个已改，后51个未改，数据库全部未改
3. `cmd_update_db` 更新到第50条时异常 → 前49条已更新，后51条未更新

**影响**：
- 定价数据不一致导致门控检查失败
- 用户看到的定价与实际收费不符
- 严重影响"赚钱"目标的可信度

**修复建议**：
```python
def cmd_apply_and_update_db():
    """原子性更新定价：先写数据库，成功后再写文件"""
    pricing_report = Path(r"d:\skills\skill-registry\pricing_report.json")
    if not pricing_report.exists():
        print("请先运行 price-all 生成定价报告")
        return
    
    with open(pricing_report, 'r', encoding='utf-8') as f:
        all_results = json.load(f)
    
    # 构建slug到路径映射
    slug_to_path = _build_slug_path_mapping()
    
    # 第1步：数据库事务（全部成功或全部回滚）
    conn = sqlite3.connect(DB_PATH)
    try:
        c = conn.cursor()
        for result in all_results:
            c.execute("""
                UPDATE skills 
                SET suggested_price = ?, pricing_category = ?, pricing_rationale = ?, 
                    pricing_tier = ?, is_paid = ?
                WHERE slug = ?
            """, (result['final_price'], result['category'], result['rationale'],
                  result['tier'], 1 if result['is_paid'] else 0, result['slug']))
        conn.commit()  # 数据库原子提交
    except Exception as e:
        conn.rollback()
        print(f"数据库更新失败，已回滚: {e}")
        return
    finally:
        conn.close()
    
    # 第2步：写文件（数据库已成功，文件失败可重试）
    updated = 0
    failed = []
    for result in all_results:
        skill_md = slug_to_path.get(result['slug'])
        if not skill_md:
            continue
        try:
            _update_skill_md_pricing(skill_md, result)
            updated += 1
        except Exception as e:
            failed.append((result['slug'], str(e)))
    
    if failed:
        print(f"警告：{len(failed)}个文件写入失败，数据库已更新")
        for slug, err in failed:
            print(f"  {slug}: {err}")
```

### C4. tools字段检查逻辑错误，条件永不触发

**文件**：`d:\skills\skill-registry\upload_gate.py`
**行号**：第253-256行

**问题描述**：
tools字段格式检查的逻辑存在严重错误，第三个条件永远为False，导致检查形同虚设。

**问题代码**：
```python
# 第253-256行
tools_match = re.search(r'^tools:\s*$', fm, re.MULTILINE)        # YAML数组格式
tools_inline = re.search(r'^tools:\s*\[', fm, re.MULTILINE)      # 内联数组格式
if not tools_match and not tools_inline and not re.search(r'^tools:', fm, re.MULTILINE):
    # 第三个条件 not re.search(r'^tools:', ...) 永远为False
    # 因为如果前两个不匹配，但tools字段存在，第三个search会匹配
    issues.append({'severity': 'BLOCKER', 'category': 'frontmatter', 'message': 'tools字段缺失'})
```

**逻辑分析**：
- 条件1：`tools:` 后面为空（YAML数组）
- 条件2：`tools: [...]`（内联数组）
- 条件3：`tools:` 存在

如果 `tools: "字符串格式"` （错误格式）：
- 条件1 = False（不匹配，因为后面有内容）
- 条件2 = False（不匹配，因为不是`[`开头）
- 条件3 = True（匹配，因为`tools:`存在）
- `not True` = False
- 整个条件 = False，不报错

**结果**：错误的tools格式（如 `tools: "Read,Write"`）不会被检测为问题。

**修复建议**：
```python
def check_tools_format(fm: str) -> Optional[str]:
    """检查tools字段格式，返回错误信息或None"""
    # 检查tools字段是否存在
    tools_exists = re.search(r'^tools:', fm, re.MULTILINE)
    if not tools_exists:
        return 'tools字段缺失'
    
    # 检查格式：必须是YAML数组或内联数组
    tools_yaml = re.search(r'^tools:\s*\n((?:\s+-\s+.+\n?)+)', fm, re.MULTILINE)
    tools_inline = re.search(r'^tools:\s*\[', fm, re.MULTILINE)
    
    if not tools_yaml and not tools_inline:
        # tools存在但格式错误
        tools_str = re.search(r'^tools:\s*(.+?)\s*$', fm, re.MULTILINE)
        if tools_str:
            return f'tools格式错误(应为YAML数组): {tools_str.group(1)}'
        return 'tools格式错误(应为YAML数组)'
    
    return None

# 在check_correctness中调用
tools_error = check_tools_format(fm)
if tools_error:
    issues.append({'severity': 'BLOCKER', 'category': 'frontmatter', 'message': tools_error})
```

### C6. scan_skillhub无限循环风险

**文件**：`d:\skills\skill-registry\market_monitor.py`
**行号**：第87-134行

**问题描述**：
`while True` 循环没有最大页数限制，如果API返回的total值异常（如0、负数、或一直返回满页数据），会导致无限循环。

**问题代码**：
```python
# 第87行
while True:
    # ...
    # 第128行
    total = data.get('data', {}).get('total', data.get('total', len(all_skills)))
    # 第131行
    if len(skills) < 50 or len(all_skills) >= total:
        break
    page += 1
```

**失败场景**：
1. API返回 `total: 100000` 但每页都返回50条 → 循环2000次
2. API返回 `total: 0` 但每页返回50条 → `len(all_skills) >= 0` 立即break（幸运）
3. API返回 `total: -1` → `len(all_skills) >= -1` 永远True，但第一页就break
4. 最危险：API持续返回满页数据且total很大 → 长时间阻塞

**修复建议**：
```python
def scan_skillhub():
    MAX_PAGES = 100  # 安全阀值
    all_skills = []
    page = 1
    total = 0
    
    while page <= MAX_PAGES:
        url = f"{SKILLHUB_PUBLIC_SKILLS}?page={page}&page_size=50&sort=downloads&order=desc"
        data = fetch_json(url)
        if not data:
            break
        
        skills = _parse_skills_response(data)
        if not skills:
            break
        
        all_skills.extend(skills)
        total = _get_total_from_response(data, len(all_skills))
        
        print(f"  获取 {len(skills)} 个skill，累计 {len(all_skills)}/{total}")
        
        if len(skills) < 50 or len(all_skills) >= total:
            break
        
        page += 1
    
    if page > MAX_PAGES:
        print(f"  警告：达到最大页数限制 {MAX_PAGES}，可能未获取全部数据")
    
    return all_skills
```

import re
from pathlib import Path
from typing import Tuple, Optional

def parse_skill_md(content: str) -> Tuple[str, str, str]:
    """统一解析SKILL.md，返回(raw_content, frontmatter, body)"""
    if content.startswith('\ufeff'):
        content = content[1:]
    
    if content.startswith('---'):
        parts = re.split(r'^---\s*$', content, maxsplit=2, flags=re.MULTILINE)
        if len(parts) >= 3:
            return content, parts[1].strip(), parts[2]
    
    return content, '', content

def extract_fm_field(fm: str, field: str) -> Optional[str]:
    """统一提取frontmatter字段值"""
    # 尝试带引号
    match = re.search(rf'^{field}:\s*["\'](.+?)["\']\s*$', fm, re.MULTILINE)
    if match:
        return match.group(1).strip()

    if match:

    # 尝试多行（description: |-）
    match = re.search(rf'^{field}:\s*\|-\s*\n((?:\s+.+\n?)+)', fm, re.MULTILINE)
    if match:
        return match.group(1).strip()
    return None

    try:
        yield conn
        conn.commit()
    except Exception:
        conn.rollback()
        raise
    finally:
        conn.close()

# 使用
with get_db_connection() as conn:
    c = conn.cursor()
    c.execute("SELECT ...")
    results = c.fetchall()

# 方案2：使用 with sqlite3.connect()
with sqlite3.connect(DB_PATH) as conn:
    c = conn.cursor()
    c.execute("SELECT ...")
    # 退出with块自动commit，但异常时不会rollback
