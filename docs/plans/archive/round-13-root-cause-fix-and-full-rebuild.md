## Round 13 核心策略：根因修复 + 全量重建

### 策略一：废除规则模板生成管线

**目标**：从代码层面杜绝模板垃圾的产生

**具体操作**：
1. 在 `generate_skill.py` 的 `run_generation_pipeline()` 中，将 `skip_l2` 参数默认值改为 `False`
2. 在 `batch_generate.py` 中，删除 `skip_l2=True, skip_dep_verify=True` 参数
3. 在 `fill_remaining_placeholders()` 开头添加硬阻断：
   ```python
   def fill_remaining_placeholders(...):
       # 禁止使用通用默认值填充
       # 如果有未填充的placeholder，直接返回失败
       remaining = re.findall(r'\{\{(\w+)\}\}', content)
       if remaining:
           return None, f"FATAL: 未填充的placeholder: {remaining}. 必须使用LLM生成,禁止模板默认值"
       return content, "OK"
   ```
4. 在 `batch_generate.py` 中添加生成方式检查：
   ```python
   if result.get('template_used') == 'pipeline_template':
       if not result.get('llm_generated'):
           logger.error(f"{slug}: 使用了模板生成,跳过L2. 这是禁止行为.")
           continue
   ```

**验证**：运行 `batch_generate.py` 确认不再产生模板填充的skill

### 策略二：SF检查器假阳性修复

**目标**：消除54.6%的"100%覆盖率"假阳性

**具体操作**：
1. 在 `compute_capability_coverage()` 中，当源能力点为空时，覆盖率返回 `None` 而非 `100%`
2. 在最终评分中，当覆盖率为 `None` 时，该项不得分（而非满分）
3. 在批量结果中，明确标记 `source_caps_empty: true`
4. 对源能力点为空的skill，使用替代验证：检查生成版本的核心能力章节数量是否≥3

```python
def compute_capability_coverage(source_points, gen_content):
    if not source_points:
        # 源能力点为空,无法验证覆盖率
        # 不返回100%假阳性,返回None并使用替代验证
        return None, "源能力点为空,使用替代验证"
    # ... 正常覆盖率计算
```

**验证**：重新运行SF批量检查，确认假阳性率从54.6%降至<10%

## 注意事项

1. **根因优先**: 先废除模板生成管线，再处理存量。不废除根因，再生成多少批都会继续产生垃圾
2. **诚实数据**: 所有质量判断基于SF批量检查结果和代码分析，不信文档描述
3. **不迎合不敷衍**: 85.4%不合格是事实，宁可花300分钟重建也不放行
4. **LLM强制**: 所有skill必须由LLM生成，禁止规则模板填充
5. **三层验证不可省略**: 每批必须运行L1+L2+SF三层验证
6. **假阳性修复优先**: SF检查器的54.6%假阳性必须先修复，否则后续验证结果不可信
7. **无源skill不放过**: 244个无源skill必须有替代质量标准，不能因为"无源"就跳过
8. **功能验证是终极标准**: 关键词匹配≠功能正确，L3功能验证是最终合格标准
9. **-free版本name字段**: name必须与slug和folder_name一致（如`name: memory-free`）
10. **P0优先**: 68个模板垃圾必须最先修复，这些是质量最差的skill
