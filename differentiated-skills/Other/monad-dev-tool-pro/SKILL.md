# Monad 开发工具 - 专业版

## 概述

Monad 开发工具专业版是企业级 Monad 区块链 DApp 开发平台,在免费版核心能力基础上扩展主网部署、批量合约管理、安全审计辅助与 CI/CD 集成。适合团队协作开发、企业级 DeFi 协议部署与大规模 NFT 项目管理.
专业版完全兼容免费版项目结构,支持从免费版无缝升级,已有测试网合约可平滑迁移至主网.
## 核心能力

### 1. 双环境部署管理

支持 Monad 主网(Chain ID: 143)与测试网(Chain ID: 10143)之间的无缝切换,内置环境隔离与安全检查机制.
**输入**: 用户提供双环境部署管理所需的指令和必要参数.
**处理**: 解析双环境部署管理的输入参数,完成核心逻辑,返回结构化响应.
**输出**: 返回双环境部署管理的响应数据,包含状态码、结果和日志.
- 执行此能力时使用`input_params`参数,支持创建/查询/导出操作
### 2. 批量合约部署

支持多合约批量部署、合约升级管理、部署回滚机制,适合复杂 DApp 项目的多合约协同部署.

### 场景二:CI/CD 自动化部署流水线

```yaml

name: Monad Contract Deploy
# ...
on:
  push:
    branches: [develop]
  release:
    types: [published]
# ...
jobs:
  test-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install Foundry
        run: curl -L https://foundry.paradigm.xyz | bash && foundryup
      - name: Run tests
        run: forge test -vvv
      - name: Deploy to testnet
        if: github.ref == 'refs/heads/develop'
        run: |
          forge script （请参考skill目录中的脚本文件）:DeployScript \
            --rpc-url https://testnet-rpc.monad.xyz \
            --private-key ${{ secrets.TESTNET_PRIVATE_KEY }} \
            --broadcast
      - name: Deploy to mainnet
        if: github.event_name == 'release'
        run: |
          forge script （请参考skill目录中的脚本文件）:DeployScript \
            --rpc-url https://rpc.monad.xyz \
            --private-key ${{ secrets.MAINNET_PRIVATE_KEY }} \
            --broadcast
      - name: Verify contracts
        run: （请参考skill目录中的脚本文件）
```

## 最佳实践

1. **主网部署前必过测试网**:所有合约必须先在测试网完成完整功能测试
2. **多签管理主网密钥**:主网操作必须使用多签钱包,避免单点故障
3. **CI/CD 自动化**:将测试、部署、验证全流程自动化,减少人为错误
4. **安全审计先行**:主网部署前必须完成安全审计,修复所有高危和中危问题
5. **Gas 优化**:使用 `via_ir` 和高 `optimizer_runs` 降低主网 Gas 成本
6. **部署记录追踪**:每次部署保存完整记录,包含合约地址、部署者、时间戳
7. **升级模式规划**:使用代理模式(UUPS/Transparent)规划合约可升级性

## 常见问题

### Q: 如何从免费版测试网合约迁移到主网?

A: 专业版支持环境配置切换。更新 `deploy.config.json` 中的环境配置,使用主网 RPC 和私钥重新部署。合约地址会变化,需要更新前端配置中的合约地址.
### Q: 批量部署时合约依赖如何处理?

A: 专业版部署脚本会自动分析合约依赖关系(`depends` 字段),按依赖顺序部署,并在后续合约构造函数中注入已部署合约地址.
### Q: 多签钱包部署如何操作?

A: 配置 `requireMultisig: true` 后,部署交易会发送到多签合约,需要足够数量的签名者确认后才会执行。专业版提供多签状态查询与催签功能.
### Q: CI/CD 中如何安全管理私钥?

A: 使用 CI/CD 平台的 Secrets 功能存储私钥(如 GitHub Secrets),通过环境变量注入。专业版推荐使用硬件钱包或 KMS 服务,CI/CD 中仅使用只读签名代理.
### Q: Gas 优化有哪些技巧?

A: 1) 开启 `via_ir` 进行深度优化;2) 调高 `optimizer_runs`(主网推荐 1000+);3) 使用 `assembly` 优化热点路径;4) 减少存储写入,多用 `immutable` 和 `constant`;5) 批量操作替代循环.
