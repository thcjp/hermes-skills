## 使用场景

### 场景一:从主题到成文

从一个主题出发,完成一篇短篇。

```bash
# 1. 生成故事构思
python3 short_story.py ideate \
  --theme "孤独" \
  --genre "都市" \
  --count 3

# 输出:
# === 故事构思 ===
# 构思1: 独居老人在智能音箱中找到了陪伴,却发现...
# 构思2: 夜班出租车司机搭载的最后一位乘客...
# 构思3: 社交媒体上的完美生活背后...

# 2. 选择构思并搭建结构
python3 short_story.py structure \
  --idea 1 \
  --structure "three-act" \
  --words 5000

# 输出:
# === 三幕式结构 ===
# 第一幕(1500字): 设定
#   - 老人独居,子女在外
#   - 收到智能音箱作为礼物
#   - 开始与音箱日常对话
#
# 第二幕(2500字): 冲突
#   - 音箱越来越了解老人
# 依赖说明
#   - 发现音箱记录了私密对话
#
# 第三幕(1000字): 解决
#   - 老人面对选择
#   - 最终决定
#   - 情感升华

# 3. 生成全文
python3 short_story.py write \
  --structure "structure.json" \
  --style "温情细腻" \
  --words 5000 \
  --output "孤独的陪伴.md"
```

## 最佳实践

1. **先构思后写作**:花时间在构思上,好故事来自好点子
2. **结构先行**:确定结构再写正文,避免跑题
3. **字数分配**:短篇字数宝贵,每段都要有存在的意义
4. **结尾有力**:短篇的结尾尤其重要,留下深刻印象
5. **少即是多**:短篇不需要太多人物,聚焦核心角色
6. **修改润色**:初稿完成后至少修改两遍,精炼语言

## 常见问题

### Q: 生成的短篇可以投稿吗?

A: 生成的内容为初稿,建议修改润色后再投稿。工具帮助你快速产出框架和初稿,但最终质量取决于作者的修改与打磨。

### Q: 如何控制字数?

A: 在 `structure` 命令中指定 `--words` 参数,工具会按比例分配各部分字数。生成后可查看实际字数,如超出范围可用 `revise --trim` 命令精简。

### Q: 可以写系列短篇吗?

A: 可以。为每篇短篇创建独立项目,在人物卡中标注关联。免费版不支持系列管理,需手动维护人物与时间线一致性。

### Q: 如何提升文学性?

A: 1) 调整 style 参数为「文学性强」;2) 增加描写比例;3) 使用隐喻与象征;4) 多用展示少用讲述(show don't tell);5) 生成后人工润色,加入个人风格。
