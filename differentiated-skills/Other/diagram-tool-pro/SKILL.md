# 图表工具专业版（Diagram Tool Pro）

## 概述

专业版不仅生成图表，更管理图表的完整生命周期：从一次描述到一批图表，从单一版本到演化追踪，从代码块到交互式页面。本Skill面向需要批量产出、统一风格、长期维护图表资产的技术团队。

设计哲学：
1. **资产化**：图表是团队资产，需要版本管理与演化追踪

## 使用场景

### 场景一：企业文档中心批量配图
技术文档中心需要为100篇API文档配上时序图。使用批量生成能力，一次性输入所有API描述，自动产出100张统一风格的时序图，并建立索引页。

### 场景二：架构演化追踪
微服务架构每季度大调整，需要对比当前与历史架构图。版本管理能力记录每次变更，diff对比高亮新增/删除/修改的节点，让架构评审有据可依。

### 场景三：品牌化输出
公司有严格的视觉规范，所有对外文档的图表必须使用品牌色。主题系统支持自定义色板，生成的图表直接符合品牌要求，无需后期调色。

### 场景四：交互式架构演示
向stakeholder演示系统设计时，一张包含30个节点的架构图过于密集。交互式HTML输出支持折叠子域、点击展开详情，让演示节奏可控。

### 场景五：多语言图表维护
开源项目需要同时维护Mermaid（GitHub渲染）与PlantUML（企业Wiki渲染）两套图表。专业版支持一次描述双语法输出，保持内容同步。

### 场景六：教学课件批量生成
教育机构为50个算法各生成流程图与状态图，共100张图。批量生成+并行渲染，10分钟完成传统方式一天的工作量。

## 最佳实践

### 批量生成优化
1. **按域分组**：将相关图表放在同一批次，便于建立关联索引
2. **模板复用**：同类图表使用同一模板，确保风格一致
3. **并行阈值**：单批次建议20-30张，超过则分批处理避免内存压力
4. **检查点机制**：每批生成后自动校验语法，失败项单独重试

### 版本管理策略
1. **语义化版本**：图表版本号跟随架构版本，如v2.1.0对应架构v2.1
2. **变更说明**：每次提交附变更摘要，便于回溯
3. **定期归档**：每季度归档历史版本，保持工作目录精简
4. **分支演化**：重大架构调整使用分支，验证通过后合并

### 主题与品牌适配
1. **色板定义**：主色不超过3种，辅色不超过2种
2. **字体统一**：中文用PingFang/微软雅黑，英文用Inter/Roboto
3. **暗色模式**：提供暗色主题，适配夜间阅读场景
4. **无障碍**：颜色对比度符合WCAG AA标准，不仅依赖颜色区分

### 交互式HTML设计
1. **渐进式展开**：默认折叠二级节点，点击展开
2. **关键路径高亮**：鼠标悬停节点时高亮关联路径
3. **面包屑导航**：大型图表提供层级面包屑
4. **搜索能力**：支持节点名称搜索与定位

## 常见问题

### Q1：批量生成时部分图表语法错误怎么办？
A：专业版内置语法校验，错误图表会标记并跳过，不阻塞其他图表生成。生成结束后输出错误报告，可单独修复后重试。

### Q2：PlantUML需要什么环境？
A：需要Java运行时（JRE 8+）与plantuml.jar。首次使用时Agent会检测环境并提示安装方式。

### Q3：自定义主题如何共享给团队？
A：主题配置为YAML文件，提交到团队Git仓库即可共享。建议放在`docs/diagram-themes/`目录。

### Q4：版本管理的存储空间会不会膨胀？
A：专业版采用增量存储，仅记录diff，50个版本约占单个图表的5倍空间。建议每季度归档历史版本。

### Q5：交互式HTML能在内网使用吗？
A：可以。HTML文件自包含所有依赖（Mermaid.js内联），无需外部CDN，适合内网Wiki嵌入。

### Q6：能否与CI/CD集成？
A：支持。批量生成与导出均可通过命令行调用，适合集成到文档构建流水线。配置示例见版本管理章节。

### Q7：SVG导出能否保持矢量可编辑？
A：可以。SVG文件保留所有矢量信息，可在Figma、Illustrator中二次编辑。

### Q8：模板库能扩展吗？
A：支持自定义模板。将团队常用图表结构保存为模板文件，放入templates目录即可被识别复用。

### Q9：批量生成50张图大概多久？
A：并行渲染下约3-5分钟。取决于图表复杂度与机器性能，专业版支持负载均衡与断点续传。

### Q10：如何处理跨图表的实体一致性？
A：专业版维护实体注册表，同一实体在不同图中使用相同名称与样式。修改实体属性时，所有关联图表自动更新。

## 错误处理

| 错误场景(现象) | 可能原因 | 解决步骤 |
|------|----------|----------|
| PlantUML渲染失败 | Java未安装或版本过低 | 安装JRE 8+，验证`java -version` |
| SVG中文乱码 | 字体缺失 | 安装中文字体，或在主题中指定可用字体 |
| 批量生成内存溢出 | 单批次过大 | 拆分为多个小批次，每批≤20张 |
| 交互式HTML白屏 | 浏览器禁用JS | 启用JS，或使用静态PNG降级方案 |
| 版本diff不准确 | 图表ID变更 | 保持节点ID稳定，仅修改标签 |
| 主题不生效 | 配置路径错误 | 检查theme.name与配置文件匹配 |
