# V2Ray代理管理（免费版）

## 概述

V2Ray代理管理工具免费版帮助你管理 V2Ray 代理的自动开关，根据网络状况自动配置系统代理。提供启动/停止、自动配置、状态查看与连接测试四大核心能力。

## 核心能力

| 能力 | 说明 |
|:-----|:-----|
| 启动/停止 | 启动或停止 V2Ray 代理服务 |
| 系统代理配置 | 自动设置或清除系统代理 |
| 自动模式 | 根据网络状况自动开关代理 |
| 状态查看 | 查看代理运行状态与端口 |
| 连接测试 | 测试代理连接质量 |

## 使用场景

### 场景一：访问外网前开启代理

开发者需要访问外网 API，手动开启代理。

```bash
# 开启代理（启动 V2Ray + 设置系统代理）
bash scripts/v2ray-proxy.sh on

# 示例
# ✅ V2Ray 已启动
# ✅ 系统代理已设置 (127.0.0.1:10808)
# 📊 状态: 运行中
```

### 场景二：访问完成后关闭代理

完成外网访问后，关闭代理恢复直连。

```bash
# 关闭代理（清除系统代理 + 停止 V2Ray）
bash scripts/v2ray-proxy.sh off

# 输出示例
# ✅ 系统代理已清除
# ✅ V2Ray 已停止
# 📊 状态: 已停止
```

### 场景三：自动模式

根据网络状况自动决定是否开启代理。

```bash
# 自动模式
bash scripts/v2ray-proxy.sh auto

# 输出示例
# 🔍 检测网络状况...
# 📊 外网连通性: 不可达
# ✅ 自动开启代理
```

## 配置示例

```bash
# V2Ray 配置
V2RAY_PATH="/path/to/v2rayN-linux-64/"
PROXY_PORT="10808"

# 命令说明
| 命令    | 说明                      |
|:--------|:--------------------------|
| start   | 仅启动 V2Ray              |
| stop    | 仅停止 V2Ray              |
| on      | 启动 + 设置系统代理       |
| off     | 清除代理 + 停止           |
| auto    | 自动模式（根据网络状况）  |
| status  | 查看状态                  |
| test    | 测试连接                  |
```

## 最佳实践

* 访问外网前用 `on` 开启代理，访问完成后用 `off` 关闭。
* 不确定是否需要代理时用 `auto` 自动判断。
* 定期用 `test` 测试连接质量，确保代理稳定。
* 代理端口建议使用 10808，避免与其他服务冲突。
* 开机自启可单独设置，代理开关由本工具控制。
* 系统代理配置会修改系统网络设置，关闭时务必清除。

## 常见问题

**Q：免费版支持多代理节点切换吗？**
A：免费版使用固定配置的节点。如需多节点切换与负载均衡，请考虑 PRO 版本。

**Q：免费版支持定时开关吗？**
A：免费版不提供定时任务。如需定时开关与计划任务，请使用 PRO 版本。

**Q：自动模式如何判断是否需要代理？**
A：自动模式会测试外网连通性，不可达时自动开启代理。

**Q：支持哪些操作系统？**
A：支持 Windows、macOS、Linux。系统代理配置方式因系统而异。

**Q：代理端口可以自定义吗？**
A：免费版使用固定端口 10808。如需自定义端口，请修改脚本中的 `PROXY_PORT` 变量。
