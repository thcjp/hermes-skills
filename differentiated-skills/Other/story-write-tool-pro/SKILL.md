## 最佳实践

1. **批量生成+人工审阅**:批量生成提高效率,但必须人工审阅每章再发布
2. **多线规划**:开书前规划好主线、副线、暗线,避免写到后面忘了暗线
3. **市场定期分析**:每月分析市场趋势,及时调整方向
4. **协作明确分工**:多人协作时明确每人负责的章节和场景,避免内容冲突
5. **读者数据驱动**:关注追读率变化,追读率下降时分析原因并调整
6. **保持存稿**:至少保持 1 周存稿,避免突发情况断更
7. **风格统一**:多人协作必须使用风格统一功能,确保读者体验一致

## 常见问题

### Q: 批量生成的章节质量如何保证?

A: 专业版在批量生成后自动进行一致性检查与质量评分。建议:1) 生成后人工审阅每章;2) 重点关注风格一致性评分;3) 对评分低于 80 分的章节重新生成或手动修改;4) 保留人工修改的版本作为后续生成的风格参考.
### Q: 多人协作如何避免剧情冲突?

A: 1) 使用剧情线管理,明确各作者负责的线;2) 共享大纲与人物卡,所有作者必须遵循;3) 合并时自动检查一致性;4) 定期召开创作会议,同步进度与方向.
### Q: 市场适配会不会让作品失去特色?

A: 市场适配是参考建议,不是强制执行。作者应保持自己的核心创意,在非核心元素上参考市场趋势。专业版提供「建议」而非「自动修改」,作者有最终决定权.
### Q: 如何与网文平台对接?

A: 专业版支持与主流网文平台的 API 对接。配置平台 API Token 后,可一键发布章节、回收读者数据。具体支持的平台取决于 API 可用性.
