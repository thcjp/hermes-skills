# 书籍插画助手（免费版）

> **从"需要插画"到"插画完成"，结构化流程让创作不再迷茫。**

书籍插画不只是"画几张图"那么简单。它涉及需求分析、风格选择、章节分配、创作指导和项目管理多个环节。书籍插画助手采用"需求-风格-分配-指导-管理"五步流程，帮助作者和插画师系统化地完成书籍插画制作。

**角色一致性原则**：

| 一致性维度 | 说明 | 检查方法 |

- 一次只提出一个改进点
- 指向具体区域而非笼统概念
- 先肯定有效部分再提建议
- 不建议完全改变风格除非被明确要求

## 使用场景

### 场景一：自出版作者的插画规划（作者角色）

**痛点**：自出版一本儿童绘本，不知道需要多少幅插画、选择什么风格、如何与插画师沟通。

**解决方案**：
```text
1. 需求分析：儿童绘本，3-6岁，32页，插画为叙事核心
2. 风格匹配：推荐温暖手绘风，提供3个参考方向
3. 分配插画：每页1幅，共32幅，封面1幅
4. 创作指导：建立角色设定表，确保角色一致性
5. 项目管理：草图→线稿→上色→定稿，每阶段审核
```

**效果**：插画规划从"不知道从哪开始"变为清晰的5步流程，与插画师沟通效率提升。

### 场景二：独立插画师的书籍项目接单（插画师角色）

**痛点**：接到书籍插画项目但需求模糊，不知道如何与客户确认需求、管理项目进度。

**解决方案**：
```text
1. 使用需求模板与客户确认：
   - 书籍类型、读者群体、插画数量
   - 风格参考、色彩倾向、截止日期
2. 制定项目计划：
   - 草图阶段：2周
   - 线稿阶段：3周
   - 上色阶段：3周
   - 修改定稿：1周
3. 每阶段提交审核，记录修改意见
4. 建立角色设定表，确保跨页一致性
```

**效果**：项目接单后的沟通和流程管理系统化，客户满意度提升，修改次数减少。

### 场景三：技术书籍配图规划（技术作者角色）

**痛点**：技术书籍需要大量配图说明概念，但不知道哪些概念需要配图、用什么风格。

**解决方案**：
```text
1. 需求分析：技术书籍，插画为辅助说明
2. 风格匹配：推荐简洁线稿/信息图风格
3. 分配策略：
   - 架构图：系统设计章节
   - 流程图：算法说明章节
   - 对比图：概念辨析章节
   - 示意图：代码说明章节
4. 创作指导：准确优先，标注清晰，风格统一
5. 项目管理：与文字同步推进，分批交付
```

**效果**：配图规划系统化，读者理解效率提升，书籍专业度增强。

### 与插画师协作建议

1. **需求先行**：用需求文档明确所有细节，避免口头沟通的模糊
2. **分阶段审核**：草图→线稿→上色，每阶段确认后再进入下一步

## 常见问题

### Q1：一本书需要多少幅插画？

取决于书籍类型和插画角色。儿童绘本通常每页1幅（20-40幅），小说通常章首图+关键情节（5-15幅），技术书籍按概念需要配图（20-50幅）。建议先明确插画的角色（叙事核心/辅助说明/氛围营造），再确定数量。

### Q2：如何确保多幅插画中角色一致？

建立角色设定表，记录每个角色的外貌特征、服装风格、比例关系。在创作前制作角色参照图，每幅插画创作时参照。关键角色建议从正面、侧面、背面三个角度制作设定图。

### Q3：插画师和作者如何高效协作？

使用结构化的需求文档明确所有细节，采用分阶段审核流程（草图→线稿→上色→定稿），每阶段确认后再进入下一步。反馈时遵循"一次一个修改点"原则，所有修改意见记录在案。

### Q4：数字插画和传统手绘插画如何选择？

取决于书籍定位和预算。数字插画修改方便、输出灵活、成本较低，适合技术书籍和商业书籍。传统手绘有独特质感和艺术价值，适合高端绘本和艺术书籍。也可以混合使用，如手绘线稿+数字上色。
