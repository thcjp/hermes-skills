## 使用场景

### 场景一:创建小说项目

开始一部新小说的创作。

```bash
# 创建小说项目
python3 story_write.py init \
  --title "星辰修仙录" \
  --genre "修仙" \
  --target-words 3000000 \
  --author "你的笔名"

# 生成项目结构:
# 星辰修仙录/
# ├── outline.md          # 总大纲
# ├── characters/         # 人物卡
# │   ├── 主角.md
# │   └── 配角.md
# ├── settings/           # 设定集
# │   ├── 世界观.md
# │   └── 修炼体系.md
# ├── chapters/           # 章节内容
# │   ├── 第001章.md
# │   └── 第002章.md
# └── progress.json       # 进度追踪
```

### 场景二:生成章节内容

根据大纲生成新章节。

```bash
# 生成下一章
python3 story_write.py write \
  --project "星辰修仙录" \
  --chapter 15 \
  --outline "主角进入秘境,遭遇第一波考验" \
  --words 3000 \
  --style "previous"

# 输出:
# 正在生成第 15 章...
# 参考大纲: 主角进入秘境,遭遇第一波考验
# 参考前文: 第 1-14 章
# 目标字数: 3000 字
#
# === 第 15 章: 秘境初探 ===
# [生成的章节内容]
#
# 一致性检查:
#   人物名称: 通过
#   设定引用: 通过
#   时间线: 通过
```

### 场景三:人物卡管理

```bash
# 创建人物卡
python3 story_write.py character create \
  --project "星辰修仙录" \
  --name "林凡" \
  --role "主角" \
  --age 18 \
  --personality "坚韧不拔,重情重义" \
  --background "废材出身,觉醒逆天灵根" \
  --abilities "星辰诀(主修),剑术(辅修)"

# 查看人物关系
python3 story_write.py character relations \
  --project "星辰修仙录" \
  --name "林凡"

# 输出:
# 林凡 的人物关系:
#   玄机子: 师父 (亦师亦友)
#   苏婉儿: 恋人 (青梅竹马)
#   赵天: 对手 (同门竞争)
#   苏明: 好友 (肝胆相照)
```

## 最佳实践

1. **大纲先行**:先完成总大纲和分卷大纲,再开始写章节
2. **人物卡完整**:主要人物必须有完整的人物卡,避免性格漂移
3. **定期一致性检查**:每写 10 章做一次全面一致性检查
4. **风格统一**:参考前文风格生成新内容,保持读者体验一致
5. **章末钩子**:每章结尾留悬念,提升读者追更欲望
6. **及时更新设定**:新增设定及时录入设定集,避免遗忘

## 常见问题

### Q: 生成的章节质量如何?

A: 生成的内容为初稿,建议作者在此基础上修改润色。工具擅长保持一致性和结构,创意和情感仍需作者把控。建议生成后人工审阅修改。

### Q: 如何保持人物性格一致?

A: 1) 完善人物卡,记录性格、口头禅、行为模式;2) 生成章节时引用人物卡;3) 定期运行一致性检查;4) 发现人物性格偏差及时修正。

### Q: 可以写非网文类型吗?

A: 可以。工具支持任意类型小说创作,调整 genre 和 style 参数即可。但优化偏向网文节奏,传统文学建议降低爽点频率、增加描写深度。

### Q: 项目数据如何备份?

A: 整个项目目录复制即可备份。建议使用 Git 进行版本管理,每次更新后提交,便于追溯修改历史。
