# UI审计工具（免费版）

## 概述

UI审计工具免费版帮助你根据成熟的 UX 原则评估界面设计。聚焦视觉层级、视觉风格与可访问性三大核心维度，生成结构化审计报告，适合个人设计师与前端开发者的日常自查。

| 视觉层级 | 标题区分度、主操作清晰度、分组/邻近性、阅读流、字号层级、色彩层级、留白、视觉权重 |

## 使用场景

### 场景一：设计稿交付前自查

设计师在交付前对界面进行快速审计。

```text
用户：帮我审计这个登录页面的设计。
助手：
审计维度：视觉层级 + 视觉风格 + 可访问性

视觉层级检查：
- 标题区分度: ✅ 通过
- 主操作清晰度: ⚠️ 登录按钮与次要按钮视觉权重接近
- 分组/邻近性: ✅ 通过
- 阅读流: ⚠️ 视线在「忘记密码」与「登录」之间跳跃
- 字号层级: ✅ 通过

视觉风格检查：
- 间距一致性: ❌ 输入框间距不一致（16px vs 12px）
- 色彩规范: ✅ 通过
- 字体系统: ✅ 通过

可访问性检查：
- 色彩对比: ❌ 次要文字对比度 3.2:1，低于 4.5:1 标准
- 触控目标: ⚠️ 「忘记密码」链接触控区域仅 36px

优先修复：
1. 统一输入框间距为 16px
2. 提升次要文字对比度至 4.5:1 以上
3. 增大「忘记密码」触控区域至 44px
```

### 场景二：前端开发的自查

开发者根据审计报告调整实现。

```text
用户：审计一下这个卡片组件的可访问性。
助手：
可访问性检查：
- 键盘可操作性: ✅ Tab 顺序正确
- 可见焦点: ❌ 焦点样式被全局 outline:none 移除
- 色彩对比: ✅ 主文字 7.2:1
- alt 文本: ⚠️ 装饰性图片缺少 alt=""
- 语义标记: ⚠️ 使用 div 而非 button 实现点击

优先修复：
1. 恢复焦点可见样式
2. 为装饰性图片添加 alt=""
3. 将可点击的 div 改为 button 元素
```

### 场景三：学习UX原则

通过审计实践学习设计原则。

```text
用户：审计这个页面，并解释每个检查项背后的原则。
助手：
（审计输出 + 原则解释）
- 分组/邻近性: 相关元素应靠近，不相关元素应远离（格式塔接近性原则）
- 色彩对比 4.5:1: WCAG AA 标准，确保视力障碍用户可读
- 触控目标 44px: Apple HIG 最小触控区域，确保手指操作准确
```

审计流程：
1. 确认审计范围与维度
2. 逐项检查（每维度 6-10 个检查项）
3. 标注状态（pass/warn/fail/na）
4. 生成优先修复建议
5. 输出结构化报告

## 配置示例

```text
审计偏好示例：
- 审计维度: 视觉层级 + 视觉风格 + 可访问性
- 检查项数量: 每维度 6-10 个
- 状态标注: pass / warn / fail / na
- 对比度标准: WCAG AA (4.5:1)
- 触控目标: 44px (Apple HIG)
- 报告格式: JSON + 文字说明
```

## 最佳实践

* 审计前明确审计范围，避免过度检查。
* 每个维度的检查项控制在 6-10 个，保证可执行性。
* 优先修复建议按影响度排序，先修高影响问题。
* 可访问性检查遵循 WCAG AA 标准（对比度 4.5:1）。
* 触控目标至少 44px，确保移动端可用性。
* 审计报告保留原始状态记录，便于修复后复查。
