# Agent身份助手（专业版）

## 概述

当企业内部同时运行着10+个AI助手——客服机器人、IT工单助手、HR问答机器人、销售陪练Agent——就会出现一个棘手问题：**这些助手说话风格不一致，价值观彼此冲突，边界划定标准不一**。有的助手对用户过度承诺，有的助手在L3边界上含糊其辞，还有的助手因为业务调整后人格未同步更新而显得突兀。

Agent身份助手专业版正是为解决这类"人格治理"问题而生。它在免费版三层身份模型的基础上，提供完整的多人格矩阵、动态路由、合规审计与A/B测试能力，让团队像管理代码资产一样管理Agent人格资产。

## 核心能力

### 能力1：多人格矩阵管理

支持在一个工作区内管理20+个Agent人格卡，按业务线、品牌、用户群体多维度分类：

| 维度 | 示例 | 用途 |
|---|---|---|
| 业务线 | 售前/售后/客服/技术支持 | 按业务场景分配人格 |
| 品牌 | 主品牌/子品牌/联名品牌 | 品牌矩阵账号管理 |
| 用户分层 | VIP/普通用户/新用户 | 差异化人格体验 |
| 地区 | 国内/海外/港澳 | 本地化人格适配 |

**输入**: 用户提供能力1：多人格矩阵管理所需的指令和必要参数。
**处理**: 解析能力1：多人格矩阵管理的输入参数,完成核心逻辑,返回结构化响应。
**输出**: 返回能力1：多人格矩阵管理的响应数据,包含状态码、结果和日志。

## 使用场景

### 场景组1：客服中心负责人 × 多业务线人格管理

- 目标层：帮用户在3轮内判断产品是否匹配需求

- 目标层：在5轮内完成情绪安抚+方案给出
- 基调：导师（耐心引导）
- 反语调：不催促、不催办

#### 场景1.3 技术支持人格设计
- 目标层：直接给可执行方案
- 基调：同事（直接、有判断力）
- 边界：L2请示才执行账号变更

### 场景组2：合规审计员 × 金融行业Agent部署

#### 场景2.1 投顾Agent合规审计
- 检测目标层是否含"保本保收益"等违规承诺

- 检测反语调是否覆盖"暴力催收话术"

### 场景组3：产品经理 × A/B测试最优人格

#### 场景3.1 新用户引导人格测试
- 对照组A：朋友基调（温暖随意）

#### 场景3.2 VIP用户服务人格测试

## 最佳实践

### 实践1：人格矩阵分阶段建设

不要一次性定义20个人格。建议按"1个核心人格 → 3个业务线人格 → N个细分场景人格"的节奏迭代。每增加一个，先用A/B测试验证效果再正式上线。

### 实践2：合规审计前置

在人格卡上线前，先由审计员角色运行合规检查。重点检查反语调清单是否覆盖行业敏感话术、L3边界是否覆盖所有禁止场景。

### 实践3：灰度发布 + 自动熔断

新人格上线时先灰度5%流量，监控关键指标。若满意度下降超过5%或投诉率上升超过2%，自动回滚至上一版本。

### 实践4：人格版本与代码版本解耦

人格卡独立于Agent代码部署，支持热更新。这样业务侧调整人格不需要重新发布代码，降低发布风险。

### 实践5：定期审计与迭代

每季度运行一次全量合规审计，识别：
- 因业务变化而过时的价值观条款
- 新出现的行业敏感话术未纳入反语调
- 边界L3未覆盖的新业务场景

## 常见问题

### Q1：专业版支持多少个人格卡？
A：单工作区上限为100张人格卡，超出可申请扩容。

### Q2：动态路由引擎的延迟是多少？
A：P99延迟<50ms，对会话体验无感知影响。

### Q3：合规审计报告支持哪些行业标准？
A：内置金融（银保监）、医疗（HIPAA）、电商（消保法）、教育（未成年人保护）等行业模板，支持自定义规则集。

### Q4：A/B测试的统计显著性如何保证？
A：内置贝叶斯统计引擎，自动计算样本量需求，达到显著性阈值后才输出结论。

### Q5：能否对接现有CRM/CDP系统？
A：支持。通过Webhook方式从CRM获取用户画像信号，也支持将路由决策日志回写至CDP。

### Q6：人格卡能否导出为标准格式？
A：支持YAML、JSON、OpenAPI Persona Spec三种导出格式。

### Q7：团队成员权限如何分配？
A：支持管理员/编辑者/审计员/观察者四种角色，可按工作区或单张人格卡粒度授权。

### Q8：灰度发布如何与A/B测试区分？
A：灰度发布是"逐步放量降低风险"，A/B测试是"对照实验寻找最优解"。两者可组合使用——先A/B测试确定胜出人格，再灰度发布逐步放量。

### Q9：人格卡变更如何回滚？
A：每次修改自动生成版本快照，支持任意版本一键回滚，回滚操作有完整审计日志。

### Q10：是否支持私有化部署？
A：企业版支持私有化部署，人格卡数据全程不出企业网络。

## 版本升级迁移指南

```bash
# 1. 导出免费版人格卡
agent-identity export --from free --output ./backup.zip
# ...
# 2. 在专业版工作区导入

agent-identity routing enable
```

## 依赖说明

### 运行环境
- **Agent平台**：支持SKILL.md的任意AI Agent（Claude Code / Cursor / Codex / Gemini CLI等）
- **操作系统**：Windows / macOS / Linux
- **运行时**：Node.js 18+（用于动态路由引擎）/ Python 3.10+（用于合规审计脚本）

### 依赖详情
| 依赖项 | 类型 | 是否必需 | 获取方式 | 版本要求 |
|----|:--:|---:|----|:--:|
| LLM API | API | 必需 | 由Agent平台内置LLM提供 | 不限 |
| Node.js | 运行时 | 必需 | 官方下载 | 18+ |
| Python | 运行时 | 必需 | 官方下载 | 3.10+ |
| YAML解析器 | 库 | 必需 | npm/pip | pyyaml 6.0+ |
| 统计引擎 | 库 | 可选 | pip | scipy 1.10+ |

### API Key 配置
- **SkillHub Token**：存储于 `d:\skills\.skillhub-credentials\api-key.txt`（已gitignore）
- **CRM/CDP集成Token**：通过环境变量 `IDENTITY_CRM_TOKEN` 注入
- **私有化部署Token**：存储于企业内网密钥管理服务
- **禁止**：在SKILL.md或脚本中硬编码Token

### 可用性分类
- **分类**：MD+EXEC（Markdown指令 + 命令行工具）
- **说明**：核心身份定义基于Markdown，动态路由与合规审计需要exec命令行执行能力
