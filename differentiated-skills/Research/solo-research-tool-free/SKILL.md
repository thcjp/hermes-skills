## 最佳实践

### 1. 研究前明确产品类型

在想法描述中明确产品类型,帮助AI助手选择正确的研究重点(SEO vs ASO):

```text
# 好:明确类型
"我想做一个Web端的AI知识管理工具"
"我想做一个iOS的健身追踪App"

# 不好:类型模糊
"我想做一个AI工具"(web? mobile? cli?)
```

### 2. Reddit搜索关键词不超过3个

Reddit搜索效果随关键词增多而下降。每次搜索使用2-3个精准关键词:

```text
# 好:精准
"knowledge management frustrating"

# 不好:冗长
"AI knowledge management tool frustrating annoying problems limitations"
```

### 3. 域名检查使用三重验证

域名可用性检查使用三重验证(whois -> dig -> RDAP),避免误判:

```bash
# 三重验证流程
whois example.dev          # 1. whois查询
dig example.dev            # 2. DNS解析
curl RDAP查询 example.dev  # 3. RDAP验证
```

### 4. 市场规模估算要保守

免费版市场规模估算基于WebSearch综合多源数据,可能偏高。对估算结果保持保守态度,结合自身可获取市场(SOM)做决策。

### 5. 研究报告是起点而非终点

`research.md`是研究的起点。基于报告发现,进一步深入调研感兴趣的维度,或运行`/validate`进行想法验证。

## 常见问题

### Q: 研究过程中部分搜索返回空结果怎么办?

A: 可能原因:(1)关键词过于具体,尝试更通用的关键词;(2)site:限定过严,去掉site:限定重试;(3)网络问题。建议调整关键词后重试,或换一个角度搜索。

### Q: 域名检查显示可用但实际被注册怎么办?

A: 某些TLD(如.app/.dev)的whois显示的是TLD创建日期而非域名注册日期,导致误判。使用三重验证(whois -> dig -> RDAP),重点检查Name Server和Registrar字段,而非创建日期。

### Q: 竞品分析遗漏了重要竞品怎么办?

A: 免费版通过WebSearch发现竞品,可能遗漏非英文市场或小众竞品。建议:(1)在想法描述中提及已知竞品;(2)搜索" alternatives"关键词;(3)手动补充已知竞品信息。

### Q: research.md已存在怎么办?

A: 如果`docs/research.md`已存在,AI助手会询问是否覆盖。建议选择合并(保留历史发现,补充新信息)或重命名旧报告后重新生成。

### Q: 产品类型判断不准确怎么办?

A: 如果AI助手推断的产品类型不准确(如将移动应用误判为web),在想法描述中明确指定:"这是一个iOS应用"或"这是一个命令行工具"。产品类型影响研究重点(SEO vs ASO),准确判断很重要。

### Q: 免费版能否搜索学术论文?

A: 免费版通过WebSearch搜索,可覆盖部分学术站点(如arxiv.org),但不专门针对学术文献优化。如需深度学术文献调研,建议使用专门的学术搜索工具或升级到专业版。
