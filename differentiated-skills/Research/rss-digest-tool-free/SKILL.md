## 快速开始

### 依赖详情

```bash
# macOS
brew install odysseus0/tap/feed

# Linux/WSL (从源码构建)
git clone https://example.com/feed.git
cd feed && cargo install --path .
```

### 第二步:导入入门订阅集

首次使用时数据库为空,需要导入一组起始订阅源:

```bash
feed import starter-tech-blogs.opml
```

### 第三步:生成你的第一份摘要

```bash
# 拉取最新内容
feed fetch

# 扫描未读条目
feed get entries --limit 50
```

然后将扫描结果交给AI助手,告诉它:"帮我从这些条目里挑出关于AI基础设施的内容并生成摘要"。

### 第四步:阅读与决策

AI助手返回摘要后,你可以:

- 让AI深入阅读某条目全文(`feed get entry <id>`)
- 按主题继续筛选(`feed search "<关键词>"`)
- 手动标记已读(`feed update entries --read <id> ...`)

## 最佳实践

### 1. 优先直接抓取URL

条目表格包含完整URL。优先让AI通过WebFetch直接抓取URL,这样全文不会进入上下文窗口,节省Token。仅在无网络抓取能力时回退到`feed get entry <id>`。

### 2. 不要让AI自动标记已读

已读状态应由用户自己决定。AI助手仅负责扫描、筛选和摘要,不替你做已读决策。

### 3. 默认使用表格输出

表格输出是Token效率最高的扫描格式。避免使用`-o json`,除非需要程序化处理。

### 4. 条目过多时按源过滤

如果未读条目堆积过多,使用`--feed <feed_id>`按订阅源过滤,分批处理:

```bash
feed get entries --feed hn --limit 20
feed get entries --feed arxiv --limit 20
```

### 5. 定期清理停滞源

通过`feed get feeds`观察各源更新频率,移除长期不更新的源,保持订阅集精简高效。

## 常见问题

### Q: 首次运行`feed get entries`返回0条结果怎么办?

A: 先运行`feed get stats`检查数据库状态。如果订阅源数为0,说明需要先导入起始订阅集。导入后运行`feed fetch`拉取内容,再重试扫描。

### Q: 摘要质量不稳定,有时很浅显怎么办?

A: 摘要质量取决于全文获取的完整性。确保AI优先通过URL直接抓取原文。如果源站点有反爬限制,回退到`feed get entry <id>`读取存储内容,但可能信息不够完整。

### Q: feed CLI安装失败怎么办?

A: 确认系统已安装Rust工具链(`rustc --version`)。如果没有,先安装rustup。macOS用户推荐使用Homebrew方式安装,避免源码编译的依赖问题。

### Q: 订阅源URL失效导致抓取报错怎么办?

A: 运行`feed get feeds`查看各源状态。对报错的源,手动验证URL是否可访问。若已永久失效,在配置中移除该源或替换为有效地址。

### Q: 搜索功能找不到我知道存在的内容怎么办?

A: 全文搜索依赖已抓取并存储的内容。如果条目从未被`feed fetch`拉取过,搜索无法命中。先执行`feed fetch`确保内容入库,再进行搜索。
