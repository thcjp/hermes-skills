# 详细参考 - swarm-coder-free

> 本文件从SKILL.md拆分而来，包含详细代码示例和扩展章节。

## 代码示例 (bash)

```bash
ls docs/plans/feature-plan.md 2>/dev/null && echo "检测到计划文件"

mkdir -p .swarm/{prompts,reports,logs}

cat > .swarm/prompts/implementer.md << 'EOF'
你是实现者子代理。你的任务是：

[INSERT_TASK_FULL_TEXT]

[INSERT_CONTEXT]

1. 实现代码
2. 编写测试
3. 提交代码
4. 自我审查

- 开始前如有疑问，立即提问
- 实现中如有疑问，立即提问
- 不要假设，要询问

- 不读取计划文件（控制器提供全文）
- 不跳过测试
- 不在main/master分支直接实现
EOF

cat > .swarm/prompts/spec-reviewer.md << 'EOF'
你是规范审查子代理。检查代码是否符合规范：

1. 所有规范要求是否实现
2. 是否有额外未要求的实现
3. 是否有遗漏的规范项

✅ 规范合规 - 所有要求已实现，无额外
或
❌ 问题：
- 缺失：[规范要求但未实现]
- 额外：[未要求但实现了]
EOF

cat > .swarm/prompts/quality-reviewer.md << 'EOF'
你是质量审查子代理。检查代码质量：

1. 代码可读性
2. 测试覆盖率
3. 错误处理
4. 性能问题
5. 安全问题

优势：[列出]
问题：[列出，标注严重度：Critical/Important/Minor]
结论：✅ 批准 / ❌ 需修复
EOF
```
