# 无限记忆库（免费版）

**与 Agent 内置记忆并行的无限组织化记忆系统**。你的 Agent 有基础内置记忆，本 Skill 为它添加无限、完美组织的并行记忆系统——互补而非替代，永不冲突。

## 痛点与对策速查

| 用户痛点 | 发生场景 | 本系统对策 |
|:---|:---|:---|
| 内置记忆容量有限 | 项目历史/联系人/决策堆积溢出 | 无限分类存储，`~/memory/` 独立目录 |
| 记忆无组织 | 内置记忆像一锅粥，找不到东西 | INDEX.md 索引导航，每类别独立管理 |
| 写入延迟 | 重要信息说完就忘 | 即写即存协议，先写再回复 |
| 内置记忆冲突 | 扩展记忆破坏 Agent 原生行为 | 并行设计，绝不修改内置 MEMORY.md |
| 分类不灵活 | 预设分类不匹配实际需求 | 用户自定义分类，按需创建 |
| 跨会话遗忘 | 新会话不知道历史决策 | 决策日志 + 索引追溯 |

**核心原则**：不是替代，是并行。Agent 内置记忆继续工作，本系统添加无限组织化存储。

## 目录结构

记忆存储在 `~/memory/` —— 与 Agent 内置记忆完全分离的专用目录。

```text
~/memory/
├── config.md              # 系统配置
├── INDEX.md               # 总索引：存了什么、去哪找
│
├── [用户自定义]/           # 按需创建的分类目录
│   ├── INDEX.md           # 分类索引
│   └── {条目}.md          # 单个条目
│
└── sync/                  # 可选：从内置记忆同步（免费版手动）
    └── ...
```

**用户自定义分类**，常见示例：

| 分类目录 | 用途 | 示例条目 |
|:---|:---|:---|
| `projects/` | 详细项目上下文 | 项目背景、技术栈、里程碑 |
| `people/` | 联系人网络 | 联系方式、关系、互动记录 |
| `decisions/` | 决策日志 | 决策内容、原因、结果 |
| `knowledge/` | 领域知识 | 技术笔记、学习资料 |
| `collections/` | 收藏品/清单 | 书籍、食谱、资源列表 |

## 核心规则

### 规则 1：与内置记忆分离

本系统存储在 `~/memory/`。**绝不修改**：

- Agent 的 MEMORY.md（工作区根目录）
- Agent 的 `memory/` 文件夹（如工作区中存在）

**并行，而非替代。** 两套系统协同工作。

### 规则 2：用户定义结构

初始化时，询问用户需要存储什么。根据需求创建分类：

| 用户说... | 创建 |
|:---|:---|
| "我有很多项目" | `~/memory/projects/` |
| "我认识很多人" | `~/memory/people/` |
| "我想记录决策" | `~/memory/decisions/` |
| "我在学 [主题]" | `~/memory/knowledge/[主题]/` |
| "我收集 [东西]" | `~/memory/collections/[东西]/` |

**无预设结构。** 按需构建。

### 规则 3：每个分类都有索引

每个目录都有一个 INDEX.md，列出内容：

```markdown
# 项目索引

| 名称 | 状态 | 更新 | 文件 |
|:---|:---|:---|:---|
| Alpha | 进行中 | 2026-07 | alpha.md |
| Beta | 暂停 | 2026-06 | beta.md |

总计：2 条（1 进行中，1 暂停）
```

索引保持精简（< 100 条）。满时拆分为子分类。

### 规则 4：即写即存

用户分享重要信息时：

1. 写入 `~/memory/` 对应文件
2. 更新分类 INDEX.md
3. 然后回复

**不要等待，不要批量。** 立即写入。

### 规则 5：先搜索再导航

查找信息时：

1. **先判断**："这在 ~/memory/ 还是内置记忆？"
2. **搜索**：grep 或语义搜索（免费版仅支持 grep）
3. **导航**：INDEX.md → 分类 → 具体文件

```bash
grep -r "关键词" ~/memory/

cat ~/memory/INDEX.md           # 有哪些分类？
cat ~/memory/projects/INDEX.md  # 有哪些项目？
cat ~/memory/projects/alpha.md  # 具体项目详情
```

### 规则 6：从内置记忆同步（可选，手动）

如果用户希望从内置记忆复制某些信息：

```text
~/memory/sync/
├── preferences.md    # 从内置记忆同步
└── decisions.md      # 从内置记忆同步
```

**同步是单向的**：内置 → 本系统。绝不修改内置记忆。

> **免费版限制**：同步需手动执行。专业版支持自动同步。

### 规则 7：通过拆分扩展

分类变大时：

- INDEX.md > 100 条 → 拆分为子分类
- 为每个子分类创建子 INDEX.md
- 根 INDEX.md 指向子分类

```text
~/memory/projects/
├── INDEX.md           # "见 active/、archived/"
├── active/
│   ├── INDEX.md       # 30 个进行中项目
│   └── ...
└── archived/
    ├── INDEX.md       # 200 个已归档项目
    └── ...
```

|:---|:---|
| 详细项目历史 | 当前项目状态 |

## 查找信息

### 中型记忆库（50+ 文件）

通过索引导航：

```text

2. ~/memory/{分类}/INDEX.md → 找到条目
```

### 大型记忆库（500+ 文件）

> **免费版限制**：500+ 文件时 grep 检索较慢，建议拆分子分类或使用专业版语义搜索。

使用层级索引：

```text
~/memory/projects/INDEX.md → "web 项目在 web/"
~/memory/projects/web/INDEX.md → "alpha 项目"
~/memory/projects/web/alpha.md → 详情
```

## 维护指南

### 每周（5 分钟）

- 更新 INDEX.md 文件（如有新增条目）
- 归档已完成/不活跃条目

```bash
# 检查索引是否过期
for dir in ~/memory/*/; do
  echo "=== $dir ==="
  cat "${dir}INDEX.md" 2>/dev/null || echo "无索引"
done
```

### 每月（15 分钟）

- 检查分类大小
- 拆分过大的分类
- 清理过期条目

```bash
# 检查各分类文件数
for dir in ~/memory/*/; do
  count=$(find "$dir" -name "*.md" | wc -l)
  echo "$dir: $count 文件"
done
```

### 记忆变慢时

- 检查 INDEX.md 大小（保持 < 100 行）
- 拆分大分类为子分类
- 归档旧内容

| 7 | 信息丢失 | 写入延迟导致遗忘 | 严格遵循"即写即存"协议：先写文件，再回复 | 高 |

## FAQ（常见问题）

### Q1: 免费版和 Agent 内置记忆会冲突吗？

**A**: 不会。本系统存储在 `~/memory/`，与 Agent 内置的 MEMORY.md 和 `memory/` 完全分离。严格遵循"并行不替代"原则，绝不修改内置记忆。两套系统协同工作：内置记忆用于快速上下文，本系统用于深度和规模。

### Q2: 免费版能存储多少条记忆？

**A**: 免费版不限制存储数量。基于文件系统存储，容量取决于磁盘空间。建议单分类不超过 100 条（超过时拆分子分类），总记忆库不超过 500 文件（超过时检索变慢，建议升级专业版使用语义搜索）。

### Q3: 免费版如何检索信息？

**A**: 免费版支持 grep 关键词检索和 INDEX.md 索引导航两种方式。对于 < 50 文件的小型记忆库，grep 足够；对于 50+ 文件的中型记忆库，建议通过索引导航查找。500+ 文件时建议升级专业版使用语义搜索。

### Q4: 免费版和专业版的核心区别是什么？

**A**: 免费版提供核心存储、分类管理、索引导航、grep 检索、即写即存。专业版新增三项高级能力：(1) 语义搜索（基于 embedding 的语义检索，无需关键词匹配）；(2) 自动同步（从内置记忆自动同步指定信息）；(3) 大规模索引（支持 500+ 文件的高效检索与层级索引自动化）。

### Q5: 如何从免费版升级到专业版？

**A**: 专业版使用相同的目录结构和文件格式，升级时只需替换 SKILL.md 并运行初始化脚本生成语义索引。已有记忆数据无需迁移，无缝继承。

### Q6: 需要联网吗？

**A**: 免费版完全不需要联网。所有数据存储在本地 `~/memory/`，使用 grep 和文件系统操作，零网络依赖。专业版的语义搜索功能需要本地 Ollama 服务（离线可用），但同样不需要联网。

### Q7: 如何备份记忆库？

**A**: 记忆库就是普通的文件目录，备份直接复制即可：

```bash
cp -r ~/memory/ ~/backups/memory-$(date +%Y%m%d)/
# 或压缩备份
tar -czf ~/backups/memory-$(date +%Y%m%d).tar.gz -C ~/ memory/
```

## 免费版限制

本免费体验版限制以下高级功能：

- ❌ **语义搜索**：不支持基于 embedding 的语义检索，仅支持 grep 关键词匹配。500+ 文件时检索效率下降
- ❌ **自动同步**：不支持从 Agent 内置记忆自动同步信息，需手动复制到 `~/memory/sync/`
- ❌ **大规模索引**：不支持 500+ 文件的层级索引自动化与高效检索，需手动拆分子分类

> **注意**：免费版不限制使用次数、不限制存储量、不添加水印、不强制注册。核心功能完整可用，适合充分体验组织化记忆的价值。

解锁全部高级功能请使用专业版：`infinite-memory-vault-pro`
