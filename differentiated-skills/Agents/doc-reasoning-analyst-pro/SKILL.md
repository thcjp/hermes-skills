# 文档推理分析师（专业版）

> **将冗长、杂乱的高风险文档转化为更清晰的推理和更干净的决策——现在支持多版本对比、结构优化与决策报告。**

文档推理分析师专业版是一款长文本推理优化器，专注于商业与法律文档工作流。在免费版核心分析能力之上，专业版解锁三大高级功能：多版本对比分析、结构改进建议、决策准备报告，为高风险决策提供全方位支持。

## 适用场景

使用本技能的场景包括：

- 分析长篇合同、备忘录、提案或谈判记录
- 对比两份以上文档版本的差异与风险变化
- 生成文档结构优化方案与重写建议
- 输出完整的决策就绪报告（含风险矩阵）
- 提升密集文本材料的推理质量
- 发现矛盾、薄弱逻辑或缺失假设
- 将大型文档压缩为决策就绪摘要
- 批量扫描多份文档的风险点

本技能**不会**：

- 替代持牌法律意见
- 保证受监管或高风险事项的正确性
- 签署法律、税务或合规决策
- 替代专业法律顾问的角色

### 2. 风险与薄弱点检测

系统检查以下问题类型：

| 检测维度 | 说明 | 示例 |
|----------|------|------|
| 模糊性 | 条款表述不清晰导致歧义 | "合理时间内"未定义 |
| 矛盾性 | 文档内部条款相互冲突 | 付款条件前后不一致 |
| 术语未定义 | 关键术语缺乏明确定义 | "知识产权"范围不清 |
| 范围缺失 | 缺少明确的适用边界 | 责任范围未限定 |
| 决策逻辑缺失 | 缺少触发条件与决策路径 | 终止条件未约定 |
| 隐藏风险转移 | 风险承担方不明确 | 不可抗力条款模糊 |
| 结构混乱 | 逻辑层次不清影响理解 | 条款编号无规律 |

**输入**: 用户提供风险与薄弱点检测所需的指令和必要参数。
**处理**: 按照skill规范执行风险与薄弱点检测操作,遵循单一意图原则。
**输出**: 返回风险与薄弱点检测的执行结果,包含操作状态和输出数据。

## 推理原则

本技能遵循七条核心推理原则，确保分析质量：

1. **保留关键细节**：不因压缩而丢失重要信息
2. **不压缩风险**：风险信号必须显式标注，不可隐藏
3. **区分事实与推断**：明确标注哪些是文档陈述、哪些是分析推断、哪些是行动建议
4. **识别显式与隐含**：区分文档明确表述的内容与需要推断的隐含内容
5. **优先标注缺失假设**：在提出结论前，先标出文档依赖但未说明的假设
6. **偏好清晰而非华丽**：用最直接的方式表达，避免修饰性语言
7. **不虚构法律或商业确定性**：当文本无法支撑确定性结论时，明确标注不确定性

## 执行协议（五步法）

当用户提供长文档时，Agent按以下顺序执行：

### Step 1：解析上下文

提取以下信息：

- 文档类型（合同/备忘录/提案/政策等）
- 目标受众（法务/管理层/执行团队等）
- 支持的决策（签署/批准/谈判/拒绝等）
- 主要义务、主张或请求
- 商业或法律敏感度等级

### Step 2：识别逻辑结构

梳理以下要素：

- 文档在说什么（核心主张）
- 为什么重要（决策相关性）
- 依赖哪些假设（显式与隐含）
- 在审查下可能哪里出问题（薄弱环节）

### Step 3：检测薄弱点

逐项检查：模糊性、矛盾性、未定义术语、范围缺失、决策逻辑缺失、隐藏风险转移、结构混乱。

### Step 4：提升实用性

根据用户请求选择处理方式：摘要、重写、对比、诊断、结构改进、决策报告，或转化为简报/备忘录/检查清单。

### Step 5：护栏机制

如果无法从文本中确立法律或商业确定性：

- 明确说明无法确定
- 标注不确定性区域
- 不虚构信心或确定性结论
- 建议交由专业顾问复核

## 性能优化策略

### 批量文档处理

对于多份文档的批量分析，建议采用以下策略：

1. **分批处理**：每批3-5份文档，避免上下文过载
2. **检查点保存**：每批分析完成后保存结果，防止中断丢失
3. **增量合并**：逐批合并分析结果，最终输出汇总报告
4. **幂等设计**：重复分析同一文档应产出一致结果

### 缓存与复用

- 对比分析中，已分析过的版本可缓存其核心逻辑提取结果
- 相同类型的文档可复用风险检测模板
- 标准输出格式可预加载，减少重复生成开销

### 成本控制

- 简单文档（<5页）使用快速分析模式
- 复杂文档（>20页）使用深度分析模式
- 多版本对比时，先提取各版本核心逻辑再对比，避免全量重复处理

## FAQ

### Q1：本技能能替代律师审查吗？

不能。本技能提供分析支持，帮助你在正式法律审查前识别风险和逻辑问题，但不替代持牌律师的专业意见。所有高风险条款应交由法律顾问复核。决策准备报告中的建议行动仅供参考。

### Q2：支持哪些文档格式？

支持任何可转化为文本的文档格式。你可以直接粘贴文本，或让Agent读取文件内容。PDF、Word、Markdown、HTML等格式均可处理。对于扫描版PDF，需先进行OCR转换。

### Q3：文档太长会被截断吗？

本技能针对长文本优化设计。对于超长文档（超过50页），建议按章节分段分析，最后合并评估结果。专业版支持增量合并，可逐步处理超长文档而不丢失上下文。

### Q4：分析结果中的⚠️标记代表什么？

⚠️标记表示风险或薄弱点。根据严重程度分为三级：
- **高风险**：可能导致经济损失、法律纠纷或合规违规
- **中风险**：需要澄清、谈判或补充条款
- **低风险**：建议优化但不紧急

### Q5：多版本对比支持几份文档？

专业版支持同时对比2-5份文档版本。超过5份时建议分组对比，先两两对比再合并结果。对比结果包含条款级差异清单、风险变化矩阵和修改建议。

### Q6：决策准备报告包含哪些内容？

决策准备报告包含：决策事项摘要、建议结论、风险矩阵（严重程度×发生概率×影响范围）、关键条款摘要、建议行动清单（含负责人与截止时间）、需法律顾问复核项。

### Q7：如何获得更精准的分析？

提供完整的上下文信息：文档类型、审查目的、支持决策、已知风险点、输出受众、期望输出模式、角色定位。上下文越完整，分析越精准。专业版支持按角色定制输出重点。

### Q8：结构改进建议会直接重写我的文档吗？

不会。结构改进建议输出的是优化方案和改写建议，你需要自行决定是否采纳。建议会标注具体修改位置、修改原因和修改后效果，便于你逐步应用。

### Q9：专业版与免费版有什么区别？

专业版在免费版核心分析能力之上，新增三大高级功能：多版本对比分析（条款级差异+风险变化矩阵）、结构改进建议（结构优化+表述改写）、决策准备报告（风险矩阵+建议行动）。此外提供多角色场景指南、性能优化策略和多平台集成示例。

### Q10：批量处理文档时有数量限制吗？

专业版不限制处理数量。建议采用分批处理策略（每批3-5份），配合检查点保存和增量合并，确保处理稳定。批量处理结果可汇总为统一报告。

### Q11：分析结果可以导出吗？

分析结果以文本格式输出，可复制至任何文档编辑器。建议保存为Markdown格式以保留结构。专业版的决策准备报告可直接作为审批流程的附件使用。

## 故障排查表

| 问题 | 可能原因 | 解决方案 | 优先级 |
|------|----------|----------|--------|
| 分析结果过于笼统 | 上下文信息不足 | 补充文档类型、审查目的、决策目标、角色定位等信息 | 高 |
| 风险点遗漏 | 文档过长未分段 | 将超长文档按章节分段，逐段分析后增量合并 | 高 |
| 输出格式不规范 | Agent未识别技能 | 在指令开头明确引用"使用文档推理分析师专业版技能" | 中 |
| 摘要丢失关键细节 | 压缩过度 | 指定"保留所有风险条款原文"或"不压缩风险信号" | 高 |
| 分析耗时过长 | 文档过大且未指定范围 | 指定分析重点章节，跳过无关内容；使用快速分析模式 | 中 |
| 推理原则未应用 | Agent未加载完整协议 | 在指令中重申"遵循五步执行协议和七条推理原则" | 低 |
| 多版本对比结果不完整 | 版本间差异过大导致匹配失败 | 先提取各版本核心逻辑再对比；指定对比重点条款 | 高 |
| 决策报告风险矩阵为空 | 未指定决策目标 | 明确"支持决策：[具体决策内容]"，提供已知压力点 | 中 |
| 结构建议不适用 | 文档类型识别错误 | 明确指定文档类型（合同/备忘录/提案/政策） | 低 |
| 批量处理中断 | 上下文过载或网络中断 | 减少每批数量至3份；启用检查点保存；从中断点恢复 | 高 |
