# 专注流优化器（免费版）

> **一套可信系统，胜过五套互相打架的清单。先诊断瓶颈，再开处方。**

本技能为AI Agent驱动的个人生产力操作系统。它不堆砌功能，而是将方向、执行、焦点、习惯、复盘整合到 `~/focus-flow/` 这一个本地目录，让Agent在你确认后写入并维护.
## 设计哲学

传统生产力工具的三大痛点：
1. **清单碎片化**：便签、待办App、日历、笔记各存一份，互相打架
2. **一刀切建议**：忽略你的能量曲线、生活约束、ADHD/远程/育儿等真实处境
3. **静默写入**：AI Agent擅自创建文件、推断偏好，破坏信任

本技能的应对：
- **单一可信系统**：所有方向、任务、承诺只进 `~/focus-flow/`，避免重复
- **先诊断再处方**：识别真正瓶颈（优先级/过载/下一步模糊/估算偏差/边界弱/能量低），给最小有效干预
- **明示授权**：仅在你明确确认后写入文件，不静默推断长期偏好

## 核心能力
### 功能1：瓶颈诊断框架

当你说"我感觉效率低"或"我 overwhelmed"时，Agent会先诊断再处方：

| 瓶颈类型 | 典型症状 | 最小有效干预 |
|---:|---:|---:|
| 优先级模糊 | 事事都重要、选择瘫痪 | 强制排出"本周北极星"，其余降级 |
| 过载 | 待办超过20条、持续焦虑 | 三层裁剪：删除/委托/推迟 |
| 下一步模糊 | 任务写成"做项目X" | 改写为"动词+对象+地点"格式 |
| 估算偏差 | 总是完不成、赶deadline | 加缓冲系数1.5x，记录估算误差 |
| 边界弱 | 被打断多、临时任务多 | 设置保护时段、明确"非紧急不打断" |
| 能量低 | 拖延、提不起劲 | 按能量曲线重排任务，难任务放能量峰值 |

**输入**: 用户提供功能1：瓶颈诊断框架所需的指令和必要参数.
**处理**: 解析功能1：瓶颈诊断框架的输入参数,完成核心逻辑,返回结构化响应.
**输出**: 返回功能1：瓶颈诊断框架的响应数据,包含状态码、结果和日志.

### 功能3：每日三件事法则

每天早晨（或前一晚）只承诺三件事，强制排序：

```markdown
## 2026-07-18 每日三件事
1. **必须完成**：[最难/最重要的一件]
2. **应当完成**：[次要的一件]
3. **可以完成**：[有空再做的一件]
# ...
## 今日保护时段
- 09:00-11:00 深度工作（关闭通知）
- 14:00-15:00 协作时段
# ...
## 今日已捕获（待下次分类）
- 想到要做X
- 别人提到Y
```

**为什么是三件？**多于三件会引发"全做一点，全没做完"的焦虑循环。其余进inbox等待分类.

### 功能5：明示授权机制（信任底线）

Agent写入 `~/focus-flow/profile.md` 前**必须**询问：
- "你刚才提到[偏好X]，是否要记录到个人档案？"

**禁止**：
- 从你的沉默推断长期偏好
- 从单次对话推断工作风格
- 在未询问的情况下修改 profile.md

**例外**：你明确说"记住X"或"以后都按X来"时，可直接写入.
## 使用场景

### 场景一：长期目标拆解（个人开发者）

**用户输入**："我想在6个月内上线一个SaaS产品，但感觉无从下手。"

**Agent执行**：
1. 诊断瓶颈：不是优先级问题，是"下一步模糊"
2. 拆解：6个月SaaS → 季度里程碑 → 月度项目 → 周度任务
3. 写入 `direction/goals-active.md`："Q3完成MVP核心功能"
4. 写入 `execution/next-actions.md`："本周完成用户调研访谈3场"
5. 在 `focus/time-blocks.md` 中为用户调研预留高能量时段
6. 询问："是否记录偏好'每周二上午做用户访谈'？"

### 场景二：能量低谷期任务调度（远程工作者）

**用户输入**："最近下午总是没精神，啥也干不动。"

**Agent执行**：
1. 诊断瓶颈：能量低，非优先级问题
2. 检查 `profile.md` 的能量曲线，确认下午是否为低能量时段
3. 重排任务：难任务移到上午，下午只安排行政/会议/回复类
4. 写入 `focus/time-blocks.md`："14-16点：邮件、行政、轻量review"
5. 在 `review/weekly.md` 记录："本周下午能量持续偏低，调整后任务完成率从40%提升至75%"
6. 询问："是否将'下午不安排深度工作'作为长期偏好？"

### 场景三：过载急救（团队负责人）

**用户输入**："我 overwhelmed，待办有30条，不知道从哪开始。"

**Agent执行**：
1. 诊断瓶颈：过载
2. 三层裁剪：
   - 删除：8条已过期或无意义的
   - 委托：6条可交给团队成员的
   - 推迟：10条移至someday
3. 剩余16条，强制排出本周北极星
4. 写入 `direction/dashboard.md`："本周北极星：完成季度汇报"
5. 其余15条按优先级排入 `execution/this-week.md`
6. 在 `focus/sessions.md` 记录："过载急救完成，从30条裁剪至16条，聚焦1件"

## 常见陷阱（必读）

| 陷阱 | 表现 | 对策 |
|:------|------:|:------|
| 动机式安慰 | 你说"我效率低"，Agent回"加油你可以的" | 要求Agent诊断结构瓶颈，而非给鸡汤 |
| 平均优先级 | 所有任务都标"高优先级" | 强制排出"本周北极星"，其余降级 |
| 系统/任务混放 | 目标、项目、任务堆在一个清单 | 严格分层：goals-active / next-actions / this-week |
| 完美主义系统 | 花一周搭建系统却不执行 | 最小可用：profile.md + capture.md 即可启动 |
| 陈旧承诺不忍删 | "也许以后会做" | 每周复盘强制清理：删除/委托/推迟 |
| 忽略真实约束 | 给远程带娃的人排"早6晚11" | 必须按 profile.md 的真实约束规划 |

## FAQ

### Q1：这个系统和Notion/Todoist/滴答清单有什么区别？
A: 它们是工具，本技能是方法论。本技能的"单一可信系统"原则要求你选定一个本地目录作为唯一真相源，避免工具切换导致的清单碎片化。你可以继续用Notion记录，但所有"待办/目标/承诺"必须以 `~/focus-flow/` 为准.
### Q2：Agent会自动创建文件吗？
A: 不会静默创建。所有文件写入都需要你的明确确认。例外是初始化时的快速开始命令（你主动执行的bash）和你说"记住X"时的偏好记录.
### Q3：我没有Linux/Mac，能用吗？
A: 可以。路径 `~/focus-flow/` 在Windows下对应 `C:\Users\<你的用户名>\focus-flow\`。所有bash命令在PowerShell/Git Bash中均可执行.
### Q4：能量曲线必须严格遵循吗？
A: 不必。能量曲线是参考，不是教条。重点是"难任务匹配高能量时段"这个原则。如果你某天状态特别好，可以临时调整.
### Q5：每周复盘必须做吗？不做会怎样？
A: 不强制，但强烈建议。复盘是"系统重新赢得信任"的关键。如果不复盘，系统会逐渐积累陈旧任务，最终被你放弃——这是90%生产力系统失效的原因.
