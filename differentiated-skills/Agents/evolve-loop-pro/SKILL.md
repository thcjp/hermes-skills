## 性能优化策略

### 多项目并行优化

1. **项目索引缓存**：维护项目ID到记忆文件的映射缓存，O(1)查找
2. **懒加载**：仅加载当前活跃项目的HOT，其他项目按需加载
3. **预取策略**：根据用户行为模式预取可能切换的项目记忆
4. **分区存储**：按项目分区存储，避免全量扫描

### 云同步优化

1. **增量同步**：仅传输变更部分，非全量同步，节省90%带宽
2. **压缩传输**：同步数据启用gzip压缩，减少70%传输量
3. **批量推送**：小变更聚合后批量推送，减少网络往返
4. **冲突解决**：自动合并策略，减少90%手动干预

### 度量采集优化

1. **异步采集**：度量数据异步写入，不阻塞主流程
2. **采样策略**：高频指标采样而非全量记录，降低开销
3. **聚合计算**：原始数据按时间窗口聚合，减少存储
4. **冷热分离**：近期度量热存储，历史度量冷归档

### 成本控制

- 低重要性记忆使用本地存储，不同步至云端
- 度量采集设置采样率，避免全量记录
- 云同步限制频率（默认5分钟），避免过度同步
- 快照按需创建，自动清理过期快照

## FAQ

### Q1：专业版与免费版的核心区别是什么？

专业版在免费版的PDCA闭环+三层记忆基础上，新增三大高级功能：六西格玛DMAIC度量体系（量化改进效果）、多项目并行记忆隔离（50+项目零污染）、跨设备云同步（多端一致性）。此外提供决策回溯时间机、自动瓶颈诊断、7种角色场景指南、性能优化策略。专业版使用GPT-4o模型路由，免费版使用GPT-4o-mini。

### Q2：六西格玛DMAIC度量如何工作？

DMAIC是六西格玛的改进方法论：Define定义改进范围、Measure测量基线、Analyze分析根因、Improve实施改进、Control控制成果。专业版将Agent的每次纠正、反思、升级都转化为度量数据，自动计算纠正率、重复犯错率、改进速度等指标，生成可量化的改进报告。

### Q3：多项目并行如何保证记忆不污染？

专业版采用严格隔离策略：每个项目有独立的memory.md、corrections.md、decisions.md，项目间默认不共享。全局偏好通过顶层memory.md继承到所有项目，项目级偏好覆盖全局偏好。跨项目查询需显式开启（opt-in），避免意外泄露。

### Q4：跨设备云同步如何处理冲突？

专业版支持三种冲突解决策略：
- **自动合并**（默认）：将冲突的记忆合并，保留两边信息
- **最新优先**：以时间戳最新的版本为准
- **手动解决**：检测到冲突时提示用户选择

可通过`.cloud-config.json`中的`conflictResolution`字段配置。建议团队协作场景使用自动合并。

### Q5：决策回溯快照会占用多少存储？

快照采用增量存储，仅记录变更部分。单个快照通常<10KB，90天保留期约占用<1MB。可配置`retention`调整保留天数，`interval`调整快照频率。建议重大决策前手动创建快照（`evolve snapshot create --label "决策描述"`），日常使用自动每日快照即可。

### Q6：如何识别改进瓶颈？

运行`evolve metrics bottleneck --top 3`自动识别改进停滞区域。诊断算法基于三个维度：重复犯错率（>10%为瓶颈）、改进速度（下降>50%为瓶颈）、记忆更新频率（30天未更新为停滞）。每个瓶颈会给出根因分析和改进建议。

### Q7：专业版支持多少个项目并行？

专业版支持最多50个项目并行管理。每个项目独立隔离，支持跨项目查询（需opt-in）。项目数超过50时建议合并相似项目或归档不活跃项目。

### Q8：团队知识传承如何实现？

通过云同步+导出导入组合实现：
1. 导出离职成员的全部记忆：`evolve export --user xxx --format full`
2. 导入到团队共享层：`evolve import --file xxx.zip --scope team-shared`
3. 新成员入职后自动拉取：`evolve cloud pull --scope team-shared`

### Q9：度量数据可以导出吗？

可以。使用`evolve metrics export --format csv/markdown/json`导出度量数据。支持按时间范围、项目、指标类型过滤。导出数据可用于管理层汇报或外部分析工具。

### Q10：专业版与免费版的数据可以互通吗？

可以。专业版完全兼容免费版的目录结构，可无缝切换。从专业版降级到免费版时，云同步、快照、度量功能将停止，但本地记忆数据保留可用。建议降级前导出快照和度量数据作为备份。

### Q11：如何确保记忆数据安全？

专业版提供多层安全保障：
- 本地存储：所有记忆存储在~/evolve-loop/，建议纳入git版本控制
- 云同步加密：TLS 1.3传输 + AES-256静态加密
- API Key安全：通过环境变量配置，不硬编码
- 访问控制：项目级隔离，跨项目查询需opt-in
- 快照备份：90天版本历史，支持回滚
