## 性能优化策略

### 指标采集优化

1. **异步采集**：指标数据异步写入，不阻塞主流程
2. **采样策略**：高频指标采样而非全量记录，降低开销
3. **聚合计算**：原始数据按时间窗口聚合，减少存储
4. **冷热分离**：近期指标热存储，历史指标冷归档

### 瓶颈诊断优化

1. **增量分析**：仅分析新增日志，非全量扫描
2. **缓存结果**：诊断结果缓存，避免重复计算
3. **并行分析**：多瓶颈并行诊断，提升速度
4. **阈值调优**：根据项目特点调整告警阈值

### 技能提取优化

1. **批量提取**：候选技能批量处理，减少交互
2. **模板化**：提取模板预设，降低定制成本
3. **质量门禁自动化**：5项检查自动化，减少人工
4. **版本管理**：提取的技能纳入git版本控制

### 成本控制

- 指标采集设置采样率，避免全量记录
- 瓶颈诊断按需触发，非持续运行
- 技能提取设置候选阈值，避免低价值提取
- 跨项目聚合限制频率（默认周报）

## FAQ

### Q1：专业版与免费版的核心区别是什么？

专业版在免费版的三类日志+状态管理+复发检测基础上，新增五大高级功能：性能指标采集（5项核心指标）、自动瓶颈诊断（根因分析）、优化建议生成（5类模板）、多平台Hook集成（Claude/Codex/Copilot）、技能自动提取（质量门禁）。此外提供复发模式可视化、跨项目知识聚合、7种角色场景指南、性能优化策略。专业版使用GPT-4o模型路由，免费版使用GPT-4o-mini。

### Q2：性能指标采集会拖慢Agent吗？

不会。专业版采用异步采集策略，指标数据在后台异步写入，不阻塞主流程。采集开销<2% CPU。高频指标采用采样策略（默认10%采样率），平衡精度与开销。可通过`.metrics-config.json`调整采样率。

### Q3：瓶颈诊断如何识别根因？

诊断算法基于三个维度：频率（发生次数）、影响（响应时间/错误率贡献）、复发（See Also关联数）。综合三维评分识别Top-N瓶颈，并基于日志上下文生成根因假设。诊断深度可配置（"symptom"仅识别症状，"root-cause"深入根因）。

### Q4：技能自动提取会生成低质量技能吗？

不会。专业版设置了5项质量门禁：已测试、自包含、无硬编码、命名规范、文档完整。任一门禁未通过则提取失败。建议开启`autoExtract: false`（默认），人工确认后再提取。提取后可通过`meta extract verify`再次验证。

### Q5：多平台Hook集成会冲突吗？

专业版支持平台优先级配置，避免冲突。建议同一项目仅启用一个平台的Hook（如团队统一用Claude Code）。若必须多平台并存，配置`priority`字段确定触发顺序。Codex CLI的Hook为实验性功能，建议先用AGENTS.md fallback。

### Q6：跨项目聚合支持多少个项目？

专业版支持无限项目聚合。建议项目数超过20时启用索引（`meta index --rebuild`）加速查询。聚合频率默认周报，可配置为日报或月报。跨项目查询默认限制返回100条，避免结果过大。

### Q7：指标数据可以导出吗？

可以。使用`meta metrics export --format csv/json/markdown`导出指标数据。支持按时间范围、指标类型、项目过滤。导出数据可用于外部分析工具或管理层汇报。

### Q8：优化建议会自动应用吗？

不会。专业版默认`autoApply: false`，优化建议仅生成不执行。用户确认后可手动应用。如需自动化，可配置`autoApply: true`，但建议仅对低风险建议（如缓存优化）启用，高风险建议（如架构调整）保持手动。

### Q9：技能提取后如何管理？

提取的技能存放在`skills/`目录，建议纳入git版本控制。每个技能有独立的SKILL.md，遵循技能命名规范。可通过`meta extract list`查看已提取技能，`meta extract update`更新技能，`meta extract archive`归档不再使用的技能。

### Q10：如何生成给管理层的技术债报告？

```bash
# 生成技术债报告
meta aggregate tech-debt --period 90d --format markdown --audience executive

# 输出包含：
# - 技术债摘要（高/中/低优先级数量）
# - 影响估算（浪费工时/周）
# - 修复成本估算（人周）
# - ROI排行（高ROI优先）
# - 建议修复顺序
```

### Q11：专业版与免费版的数据可以互通吗？

可以。专业版完全兼容免费版的.learnings/目录结构，可无缝切换。从专业版降级到免费版时，性能采集、瓶颈诊断、技能提取功能将停止，但日志数据保留可用。建议降级前导出指标数据和已提取技能作为备份。

### Q12：如何确保日志数据安全？

专业版提供多层安全保障：
- 本地存储：所有日志存储在.learnings/，建议纳入git版本控制
- 敏感信息过滤：检测触发器自动过滤凭证、密码、API Key
- 访问控制：跨项目查询需opt-in
- 导出脱敏：导出报告时自动脱敏敏感路径
