### 3. 偏见深度检测（专业版独有）

自动识别10+种认知偏见：

| 偏见类型 | 检测信号 | 规避建议 |
|----------|----------|----------|
| 沉没成本 | "已经投入这么多了" | 评估未来成本，不考虑已投入 |
| 锚定效应 | 第一个数字影响过大 | 多角度独立估值 |
| 确认偏误 | 只找支持证据 | 主动寻找反证 |
| 可得性偏差 | 近期事件权重过高 | 用历史数据校准 |
| 过度自信 | 置信度与准确度不匹配 | 用Base Rate校准 |
| 框架效应 | 表述方式影响选择 | 用多种表述重新框架 |
| 损失厌恶 | 损失感>收益感 | 用期望值而非感受 |
| 近因偏差 | 最近发生的事权重高 | 拉长时间窗口 |
| 幸存者偏差 | 只看成功案例 | 研究失败案例 |
| 光环效应 | 单一优点覆盖全局 | 分维度独立评估 |

**检测输出示例**：
```text
偏见检测报告：

决策：是否继续投入X项目
检测到偏见：
1. [高] 沉没成本偏误 - "已经投入300万"被作为继续理由
   规避建议：仅评估未来300万投入的ROI，不考虑已投入的300万
2. [中] 确认偏误 - 仅引用支持继续的3个案例，忽略2个失败案例
   规避建议：补充失败案例分析
3. [低] 近因偏差 - 上周的小成功权重过高
   规避建议：用3个月数据评估，而非1周

偏见控制评分：0.65（目标>0.90）
```

**输入**: 用户提供偏见深度检测（专业版独有）所需的指令和必要参数。
**处理**: 按照skill规范执行偏见深度检测（专业版独有）操作,遵循单一意图原则。

### 4. 决策树可视化（专业版独有）

将复杂决策可视化为分支树：

```bash
# 生成决策树
soul tree build --decision "是否进入新市场"

# 输出决策树（Markdown格式）
soul tree render --format markdown --max-depth 5

# 敏感度分析
soul tree sensitivity --variable "市场规模" --range "100万-500万"
```

**决策树示例**：
```text
是否进入新市场？
├── 进入（概率0.6）
│   ├── 大成功（概率0.3）→ 收益+500万
│   ├── 小成功（概率0.5）→ 收益+100万
│   └── 失败（概率0.2）→ 损失-200万
│   期望值：+140万
├── 不进入（概率0.4）
│   ├── 竞争对手进入（概率0.7）→ 相对损失-80万
│   └── 竞争对手不进入（概率0.3）→ 无变化
│   期望值：-56万

敏感度分析：
- 若大成功概率降至0.2，进入期望值=+70万（仍优于不进入）
- 若失败损失增至-400万，进入期望值=+10万（接近临界）
```

## 使用场景

### 场景一：企业级战略决策（CEO角色）

**痛点**：重大战略决策缺乏结构化分析，决策质量无法评估，事后复盘流于形式。

**使用方式**：
```bash
# 决策前生成决策树
soul tree build --decision "是否收购公司X" --max-depth 5

# 决策中检测偏见
soul bias detect --context "收购决策" --depth deep

# 输出：
# 检测到偏见：
# 1. [高] 锚定效应 - 收购价以X公司报价为锚
# 2. [中] 过度自信 - 协同效应预测过于乐观
# 3. [低] 确认偏误 - 仅引用成功收购案例

# 决策后质量评分
soul quality score --decision "收购X" --dimensions all

# 输出：
# 决策质量评分：0.78（目标>0.85）
# - 过程合理性：0.85（框架匹配良好）
# - 信息充分性：0.70（财务尽调不完整）
# - 偏见控制：0.65（检测到2项高偏见）
# - 框架匹配：0.85（决策树+风险评估）
# - 结果可预测性：0.85（待复盘验证）
```

**效果**：战略决策从"拍脑袋"到"有结构+有评分+有偏见检测"。

### 场景二：跨团队决策风格统一（CTO角色）

**痛点**：不同团队的决策风格不一，技术团队偏保守，产品团队偏激进，冲突频发。

**使用方式**：
```bash
# 查看跨域权重配置
soul weight show

# 输出：
# - tech: stability=0.7, speed=0.3
# - product: speed=0.6, quality=0.4
# 冲突点：技术求稳，产品求快

# 设置跨域冲突解决规则
soul weight resolve --rule "when tech.stability > 0.7 AND product.speed > 0.5, use weighted_average with tech=0.6, product=0.4"

# 团队共享决策记忆
soul cloud sync --scope team --direction both
```

**效果**：跨团队决策冲突从"吵架"到"有规则"，决策效率提升约40%。

### 场景三：高风险不可逆决策的偏见规避（创业者角色）

**痛点**：创业过程中充满高风险不可逆决策，认知偏见容易导致致命错误。

**使用方式**：
```bash
# Pre-mortem分析（专业版独有）
soul framework pre-mortem --decision "All-in投入产品X"

# 输出：
# Pre-mortem：假设1年后产品X失败了，可能原因：
# 1. 市场需求被高估（过度自信）
# 2. 竞争对手反应比预期快（可得性偏差）
# 3. 技术难度被低估（锚定效应）
# 建议：为每个失败原因设计早期预警指标

# 偏见深度检测
soul bias detect --depth deep --alert-on high
```

**效果**：高风险决策的盲点提前暴露，减少致命错误。

## 性能优化策略

### 决策树优化

1. **深度限制**：默认max-depth=5，避免树过大
2. **剪枝策略**：低概率分支自动剪枝（<5%）
3. **缓存计算**：期望值计算结果缓存
4. **懒加载**：子树按需展开

### 偏见检测优化

1. **增量检测**：仅检测新增决策上下文
2. **规则缓存**：偏见规则编译后缓存
3. **并行检测**：10+种偏见并行检测
4. **阈值过滤**：低置信度检测结果过滤

### 云同步优化

1. **增量同步**：仅传输变更部分
2. **压缩传输**：gzip压缩减少70%传输量
3. **批量推送**：小变更聚合后批量推送
4. **冲突自动解决**：减少90%手动干预

### 成本控制

- 决策树深度限制，避免过度分析
- 偏见检测按需触发（非持续运行）
- 云同步限制频率（默认5分钟）
- 快照按需创建，自动清理过期快照

## FAQ

### Q1：专业版与免费版的核心区别是什么？

专业版在免费版的混合命名空间+四层记忆+决策框架+复盘+置信度基础上，新增六大高级功能：决策质量评分（5维度量化）、偏见深度检测（10+种偏见）、决策树可视化（分支+概率+敏感度）、多领域权重学习（跨域自动平衡）、跨设备云同步、决策回溯。专业版使用GPT-4o模型路由，免费版使用GPT-4o-mini。

### Q2：决策质量评分如何计算？

5维度独立评分（0-1分），加权平均得总分。维度包括：过程合理性、信息充分性、偏见控制、框架匹配、结果可预测性。每个维度有明确的评分标准。结果可预测性需等决策结果出来后才能评分，其他维度可实时评分。

### Q3：偏见检测会误报吗？

会。专业版采用"高/中/低"三级置信度标注检测结果。建议关注"高"置信度的偏见，"低"置信度的可能为误报。可通过调整检测阈值降低误报率。偏见检测是辅助工具，最终判断仍需用户。

### Q4：决策树可视化支持多深？

默认max-depth=5，可配置至10。深度过大时建议启用剪枝（过滤<5%概率的分支）。决策树支持敏感度分析，可评估关键变量变化对期望值的影响。

### Q5：多领域权重学习如何工作？

系统从历史决策中学习各域的权重配置（如product: quality=0.7, speed=0.3）。当跨域决策冲突时（如product求快vs tech求稳），自动按学习到的权重计算加权平均。冲突解决规则可手动覆盖。

### Q6：跨设备云同步如何处理冲突？

三种策略：自动合并（默认）、最新优先、手动解决。建议团队协作场景使用自动合并。冲突解决后会记录在决策记忆中，保留历史可追溯。

### Q7：决策回溯快照会占用多少存储？

快照采用增量存储，单个快照通常<10KB，180天保留期约占用<2MB。可配置retention调整保留天数。建议重大决策前手动创建快照。

### Q8：专业版支持多少个域？

免费版支持4个固定域（product/tech/business/personal）。专业版支持自定义域，无数量限制。建议域数不超过10个，避免管理复杂度。

### Q9：Pre-mortem分析如何工作？

Pre-mortem假设决策已失败，倒推可能的原因。例如"假设1年后产品X失败了，可能原因是什么？"。这帮助识别决策盲点，为每个可能失败原因设计早期预警指标。Pre-mortem是专业版独有框架。

### Q10：质量评分可以导出吗？

可以。使用`soul quality export --format csv/json/markdown`导出评分数据。支持按时间范围、维度、用户过滤。导出数据可用于管理层汇报或外部分析。

### Q11：专业版与免费版的数据可以互通吗？

可以。专业版完全兼容免费版的目录结构，可无缝切换。从专业版降级到免费版时，质量评分、偏见检测、决策树功能将停止，但决策记忆数据保留可用。建议降级前导出质量评分和快照作为备份。

### Q12：如何确保决策数据安全？

专业版提供多层安全保障：
- 本地存储：所有决策存储在~/soul-decision/，建议纳入git版本控制
- 云同步加密：TLS 1.3传输 + AES-256静态加密
- API Key安全：通过环境变量配置
- 访问控制：项目级隔离，跨项目查询需opt-in
- 快照备份：180天版本历史，支持回滚
