# 效率倍增器（Productivity Boost）

**不是更多工具，而是更少焦虑。** 从最小可行系统起步，基于能量动态调整，用反过载分诊和强制回顾重建信任。所有数据存储在本地 `~/productivity/`，零网络请求，不自动读取日历/邮件，不从观察推断偏好。

## 核心能力

### 1. 能量感知规划
将任务类型（深度创造/协作沟通/机械事务）与能量水平（高/中/低）匹配。通过能量日志记录发现个人高低能时段，避免高能做低价值事、低能硬撑深度工作。高能时段被杂事占用时主动标记冲突并建议调整。

**输入**: 用户提供能量感知规划所需的指令和必要参数。
**处理**: 按照skill规范执行能量感知规划操作,遵循单一意图原则。
**输出**: 返回能量感知规划的执行结果,包含操作状态和输出数据。

- 执行此能力时使用`input_params`参数,支持创建/查询/导出操作
### 2. 最小可行系统起步
从 5 个文件（memory.md、inbox.md、next-actions.md、this-week.md、done.md）开始，只有当当前结构不够用时才扩展。避免一上来建 20 个文件夹导致三天崩塌。

**输入**: 用户提供最小可行系统起步所需的指令和必要参数。
**处理**: 按照skill规范执行最小可行系统起步操作,遵循单一意图原则。
**输出**: 返回最小可行系统起步的执行结果,包含操作状态和输出数据。

- 执行此能力时使用`input_params`参数,支持创建/查询/导出操作
### 3. 反过载分诊
当用户说"我 overwhelmed"时，优先砍范围而非加任务。三步法：紧急砍范围（推迟/删除/委派）→ 重置下一步（选 3 个最重要）→ 重建信任（本周只承诺 3 件事，设每日 must-win）。

**输入**: 用户提供反过载分诊所需的指令和必要参数。
**处理**: 按照skill规范执行反过载分诊操作,遵循单一意图原则。
**输出**: 返回反过载分诊的执行结果,包含操作状态和输出数据。

- 执行此能力时使用`input_params`参数,支持创建/查询/导出操作
### 4. 情境适配指南
针对 ADHD、倦怠、远程工作、管理者、学生、自由职业、创意工作者、创业者、育儿家长、内疚与恢复等 10+ 情境提供差异化策略。通用建议假设"正常"状态，特殊情境需要不同方法。

**输入**: 用户提供情境适配指南所需的指令和必要参数。
**处理**: 按照skill规范执行情境适配指南操作,遵循单一意图原则。
**输出**: 返回情境适配指南的执行结果,包含操作状态和输出数据。

- 执行此能力时使用`input_params`参数,支持创建/查询/导出操作
### 5. 目标-项目-任务严格分层 + 回顾节奏
目标描述结果、项目打包工作、任务是可见动作、习惯是重复行为，四层刻意分离永不混在一个清单。每日 5 分钟、每周 30 分钟、每月 1 小时的强制回顾重建系统信任，回顾比重计划更重要。

**输入**: 用户提供目标-项目-任务严格分层 + 回顾节奏所需的指令和必要参数。
**处理**: 按照skill规范执行目标-项目-任务严格分层 + 回顾节奏操作,遵循单一意图原则。
**输出**: 返回目标-项目-任务严格分层 + 回顾节奏的执行结果,包含操作状态和输出数据。

## 适用场景

| 场景类型 | 输入 | 输出 | 是否适用 |
|:---|:---|:---|:---|
| 任务清单过长焦虑 | "我 overwhelmed 了，30 项任务" | 分诊后的 3 项 must-win + 分类归档 | ✅ 适用 |
| 系统建了不维护 | 完美系统用两天就放弃 | 简化到最小可行 5 文件 | ✅ 适用 |
| 能量与任务错配 | 高能做杂事，低能硬撑深度工作 | 基于能量日志的任务排期 | ✅ 适用 |
| 情境化建议需求 | "我有 ADHD，普通方法没用" | 适配情境的最小系统与策略 | ✅ 适用 |
| 目标转计划 | 模糊目标 | 目标→项目→里程碑→下一步动作 | ✅ 适用 |
| 回顾重建信任 | 周/月回顾需求 | 清理+重连+调整的结构化回顾 | ✅ 适用 |

**不适用场景**：
- 一次性任务管理（如今天买个菜）→ 用便签即可
- 团队级项目管理（多人协作、甘特图）→ 用 Notion/Jira 等专业工具
- 需要自动同步日历/邮件的场景 → 本技能不做后台监控与自动读取
- 强制特定方法论（如纯 GTD）→ 本技能是方法论融合，非单一体系

## 使用流程

### Step 1：创建最小可行系统

```bash
mkdir -p ~/productivity
# 5 个核心文件
touch ~/productivity/memory.md          # 工作风格、约束、能量、偏好
touch ~/productivity/inbox.md           # 快速捕获（未分类）
touch ~/productivity/next-actions.md    # 下一步具体动作
touch ~/productivity/this-week.md       # 本周承诺
touch ~/productivity/done.md            # 已完成（保留有价值记录）
```

### Step 2：诊断真实瓶颈

判断问题是优先级、过载、下一步不清、估算差、边界弱还是低能量，给最小有效干预而非全面改造。

### Step 3：按需扩展（仅当不够用时）

| 触发条件 | 扩展内容 |
|:---|:---|
| 有多个并行项目 | 加 `projects/` 目录 |
| 需要长期目标跟踪 | 加 `goals/` 目录 |
| 习惯跟踪需求 | 加 `habits/` 目录 |
| 深度工作规划 | 加 `focus/` 目录 |
| 每日计划需求 | 加 `planning/` 目录 |

### Step 4：能量感知规划

1. 记录一周能量日志（时段/能量/做了什么/感受）
2. 识别个人能量模式（如上午高能、午后低能）
3. 深度任务 → 高能时段；机械任务 → 低能时段
4. 高能时段被杂事占用 → 标记冲突，建议调整

### Step 5：执行回顾节奏

- **每日 5 分钟**：确认明日 must-win，更新 next-actions
- **每周 30 分钟**：清空 inbox、清理过期任务、重命名模糊项、重连任务与优先级
- **每月 1 小时**：反思目标进展、调整系统结构、归档完成项

**结果处理**: 执行完成后,查看输出结果确认操作状态。成功时输出包含处理摘要和结果数据;失败时根据错误信息排查问题,查阅错误处理章节获取恢复步骤。

代理响应："已分诊。今日只做 3 件事：1. 完成支付模块代码审查 2. 回复客户合同问题 3. 准备明日演示。其余 27 项已推迟/委派/归档。"

2. 识别能量模式：上午高能，午后低能

代理响应："基于你的能量模式安排：上午高能做架构设计（最重要），午后低能做机械事务，下午回升做协作。"

## 错误处理

| 场景 | 原因 | 处理方式 |
|:---|:---|:---|
| 系统建了不维护 | 过度设计，结构太复杂 | 回到最小可行 5 文件，只在不够用时扩展 |
| 任务清单持续膨胀 | 未执行反过载分诊 | 执行分诊三步法：砍范围→重置下一步→重建信任 |
| 能量错配严重 | 未记录能量日志或模式变化 | 重新记录一周能量日志，重新匹配任务时段 |
| 回顾不执行 | 回顾阻力大或时间过长 | 缩短到 5 分钟；设固定提醒；降低完美主义 |
| 情境策略不适配 | 未确认情境标签 | 明确情境（ADHD/倦怠/远程等），加载对应情境指南 |
| 目标长期无进展 | 目标无具体下一步动作 | 执行目标→项目→任务拆解，确保每个目标有可见动作 |
| 保留过期承诺 | 删除不舒服导致系统失信任 | 果断清理，过期承诺移入 someday/ 可找回 |
| 从观察推断偏好写入 memory.md | 违反"只存明确批准的偏好"规则 | 撤回未批准的写入，写入前必须向用户确认 |

## 常见问题

**Q1：系统建了但不维护怎么办？**
A：说明系统太复杂。回到最小可行（5 文件），只在不够用时扩展。回顾比重计划重要——先养成每周 30 分钟回顾习惯，再考虑结构优化。

**Q2：能量感知适合所有人吗？**
A：适合。每个人的能量模式不同，但都有高/低时段。先记录一周能量日志，找出自己的模式，再匹配任务。模式会随季节/作息变化，需定期更新。

**Q3：ADHD/倦怠等情境真的需要不同方法吗？**
A：需要。ADHD 执行功能弱，靠意志力的方法必败，需外化记忆、降低启动摩擦、短冲刺。倦怠时需先减负再优化。通用建议假设"正常"状态，不适合特殊情境。

**Q4：反过载分诊会不会丢重要任务？**
A：不会。分诊是分类（推迟/委派/删除），不是遗忘。删除的明确不重要，推迟的进入 someday/ 可找回。关键是果断清理过期承诺以重建系统信任。

**Q5：能和现有工具（Notion/Todoist）配合吗？**
A：可以。本系统是方法论，文件是载体。可用现有工具实现同样结构，关键是目标-项目-任务分层 + 回顾节奏。本技能不与特定工具绑定。

## 已知限制

1. **不自动读取日历/邮件/行为数据**：本技能不做后台监控行为，不自动读取日历或邮件。所有信息需用户主动输入或捕获到 inbox.md。
2. **不从观察推断偏好**：仅在用户明确要求或明确批准时存工作风格信息到 memory.md。绝不从沉默推断长期偏好。
3. **非团队级项目管理工具**：本技能面向个人效率系统，不提供多人协作、甘特图、资源分配等团队级功能。
4. **情境指南为建议非诊断**：ADHD/倦怠等情境指南是策略建议，不构成医学诊断。严重情况应寻求专业帮助。
5. **依赖用户主动回顾**：系统信任在回顾中重建，若用户不执行周/月回顾，系统会逐渐失信任。本技能无法强制用户回顾，只能提供结构与提醒。
