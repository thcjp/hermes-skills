# 效率倍增器（Productivity Boost）

**不是更多工具，而是更少焦虑。** 直击四大效率顽疾：任务清单过长、完美系统三天崩塌、能量与任务错配、忽视真实情境约束。从最小可行系统起步，基于能量动态调整，用反过载分诊和强制回顾重建信任。

## 痛点与对策速查

| 用户痛点 | 发生场景 | 本系统对策 |
|:---|:---|:---|
| 任务清单焦虑 | 清单越列越长，看一眼就累 | 反过载分诊：优先砍范围而非加任务 |
| 系统三天崩塌 | 建了完美系统却无法维护 | 最小可行系统起步（5 文件够用） |
| 能量错配 | 高能时段做低价值事，低能时硬撑深度工作 | 能量感知规划：匹配任务类型与能量 |
| 忽视情境约束 | 通用建议不适合 ADHD/倦怠/远程 | 10+ 情境指南适配真实约束 |
| 目标任务混淆 | 目标、项目、任务混在一个清单 | 严格三层分离：目标→项目→任务 |
| 回顾缺失 | 只顾执行不复盘，系统失信任 | 强制回顾节奏：日/周/月 |
| 习惯难坚持 | 靠意志力硬撑，三天放弃 | 摩擦管理：减少阻力而非增加动力 |
| 深度工作被打断 | 永远在响应，无专注时间 | 深度工作保护 + 干扰日志 |

## 何时使用

| 触发情境 | 说明 |
|:---|:---|
| 用户要真实的效率系统 | 非一次性激励，需可持续操作系统 |
| 任务清单过长焦虑 | "我 overwhelmed 了" |
| 系统建了不维护 | 完美系统用两天就放弃 |
| 能量与任务错配 | 高能做杂事，低能硬撑深度工作 |
| 需要情境化建议 | ADHD/倦怠/远程/管理者等特定情境 |
| 目标转计划 | 把模糊目标拆解为可执行步骤 |
| 需要回顾 | 周/月回顾重建系统信任 |

## 最小可行系统（差异化核心）

**不要一上来建 20 个文件夹。** 从 5 个文件起步，按需扩展：

## 能量感知规划（差异化核心）

**核心原则：高能做高价值深度工作，低能做机械性事务。** 避免能量错配。

| 能量水平 | 适合任务类型 | 示例 |
|:---|:---|:---|
| 🔋🔋🔋 高 | 深度创造、战略思考、难决策 | 架构设计、写作、关键决策 |
| 🔋🔋 中 | 协作、沟通、中等复杂度 | 会议、代码审查、反馈 |
| 🔋 低 | 机械性、事务性、低风险 | 回邮件、整理文档、例行更新 |

### 能量日志模板

```markdown
## 能量日志

| 时段 | 能量 | 做了什么 | 感受 |
|:---|:---|:---|:---|
| 09:00-11:00 | 🔋🔋🔋 | 架构设计 | 心流，高效 |
| 11:00-12:00 | 🔋🔋 | 代码审查 | 专注 |
| 14:00-15:00 | 🔋 | 回邮件 | 勉强 |
| 15:00-17:00 | 🔋🔋 | 会议 | 还行 |

规律：上午高能，午后低能，下午回升
```

### 能量匹配规则

```text
计划任务时：
1. 评估任务类型（深度/协作/机械）
2. 查能量日志找出高低能时段
3. 深度任务 → 高能时段
4. 机械任务 → 低能时段
5. 若高能时段被杂事占用 → 标记冲突，建议调整
```

## 反过载分诊（差异化核心）

**当用户说"我 overwhelmed"时，优先砍范围，而非加任务。**

### 分诊三步法

```text
第 1 步：紧急砍范围
  - 哪些任务可以今天不做？（推迟）
  - 哪些任务可以不做？（删除）
  - 哪些可以委派？（转交）

第 2 步：重置下一步
  - 从剩余任务中选 3 个最重要的
  - 其余移入 someday/ 或 waiting/

第 3 步：重建信任
  - 确认本周只承诺 3 件事
  - 设置每日 must-win（1 件）
  - 其余作为 bonus
```

### 分诊决策矩阵

| 任务属性 | 决策 |
|:---|:---|
| 重要且紧急 | 做（今日 must-win） |
| 重要不紧急 | 排期（本周） |
| 紧急不重要 | 委派或批量处理 |
| 不重要不紧急 | 删除或 someday/ |

## 情境适配指南（差异化核心）

不同情境需要不同策略。通用建议往往失效。

### 情境速查

| 情境 | 核心挑战 | 关键策略 |
|:---|:---|:---|
| ADHD | 注意力分散、执行启动难 | 外化记忆、降低启动摩擦、短冲刺 |
| 倦怠 | 能量枯竭、意义感缺失 | 先减负再优化、保护恢复时间 |
| 远程工作 | 边界模糊、孤独 | 固定工作区、仪式化开关、主动社交 |
| 管理者 | 会议碎片、上下文切换 | 批量会议、保护深度块、委派 |
| 学生 | 截止日期密集、拖延 | 番茄钟、拆解小步、奖励机制 |
| 自由职业 | 不稳定、自我管理 | 固定例程、客户分级、缓冲日 |
| 创意工作者 | 灵感不规律、交付压力 | 捕获系统、创意时间块、原型思维 |
| 创业者 | 不确定性、多角色 | 周复盘、最小可行决策、精力分配 |
| 育儿家长 | 中断频繁、时间碎片 | 微任务、低期望、利用碎片 |
| 内疚与恢复 | 完美主义、自我批评 | 自我同情、小胜积累、重新开始 |

### ADHD 情境示例

```text
核心挑战：启动困难、注意力漂移

策略：
1. 外化记忆：一切写入 inbox.md，不靠脑子记
2. 降低启动摩擦：任务拆到"5 分钟能开始"
3. 短冲刺：25 分钟番茄钟，不要求长专注
4. 视觉化：用清单而非段落，勾选有快感
5. 外部结构：固定例程，减少决策消耗
6. 接纳漂移：走神后温和拉回，不自责

避免：
- 要求 2 小时深度专注（不现实）
- 复杂多步系统（维护成本太高）
- 依赖意志力（ADHD 执行功能弱）
```

## 目标-项目-任务严格分层

| 层级 | 定义 | 示例 | 反模式 |
|:---|:---|:---|:---|
| 目标 | 期望的结果状态 | "本季度用户增长 20%" | ❌ 模糊愿望 |
| 项目 | 达成目标所需的工作包 | "优化注册流程" | ❌ 单个动作 |
| 任务 | 下一步可见动作 | "分析注册漏斗数据" | ❌ 无动词项目 |
| 习惯 | 支撑系统的重复行为 | "每天晨间计划 10 分钟" | ❌ 一次性目标 |

**规则**：永不留一个目标作为模糊愿望而无具体项目或下一步动作。

## 回顾节奏（差异化核心）

**回顾比重计划更重要。** 系统信任在回顾中重建。

| 频率 | 时长 | 内容 |
|:---|:---|:---|
| 每日 | 5 分钟 | 确认明日 must-win，更新 next-actions |
| 每周 | 30 分钟 | 清理过期任务、重命名模糊项、重连任务与优先级 |
| 每月 | 1 小时 | 反思目标进展、调整系统结构、归档完成项 |

### 每周回顾清单

```markdown
## 每周回顾（30 分钟）

### 清理（10 分钟）
- [ ] 清空 inbox/，分类到正确位置
- [ ] 过期任务：推迟、删除或重新定义
- [ ] 模糊任务重命名为具体动作

### 重连（10 分钟）
- [ ] 任务是否对齐目标？
- [ ] 本周完成的最重要 3 件事？
- [ ] 下周 must-win 是什么？

### 调整（10 分钟）
- [ ] 系统哪里不顺手？
- [ ] 需要加/减什么结构？
- [ ] 能量模式有变化吗？
```

## 核心规则

### 1. 一个系统，非五个竞争系统
一个可信结构优于散乱笔记+随机任务清单+重复计划。

### 2. 从真实瓶颈开始
诊断问题是优先级、过载、下一步不清、估算差、边界弱还是低能量。给最小有效干预，而非全面改造。

### 3. 目标项目任务刻意分离
目标描述结果，项目打包工作，任务是可见动作，习惯是重复行为。

### 4. 适配真实约束
情境指南当用户现实比通用建议更重要时使用。能量、育儿、截止日期、倦怠、ADHD 约束应塑造计划。

### 5. 回顾重于频繁重计划
每周回顾重建系统信任。清理过期、重命名模糊、重连任务与优先级。若用户不断重计划却无进展，简化并回顾。

### 6. 只存明确批准的偏好
仅在用户明确要求或明确批准时存工作风格信息。写入 memory.md 前确认。绝不从沉默推断长期偏好。

## 常见陷阱

| 陷阱 | 为什么失败 | 更好做法 |
|:---|:---|:---|
| 问题是结构性的却给激励话语 | 激励不解决结构问题 | 诊断真实瓶颈 |
| 每个任务同等优先级 | 失去焦点 | 标记 must-win |
| 目标项目任务混在一个清单 | 混乱 | 严格三层分离 |
| 建完美系统但不维护 | 三天崩塌 | 最小可行起步 |
| 推荐忽视情境的例程 | 不适用 | 用情境指南 |
| 保留过期承诺因删除不舒服 | 系统失信任 | 果断清理 |

## 常见问题 FAQ

**Q1：系统建了但不维护怎么办？**
A：说明系统太复杂。回到最小可行（5 文件），只在不够用时扩展。回顾比重计划重要——先养成每周 30 分钟回顾习惯。

**Q2：能量感知适合所有人吗？**
A：适合。每个人的能量模式不同，但都有高 低时段。先记录一周能量日志，找出自己的模式，再匹配任务。

**Q3：ADHD/倦怠等情境真的需要不同方法吗？**
A：需要。ADHD 执行功能弱，靠意志力的方法必败。倦怠时需要先减负。通用建议假设"正常"状态，不适合特殊情境。

**Q4：反过载分诊会不会丢重要任务？**
A：不会。分诊是分类（推迟/委派/删除），不是遗忘。删除的明确不重要，推迟的进入 someday/ 可找回。

**Q5：能和现有工具（Notion/Todoist）配合吗？**
A：可以。本系统是方法论，文件是载体。可用现有工具实现同样结构，关键是目标-项目-任务分层+回顾节奏。

| 能量错配 | 检查能量日志 | 重新匹配任务与能量时段 |
| 回顾不执行 | 检查回顾阻力 | 缩短到 5 分钟；设提醒 |
| 情境不适配 | 确认情境标签 | 加载对应情境指南 |
| 目标无进展 | 检查是否有下一步动作 | 目标→项目→任务拆解 |
