## 核心能力
### 功能1：瓶颈诊断框架（强化版）
| 瓶颈类型 | 典型症状 | 最小有效干预 | 专业版增强 |
|:---:|:---:|:---:|:---:|
| 优先级模糊 | 事事都重要、选择瘫痪 | 强制排出"本周北极星" | 历史优先级决策追溯 |
| 过载 | 待办超过20条、持续焦虑 | 三层裁剪：删除/委托/推迟 | 委托追踪+负载监控 |
| 下一步模糊 | 任务写成"做项目X" | 改写为"动词+对象+地点" | AI辅助任务拆解 |

## 多角色场景指南

### 角色一：独立创业者

**典型痛点**：一个人干五个人的活，优先级混乱，长期过载.
**推荐层级组合**：方向层+执行层+焦点层+复盘层

**专属配置**：
```markdown
## 独立创业者档案
- 角色：CEO+产品+技术+运营
- 约束：无团队、无外部监督、决策孤岛
- 能量曲线：早晨高（创作）、下午低（行政）
- 风险：长期过载导致倦怠
# ...
## 周度强制规则
1. 每周一上午排"本周北极星"（仅1件）
2. 每天三件事，不超过3件
3. 行政任务集中在周五下午（低能量时段）
4. 每月最后一周做"方向校准"（避免陷入执行细节）
```

**典型场景**：
- 用户："我同时在做产品、技术、运营，每天被紧急事项淹没。"
- Agent诊断：过载+优先级模糊
- 干预：
  1. 强制排出本周北极星（仅1件）
  2. 行政任务批量推迟到周五下午
  3. 每天三件事，超出进someday
  4. 启用估算误差统计，校准你的"我能搞定"错觉

### 角色二：远程工作者

**典型痛点**：工作生活边界模糊、孤独感、会议碎片化.
**推荐层级组合**：焦点层+复盘层（强化）

**专属配置**：
```markdown
## 远程工作者档案
- 角色：远程工程师/设计师
- 约束：家庭环境、跨时区协作、无办公室边界
- 能量曲线：因人而异，需个性化建模
- 风险：边界弱、过度工作、会议碎片化
# ...
## 远程专属规则
1. 设立"虚拟通勤"仪式（开始/结束工作）
2. 上午保护时段（关闭IM通知）
3. 会议集中下午（避免碎片化）
4. 每周一次"深度工作日"（无会议）
```

### 角色三：团队负责人

**典型痛点**：管理任务挤压执行时间、委托质量低、上下文切换频繁.
**推荐层级组合**：执行层（委托）+团队层+焦点层

**专属配置**：

## 性能优化策略

### 估算校准优化

1. **缓冲系数自动调整**：根据历史误差率动态调整缓冲系数
2. **任务分类校准**：不同任务类型使用不同缓冲系数（编码1.5x、会议准备2.0x）
3. **个体差异建模**：建立个人估算偏差模型，而非使用通用系数
4. **周期性校准**：每月重新计算缓冲系数，适应能力变化

### 能量曲线优化

1. **动态建模**：能量曲线随睡眠、情绪、季节动态调整
2. **异常检测**：自动识别能量骤降日，提前调整任务安排
3. **匹配度优化**：监控任务-能量匹配度，目标>80%
4. **趋势预测**：基于历史数据预测下周能量曲线

### 专注会话优化

1. **会话时长优化**：根据个人专注极限设置最佳会话时长
2. **干扰源治理**：按频次排序干扰源，逐一治理
3. **保护规则**：高价值时段自动启用保护规则（关闭通知）
4. **模式复用**：识别高产会话模式，主动复制

### 团队委托优化

1. **委托质量监控**：准时率/阻塞率/重做率三维评估
2. **需求对齐机制**：委托前15分钟需求对齐，减少50%阻塞
3. **配对委托**：技能不匹配任务采用老带新模式
4. **检查点优化**：根据任务时长设置合理检查点（非每日催促）

### 成本控制

- 性能分析数据保留90天，超期自动归档
- 团队协作数据按项目分区，避免跨项目混淆
- 多平台集成按需启用，非必要不开启
- 每月清理已完成任务，保持执行层精简

## FAQ

### Q1：免费版与专业版有什么区别？

免费版提供核心五层架构（捕获/方向/执行/焦点/复盘）、瓶颈诊断框架、能量曲线建模、每日三件事法则、周度复盘模板、3类基础场景。专业版解锁团队协作模式、深度性能分析（估算误差统计/能量曲线趋势/专注会话模式挖掘）、七大角色场景指南（独立创业者/远程工作者/团队负责人/ADHD/育儿期/倦怠恢复期/学生）、多平台集成（日历/邮件/协作工具/开发工具）、版本迁移指南（从Notion/Todoist/滴答清单迁移）、扩展FAQ与故障排查表.
### Q2：能量曲线趋势分析如何工作？

专业版记录你每天的能量水平（通过任务完成情况、专注会话时长、自评等指标反推），建立个人能量曲线模型。系统会自动检测异常（如某天能量骤降），并基于历史数据预测下周能量曲线。你还可以查看任务-能量匹配度，识别被错误安排在低能量时段的难任务.
### Q3：估算误差统计如何帮助我？

专业版记录每个任务的估算时长与实际时长，按任务类型（编码/文档/会议准备等）分别统计误差率。系统会自动建议缓冲系数（如"你系统性低估会议准备，建议加2.0x缓冲"），并在生成新计划时自动应用这些系数。这能显著减少"总是完不成"的焦虑.
### Q4：团队委托追踪与普通待办有什么区别？

普通待办只记录"我要做什么"，委托追踪记录"我委托给谁、何时委托、预期完成、检查点、实际完成、阻塞原因"。专业版还提供委托质量分析（准时率/阻塞率/重做率），帮助识别委托质量问题（如需求不清晰是阻塞主因）.
### Q5：ADHD场景与普通场景有什么不同？

ADHD场景采用更细的任务颗粒度（15-30分钟，而非1-2小时）、5分钟"开始就行"启动仪式、外部化记忆（所有任务进capture.md）、25分钟检查点（应对时间盲）、过度专注保护（设闹钟防忘了吃饭）。禁止使用长周期规划（容易引发焦虑）和超过5项的待办（会瘫痪）.
### Q6：倦怠恢复期为什么禁止长期目标规划？

倦怠期的大脑处于能量耗竭状态，长期目标规划会加重"我应该但我做不到"的自我否定。专业版的倦怠恢复配置强制将每日任务上限设为1件、仅允许低认知负荷任务、复盘重点改为"今天做对了什么"（而非未完成）。恢复分4个阶段（1-2周维持/3-4周轻任务/5-8周三件/9周+逐步恢复深度工作）.
### Q7：从Notion迁移会丢失数据吗？

不会。专业版迁移助手会先做映射规则review，确认Notion的每个database对应到focus-flow的哪个文件。迁移后核对数量，确保无丢失。常见映射：Notion的"Goals"→goals-active.md，"Tasks"→next-actions.md，"Projects"→dashboard.md，"Habits"→sessions.md.
### Q8：多平台集成会同步我的所有任务吗？

不会全量同步。你可以配置同步规则：例如仅将time-blocks.md同步至日历（创建事件并标记忙碌），仅将含"紧急"的邮件拉取至capture.md。委托类邮件自动写入delegated.md。专业版支持双向同步与冲突解决（默认以focus-flow为准）.
### Q9：性能分析数据会泄露隐私吗？

所有性能分析数据存储在本地 `~/focus-flow/review/analytics.md`，不上传云端。多平台同步默认仅同步方向层与执行层，不同步性能分析数据。你可以通过config.json完全关闭多平台同步.
### Q10：可以只使用部分专业版功能吗？

可以。专业版功能是模块化设计，可通过config.json的features字段按需启用。例如只启用teamSync不启用analytics，或只启用migrationAssistant用于一次性迁移。专业版许可证覆盖所有功能，但使用哪些由你决定.
### Q11：每周复盘必须做吗？不做会怎样？

不强制，但强烈建议。复盘是"系统重新赢得信任"的关键。如果不复盘，系统会逐渐积累陈旧任务，最终被你放弃——这是90%生产力系统失效的原因。专业版支持精简复盘模板（3问：完成/未完成原因/下周北极星），15分钟内可完成.
### Q12：如何确保子Agent也能获取我的偏好？

在派生子Agent的任务提示中显式包含关键上下文，或让子Agent直接读取profile.md。专业版支持通过性能分析让子Agent按需检索相关偏好，而非全量继承.

- 建议将API Key存储在 `~/.focus-flow/credentials/` 目录（已gitignore）
