## 最佳实践

### 输入安全校验

生成代理前必须校验所有用户输入，防止命令注入：

| 输入项 | 校验规则 | 拒绝条件 |
|:---:|:---:|:---:|
| `geo` | 2个大写ASCII字母（ISO 3166-1） | 非字母、长度不等于2、小写 |
| `minutes` | 正整数，范围1-1440 | 非数字、超出范围 |
| `connection_type` | 枚举值：any/datacenter/residential | 非枚举值 |
| `format` | 枚举值：text/json | 非枚举值 |
| 目标URL | http/https开头，无shell元字符 | 含特殊字符、内网地址 |

### 代理使用规范

```bash
# 用户端请求：使用socks5h://（DNS通过代理解析，保护DNS隐私）
curl --proxy socks5h://username:password@ip:port https://target.com
# ...
# 注意：socks5h中的h表示DNS解析在代理端完成
# 如果使用socks5://则DNS在本地解析
```

### URL安全校验清单

通过代理请求URL前，必须通过以下全部检查：

1. URL必须以 `http://` 或 `https://` 开头
2. 不含shell元字符（`$`、`` ` ``、`(`、`)`、`;`、`&`、`|`、`<`、`>`）
3. 拒绝原始IP地址，只允许域名
4. 拒绝内网主机名（`*.local`、`*.internal`、`*.localhost`等）
5. 域名必须能通过本地DNS解析
6. 解析后的IP必须是公网地址（拒绝 `127.0.0.0/8`、`10.0.0.0/8`、`192.168.0.0/16`等）

### 错误恢复策略

| 错误类型 | HTTP状态码 | 原因 | 恢复策略 |
|:------|------:|:------|:------|
| 认证失败 | 401 | API密钥无效 | 检查密钥格式，确认使用X-API-Key头部 |
| 额度不足 | 402 | 余额耗尽 | 充值或使用x402按需支付 |
| 服务不可用 | 503 | 无可用节点 | 额度自动退还，稍后重试 |
| 连接超时 | - | 代理已过期 | 重新生成新代理 |
| DNS解析失败 | - | 本地DNS问题 | 切换为socks5h://远程DNS解析 |

## 常见问题

### Q1：socks5://和socks5h://有什么区别？

A：`socks5://` 的DNS解析在本地完成，`socks5h://` 的DNS解析在代理端完成。对于用户端代码，推荐使用 `socks5h://` 以保护DNS隐私。Agent端发起请求时使用 `socks5://` 以确保本地DNS校验生效.
### Q2：代理过期后还能使用吗？

A：不能。代理凭证有明确的过期时间（`expiresAt`字段），过期后连接会失败。需要重新生成新代理。免费版不会自动续期，如需自动轮换请使用专业版.
### Q3：住宅节点和数据中心节点有什么区别？

A：住宅节点使用家庭宽带IP，更不容易被目标网站识别和封锁，适合反爬严格的场景。数据中心节点速度更快但更容易被识别。住宅节点消耗更多额度.
### Q4：免费版有哪些限制？

A：免费版不限制使用次数，但不支持多代理并发管理、代理自动轮换、请求重试与熔断、代理性能监控等高级功能。如需这些能力请使用专业版.
### Q5：如何查看剩余额度？

A：向Agent说"查看我的余额"，Agent会调用余额查询接口并返回剩余额度数。新注册账户通常附带100免费额度，约可生成5个60分钟代理.
