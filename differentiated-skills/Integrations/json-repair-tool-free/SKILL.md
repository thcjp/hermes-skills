# JSON修复工具（免费版）

本工具自动修复JSON文件中的常见语法错误，包括尾随逗号、单引号、未引号键名、注释、十六进制数字等，帮助开发者快速恢复JSON的可解析性。

## 概述

JSON标准较为严格，不支持尾随逗号、单引号、注释等宽松语法。然而实际开发中，手写JSON或从其他格式转换时常产生这些"宽松"错误，导致解析失败。本工具围绕"修复"场景，自动识别并修复5类常见错误，将宽松JSON转换为标准JSON。

## 常见问题

### Q1：修复后JSON仍不合法？
A：可能存在本工具不支持的错误类型（如未闭合括号、字符串内换行）。查看修复报告的`valid`字段，若为false，说明剩余错误需人工处理。专业版支持更多错误类型。

### Q2：备份文件在哪？
A：备份文件与原文件同目录，名称为`原文件名.bak`。若使用了`--no-backup`参数，则不创建备份。建议始终保留备份机制。

### Q3：单引号修复误伤了字符串内容？
A：本工具的单引号修复针对键名与字符串边界，不修改字符串内部的单引号。若字符串内部含单引号（如`"It's ok"`），不会被误改。若仍出现误伤，检查JSON结构是否规范。

### Q4：注释移除后格式混乱？
A：注释移除可能留下空行或多余空格。修复后建议配合格式化工具（json-beautifier）重新整理缩进。本工具的修复后验证仅检查语法合法性，不保证格式美观。

### Q5：十六进制数字修复后值变了？
A：十六进制数字（如`0xFF`）会被转为十进制（`255`），值本身不变。若JSON中本意是字符串"0xFF"，应使用引号包裹。修复前检查数字字段是否需要保持字符串形式。
