# 链上数据查询（免费版）

## 概述

本 Skill 帮助 Agent 将自然语言提问路由到合适的链上数据服务，返回实时数据而非仅推荐。免费版聚焦"无钱包、免费层"场景：用户无需连接钱包即可查询代币余额、NFT、协议子图等基础数据。Skill 自动识别用户意图，选择对应服务并执行查询，返回结构化 JSON.

## 使用场景

1. **地址余额查询**：输入钱包地址查询 ETH/USDC/NFT 持仓，无需手动拼接 GraphQL.
2. **协议子图定位**：为某个协议（如 Uniswap V3）找到合适的子图，获取 query hints.
3. **DeFi 借贷数据**：查询 Aave 借贷池的利率、清算、TVL 等基础指标.
4. **链上数据快速验证**：核验某地址是否持有某代币、某笔交易是否上链.

## 最佳实践

1. **免费层优先规划**：每日 3 次免费查询，建议优先用于高价值查询，低价值查询用本地缓存复用.
2. **提问要具体**：包含链名、协议名、地址等具体信息，路由置信度更高。如"以太坊主网 0x.. 的 USDC 余额"优于"查余额".
3. **复用 query_ready**：返回的 `query_ready` 参数可直接用于后续查询，无需重复构造.
4. **利用缓存**：返回结果含 `cache_for_seconds`（默认 86400 秒），同一查询 24 小时内可复用.
5. **免费层无钱包**：免费版从不请求钱包，付费端点未配置时返回 402 即停止，无资金流动.
## 常见问题

### Q1：免费查询次数用完怎么办？

A：免费版每日 3 次。建议：(1) 复用 `cache_for_seconds` 缓存；(2) 次日重试；(3) 升级专业版解锁付费端点与更高配额.
### Q2：查询返回的数据是实时的吗？

A：是。Skill 返回链上实时数据而非仅推荐。部分聚合数据可能有少量延迟，结果中会标注数据源与缓存时间.
### Q3：路由置信度低怎么办？

A：(1) 补充链名、协议名等上下文；(2) 拆分复杂问题为多个单一问题；(3) 涉及多个服务时，Skill 会同时调用并合并结果.
### Q4：免费版能查询哪些链？

A：覆盖 EVM 系（以太坊、Arbitrum、Base 等）、Solana、TON。具体链支持取决于所选服务的覆盖范围.
### Q5：返回的 query_ready 如何使用？

A：`query_ready` 包含 `tool` 与 `args`，可直接传给对应服务执行，无需重新构造查询参数.
### Q6：免费版支持哪些链的查询？

A：覆盖 EVM 系（以太坊、Arbitrum、Base、Polygon 等）、Solana、TON。具体链支持取决于所选服务的覆盖范围，结果中会标注数据来源链.
### Q7：如何提高路由准确率？

A：(1) 提问包含链名、协议名、地址等具体信息；(2) 避免模糊表述如"查一下余额"；(3) 涉及多个服务时 Skill 会自动合并结果.
### Q8：免费版查询会消耗 Gas 吗？

A：不会。免费版仅读取链上数据，不发送交易，不消耗 Gas，无需钱包连接.
## 错误处理

| 错误场景(现象) | 可能原因 | 解决步骤 | 优先级 |
|------:|------:|------:|------:|
| 路由置信度低 | 提问不具体 | 补充链名、协议名、地址 | P2 |
| 返回空数据 | 地址无活动 / 协议不支持 | 确认地址与协议匹配 | P3 |
| 免费配额耗尽 | 当日 3 次已用完 | 次日执行ping命令测试网络连通性,检查防火墙和代理设置连接后重新执行命令或复用缓存 | P2 |
| 查询超时 | 网络波动 / 服务繁忙 | 稍后执行ping命令测试网络连通性,检查防火墙和代理设置连接后重新执行命令，复用缓存 | P3 |
| 子图未找到 | 协议无对应子图 | 尝试相近协议名或全链搜索 | P3 |
