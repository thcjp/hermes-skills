# 设计偏好工具箱（免费版）

## 概述

设计偏好工具箱免费版是一款面向个人创作者的视觉偏好自学习工具。它会观察你在设计过程中的选择、反馈和反应，自动提炼出你的审美倾向，并在后续设计中主动应用这些偏好，让每一次设计输出都更贴近你的个人风格。

本版本采用纯 Markdown 指令驱动，无需安装额外软件，零配置即可开始使用。所有偏好数据存储在本地文件中，完全由你掌控。

| 能力 | 说明 |

| 双媒介支持 | 覆盖 UI 设计与图形设计两大类别 |

| 即时应用 | 后续设计任务自动调用已记录的偏好 |

### 场景二：UI 组件设计中的偏好记忆

在设计多个 UI 组件时，避免每次都重复说明自己的审美要求。

```python
# 模拟偏好读取与应用流程
def load_preferences(profile_path="~/.design-preferences/profile.md"):
    """加载用户视觉偏好"""
    prefs = {
        "aesthetic": "暗色系 / 高对比",
        "typography": "等宽字体优先",
        "color": {"bg": "#1a1a2e", "accent": "#e94560"},
        "avoid": ["淡紫色渐变", "白底浅色方案"]
    }
    return prefs

def apply_to_component(component, prefs):
    """将偏好应用到组件设计"""
    component["style"]["background"] = prefs["color"]["bg"]
    component["style"]["accentColor"] = prefs["color"]["accent"]
    return component
```

## 快速开始

### 第一步：了解偏好分区

偏好文件分为四个核心分区，初始状态均为空，随使用逐步填充：

### By Medium        # 按媒介分类的偏好

## 最佳实践

1. **让偏好自然积累**：不要刻意制造选择，让真实设计决策驱动偏好学习。
2. **定期审查偏好**：每周检查一次偏好文件，删除过时或不准确的记录。
3. **保持分区整洁**：每个偏好条目保持一行，超长内容拆分为多条。
4. **明确"Never"分区**：把自己坚决不想要的元素写入 Never 分区，避免重复犯错。
5. **媒介分类记录**：不同媒介的偏好可能冲突，按媒介分别记录更准确。

```text
最佳实践检查清单:
[ ] 偏好文件每周审查一次
[ ] Never 分区至少有 3 条记录
[ ] 每个媒介都有独立的偏好记录
[ ] 偏好条目保持一行简洁
[ ] 过时偏好及时清理
```

## 常见问题

### Q: 偏好文件为空怎么办？

A: 这是正常状态。空分区表示尚未积累足够偏好。继续正常使用，工具会在 2 次一致选择后自动填充。

### Q: 工具记录了我不想要的偏好怎么办？

A: 直接编辑偏好文件，删除或修改对应条目即可。工具不会覆盖你手动编辑的内容。

### Q: 免费版支持视频和印刷设计吗？

A: 免费版聚焦 UI 和图形设计两类媒介。视频、印刷等多媒介支持请升级至专业版。

### Q: 偏好数据存储在哪里？

A: 默认存储在 `~/.design-preferences/profile.md`，可自定义路径。所有数据保留在本地。

### Q: 可以在多个项目间共享偏好吗？

A: 可以。复制偏好文件到新项目目录即可。专业版支持多项目偏好管理与团队共享。
