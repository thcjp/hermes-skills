- 执行此能力时使用`input_params`参数,支持创建/查询/导出操作

### 参数配置与调用
用`config_options`参数进行配置。

**输入**: 用户提供参数配置与调用所需的指令和必要参数。

### 结果处理与输出

## 使用场景

### 场景一：Web应用架构设计

为个人或小型Web应用设计云架构方案。

```text
用户指令：为一个日活1000人的Web应用设计AWS架构方案。
技术栈：Python + PostgreSQL + Redis
需求：高可用、自动扩展、成本可控

云架构师输出：
【架构方案 - AWS】

架构组件：
├── 接入层
│   ├── Route 53（DNS解析）
│   ├── CloudFront（CDN加速）
│   └── Application Load Balancer（负载均衡）
├── 计算层
│   ├── EC2 Auto Scaling Group（2-4台 t3.medium）
│   └── ECS Fargate（可选容器化方案）
├── 数据层
│   ├── RDS PostgreSQL（db.t3.micro，单可用区）
│   └── ElastiCache Redis（cache.t3.micro）
├── 存储层
│   └── S3 Standard（静态资源存储）
└── 安全层
    ├── IAM Roles（最小权限原则）
    ├── Security Groups（网络隔离）
    └── AWS WAF（基础防护）

【月度成本估算】
- EC2 (2台 t3.medium): ~$60
- RDS PostgreSQL (db.t3.micro): ~$15
- ElastiCache (cache.t3.micro): ~$12
- ALB: ~$18
- CloudFront: ~$5
- Route 53: ~$1
- S3: ~$2
- 数据传输: ~$10
预估总计: ~$123/月

【优选实践建议】
1. 使用Auto Scaling应对流量波动
2. RDS启用自动备份
3. S3启用版本控制
4. 使用IAM Role而非Access Key
5. 配置CloudWatch告警监控成本
```

### 场景二：云服务选型

根据应用需求推荐合适的云服务。

```text
用户指令：我需要选择Azure的存储服务，需求如下：
- 存储大量图片（约1TB）
- 访问频率不高（每月几次）
- 成本敏感

云架构师输出：
【Azure存储服务选型建议】

推荐方案：Blob Storage（Hot → Cool → Archive分层）

选型分析：
| 服务 | 适用场景 | 月度成本(1TB) | 推荐度 |
| --- | --- | --- | --- |
| Blob Hot | 频繁访问 | ~$18 | 不推荐 |
| Blob Cool | 不频繁访问 | ~$7 | 推荐 |
| Blob Archive | 极少访问 | ~$1 | 可选 |

推荐策略：
- 近期需要访问的图片 → Cool层
- 超过6个月不访问 → Archive层
- 设置生命周期策略自动分层

预估月度成本：~$7（Cool层1TB）
```

### 场景三：成本优化建议

对现有架构进行成本优化分析。

```text
用户指令：我的AWS月度账单$500，主要花费在EC2上，请提供优化建议。

云架构师输出：
【成本优化分析】

当前花费分布：
- EC2 (5台 m5.large): $350
- RDS: $80
- 其他: $70

优化建议：
1. EC2实例优化
   - 评估Reserved Instance（可节省30-40%）
   - 考虑Spot Instance用于非关键任务
   - 使用Auto Scaling避免过度配置
   
2. 预估优化后成本
   - RI购买后EC2: ~$220（节省$130）
   - 总计: ~$370/月（节省26%）

3. 监控建议
   - 启用AWS Cost Explorer
   - 设置预算告警（阈值$400）
   - 定期审查资源利用率
```

## 优选实践

1. **需求明确**：提供清晰的用户规模、技术栈与核心需求
2. **成本优先**：小型项目优先考虑成本，使用按需付费
3. **安全基础**：始终配置IAM、安全组、加密等基础安全措施
4. **监控告警**：设置成本与性能监控告警，及时发现问题
5. **渐进扩展**：先小规模部署，根据需求逐步扩展
6. **区域选择**：选择离用户最近的区域，降低延迟与传输成本

## 常见问题

### Q1：免费版支持多云架构设计吗？

免费版仅支持单一云平台的架构设计。如需多云架构策略，建议升级至PRO版本。

### Q2：成本估算准确吗？

成本估算基于云厂商公开定价，实际成本可能因使用模式、区域、折扣等因素有所不同。建议作为参考而非精确预算。

### Q3：可以设计Serverless架构吗？

可以。免费版支持Serverless架构设计，包括AWS Lambda、Azure Functions、GCP Cloud Functions等。

### Q4：支持私有云或混合云吗？

免费版主要聚焦公有云架构设计。私有云或混合云场景建议升级至PRO版本。

### Q5：架构方案可以直接实施吗？

免费版输出架构设计方案与建议，实际实施需要云工程师根据方案进行配置部署。
