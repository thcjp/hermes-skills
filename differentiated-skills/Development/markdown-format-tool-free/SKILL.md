# Markdown 格式化工具免费版

## 概述

Markdown 格式化工具免费版帮助个人用户将纯文本或已有 Markdown 文件转换为结构清晰、读者友好的格式。工具的核心原则是:只调整格式和修正明显错别字,绝不增删或改写内容。

免费版聚焦单文件处理,提供内容分析、格式优化、排版修正三步流程,适合个人博客、技术笔记等场景。

## 核心能力

### 1. 内容类型自动识别

| 指标 | 判定结果 |
|---|----|
| 包含 `---` YAML frontmatter | Markdown |
| 包含 `#`、`##`、`###` 标题 | Markdown |
| 包含 `**bold**`、列表、代码块 | Markdown |
| 以上均无 | 纯文本 |

**输入**: 用户提供内容类型自动识别所需的指令和必要参数。
**处理**: 解析内容类型自动识别的输入参数,完成核心逻辑,返回结构化响应。
**输出**: 返回内容类型自动识别的响应数据,包含状态码、结果和日志。

### 2. Frontmatter 生成

自动为文档生成 YAML frontmatter,包含以下字段:

| 字段 | 说明 |
|:-----|:-----|
| `title` | 文档标题(交互式选择) |
| `slug` | 从文件名或标题推断 |
| `summary` | 一句话摘要(50-80字) |
| `description` | 详细描述(100-200字) |

**输入**: 用户提供Frontmatter 生成所需的指令和必要参数。
**处理**: 解析Frontmatter 生成的输入参数,完成核心逻辑,返回结构化响应。
**输出**: 返回Frontmatter 生成的响应数据,包含状态码、结果和日志。

## 最佳实践

### 1. 遵守核心原则

格式化的核心原则:

| 该做 | 不该做 |
|---:|:---|
| 保留作者语气和每个词 | 不添加句子或注释 |
| 加粗关键结论 | 不删除或缩短内容 |
| 平行项提取为列表 | 不改写作者原文 |
| 对比数据转为表格 | 不加主观性标题 |
| 修正明显错别字 | 不过度格式化 |

### 2. 标题层级规范

```markdown
# 文档标题 (H1, 仅在 frontmatter 无 title 时)
# ...
## 主要章节 (H2)
# ...
### 子章节 (H3)
# ...
#### 细节 (H4, 谨慎使用)
```

### 3. 摘要写作原则

| 原则 | 说明 |
|:------:|--------|
| 传达核心价值 | 不只是描述主题 |
| 使用具体细节 | 数字、结果、方法 |
| summary 简洁有力 | 50-80 字,自包含 |
| description 可展开 | 100-200 字,含支撑细节 |

**禁止模式:**
- "本文介绍了..."
- 纯主题描述无价值主张
- 用不同词重复标题

## 常见问题

### Q1: 免费版支持批量格式化吗?

不支持。免费版一次处理一个文件。如需批量处理,请使用 PRO 版。

### Q2: 格式化会修改原文内容吗?

不会。工具只调整格式和修正明显错别字,绝不增删或改写内容。

### Q3: 支持哪些文件格式?

支持纯文本(`.txt`)和 Markdown(`.md`)文件。输出统一为 Markdown 格式。

### Q4: 中英文混排间距如何处理?

默认自动在中英文之间添加空格,可通过配置 `spacing: false` 关闭。

### Q5: 生成的标题不满意怎么办?

标题选择是交互式的,可以输入自定义标题替代候选选项。
