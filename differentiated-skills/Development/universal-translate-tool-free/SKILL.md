# 通用翻译工具(免费版)

## 概述

`universal-translate-tool-free` 为个人用户提供日常多语言翻译能力。它支持任意语言互译、自动源语言检测,并保留 Markdown 格式与技术术语,适合个人阅读外文资料、编写多语言项目文档与日常对话翻译。

本工具通过自然语言指令驱动 Agent 完成翻译,无需安装额外脚本或调用外部翻译 API。

## 核心能力

| 能力 | 说明 |
| --- | --- |
| 文本翻译 | 任意语言之间互译 |
| 自动源语言检测 | 自动识别源语言并标注 |
| 格式保留 | Markdown 标题、列表、代码块、表格、链接全保留 |
| 术语保留 | 变量名、函数名、CLI 命令、URL 不翻译 |
| 语气保留 | 正式保持正式,口语保持口语 |
| 单文件翻译 | 翻译单个 Markdown 文件并保存为带语言后缀的版本 |
**技术实现要点**：核心能力基于`input_params`参数与`output_format`配置实现,支持创建/查询/修改/删除等操作模式,通过`config_options`进行运行时配置。

## 使用场景

### 场景 1:日常文本翻译

```
把这段翻译成日语:你好,你最近怎么样?
```

输出:

```text
**中文 → 日语**
こんにちは、最近いかがですか?
```

### 场景 2:自动检测源语言

```
Xin chào, tôi là Ha Le
```

输出:

```text
**检测到:越南语 → 英语**
Hello, I am Ha Le
```

### 场景 3:Markdown 文件翻译

```
把 README.md 翻译成中文
```

工具会:
1. 读取文件
2. 翻译时保留所有 Markdown 格式
3. 保存为 `README.zh.md`
4. 输出:`已翻译 README.md → README.zh.md(中文,245 行)`

### 场景 4:代码注释翻译

翻译代码文件时,只翻译注释与字符串,不翻译代码:

```python
# 翻译前
variable_name = "Hello"  # 此变量保存问候语

# 翻译后(中文)
variable_name = "你好"  # 此变量保存问候语
```

## 最佳实践

1. **明确目标语言**:始终指定目标语言,避免工具猜测。若文本已是英语且未指定目标,工具会询问。
2. **保留格式**:翻译 Markdown 时,标题层级、列表缩进、代码块语言标识全部保留。
3. **术语不翻译**:变量名、函数名、CLI 命令、URL、路径保持原样。
4. **专有名词处理**:人名、品牌、地名保留原文,除非有标准本地化译名。
5. **歧义词选最常见译法**:根据上下文选择最常见翻译,必要时在括号中保留原文。
6. **代码文件只翻译注释与字符串**:不翻译代码本身,避免破坏可运行性。
7. **不添加内容**:仅翻译已有内容,不主动添加解释或补充。

## 常见问题

### Q1: 工具支持多少种语言?

理论上支持所有有书面文字的语言互译,包括主流语言与小语种。常见语言代码见上方表格。

### Q2: 翻译 Markdown 文件时格式会丢失吗?

不会。工具会保留所有 Markdown 格式:标题、加粗、斜体、代码块、表格、列表、链接。代码块内容不翻译。

### Q3: 技术术语会被翻译吗?

不会。变量名、函数名、CLI 命令、URL、文件路径保持原文。例如 `npm install` 不会被翻译。

### Q4: 翻译质量如何保证?

工具会根据上下文选择最合适的译法,保留原文语气(正式/口语)。对于无良好译法的词,保留原文并在括号中标注。

### Q5: 免费版与 Pro 版的区别?

免费版支持单文件翻译与日常文本翻译;Pro 版扩展批量翻译、对话模式、术语库管理、API 集成与企业级本地化流程。
