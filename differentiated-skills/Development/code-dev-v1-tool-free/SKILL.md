# 结构化开发工具免费版

## 概述

结构化开发工具免费版为个人开发者提供版本化的编码工作流管理。工具遵循"请求 → 规划 → 执行 → 验证 → 交付"的五步结构化流程,每个步骤都有明确的检查点,支持回溯与状态追踪。

免费版聚焦个人开发场景,提供偏好记忆、任务拆分、检查点管理等核心能力。

## 核心能力

### 1. 五步结构化工作流

```text
请求 → 规划 → 执行 → 验证 → 交付
  ↑                              |
  └──── 检查点回溯 ←─────────────┘
```

| 步骤 | 输入 | 输出 | 检查点 |
|:-----|:-----|:-----|:-------|
| 请求 | 用户需求 | 明确目标 | 需求确认 |
| 规划 | 开发目标 | 任务清单 | 方案评审 |
| 执行 | 任务清单 | 代码变更 | 步骤验证 |
| 验证 | 代码变更 | 测试结果 | 质量检查 |
| 交付 | 测试结果 | 交付物 | 交付确认 |

**输入**: 用户提供五步结构化工作流所需的指令和必要参数。
**处理**: 按照skill规范执行五步结构化工作流操作,遵循单一意图原则。
**输出**: 返回五步结构化工作流的执行结果,包含操作状态和输出数据。

## 使用场景

### 场景一: 个人项目结构化开发

从需求到交付的完整结构化流程。

```bash
# 1. 提出请求(需求确认检查点)
请帮我实现一个用户注册功能
要求: 邮箱注册、密码加密、邮件验证

# 2. 规划(方案评审检查点)
# 工具拆分任务:
# 步骤1: 创建用户数据模型
# 步骤2: 实现密码哈希
# 步骤3: 实现注册端点
# 步骤4: 实现邮件发送
# 步骤5: 编写测试

# 3. 逐步执行(每步创建检查点)
请执行步骤1: 创建用户数据模型
# → 检查点: 模型验证通过

# 4. 验证(质量检查检查点)
请运行步骤1的单元测试

# 5. 交付(交付确认检查点)
请运行完整测试套件并交付
```

## 常见问题

### Q1: 免费版的检查点最多保存多少个?

免费版每个任务保存全部检查点,建议任务完成后归档到单独目录,避免占用过多空间。

### Q2: 检查点回溯会丢失已有代码吗?

不会。回溯是回到该检查点的规划状态,已有代码保留,用户可基于当前状态重新规划。

### Q3: 偏好记忆会自动学习吗?

不会。仅存储用户明确要求记住的内容,不会自动推断或学习偏好。

### Q4: 可以同时管理多个任务吗?

免费版支持基本的多任务状态追踪,每个任务有独立的检查点目录。建议一次专注一个任务。

### Q5: 工具会访问网络吗?

不会。本工具完全在本地运行,不发起任何网络请求。
