# 代码委派工具免费版

## 概述

代码委派工具免费版为个人开发者提供将编程任务委派给本地代码 CLI 执行的能力。工具采用异步流程设计,主 Agent 在委派任务后立即响应用户,保持对话流畅。委派的任务在后台执行,用户下次交互时查看结果。

核心原则:主 Agent 永远不直接写代码,所有编程任务都通过委派执行。

## 核心能力

### 1. 异步委派流程

```text
步骤1: exec 委派命令      → 获取会话 ID
步骤2: 立即回复用户       → "正在处理,请稍候"
步骤3: 结束当前轮次       → 不再调用工具
步骤4: 用户下次消息时     → 检查委派状态
步骤5: 完成则返回结果     → 未完成则告知"仍在处理"
```

**输入**: 用户提供异步委派流程所需的指令和必要参数。
**处理**: 按照skill规范执行异步委派流程操作,遵循单一意图原则。
**输出**: 返回异步委派流程的执行结果,包含操作状态和输出数据。
- 执行此能力时使用`input_params`参数,支持创建/查询/导出操作

### 2. 会话上下文保持

| 模式 | 用途 | 参数 |
|:-----|:-----|:-----|
| 新会话 | 新任务、测试验证 | 无需额外参数 |
| 继续会话 | 修复迭代、延续上下文 | `--continue` |

**输入**: 用户提供会话上下文保持所需的指令和必要参数。
**处理**: 按照skill规范执行会话上下文保持操作,遵循单一意图原则。
**输出**: 返回会话上下文保持的执行结果,包含操作状态和输出数据。

### 3. 独立测试验证

代码编写完成后,使用全新会话(不带 `--continue`)独立测试:

```text
步骤1: 委派编写代码     → 作者会话(可用 --continue 迭代)
步骤2: 委派测试代码     → 测试会话(全新,不带 --continue)
```

**输入**: 用户提供独立测试验证所需的指令和必要参数。
**处理**: 按照skill规范执行独立测试验证操作,遵循单一意图原则。
**输出**: 返回独立测试验证的执行结果,包含操作状态和输出数据。

## 使用场景

### 场景一: 个人项目功能开发

开发者需要实现一个新功能,委派给代码 CLI 执行。

```bash
# 委派任务
请委派以下任务到项目 my-project:
任务: 在 src/auth/ 目录下实现 JWT 认证模块
要求: 包含签发、验证、刷新三个函数
语言: Python
```

委派命令:

```bash
cd "projects/my-project" && claude -p \
  "在 src/auth/ 目录下实现 JWT 认证模块,包含签发、验证、刷新三个函数,使用 Python" \
  --output-format text \
  --max-turns 10
```

### 场景二: Bug 修复与迭代

用户报告 Bug,委派修复并迭代验证。

```bash
# 委派修复(使用 continue 延续上下文)
请委派修复以下 Bug:
项目: projects/web-app
问题: WebSocket 连接存在内存泄漏
使用 continue 模式延续上次会话上下文
```

### 场景三: 独立代码测试

代码编写完成后,用全新会话独立测试。

```bash
# 委派测试(全新会话)
请委派测试以下项目:
项目: projects/my-project
任务: 运行测试套件,报告通过/失败结果
注意: 使用全新会话,不带 continue
```

## 最佳实践

### 1. 任务描述清晰

好的任务描述应包含:

| 要素 | 说明 | 示例 |
|:-----|:-----|:-----|
| 目标 | 做什么,改哪个文件 | "实现 src/auth/login.py" |
| 上下文 | 函数名、错误信息、期望行为 | "修复 verify_token 函数的过期检查" |
| 约束 | 语言、框架、代码风格 | "使用 Python,遵循 PEP 8" |
| 单一性 | 一次一个任务 | 不要在一个任务中混合多个功能 |

### 2. 安全最佳实践

| 规则 | 说明 |
|:-----|:-----|
| 隔离项目目录 | 永远在专用 projects/ 目录运行 |
| 限制项目范围 | cd 前缀确保在正确目录操作 |
| 审查委派输出 | 始终通过主 Agent 转达结果 |
| 禁用危险参数 | 不使用 --dangerously-skip-permissions |

### 3. 失败处理原则

```text
如果委派失败或超时,主 Agent 不直接写代码。
告诉用户:"编程任务未完成,要再试一次吗?"
用更长超时或更简单的任务描述重试。
```

## 常见问题

### Q1: 委派任务时主 Agent 会阻塞吗?

不会。采用异步流程,委派后主 Agent 立即回复用户并结束轮次,用户下次消息时才检查状态。

### Q2: 可以同时委派多个任务吗?

免费版支持多个任务并行委派,但建议一次专注一个任务,便于管理和验证。

### Q3: --continue 和全新会话有什么区别?

`--continue` 延续上次会话的上下文,适合迭代修复;全新会话独立读取代码,适合测试验证,避免上下文偏见。

### Q4: 委派失败后主 Agent 会自己写代码吗?

不会。失败后会告知用户,询问是否重试。只有用户明确要求"你自己来做"时才会直接写代码(不推荐)。

### Q5: 如何确保委派的安全性?

始终在隔离的项目目录运行,不指向主目录或含敏感信息的仓库,通过主 Agent 转达所有结果。
