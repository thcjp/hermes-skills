# 代码分析工具包免费版

## 概述

代码分析工具包免费版为个人开发者提供 Git 历史自查能力。工具将提交历史聚合为描述性统计(提交频率、文件变更模式、提交规范使用率、Bug 修复率等),并生成自我反思报告.
重要原则:本工具输出的是对 Git 历史的描述性画面,不是对个人能力的评判。代码审查、设计、指导、运维等贡献在 Git 历史中不可见,应将分析结果作为讨论起点,而非结论.

## 常见问题

### Q1: 免费版可以分析其他开发者的提交吗?

不可以。免费版默认仅分析当前本地 Git 用户(自我范围)。分析其他人需要明确同意机制,请使用 PRO 版.
### Q2: 报告会给出评分或等级吗?

不会。报告是描述性的,不包含综合评分、字母等级或评判性结论。这是设计决策,防止被用作个人评分卡.
### Q3: --i-have-consent 参数可以跳过吗?

不能。这是硬性门禁,没有环境变量绕过方式。不提供此参数,工具会打印使用须知并退出.
### Q4: 分析大仓库会很慢吗?

是的。10万+提交的仓库分析可能需要较长时间,建议限制日期范围(`-s` 和 `-u` 参数).
### Q5: Python 复杂度分析支持其他语言吗?

不支持。复杂度分析依赖 radon,仅适用于 `.py` 文件。其他语言文件的复杂度不分析.
### Q6: 报告可以用于团队复盘吗?

免费版仅支持自我分析。团队复盘(多作者)需要 PRO 版的同意管理机制.
