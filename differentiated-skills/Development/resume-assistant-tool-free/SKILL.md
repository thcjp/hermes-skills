# 简历助手工具(免费版)

## 概述

简历助手工具(免费版)为个人求职者提供简历润色、基础评分与Markdown导出能力。当你在请求中提及 简历、求职、润色、评分 等内容时,本工具会自动激活,根据自然语言意图路由到对应工作流,输出经过40+项清单审查的润色版本与100分制评分。

本版本聚焦个人单份简历的润色与评分,适合求职准备、简历自查与写作学习。如需针对具体岗位定制、多格式导出(Word/HTML/LaTeX/PDF)与多模板,请升级至 PRO 版本。

## 核心能力

| 能力 | 说明 |
|---|---|
| 简历润色 | 40+项清单审查,8大类别,输出润色版本 |
| 基础评分 | 100分制评分,5个维度,给出优势与改进点 |
| Markdown导出 | 输出结构化Markdown,版本控制友好 |
| 意图路由 | 自然语言自动识别意图并路由到对应工作流 |
| 中英双语 | 支持中文与英文简历,中英混排规范 |
**技术实现要点**：核心能力基于`input_params`参数与`output_format`配置实现,支持创建/查询/修改/删除等操作模式,通过`config_options`进行运行时配置。

**输出**: 返回核心功能执行的响应数据,包含状态码、结果和日志。
- 执行此能力时使用`input_params`参数,支持创建/查询/导出操作

### 参数配置与调用

**输入**: 用户提供参数配置与调用所需的指令和必要参数。

## 使用场景

### 场景一:快速润色简历

用户粘贴简历,工具运行40+项清单审查并输出润色版本。

```text
用户: 帮我润色简历
[粘贴简历内容]
# ...
工具: 运行40+项清单审查...
# ...
📊 简历评分: 68/100 (等级: C)
主要问题:
1. ❌ 没有量化成果
2. ⚠️ 动词力度不足
3. ⚠️ 缺少目标岗位关键词
# ...
[输出润色后的完整简历]
# ...
变更摘要(按优先级):
🔴 关键: 补充量化成果
🟡 重要: 替换为强动词
🟢 次要: 统一格式
💡 建议: 补充技能标签
```

清单审查覆盖8大类别:

| 类别 | 审查项示例 |
|:-----|:-----|
| 联系方式 | 邮箱专业、电话格式、领英链接 |
| 个人简介 | 简洁有力、突出价值、避免空话 |
| 工作经历 | 量化成果、强动词、STAR结构 |
| 教育背景 | 完整、相关课程、荣誉 |
| 技能 | 关键词匹配、分组、熟练度 |
| 语法 | 拼写、时态、主谓一致 |
| 格式 | 一致性、长度、章节顺序 |
| ATS兼容 | 标准标题、可解析格式 |

### 1. 自然语言交互(推荐)

无需记忆命令,直接描述需求:

```text
"润色我的简历"
"评估我的简历并告诉我哪里需要改进"
"把简历导出为Markdown"
"我的简历有什么问题?"
```

工具会自动识别意图并路由:

| 你说的 | 工具做的 |
|:---:|:---:|
| "润色/修复/改进我的简历" | 运行40+项清单审查 → 输出润色版本 |
| "评估/打分/评价我的简历" | 100分制评分 → 优势与改进计划 |
| "导出为Markdown" | 输出结构化Markdown |
| "我的简历有什么问题?" | 先评分 → 建议下一步 |

### 2. 斜杠命令

```text
/resume polish
请润色我的简历:
[粘贴简历]
```

### 3. 评分维度参考

| 维度 | 分值 | 评估内容 |
|:------|------:|:------|
| 内容质量 | 30 | 成就、动词、相关性、完整性 |
| 结构格式 | 25 | 布局、一致性、长度、章节顺序 |
| 语言语法 | 20 | 语法、拼写、语气、清晰度 |
| ATS优化 | 15 | 关键词、标准标题、格式兼容 |
| 印象冲击 | 10 | 6秒测试、职业故事、专业度 |

等级: A+(95-100) → A(90-94) → B+(85-89) → B(80-84) → C+(75-79) → C(70-74) → D(60-69) → F(<60)

## 最佳实践

### 1. 从评分开始

已有简历时先评分,了解基线再决定改进方向。

### 2. 量化你的成果

```text
# 弱表述
"负责前端开发,优化了性能"
# ...
# 强表述(量化)
"主导React 18升级,首屏加载时间从4.2s降至2.1s,提升50%"
```

### 3. 使用强动词

| 弱动词 | 强动词 |
|:------:|--------|
| 负责 | 主导 / 牵头 |
| 做了 | 实现 / 构建 |
| 帮助 | 赋能 / 加速 |
| 参与了 | 推动 / 落地 |

### 4. 保持简洁

- 简历长度:1页(初级)或2页(高级)
- 每段经历3-5个要点
- 删除10年以前无关经历

### 5. ATS友好

- 使用标准标题(工作经历、教育、技能)
- 避免图片、表格、文本框(部分ATS解析困难)
- 关键词自然融入,不要堆砌

## 常见问题

### Q1:免费版支持哪些导出格式?

免费版仅支持Markdown导出。如需Word/HTML/LaTeX/PDF与多模板,请使用PRO版。

### Q2:免费版能否针对具体岗位定制?

免费版提供基础润色与评分,不针对具体岗位做关键词优化。岗位定制与JD分析请使用PRO版。

### Q3:支持哪些语言?

支持中文与英文简历,中英混排规范。`language` 参数可指定 `en` 或 `zh`。

### Q4:免费版与PRO版差异?

| 维度 | 免费版 | PRO版 |
|----|:--:|---:|
| 命令 | polish, score | polish, customize, export, score |
| 导出格式 | 仅Markdown | Word/HTML/LaTeX/PDF/Markdown |
| 模板 | 无 | 4种专业模板 |
| 岗位定制 | 不支持 | JD分析与关键词优化 |
| 评分深度 | 基础评分 | 详细评分+岗位匹配度 |
| 支持 | 社区支持 | 优先支持 |

### Q5:简历数据会上传到外部吗?

不会。本工具基于Markdown指令驱动Agent,简历内容仅在本地Agent与LLM间处理,不上传任何外部服务。
