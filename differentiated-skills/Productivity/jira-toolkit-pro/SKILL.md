## 使用场景

### 场景 1: 查看事务详情

通过自然语言查看指定事务

```bash
用户: "看一下 PROJ-123"
Agent: jira issue view PROJ-123
输出: 事务标题、状态、描述与评论
```

### 场景 2: 创建新事务

描述需求创建 Jira 事务

```bash
用户: "创建一个Bug:登录页面404"
Agent: jira issue create -tBug -s"登录页面404" -b"用户反馈登录返回404错误"
```

### 场景 3: 企业批量操作

PRO 版本支持批量执行操作,适合企业级规模化管理需求。以下示例展示如何批量处理多个目标:

```bash
# 示例
targets=("target1" "target2" "target3" "target4" "target5")
for target in "${targets[@]}"; do
  echo "Processing: $target"
  # 执行批量操作逻辑
  result=$(process_target "$target")
  echo "  Result: $result"
done

# 批量结果汇总
echo "Batch operation completed: ${#targets[@]} items processed"
```

```python
# Python 批量处理示例
import concurrent.futures

def process_item(item):
    # 处理单个项目
    # 执行处理逻辑
    return {"item": item, "status": "success"}

items = ["item1", "item2", "item3", "item4", "item5"]

# 使用线程池并发处理
with concurrent.futures.ThreadPoolExecutor(max_workers=5) as executor:
    results = list(executor.map(process_item, items))

# 输出结果
for r in results:
    print(f"  {r["item"]}: {r["status"]}")
```

## 常见问题

### Q: CLI 和 MCP server 用哪个?

A: 有 jira 命令用 CLI,有 Atlassian MCP工具用 MCP server,都没有则引导安装。

### Q: 事务 key 格式?

A: 遵循 [A-Z]+-[0-9]+ 格式,如 PROJ-123。

### Q: PRO 版本与 FREE 版本如何切换?

A: PRO 版本完全兼容 FREE 版本的所有功能。卸载 PRO 版本后可自动回退至 FREE 版本,数据与配置保持不变。升级时只需安装 PRO 版本即可,原有配置自动迁移,无需额外操作。

### 已知限制

A: 批量操作受以下因素限制:
- 最大并发数:默认 10,可通过配置调整
- API 速率限制:取决于目标服务的限制
- 系统资源:CPU、内存与网络带宽
建议在非高峰时段执行大规模批量操作。

### Q: 如何获取 PRO 版本的技术支持?

A: PRO 版本用户可通过以下渠道获取优先技术支持:
- 提交工单:优先处理 PRO 版本请求
- 企业微信群:加入 PRO 用户专属支持群
