## 使用场景

### 场景 1: 生成标准周报

根据工作记录生成结构化周报

用户可通过自然语言指令触发此场景，工具将自动执行相应操作并返回结构化结果.
## 输入格式
| 参数名 | 类型 | 必填 | 说明 |
|:-----|:-----|:-----|:-----|
| input | string | 是 | 周报写作器基础版处理的输入数据或指令 |
| options | object | 否 | 附加配置选项,如模式选择、格式偏好等 |
| callback_url | string | 否 | 异步处理完成后的回调通知URL |

```bash
# 输入: 本周 Git 提交 + 任务记录
# 输出: weekly-report-2026-W06.md
## 本周完成
1. 完成用户认证 API 开发 (PROJ-123)
2. 修复登录页面 Bug (PROJ-145)
## 下周计划
1. 性能优化与压测
2. 编写技术文档
## 问题与风险
- 数据库迁移可能延期
```

### 场景 2: 从 Git 生成周报

自动从 Git 提交记录生成周报

```bash
# 采集 Git 记录
git log --since="last week" --author="$(git config user.name)" --oneline
# 整合到周报模板
# 自动分类: feature/bugfix/docs/refactor
```
