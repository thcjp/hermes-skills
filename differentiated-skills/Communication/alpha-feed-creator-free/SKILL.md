# 内容采集器(免费版)

## 概述

内容采集器免费版是一款面向个人内容创作者的轻量化 AI 资讯采集工具。它能够按照你预先配置的白名单账号和关键词,从 X(原 Twitter)平台首页信息流中抓取当日 AI 相关内容,进行热度打分和基础分类排名,最终生成结构化的 Markdown 日报文件,方便你直接写入本地笔记库(如 Obsidian)进行二次创作。

免费版聚焦「单源采集 + 基础排名 + 本地落盘」三个核心环节,不依赖任何付费 API,适合个人开发者、独立博主和学习者零成本上手。如果你需要多源采集、群组推送和企业级批量运营能力,请升级至 Pro 版。

向 Agent 发送指令:


Agent 将按以下顺序执行:

1. 读取配置文件
2. 依次采集:首页信息流 → 白名单账号 → 关键词匹配

5. 按结构生成排名结果
6. 写入日报文件(命名格式 `YYYY-MM-DD_HHMM.md`)

## 最佳实践

1. **白名单精简原则**: 免费版建议白名单控制在 5 个账号以内,避免采集时间过长影响日报生成效率。

2. **关键词聚焦**: 关键词数量建议 3-8 个,过于宽泛的关键词(如「AI」)会产生大量噪声,降低排名质量。

3. **浏览器标签管理**: 采集过程中严格控制并发标签数(默认上限 7),完成后及时关闭临时标签,结束时应为 0 个标签。

4. **登录态维护**: 若某个来源需要登录,Agent 会暂停并通知你,等待最多 3 分钟。建议提前在浏览器中保持登录态。

5. **日报命名规范**: 统一使用 `YYYY-MM-DD_HHMM.md` 格式命名,便于按日期排序和检索历史日报。

6. **Obsidian 目录结构**: 推荐使用三级目录结构:
   - `01-日报/` 存放最终报告
   - `02-运行记录/` 存放调试与验证记录
   - `03-文档/` 存放安装与运维文档

7. **降级处理**: 当 API 不可用时,自动回退到浏览器采集;当浏览器也不可用时,跳过该来源并在日报中标注降级说明。

## 常见问题

### Q1: 采集时报错「需要登录」怎么办?

Agent 会暂停采集并通知你登录。请在浏览器中完成对应平台的登录,Agent 会每间隔一段时间检查登录状态,最多等待 3 分钟。若超时仍未登录,将跳过该来源继续采集其余来源,最终在日报中标注缺失项。

### Q2: 免费版支持哪些平台?

免费版当前仅支持 X(原 Twitter)平台的内容采集,包括首页信息流、白名单账号主页和关键词搜索三条采集通道。如需扩展到更多平台(如知乎、即刻、微博等),请升级到 Pro 版。

### Q3: 日报生成后如何查看?

日报以 Markdown 文件形式写入配置中指定的 Obsidian 路径(默认 `01-日报/` 目录)。你可以在 Obsidian 中直接打开浏览,也可用任意 Markdown 编辑器查看。Agent 完成后会在回复中附上完整文件路径。

### Q4: 浏览器标签占用过多怎么办?

免费版强制限制并发浏览器标签上限为 7 个。采集流程会在每个来源完成后立即关闭对应标签,整个流程结束时标签数应为 0。如果发现残留标签,可手动关闭后重新运行。

### Q5: 排名算法是如何计算的?

免费版采用基础热度算法,主要参考互动数据(转发数、点赞数)进行加权打分。该算法不涉及复杂语义分析,适合快速筛选热门内容。Pro 版提供基于语义相似度和内容质量的多维排名算法。

### Q6: 如何配置 Obsidian Vault 路径?

通过环境变量 `OBSIDIAN_CONFIG_PATH` 指定你的配置文件路径。若未设置,Agent 会使用默认路径并提示你确认。建议使用真实的 Obsidian Vault 路径而非工作区镜像,以便日报直接进入笔记库。

## 依赖说明

本 Skill 以 Markdown 指令驱动 Agent 执行,本身不需要额外 API Key。采集过程中若使用平台官方 API,需由 Agent 环境预配置对应平台的访问凭证;若回退到浏览器方式,则无需 API Key,但需保持对应平台的登录态。

### 可用性分类

- **分类**: MD+EXEC(纯 Markdown 指令,部分功能需要 `exec` 命令行执行能力)
- **说明**: 以自然语言指令驱动 Agent 完成采集、排名与文件写入,浏览器操作作为可选回退路径
- **适用规模**: 单用户、单源、日采集量 50 条以内
- **升级建议**: 如需多源采集、群组推送、批量调度,请升级至 `alpha-feed-creator-pro`
