# 书签智能中心 免费版

## 概述

免费版书签智能中心帮助个人用户从社交平台抓取收藏的书签内容，自动提取链接背后的完整文章，并基于关键词进行基础分析。用户手动触发运行，处理结果保存至本地知识库，便于后续检索与回顾。本版本适合希望整理收藏内容、建立个人知识库的独立用户.
与专业版相比，免费版聚焦手动触发与关键词分析，不包含自动化后台监控、AI 深度分析、多渠道通知与团队协作等高级能力。但两者的书签抓取接口与存储格式完全一致，便于后续平滑升级.
## 核心能力

| 能力 | 说明 |
|---|---|
| 书签抓取 | 从社交平台获取最新收藏的书签列表 |
| 文章全文提取 | 自动访问书签中的链接，提取完整文章内容 |
| 关键词分析 | 基于关键词匹配提取文章核心概念 |
| 本地知识库 | 将分析结果以 JSON 文件存入本地目录 |
| 手动触发 | 用户主动运行，按需处理 |
| 测试模式 | 预览将要处理的内容而不实际执行 |
**技术实现要点**：核心能力基于`input_params`参数与`output_format`配置实现,支持创建/查询/修改/删除等操作模式,通过`config_options`进行运行时配置.

## 快速开始

1. 安装依赖.
```bash
cd skills/bookmark-smart-hub
npm install
```

2. 运行配置向导，填入社交平台凭证.
```bash
npm run setup
```

向导将引导你完成：
- 检查必需工具是否已安装
- 配置社交平台访问凭证（auth_token 与 ct0）


4. 查看处理结果.
```bash
ls ~/knowledge/bookmarks/
# bookmark-001.json  bookmark-002.json  ...
```

## 示例

免费版配置文件位于工具目录下的 `config.json`.
```json
{
  "credentialsFile": ".env",
  "bookmarkCount": 20,
  "storageDir": "~/knowledge/bookmarks",
  "notifyTelegram": false,
  "analysisMode": "keyword",
  "contextProjects": [
    "个人学习笔记",
    "技术博客素材"
  ]
}
```

**配置说明**

| 字段 | 说明 | 默认值 |
|:-----|:-----|:-----|
| `bookmarkCount` | 每次处理的书签数量 | 20 |
| `storageDir` | 分析结果存储目录 | `~/knowledge/bookmarks` |
| `notifyTelegram` | 是否发送通知（免费版不支持） | `false` |
| `analysisMode` | 分析模式（免费版仅 `keyword`） | `keyword` |
| `contextProjects` | 个人关注领域，辅助关键词匹配 | `[]` |

**凭证配置（.env 文件）**

```bash
AUTH_TOKEN=your_auth_token_here
CT0=your_ct0_token_here
```

> 安全提示：凭证等同于账号密码，切勿分享或提交至版本控制。`.env` 文件权限已设为 600（仅所有者可读写）.
## 最佳实践

- **定期运行**：建议每周手动运行一次，避免书签堆积过多导致处理时间过长.
- **合理设置数量**：`bookmarkCount` 建议 20-50 条，过多会触发平台限流.
- **测试先行**：首次使用先运行 `npm test` 预览，确认无误后再正式执行.
- **项目描述具体**：`contextProjects` 填写具体项目描述，关键词匹配更精准.
- **本地存储**：结果存储在本地，建议定期备份 `storageDir` 目录.
- **凭证更新**：社交平台凭证会过期，失效时重新获取并更新 `.env` 文件.
- **内容筛选**：免费版基于关键词分析，结果可能不够精准，建议人工复核后归档.
## 常见问题

### Q1：提示"凭证缺失"或"未授权"？

社交平台访问凭证已过期。重新登录平台，从浏览器开发者工具中获取最新的 `auth_token` 与 `ct0`，更新 `.env` 文件.
**手动验证凭证**

```bash
cd skills/bookmark-smart-hub
source .env
bird whoami --json
```

若返回用户信息则凭证有效，否则需要更新.
### Q2：提示"bird 命令未找到"？

需要安装 bird 命令行工具.
```bash
npm install -g bird
```

### Q3：免费版每次能处理多少书签？

免费版建议每次处理 20 条以内，每小时最多运行 1 次。`bookmarkCount` 可调高但可能触发平台限流.
### Q4：免费版有 AI 分析吗？

免费版仅提供关键词分析（基于关键词匹配提取核心概念）。如需 AI 深度分析（概念关联、行动建议、项目关联），请升级至专业版.
### Q5：提取的文章内容不完整？

部分网站有付费墙或反爬机制，可能无法提取完整内容。免费版会尽量提取可访问的内容，并在结果中标注提取状态.
### Q6：如何备份数据？

直接复制 `storageDir` 目录即可。每个书签的分析结果都是独立的 JSON 文件，便于迁移与备份.
### Q7：免费版支持自动监控吗？

不支持。免费版需手动运行。如需后台自动监控新书签并实时通知，请升级至专业版.
### Q8：数据会上传到云端吗？

不会。所有数据存储在本地，仅在你主动运行时访问社交平台抓取书签和链接内容。凭证不会离开本地机器.
