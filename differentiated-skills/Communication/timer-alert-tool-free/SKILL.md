## 使用场景

### 场景一：烹饪定时

用户说"帮我设一个 12 分钟的定时器，提醒我意大利面好了"。Agent 在后台启动 12 分钟定时器，到时后通知用户。

```bash
bash background:true command:"node {baseDir}/timer.js 12m '意大利面好了！'"
```

完成后 Agent 响应：

```text
⏰ 定时提醒！12 分钟计时已到：意大利面好了！
```

### 场景二：番茄钟工作法

用户说"帮我启动番茄钟工作法"。Agent 先后启动 25 分钟专注定时器与 5 分钟休息定时器，交替提醒。

```bash
# 专注阶段（25 分钟）
bash background:true command:"node {baseDir}/timer.js 25m '番茄钟专注时间结束！该休息了。'"

# 休息阶段（5 分钟，专注结束后启动）
bash background:true command:"node {baseDir}/timer.js 5m '休息结束！回到工作。'"
```

### 场景三：多任务并行计时

用户说"帮我同时设三个定时器：茶 5 分钟、鸡蛋 10 分钟、会议 30 分钟"。Agent 并行启动三个后台定时器。

```bash
bash background:true command:"node {baseDir}/timer.js 5m '茶泡好了'"
bash background:true command:"node {baseDir}/timer.js 10m '鸡蛋煮好了'"
bash background:true command:"node {baseDir}/timer.js 30m '会议马上开始'"
```

## 最佳实践

### 1. 完成通知格式（关键）

定时器完成后收到 `System:` 消息。**响应必须直接以提醒开头**，不能以 `HEARTBEAT_OK` 开头——以 `HEARTBEAT_OK` 开头且少于 300 字符的响应会被自动过滤，用户收不到提醒。

```text
# 错误（会被过滤，用户收不到）
HEARTBEAT_OK
⏰ 定时器完成了！

# 正确（用户能收到）
⏰ 定时提醒！你的 12 分钟计时已到：意大利面好了！
```

### 2. 后台运行

定时器必须使用 `background:true` 在后台运行，否则会阻塞当前会话。后台定时器有独立 sessionId，完成后退出码为 0。

### 3. 多计时器管理

并行运行多个定时器时，用 `process action:list` 查看所有 sessionId。每个定时器独立计时，完成时分别通知。建议为每个定时器设置有区分度的提醒文案。

### 4. 番茄钟循环

标准番茄钟为 25 分钟专注 + 5 分钟休息。每 4 个番茄钟后安排一次长休息（15-30 分钟）。专注定时器结束后立即启动休息定时器，保持节奏。

### 5. 提醒文案要具体

提醒文案应包含足够信息让用户立即理解。避免模糊的"时间到了"——写明是什么任务的定时器完成了。

### 6. 及时清理已完成定时器

已完成的定时器会自动退出（码 0）。手动终止的定时器退出码 130。定期用 `process action:list` 检查，避免残留进程。

### 7. 提醒文案包含上下文

好的提醒文案应包含：计时器用途 + 剩余状态 + 后续动作。例如"Pasta 定时完成！请关火滤水，准备酱汁"比"时间到了"更有帮助。

### 8. 长时间定时器分段

超过 1 小时的定时器建议分段设置中间检查点。例如 3 小时工作拆为 3 个 1 小时定时器，每个检查点评估进度并决定是否调整计划。

## 常见问题

### Q1：定时器完成后用户没收到通知？
A：检查 Agent 响应格式——不能以 `HEARTBEAT_OK` 开头。响应应直接以提醒内容开头，如 `⏰ 定时提醒！...`。`HEARTBEAT_OK` 开头且少于 300 字符的响应会被自动过滤。

### Q2：定时器没有在后台运行？
A：必须使用 `bash background:true` 启动。如果省略 `background:true`，定时器会阻塞当前会话直到完成。

### Q3：如何取消正在运行的定时器？
A：先用 `process action:list` 找到 sessionId，再执行 `process action:kill sessionId:XXX`。被终止的定时器退出码为 130。

### Q4：多个定时器同时完成怎么处理？
A：每个定时器完成后分别触发独立的系统通知。Agent 按通知到达顺序依次响应，每个响应独立处理。

### Q5：定时器精度如何？
A：短定时器（< 1 分钟）每秒记录进度，长定时器每 10 秒记录。精度在 ±1 秒内。macOS 完成时会播放声音提示（需 `afplay` 可用）。

### Q6：如何查看定时器运行日志？
A：`process action:log sessionId:XXX` 查看特定定时器的进度日志，包含每秒/每 10 秒的计时记录。

### 已知限制
A：免费版不支持循环定时（间隔重复提醒）、提醒模板库、多级升级提醒、指定时间点触发与实时监控面板。这些能力在专业版提供。
