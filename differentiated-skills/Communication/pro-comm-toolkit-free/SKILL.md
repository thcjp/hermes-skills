# 职场沟通工具箱（免费版）

## 概述

职场沟通工具箱免费版是一款面向个人用户的专业写作辅助工具。它通过结构化模板和清晰的写作规则，帮助你快速撰写高质量的职场沟通内容——无论是邮件、会议纪要、即时消息，还是向非技术人员解释技术概念.
核心理念：**关键信息先行，可扫描的格式，明确的行动请求。**

每一条专业消息都应回答三个问题：对方需要知道什么？为什么重要？需要采取什么行动（如有）？

## 核心能力

### 1. 邮件写作模板

提供状态更新、请求、升级、介绍等常见邮件类型的结构化模板，确保邮件主题清晰、正文简洁、行动明确.
**输入**: 用户提供邮件写作模板所需的指令和必要参数.
**处理**: 解析邮件写作模板的输入参数,完成核心逻辑,返回结构化响应.
**输出**: 返回邮件写作模板的响应数据,包含状态码、结果和日志.
- 执行此能力时使用`input_params`参数,支持创建/查询/导出操作
### 2. 会议结构化工具

涵盖会议议程模板和会议纪要模板，帮助你在会前明确目标、会中高效记录、会后清晰跟进.
**输入**: 用户提供会议结构化工具所需的指令和必要参数.
**处理**: 解析会议结构化工具的输入参数,完成核心逻辑,返回结构化响应.
**输出**: 返回会议结构化工具的响应数据,包含状态码、结果和日志.
- 执行此能力时使用`input_params`参数,支持创建/查询/导出操作
### 3. 技术概念通俗化

将复杂的技术术语转化为非技术人员能理解的语言，降低跨部门沟通成本.
**输入**: 用户提供技术概念通俗化所需的指令和必要参数.
**处理**: 解析技术概念通俗化的输入参数,完成核心逻辑,返回结构化响应.
**输出**: 返回技术概念通俗化的响应数据,包含状态码、结果和日志.

### 4. 即时消息规范

针对 Slack/Teams 等即时通讯工具，提供消息长度、格式、频道选择的最佳实践.

## 最佳实践

### 四大写作原则

| 原则 | 说明 | 示例 |
|---:|---:|---:|
| 主题行讲清故事 | 主题应包含项目和目的 | "项目X: 周五前需要决策" 优于 "问题" |
| 要点优于段落 | 无人愿意阅读大段文字 | 使用 `-` 列表替代长段落 |
| 具体请求 | 明确时间和动作 | "请周四前评审" 优于 "有空看看" |
| 匹配渠道 | 不同渠道适用不同场景 | 快速沟通用即时消息，正式记录用邮件 |

### 避免清单

- 不要在没有明确目的的情况下发送消息
- 不要使用"只是跟进一下"而不说明跟进什么
- 不要用段落代替要点
- 不要把请求埋在消息末尾
- 不要对非技术人员使用术语
- 不要在即时消息中发送大段文字（使用线程）
- 不要不必要地回复全部
- 不要使用被动语态（用"我们决定"而非"已决定"）

## 常见问题

### Q: 免费版支持哪些邮件类型？

免费版支持以下常用类型：状态更新、请求、升级、介绍、感谢、道歉。如需更多高级类型（如危机公关、跨文化沟通），请考虑升级到 PRO 版本.
### Q: 可以自定义模板吗？

免费版提供预设模板，暂不支持自定义模板创建。PRO 版本支持完整的自定义模板管理.
### Q: 支持哪些语言？

免费版支持中文和英文输出。PRO 版本额外支持日语、韩语等多语言输出.
### Q: 如何选择邮件 vs 即时消息？

| 场景 | 推荐渠道 |
|:---:|:---:|
| 快速确认 | 即时消息 |
| 正式记录 | 邮件 |
| 紧急事项 | 即时消息 + 提及 |
| 详细方案 | 邮件 |
| 团队讨论 | 即时消息频道 |
