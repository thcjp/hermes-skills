# 中英翻译中枢（免费版）

> 一句话输入，秒级返回准确翻译。自动识别语言方向，保留代码与格式，内置术语库保证一致性。

## 核心功能

### 1. 源语言自动识别

无需用户指定方向，自动识别输入是中文还是英文：

```text
输入："Hello world"
识别：英文 → 输出中文："你好，世界"

输入："今天天气不错"
识别：中文 → 输出英文："The weather is nice today."
```

### 2. 普通文本翻译

直接对用户输入的文本进行中英文互译，支持长文本分段：

- 自动按段落分割长文本，保持上下文一致
- 翻译后保持原有段落结构
- 支持中英混合文本的智能处理

### 3. 代码注释翻译

仅翻译代码中的注释部分，完整保留代码逻辑与结构：

```python
# 输入
def calculate_total(items):
    # Calculate the sum of all items
    total = 0
    for item in items:
        total += item.price  # Add item price to total
    return total

# 输出
def calculate_total(items):
    # 计算所有项目的总和
    total = 0
    for item in items:
        total += item.price  # 将项目价格加到总数
    return total
```

支持的语言：Python、JavaScript、TypeScript、Java、Go、Rust、C/C++、Shell等。

### 4. Markdown格式保留

翻译Markdown文档时完整保留格式结构：

| 格式元素 | 处理方式 |
|----------|----------|
| 标题（#/##/###） | 保留层级，翻译标题文字 |
| 列表（-/* / 1.） | 保留列表符号与缩进 |
| 代码块（```） | 不翻译代码内容，仅翻译语言标识后的注释 |
| 链接（[text](url)） | 翻译显示文本，保留URL |
| 表格（\| \|） | 翻译单元格内容，保留表格结构 |
| 引用（>） | 保留引用符号，翻译内容 |
| 行内代码（`code`） | 不翻译，原样保留 |

### 5. 基础术语对照

内置三类常用术语库，保证翻译一致性：

```yaml
# 示例
deployment: 部署
repository: 仓库

authorization: 授权

serialization: 序列化

# 商务术语示例

deliverable: 交付物
milestone: 里程碑
baseline: 基线

# 日常术语示例
deadline: 截止日期
feedback: 反馈
standup: 站会
```

## 使用场景

### 场景一：代码注释本地化（开发者）

**痛点**：阅读英文开源项目时，注释需要逐句翻译，打断阅读节奏；团队中文成员协作时，英文注释理解成本高。

**对策**：用本工具批量翻译代码注释，保留代码逻辑，仅翻译注释部分。

**效果**：代码阅读效率提升约40%，团队新成员理解开源代码的时间缩短约一半。

### 场景二：技术文档翻译（技术写作者）

**痛点**：将英文技术文档翻译为中文，需手动保留Markdown格式，逐段复制粘贴，效率低且格式容易错乱。

**对策**：用本工具翻译Markdown文档，自动保留所有格式元素。

**效果**：文档翻译效率提升约3倍，格式错误率从约30%降至接近0。

### 场景三：商务邮件翻译（职场人士）

**痛点**：与海外同事或客户邮件往来时，需借助翻译工具，但通用翻译不够专业，术语不统一。

**对策**：用本工具翻译邮件，内置商务术语库保证专业性。

**效果**：邮件撰写时间缩短约50%，术语一致性显著提升，专业度提高。

## 翻译原则

1. **准确性优先**：保证翻译内容准确传达原意，不漏译、不增译
2. **符合语境**：根据使用场景选择合适的语气（技术文档正式、日常对话自然）
3. **保留格式**：原文本有格式（Markdown/代码块）时，翻译后保持格式不变
4. **术语一致**：同一术语在整个翻译过程中保持统一，遵循内置术语库
5. **代码安全**：代码内容仅翻译注释，不修改代码逻辑与结构

## 特殊场景处理

- **专业术语不确定时**：主动向用户确认，提供2-3个备选译法
- **包含敏感内容时**：拒绝翻译并说明原因（如涉及违法违规内容）
- **中英混合文本时**：智能识别，仅翻译需要翻译的部分
- **代码内容时**：仅翻译注释，不修改代码逻辑
- **专有名词时**：保留原文，必要时在括号中附上译名

## FAQ

### Q1：免费版支持批量翻译吗？

支持小批量（单次最多5段文本或1个文件）。需要大规模批量翻译请使用专业版。

### Q2：翻译质量如何保证？

基于以下机制保证质量：(1) 内置三类术语库保证一致性；(2) 按场景选择语气；(3) 保留格式与代码结构；(4) 专业术语主动确认。免费版适用于日常使用，专业版提供更高质量的术语定制。

### Q3：支持哪些编程语言的注释翻译？

支持主流编程语言：Python、JavaScript、TypeScript、Java、Go、Rust、C/C++、C#、Shell、Ruby、PHP等。能识别各语言的注释符号（#、//、/* */、-- 等）。

### Q4：翻译结果可以直接使用吗？

可以。翻译结果会直接输出在对话中，无需复制粘贴。Markdown格式完整保留，可直接用于文档。代码注释翻译后可直接替换原代码。

### Q5：术语库可以自定义吗？

免费版使用内置的三类术语库（技术/商务/日常），不支持自定义。需要项目专属术语库请使用专业版。

## 错误处理

| 问题 | 可能原因 | 解决方案 | 优先级 |
|------|----------|----------|--------|
| 翻译方向识别错误 | 中英混合文本导致误判 | 明确指定方向，如"翻译成英文：..." | 中 |
| 代码注释翻译遗漏 | 注释符号非标准 | 检查注释符号；明确指出需翻译的注释 | 中 |
| Markdown格式丢失 | 文本中包含非标准Markdown | 检查Markdown语法；用标准格式重写 | 低 |
| 术语翻译不一致 | 术语库未覆盖 | 记录该术语；专业版可自定义术语库 | 低 |
| 翻译质量不理想 | 文本过于专业或领域特殊 | 提供领域上下文；使用专业版的领域术语库 | 中 |
| 长文本翻译超时 | 文本过长 | 分段翻译；专业版支持长文档批量处理 | 低 |
