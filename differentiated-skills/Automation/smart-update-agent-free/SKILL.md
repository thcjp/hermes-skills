# 智能更新管家（免费版）

> **每日自动检查并更新Agent运行时与已安装技能。版本对比一目了然，告别手动更新与版本滞后。**

保持Agent运行时与技能的最新状态是确保功能正常与安全的基础。本技能设置每日定时任务，自动检查更新、执行升级、并推送更新摘要。

## 核心理念

**更新三原则**：
1. **每日检查**：不在更新发生时才发现，而是主动每日检查
2. **先检查后更新**：dry-run预览变更，确认后再执行
3. **更新后验证**：每次更新后运行健康检查，确保未引入问题

告诉Agent设置自动更新：

```text
为我设置每日自动更新，检查Agent运行时和所有已安装技能的更新
```

Agent会自动生成cron配置并启用。

### 手动设置

```bash
# 添加每日自动更新定时任务
cron add \
  --name "每日自动更新" \
  --cron "0 4 * * *" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --wake now \
  --deliver \
  --message "执行每日自动更新：1.检查Agent运行时更新 2.更新所有已安装技能 3.报告更新结果"
```

### 配置选项

| 选项 | 默认值 | 说明 |
|------|--------|------|
| 执行时间 | 凌晨4:00 | 使用`--cron`修改 |
| 时区 | 系统默认 | 使用`--tz`设置 |
| 通知方式 | 主会话 | 更新摘要发送位置 |
| 会话模式 | isolated | 独立会话执行，不干扰当前工作 |

## 更新日志记录

每次更新自动记录日志，便于追溯：

```bash
#!/bin/bash
set -euo pipefail

LOG_FILE="logs/$(date +%Y-%m-%d)-update.log"
TIMESTAMP=$(date -u +"%Y-%m-%dT%H:%M:%SZ")

log() { echo "[$TIMESTAMP] $1" >> "$LOG_FILE"; }

log "开始每日更新检查..."

# 检查Agent运行时更新
CURRENT=$(agent --version 2>/dev/null || echo "unknown")
npm update -g agent-runtime@latest >> "$LOG_FILE" 2>&1
NEW=$(agent --version 2>/dev/null || echo "unknown")

if [ "$CURRENT" != "$NEW" ]; then
  log "Agent运行时已更新：$CURRENT → $NEW"
  agent doctor >> "$LOG_FILE" 2>&1
else
  log "Agent运行时已是最新：$CURRENT"
fi

# 更新所有技能
log "检查技能更新..."
skill registry update --all >> "$LOG_FILE" 2>&1

log "更新完成"
```

## 故障排查

### 更新未执行

1. 验证cron已启用：检查配置中 `cron.enabled = true`
2. 确认网关持续运行：cron依赖后台网关
3. 检查cron任务是否存在：`cron list`

### 更新失败

如果更新失败，摘要会包含错误信息。常见修复：

- **权限错误**：确保执行用户对技能目录有写权限
- **网络错误**：检查网络连接与代理设置
- **包冲突**：运行 `agent doctor` 诊断依赖问题
- **磁盘空间不足**：检查磁盘空间，清理旧版本缓存

### 暂时禁用自动更新

```bash
# 移除cron任务
cron remove "每日自动更新"
```

或在配置中临时禁用：

```json
{
  "cron": {
    "enabled": false
  }
}
```

## FAQ

### Q1：免费版每天会自动更新几次？

免费版每天执行1次自动更新检查，通过cron配置在指定时间运行。你可以随时手动触发更新，不受次数限制。

### Q2：更新会在什么时间执行？

默认在凌晨4:00执行，可通过`--cron`参数修改为任意时间。建议选择非工作时段，避免更新期间影响使用。

### Q3：更新会中断正在运行的任务吗？

不会。cron任务在isolated独立会话中执行，不影响当前正在进行的对话或任务。更新完成后通过通知告知结果。

### Q4：如何知道某个技能更新了什么？

更新摘要会显示版本变化（如2.0.3→2.0.4）。如需查看详细变更，可访问该技能的changelog或使用`skill registry info <skill-name>`查看更新说明。

### Q5：免费版与专业版有什么区别？

免费版提供每日自动更新检查、Agent运行时与技能批量更新、更新摘要通知、手动命令与基础故障排查。专业版新增多环境更新编排（dev/staging/prod）、更新前回滚备份、金丝雀发布、依赖冲突分析、Breaking Change检测、合规审计报告、更新窗口策略与完整故障排查表。
