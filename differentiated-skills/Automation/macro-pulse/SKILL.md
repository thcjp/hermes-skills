# 宏观脉搏

每日定时巡检免费宏观数据源，整合过去 24 小时发布的数据与政策，配小白向科普解读后推送。

## 执行流程

### 1. 启动浏览器与加载知识库

```text
browser action=start profile=macro-pulse
read path=<skill-dir>/references/indicators.md
```

`references/indicators.md` 是科普知识库，含常见指标的标准解释模板。**必须先读**，避免重复网络搜索。

### 2. 多源巡检（按优先级）

| 类别 | 数据源 | URL | 关注点 |
|---|---|---|---|
| 国际 | Trading Economics 经济日历 | https://tradingeconomics.com/calendar | 红色高重要性事件 |
| 国际 | FRED 数据发布 | https://fred.stlouisfed.org/releases | 美联储数据 |
| 国内 | 国家统计局 | http://www.stats.gov.cn/ | GDP/CPI/PPI/PMI/工业增加值/社零 |
| 国内 | 央行官网 | http://www.pbc.gov.cn/ | LPR/MLF/逆回购/货币政策报告 |
| 国内 | 证监会 | http://www.csrc.gov.cn/ | 监管政策 |
| 资讯 | 财联社 | https://www.cls.cn/ | 实时金融新闻 |
| 资讯 | 华尔街见闻 | https://wallstreetcn.com/ | 市场资讯 |

### 3. 数据源健康表与降级

每个源访问后记录状态，**单源失败不阻断整体**：

| 源 | 状态 | 失败原因 | 降级动作 |
|:-----|:-----|:-----|:-----|
| Trading Economics | ok / fail / partial | 超时/反爬/结构变更 | fail 则用 FRED 补国际数据 |
| 国家统计局 | ok / fail / partial | 同上 | fail 则用财联社新闻补国内数据 |
| ... | ... | ... | ... |

连续 3 天 fail 的源写入 `references/source-health.md` 的黑名单，下次巡检最后访问。

### 4. 时区与时间窗（强制）

- 时间窗：过去 24 小时，**基准时区 GMT+8**
- 国际数据按发布地时区记录，但展示时统一转为 GMT+8
- 跨日数据（如美股收盘指标）标注"前一交易日"
- 避免把"昨日已发"当"今日新增"

### 5. 整合与科普（强制闭环）

输出结构：

```text
【过去24小时宏观数据】📊
巡检时间：YYYY-MM-DD HH:MM (GMT+8)
数据源健康：TE ok / FRED ok / 统计局 fail→已降级
# ...
🔥 今日重点关注（超预期/极值）
- [指标] [发布值] vs [预期值] — [一句话影响]
  💡 [指标名]：[一句话定义]
     - 为什么重要：[简短说明]
     - 怎么看：[正常范围/关键阈值]
# ...
🌍 国际数据
- [指标] [实际值] [预期值] [前值] [影响]
  💡 [三段式科普]
# ...
🇨🇳 国内数据
- [指标] [实际值] [预期值] [前值] [影响]
  💡 [三段式科普]
# ...
📜 政策动态
- [标题] — [简要说明]
# ...
📰 重要资讯
- [标题] — [简要说明]
# ...
⚠️ 数据源告警
- 统计局官网连续 2 天 fail，已用财联社补充
```

### 科普三段式格式（强制）

每个指标必须包含，无例外：

```text
💡 [指标名]：[一句话定义]
   - 为什么重要：[简短说明]
   - 怎么看：[正常范围/关键阈值]
```

### 未知指标处理流程

`references/indicators.md` 找不到时：

1. **多源搜索验证**
   - 搜索1：`[指标名] 是什么 意思`
   - 搜索2：`[指标名] 经济指标 解释`
   - 搜索3：`[指标名] investing.com` 或 `trading economics`
2. **交叉验证**：对比多源，优先权威源（央行/统计局/知名财经媒体），避免单一来源片面
3. **整理三段式科普**
4. **回写知识库**：若是常见指标，追加到 `references/indicators.md`，下次免搜索

### 6. 阈值预警

满足任一条件则标 🔥 置顶：

| 条件 | 判定 |
|---:|---:|
| 超预期 | 实际值与预期值偏差 > 5%（CPI/PPI 等百分比指标）或 > 10%（绝对值指标） |
| 历史极值 | 实际值进入近 12 个月 top/bottom 3 |
| 政策超预期 | 央行意外加息/降息、未在预期内的政策出台 |

## 推送

使用 message 工具推送整理好的报告。支持多通道：

```text
message action=send target=<user> message="<报告内容>"
```

可选通道：IM（默认）、邮件、Webhook（接入飞书/钉钉/企微）。

## Cron 调度

```json
{
  "name": "macro-pulse-daily",
  "schedule": {
    "kind": "cron",
    "expr": "0 22 * * *",
    "tz": "Asia/Shanghai"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "执行宏观数据巡检，整合过去24小时发布的数据与政策，配科普解读后推送"
  },
  "sessionTarget": "isolated",
  "enabled": true
}
```

时区固定 `Asia/Shanghai`（GMT+8），与数据时间窗对齐。

## 手动触发

```text
执行宏观数据巡检，整合过去24小时发布的数据与政策，配科普解读后推送
```

可选参数：
- `--window 48h`：扩大时间窗到 48 小时（周末补跑用）
- `--source-only domestic`：只跑国内源
- `--no-science`：跳过科普（仅资深用户）

## 错误处理

| 序号 | 错误场景 | 原因 | 处理方式 | 优先级 |
|:---:|:---:|:---:|:---:|:---:|
| 1 | 输入参数缺失 | 用户未提供必要参数 | 提示用户提供所需参数后执行ping命令测试网络连通性,检查防火墙和代理设置连接后重新执行命令 | P0 |
| 2 | 执行超时 | 处理时间过长 | 检查输入数据量,分批处理 | P1 |
| 3 | 输出格式错误 | 结果不符合预期格式 | 检查`output_format`参数配置 | P1 |

## FAQ

**Q1: 数据源被反爬怎么办？**
A: 单源失败不阻断。降级到备用源（如 TE 失败用 FRED），并在报告末尾标注降级。连续 3 天 fail 进黑名单，下次最后访问。

**Q2: 国际数据时区怎么处理？**
A: 按发布地时区记录原始时间，展示时统一转 GMT+8 并标注。例如"美国 CPI（北京时间 20:30 发布）"。

**Q3: 科普知识库怎么维护？**
A: 未知指标自动多源搜索后回写。用户也可手动编辑 `references/indicators.md` 追加条目。建议每季度审查一次，剔除过时解释。

**Q4: 推送失败怎么办？**
A: 重试 3 次（间隔 30s）。仍失败则保存报告到 `reports/<date>.md`，下次推送时附带"昨日补发"。

**Q5: 如何只关注特定指标？**
A: 在 `references/watchlist.md` 配置关注指标清单，巡检时优先展示清单内指标，其余折叠。

## 故障排查

| 现象 | 排查路径 |
|:------|------:|
| 浏览器启动失败 | 检查 profile 是否被占用 → 重启 browser → 检查系统资源 |
| 全部数据源 fail | 检查网络 → 检查代理配置 → 降级为"今日无数据"通知 |
| 科普内容不准 | 检查 indicators.md 是否被污染 → 用未知指标流程重新验证 |
| 时区错乱 | 确认系统时区 → 确认 cron tz 为 Asia/Shanghai → 强制 GMT+8 转换 |
| 推送超时 | 检查 IM webhook 有效性 → 缩短报告长度 → 分段推送 |
| 指标值与官方不符 | 交叉验证 2 个源 → 以官方源（统计局/央行）为准 → 标注修订 |

### 运行环境
- **Agent 平台**: 任意支持 SKILL.md 的 AI Agent
- **操作系统**: Windows / macOS / Linux
- **浏览器**: 内置 browser 工具或 Playwright/Puppeteer（用于巡检需 JS 渲染的源）

### 依赖详情
| 依赖项 | 类型 | 是否必需 | 获取方式 |
|---:|:---|---:|---:|
| browser 工具 | 浏览器自动化 | 必需 | Agent 平台内置 |

| message 工具 | 推送通道 | 必需 | Agent 平台内置 |

| LLM API | API | 必需 | 由 Agent 内置 LLM 提供 |

### API Key 配置
- 大部分数据源免费访问无需 Key
- 可选：FRED API Key（提升数据稳定性，https://fred.stlouisfed.org/docs/api/api_key.html）
- 推送 Webhook URL（飞书/钉钉/企微）由用户提供

### 可用性分类
- **分类**: MD+EXEC（Markdown 指令 + 必须通过 exec/browser 执行浏览器巡检与推送）
- **说明**: 基于自然语言指令驱动 Agent 巡检多源、整合科普、推送报告，含降级与自愈
