# 轻量工作流

自动化的5分钟入门。核心信条：**先判断值不值得，再挑最小模板，立刻跑起来。**

## 30秒判断：该不该自动化

打三个分（各1-3分），加起来≥7分就值得：

| 维度 | 1分 | 2分 | 3分 |
|:-----|:----|:----|:----|
| 频次 | 每月 | 每周 | 每天 |
| 单次耗时 | <5分钟 | 5-15分钟 | >15分钟 |
| 步骤一致性 | 每次不同 | 部分相同 | 完全相同 |

- ≥7分：立刻做
- 5-6分：下个迭代
- <5分：手工就行

**额外门槛**：每周做不超过2次、且每次<5分钟的，别自动化，得不偿失。

## 三个最小可执行模板

### 模板1：表单提交 → 自动存表格 + 通知

```text
触发：网站表单提交（Google Forms / Typeform）
步骤1：在Google表格新增一行（映射字段）
步骤2：给我发Slack/邮件"新提交：[姓名]"
错误处理：步骤1失败则邮件告警

工具：Zapier免费档即可
搭建时间：15分钟
```

### 模板2：付款成功 → 自动开票 + 发收据

```text
触发：Stripe收到付款（webhook）
步骤1：在财务工具创建发票（QuickBooks / Xero）
步骤2：给客户发收据邮件
步骤3：Slack通知我"收款 $X"
错误处理：步骤1失败则紧急告警

工具：Make入门档
搭建时间：30分钟
```

### 模板3：定时 → 拉数据 → 发消息

```text
触发：每周一9:00（cron）
步骤1：从API拉数据（GET请求）
步骤2：计算关键指标
步骤3：发邮件周报给干系人
错误处理：API失败则发"周报延迟"通知

工具：n8n自部署 / Make
搭建时间：20分钟
```

### 触发器

| 类型 | 用什么时候 |
|:-----|:-----------|
| 定时（cron） | 每天/每周固定跑 |

|:-----|:-----|

### 常见坑

| 坑 | 怎么避 |
|:---|:-------|
| 重复触发 | 用唯一ID去重 |
| 字段错位 | 上线前用真实数据测一遍 |

| 时区错 | cron写明时区 |
| 静默失败 | 必加失败告警到Slack |

## 5分钟搭建流程

1. **选模板**（1分钟）：从上面3个挑最像你需求的
2. **选工具**（1分钟）：简单2步用Zapier，复杂用Make，高频用n8n
3. **配触发器**（1分钟）：连账号、测触发
4. **配动作**（1分钟）：映射字段、测动作
5. **加错误告警**（1分钟）：失败时通知自己

**上线前必做**：用真实数据跑一次全流程。

## FAQ

**Q：我从来没做过自动化，从哪开始？**
A：用模板1（表单→表格+通知），Zapier免费档，15分钟搞定。这是最经典的入门。

**Q：自动化失败了怎么办？**
A：每个工作流必须配"失败告警"。最简单：失败时给自己发邮件/Slack。

**Q：要不要先学编程？**
A：不用。Zapier/Make/n8n都是无代码拖拽。编程只在极复杂场景才需要。

**Q：工具选错了能换吗？**
A：能但痛苦。先用决策树选对：2-3步用Zapier，复杂用Make，高频自部署用n8n。

## 故障排查

| 症状 | 解决 |
|:-----|:-----|
| 不触发 | 检查触发条件、OAuth状态、webhook签名 |
| 动作失败 | 检查API限流、权限、字段映射 |
| 数据错位 | 重新映射字段，跑真实数据测试 |
| 重复执行 | 按唯一ID去重 |
