## 快速开始

### 基础搭建（<60秒）

配置邮件与文档处理环境：

```bash
# 安装完整依赖
pip install python-docx openpyxl pdfdocx2pdf pandas python-pptx
pip install secure-smtplib icalendar matplotlik
```

```text
用户："批量发送薪资条邮件，收件人列表在salary.xlsx里，邮件模板用salary_template.txt，附件是各自的薪资条PDF"
```

### 标准搭建（<120秒）

配置邮件服务器与工作流：

```json
{
  "skills": {
    "entries": {
      "office-automation": {
        "config": {
          "email": {
            "smtp_server": "smtp.company.com",
            "smtp_port": 465,
            "username": "hr@company.com",
            "password": "${EMAIL_PASSWORD}"
          },
          "schedule": {
            "calendar_provider": "outlook",
            "conflict_detection": true
          },
          "report": {
            "chart_engine": "matplotlib",
            "output_format": ["pdf", "xlsx"]
          }
        }
      }
    }
  }
}
```

### 完整搭建（<300秒）

配置工作流引擎与云存储：

```text
用户："创建一个工作流：每月1号自动从CRM导出销售数据，生成PDF报表，通过邮件发送给管理层，并同步到百度网盘"

Agent执行：
1. 配置定时触发器（每月1号9:00）
2. 定义工作流步骤：
   Step 1: 连接CRM导出数据
   Step 2: 生成Excel报表
   Step 3: 创建可视化图表
   Step 4: 合并为PDF报表
   Step 5: 邮件发送给管理层
   Step 6: 同步至百度网盘备份
3. 测试执行并验证
```

## 使用场景

### 场景一：HR入职流程自动化（HR经理）

**痛点**：新员工入职涉及合同生成、欢迎邮件发送、账号创建申请、培训日程安排，跨4个系统手动操作耗时半天。

**解决方案**：

```text
用户："为新员工张三执行入职流程：生成劳动合同、发送欢迎邮件、安排培训日程、创建入职 checklist"

Agent执行工作流：
1. 读取张三的入职信息（Excel）
2. 生成劳动合同（Word→PDF）
3. 发送欢迎邮件（含入职指南附件）
4. 安排3天培训日程（检查讲师空闲时间）
5. 生成入职checklist（账号/工位/设备）
6. 同步至飞书和Outlook日历
7. 生成入职流程执行报告
```

**效果**：单员工入职流程从4小时缩短至15分钟，流程标准化达100%。

### 场景二：财务月报自动生成与分发（财务主管）

**痛点**：每月需从ERP导出数据、清洗整理、制作图表、生成PDF报表、邮件发送10位管理层，全流程耗时2天。

**解决方案**：

```text
用户："创建月度财务报表工作流：每月1号从ERP导出数据→生成三张报表（资产负债表/利润表/现金流量表）→制作趋势图表→合并PDF→邮件发送管理层"

Agent执行：
1. 配置定时触发（每月1号8:00）
2. 连接ERP导出原始数据
3. 数据清洗与科目分类
4. 生成三张财务报表（Excel）
5. 制作同比环比趋势图
6. 合并为月度财务报告PDF
7. 邮件发送给10位管理层（含附件）
8. 同步至OneDrive备份
9. 生成执行报告
```

**效果**：月报流程从2天缩短至1小时，报表准确率达100%，管理层早9点前收到。

## 性能优化策略

### 批量处理优化

1. **分批执行**：大批量任务分批处理（每批50-100个），避免内存溢出
2. **并行处理**：独立任务并行执行，提升吞吐量（专业版支持）
3. **增量处理**：仅处理变更文件，避免全量重算
4. **检查点机制**：长流程设置检查点，失败可从断点恢复

### 邮件发送优化

1. **SMTP连接池**：复用SMTP连接，避免反复握手
2. **批量发送**：单次连接发送多封邮件
3. **失败重试**：网络异常自动重试3次，间隔递增
4. **发送限速**：控制发送频率，避免被标记为垃圾邮件

### 报表生成优化

1. **数据缓存**：重复查询的数据缓存复用
2. **图表预生成**：常用图表提前生成，按需引用
3. **模板复用**：报表模板缓存，减少重复解析
4. **异步生成**：大报表异步生成，不阻塞主流程

### 工作流优化

1. **DAG优化**：分析任务依赖，最大化并行执行
2. **资源预估**：预估每步资源消耗，合理调度
3. **超时控制**：设置合理超时，避免长时间阻塞
4. **状态追踪**：实时追踪工作流状态，异常告警

## FAQ

### Q1：免费版与专业版有什么区别？

免费版提供文档处理、数据清洗、格式转换三大核心模块。专业版新增邮件批量发送、日程智能管理、报表自动生成、工作流编排、PPT批量制作、云存储集成六大高级模块。此外提供6种角色场景指南、性能优化策略和多平台集成示例。

### Q2：邮件发送会被标记为垃圾邮件吗？

专业版内置防垃圾邮件策略：(1) 控制发送频率（建议每分钟不超过30封）；(2) 个性化邮件内容（避免完全相同）；(3) 配置正确的SPF/DKIM记录；(4) 使用企业邮箱而非个人邮箱；(5) 提供退订链接。正常使用下不会被标记。

### Q3：工作流引擎支持哪些触发方式？

支持四种触发方式：(1) 定时触发（cron表达式）；(2) 事件触发（邮件到达/文件变更）；(3) 手动触发（指令驱动）；(4) 上游完成触发（DAG依赖）。可组合使用实现复杂自动化场景。

### Q4：日程管理支持哪些日历平台？

支持Outlook、飞书日历、钉钉日历、Google Calendar、Apple Calendar。通过iCal协议实现跨平台同步。冲突检测基于参与者的共享空闲时间自动计算。

### Q5：报表生成支持哪些图表类型？

支持折线图、柱状图、饼图、散点图、面积图、热力图、雷达图等10+种图表类型。基于matplotlib渲染，可自定义颜色、字体、标题。图表可嵌入Word/PPT/PDF。

### Q6：云存储同步是实时的吗？

默认为定时同步（可配置频率，最短1分钟）。OneDrive支持近实时同步（基于Webhook）。大文件同步支持断点续传。同步冲突时默认保留最新版本，可配置为手动解决。

### Q7：工作流失败后如何恢复？

专业版支持检查点机制：(1) 每个步骤完成后记录状态；(2) 失败时从最后成功的检查点恢复；(3) 支持手动重试单个步骤；(4) 失败自动通知相关负责人；(5) 提供 detailed 错误日志。

### Q8：PPT生成支持哪些模板？

支持.pptx格式的PowerPoint模板。可定义占位符（{{标题}}、{{图表}}、{{数据}}），通过数据驱动批量生成。支持插入自动生成的图表、表格和图片。兼容企业品牌模板。

### Q9：邮件附件大小有限制吗？

单封邮件附件建议不超过25MB（兼容大多数邮件服务器）。超过时自动拆分为多封邮件或使用云存储链接替代。批量发送时附件总大小建议不超过2GB。

### Q10：如何调试工作流？

推荐：(1) 使用dry-run模式预览执行流程；(2) 查看工作流执行日志；(3) 单步执行验证每个节点；(4) 使用测试数据验证；(5) 配置debug模式输出详细信息。

### Q11：支持数据库直连生成报表吗？

支持连接 `PostgreSQL`、MySQL、SQLite等数据库。通过SQL查询采集数据，自动生成报表。连接配置存储在环境变量中，不硬编码。`PostgreSQL` 连接示例：

```python
# PostgreSQL连接（代码块中无需反引号）
import psycopg2
conn = psycopg2.connect(
    host=os.getenv('DB_HOST'),
    database=os.getenv('DB_NAME'),
    user=os.getenv('DB_USER'),
    password=os.getenv('DB_PASSWORD')
)
```

### Q12：多平台日程冲突如何解决？

专业版支持跨平台冲突检测：(1) 聚合所有平台的日历事件；(2) 统一时间轴分析重叠；(3) 推荐无冲突时段；(4) 确认后同步至所有平台。同步时保留各平台的事件ID，避免重复。
