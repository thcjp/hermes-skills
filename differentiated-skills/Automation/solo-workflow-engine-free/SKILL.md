## 核心能力
### Step 1：识别自动化机会

并非所有任务都适合自动化。先找到最高价值的机会。

**自动化审计（花1小时）**：

1. 追踪你一周内做的每件事（用记事本或简单表格）
2. 对每件事记录：
   - 耗时多少
   - 频率如何（每日/每周/每月）
   - 是重复的还是需要判断
3. 计算每个任务的时间成本：

```text
时间成本 = (单次耗时分钟 × 月频率) / 60
# ...
示例：15分钟的任务 × 每月20次 = 每月5小时
```

4. 按时间成本从高到低排序

**适合自动化的任务特征**：

- 重复性高（每次步骤相同）
- 规则驱动（不需要复杂判断）
- 频率高（每日或每周）
- 耗时长（10分钟以上）

**自动化机会示例**：

| 适合自动化 | 不适合自动化 |
|:------|:------|
| 给客户发周报（固定格式、固定时间） | 做客户访谈（需要细腻判断） |
| 付款后自动创建发票 | 写定制方案（需要创造力） |
| 表单提交自动加入CRM | 做产品决策（需要综合考虑） |
| 定时发布社交媒体内容 | 回答客户咨询（需要理解上下文） |

**低垂果实清单（从这里开始）**：

- 表单提交自动邮件通知
- 表单回复自动保存到表格
- 提前排期社交媒体发布
- 付款确认后自动创建发票
- 工具间数据同步（CRM ↔ 邮件工具 ↔ 表格）

**输入**: 用户提供Step 1：识别自动化机会所需的指令和必要参数。
**处理**: 解析Step 1：识别自动化机会的输入参数,完成核心逻辑,返回结构化响应。
**输出**: 返回Step 1：识别自动化机会的响应数据,包含状态码、结果和日志。

### Step 3：设计工作流

在构建之前，先在纸上或白板上画出工作流。

**工作流设计模板**：

```text
触发器：什么事件启动工作流？
  示例："Google表格新增一行"
# ...
条件（可选）：每次都执行，还是满足条件才执行？
  示例："仅当状态列 = '已批准'"
# ...
操作：执行什么动作？
  步骤1：[动作]
  步骤2：[动作]
  步骤3：[动作]
# ...
错误处理：某步失败时怎么办？
  示例："发Slack消息通知我"
```

**示例工作流（线索收集 → CRM → 邮件）**：

```text
触发器：网站新表单提交
# ...
条件：邮箱字段不为空
# ...
操作：
  步骤1：添加线索到CRM（如Airtable或HubSpot）
  步骤2：通过邮件工具发送欢迎邮件（如ConvertKit）
  步骤3：在项目管理工具创建跟进任务（如Notion），3天后跟进
  步骤4：发Slack通知："新线索：[姓名]"
# ...
错误处理：如果步骤1失败，发邮件警报给我
```

**设计原则**：

- 保持简单——从2-3步开始，以后再加复杂度
- 每步单独测试后再串联
- 需要时在操作间加延迟（有些API较慢）
- 必须包含错误通知，这样出问题时你会知道

**输入**: 用户提供Step 3：设计工作流所需的指令和必要参数。
**处理**: 解析Step 3：设计工作流的输入参数,完成核心逻辑,返回结构化响应。
**输出**: 返回Step 3：设计工作流的响应数据,包含状态码、结果和日志。

### Step 4：构建与测试

在你选的工具中实现工作流。

**构建流程（Zapier示例）**：

1. 选择触发器应用（如Google Forms、Typeform、网站表单）
2. 连接你的账户（OAuth认证）
3. 测试触发器（提交一个测试表单，确认数据能过来）
4. 添加操作（如"添加行到Google表格"）
5. 映射字段（匹配表单字段到表格列）
6. 测试操作（运行测试，确认行被正确添加）
7. 重复添加更多操作
8. 开启工作流（Zapier叫"turn on Zap"）

**测试清单**：

- [ ] 通过触发器提交测试数据
- [ ] 验证每个操作正确执行
- [ ] 检查数据映射到正确字段
- [ ] 用边界情况测试（空字段、特殊字符、长文本）
- [ ] 测试错误处理（故意制造失败，看告警是否工作）

**常见问题与修复**：

| 问题 | 原因 | 修复 |
|:---:|:---:|:---:|
| 工作流不触发 | 触发条件太窄 | 检查过滤设置，放宽条件 |
| 操作失败 | API限流或权限不足 | 在操作间加延迟，重新认证 |
| 数据缺失或错误 | 字段映射错误 | 仔细检查哪些字段被映射 |
| 工作流执行多次 | 重复触发 | 基于唯一ID去重 |

**铁律**：用真实数据测试后再依赖自动化。不要等真实客户遇到了才发现bug。

## FAQ

### Q1：我没有任何技术背景，能用这个吗？

完全可以。独立者引擎专注于无代码自动化工具（Zapier/Make/n8n），这些工具通过可视化界面连接各种应用，不需要写代码。Agent会根据你的描述，帮你选择工具、设计方案并指导构建。

### Q2：Zapier、Make、n8n该怎么选？

如果你是新手，从Zapier开始，它最简单。当你需要更复杂的工作流（多分支、循环、数据处理）时，升级到Make。如果你有技术背景且想完全控制，选n8n自托管（免费）。预算紧张时n8n自托管是最佳选择。

### Q3：免费版包含七步框架的哪些步骤？

免费版包含前四步核心流程：Step 1识别机会、Step 2选择工具、Step 3设计方案、Step 4构建测试。这四步足以帮你从零开始构建可用的自动化工作流。专业版解锁Step 5维护监控、Step 6高级场景、Step 7 ROI计算，以及完整故障排查表。

### Q4：自动化后如何确保不出错？

每个工作流都必须包含错误处理：在关键步骤设置失败通知（如Slack/邮件告警），这样出问题时你会立即知道。上线前用真实数据完整测试，包括边界情况（空字段、特殊字符）。专业版提供完整的故障排查表，覆盖常见问题。

### Q5：免费版和专业版有什么区别？

免费版提供四步核心流程和三个基础场景。专业版解锁七步完整框架（新增维护监控、高级场景、ROI计算），提供高级自动化方案（客户入驻、内容分发、客户健康监控、发票追踪）、多角色场景指南、常见错误清单和完整故障排查表。专业版还提供自动化维护计划。
